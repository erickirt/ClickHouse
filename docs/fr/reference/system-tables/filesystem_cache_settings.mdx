<div id="description">
  ## Description
</div>

Contient des informations sur l’ensemble des paramètres du cache du système de fichiers

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/fr/reference/data-types/string)) — nom du cache
* `path` ([String](/fr/reference/data-types/string)) — chemin du répertoire du cache
* `max_size` ([UInt64](/fr/reference/data-types/int-uint)) — taille maximale du cache
* `max_elements` ([UInt64](/fr/reference/data-types/int-uint)) — nombre maximal d&#39;éléments du cache, par ex. des segments de fichier (limite le nombre de fichiers sur le système de fichiers)
* `max_file_segment_size` ([UInt64](/fr/reference/data-types/int-uint)) — taille maximale d&#39;un segment de fichier unique
* `boundary_alignment` ([UInt64](/fr/reference/data-types/int-uint)) — alignement des segments de fichier
* `cache_on_write_operations` ([UInt8](/fr/reference/data-types/int-uint)) — active le cache en écriture immédiate (mise en cache lors de INSERT et MERGE)
* `cache_policy` ([String](/fr/reference/data-types/string)) — politique d&#39;éviction du cache
* `slru_size_ratio` ([Float64](/fr/reference/data-types/float)) — ratio de taille de la politique de cache SLRU entre les éléments protégés et ceux en probation
* `background_download_threads` ([UInt64](/fr/reference/data-types/int-uint)) — nombre de threads de téléchargement en arrière-plan. La valeur 0 désactive le téléchargement en arrière-plan
* `background_download_queue_size_limit` ([UInt64](/fr/reference/data-types/int-uint)) — taille de la file d&#39;attente de téléchargement en arrière-plan. La valeur 0 désactive le téléchargement en arrière-plan
* `background_download_max_file_segment_size` ([UInt64](/fr/reference/data-types/int-uint)) — taille maximale pouvant être téléchargée en arrière-plan
* `load_metadata_threads` ([UInt64](/fr/reference/data-types/int-uint)) — nombre maximal de threads utilisés pour charger les métadonnées du cache au démarrage du serveur
* `load_metadata_asynchronously` ([UInt8](/fr/reference/data-types/int-uint)) — active le chargement asynchrone des métadonnées au démarrage du serveur
* `keep_free_space_size_ratio` ([Float64](/fr/reference/data-types/float)) — ratio d&#39;espace libre que le cache tente de maintenir en arrière-plan
* `keep_free_space_elements_ratio` ([Float64](/fr/reference/data-types/float)) — ratio d&#39;éléments libres que le cache tente de maintenir en arrière-plan
* `keep_free_space_remove_batch` ([UInt64](/fr/reference/data-types/int-uint)) — taille du lot de suppression des éléments du cache effectuée par le thread d&#39;arrière-plan qui maintient le ratio d&#39;espace libre/d&#39;éléments libres
* `invalidated_entries_cleanup_interval_ms` ([UInt64](/fr/reference/data-types/int-uint)) — intervalle d&#39;inactivité, en millisecondes, de la tâche d&#39;arrière-plan qui purge les entrées invalidées (supprimées de manière différée) de la file de priorité
* `invalidated_entries_cleanup_threshold` ([UInt64](/fr/reference/data-types/int-uint)) — nombre d&#39;entrées invalidées accumulées dans la file de priorité qui déclenche leur suppression en arrière-plan
* `invalidated_entries_cleanup_remove_batch` ([UInt64](/fr/reference/data-types/int-uint)) — nombre maximal d&#39;entrées invalidées de la file de priorité supprimées sous un seul verrou d&#39;écriture à chaque itération en arrière-plan
* `enable_filesystem_query_cache_limit` ([UInt8](/fr/reference/data-types/int-uint)) — active la limitation de la taille maximale du cache pouvant être écrite dans le cadre d&#39;une requête
* `cache_hits_threshold` ([UInt64](/fr/reference/data-types/int-uint)) — paramètre obsolète
* `enable_bypass_cache_with_threshold` ([UInt8](/fr/reference/data-types/int-uint)) — Non documenté. Utilisation non recommandée
* `bypass_cache_threshold` ([UInt64](/fr/reference/data-types/int-uint)) — Non documenté. Utilisation non recommandée
* `write_cache_per_user_id_directory` ([UInt8](/fr/reference/data-types/int-uint)) — paramètre interne de ClickHouse Cloud
* `allow_dynamic_cache_resize` ([UInt8](/fr/reference/data-types/int-uint)) — Autorise le redimensionnement dynamique du cache du système de fichiers
* `max_size_ratio_to_total_space` ([Float64](/fr/reference/data-types/float)) — Rapport entre `max_size` et l’espace disque total
* `skip_cache_on_disk_failure` ([UInt8](/fr/reference/data-types/int-uint)) — Si true, ignore silencieusement les opérations du cache du système de fichiers en cas d’erreurs d’E/S disque. Si false (par défaut), les erreurs d’E/S disque sont propagées comme des échecs au démarrage.
* `use_split_cache` ([UInt8](/fr/reference/data-types/int-uint)) — Utilise la séparation des fichiers entre system et data.
* `split_cache_ratio` ([Float64](/fr/reference/data-types/float)) — Rapport entre le segment system et la taille totale du cache pour split&#95;cache.
* `overcommit_eviction_evict_step` ([UInt64](/fr/reference/data-types/int-uint)) — Pas d’éviction en octets pour la politique d’éviction overcommit. Utilisé pour les paramètres keep&#95;free&#95;space&#95;*&#95;ratio
* `check_cache_probability` ([Float64](/fr/reference/data-types/float)) — Fonctionne uniquement avec un build de Débogage ou un sanitizer. Vérifie l’intégrité du cache en le parcourant entièrement et en contrôlant l’état de chaque élément du cache
* `expose_prometheus_eviction_metrics` ([UInt8](/fr/reference/data-types/int-uint)) — Expose les métriques Prometheus relatives à l’activité d’éviction du cache du système de fichiers (`filesystem_cache_evictions_total` etc.). Désactivé par défaut. Peut être activé ou désactivé à l’exécution via `SYSTEM RELOAD CONFIG`.
* `expose_prometheus_eviction_metrics_per_user` ([UInt8](/fr/reference/data-types/int-uint)) — Expose également des métriques d’éviction par ID utilisateur. Nécessite `expose_prometheus_eviction_metrics`. La cardinalité augmente avec le nombre d’utilisateurs distincts effectuant des évictions.
* `is_initialized` ([UInt8](/fr/reference/data-types/int-uint)) — Indique si le cache a été initialisé avec succès
* `current_size` ([UInt64](/fr/reference/data-types/int-uint)) — Taille actuelle du cache
* `current_elements_num` ([UInt64](/fr/reference/data-types/int-uint)) — Nombre actuel d’éléments du cache (segments de fichier)

{/*AUTOGENERATED_END*/}
