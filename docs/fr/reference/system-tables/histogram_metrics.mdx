<div id="description">
  ## Description
</div>

Cette table contient des métriques d’histogramme qui peuvent être calculées instantanément et exportées au format Prometheus. Elle est toujours à jour. Elle remplace `system.latency_log`, désormais obsolète.

<div id="columns">
  ## Colonnes
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/fr/reference/data-types/string)) — Nom de la métrique.
* `value` ([Float64](/fr/reference/data-types/float)) — Valeur de la métrique.
* `description` ([String](/fr/reference/data-types/string)) — Description de la métrique.
* `labels` ([Map(String, String)](/fr/reference/data-types/map)) — Labels de métrique.

**Alias :**

* `name` — Alias de `metric`.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## Exemple
</div>

Vous pouvez utiliser une requête comme celle-ci pour exporter toutes les métriques sous forme d’histogramme au format Prometheus.

```sql
SELECT
  metric AS name,
  toFloat64(value) AS value,
  description AS help,
  labels,
  'histogram' AS type
FROM system.histogram_metrics
FORMAT Prometheus
```

<div id="metric_descriptions">
  ## Descriptions des métriques
</div>

| Métrique                                                     | Description                                                                                                                                                               |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `keeper_response_time_ms_bucket`                             | Temps de réponse de Keeper, en millisecondes.                                                                                                                             |
| `keeper_client_queue_duration_milliseconds_bucket`           | Temps passé par les requêtes à attendre leur mise en file d’attente, puis dans la file avant d’être traitées par le client Keeper, en millisecondes.                      |
| `keeper_receive_request_time_milliseconds_bucket`            | Temps nécessaire pour recevoir et analyser une requête du client dans le gestionnaire TCP de Keeper, en millisecondes.                                                    |
| `keeper_dispatcher_requests_queue_time_milliseconds_bucket`  | Temps qu’une requête passe dans la file d’attente des requêtes du dispatcher de Keeper, en millisecondes.                                                                 |
| `keeper_write_pre_commit_time_milliseconds_bucket`           | Temps nécessaire pour prétraiter une requête d’écriture avant le commit Raft, en millisecondes.                                                                           |
| `keeper_write_commit_time_milliseconds_bucket`               | Temps nécessaire pour traiter une requête d’écriture après le commit Raft, en millisecondes.                                                                              |
| `keeper_dispatcher_responses_queue_time_milliseconds_bucket` | Temps qu’une réponse passe dans la file d’attente des réponses du dispatcher de Keeper, en millisecondes.                                                                 |
| `keeper_send_response_time_milliseconds_bucket`              | Temps nécessaire pour envoyer une réponse au client dans le gestionnaire TCP de Keeper (inclut la mise en file d’attente et l’écriture dans le socket), en millisecondes. |
| `keeper_read_wait_for_write_time_milliseconds_bucket`        | Temps pendant lequel une requête de lecture attend la fin de la requête d’écriture dont elle dépend, en millisecondes.                                                    |
| `keeper_read_process_time_milliseconds_bucket`               | Temps nécessaire pour traiter une requête de lecture dans Keeper, en millisecondes.                                                                                       |
| `keeper_batch_size_elements_bucket`                          | Taille du lot envoyé à Raft, en éléments.                                                                                                                                 |
| `keeper_batch_size_bytes_bucket`                             | Taille du lot envoyé à Raft, en octets.                                                                                                                                   |
| `filesystem_cache_evicted_segment_hits_bucket`               | Distribution du nombre de hits du cache pour les segments de fichier au moment de leur éviction, libellée par nom de cache.                                               |
| `filesystem_cache_evicted_segment_size_bytes_bucket`         | Distribution des tailles en octets des segments de fichier évincés, libellée par nom de cache.                                                                            |
| `filesystem_cache_evicted_segment_hits_by_user_bucket`       | Distribution du nombre de hits du cache pour les segments de fichier évincés, libellée par nom de cache et ID utilisateur.                                                |
| `filesystem_cache_evicted_segment_size_bytes_by_user_bucket` | Distribution des tailles en octets des segments de fichier évincés, libellée par nom de cache et ID utilisateur.                                                          |
