Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="allow_feature_tier">
  ## allow_feature_tier
</div>

<SettingsInfoBlock type="UInt32" default_value="0" changeable_without_restart="Oui" />

Contrôle si l&#39;utilisateur peut modifier les paramètres associés aux différents niveaux de fonctionnalités.

* `0` - Les modifications de tout paramètre sont autorisées (experimental, beta, production).
* `1` - Seules les modifications des paramètres des fonctionnalités beta et de production sont autorisées. Les modifications des paramètres experimental sont rejetées.
* `2` - Seules les modifications des paramètres des fonctionnalités de production sont autorisées. Les modifications des paramètres experimental ou beta sont rejetées.

Cela équivaut à définir une contrainte `readonly` sur toutes les fonctionnalités `EXPERIMENTAL` / `BETA`.

<Note>
  Une valeur de `0` signifie que tous les paramètres peuvent être modifiés.
</Note>

<div id="allow_impersonate_user">
  ## allow_impersonate_user
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Active ou désactive la fonctionnalité IMPERSONATE (EXECUTE AS target&#95;user). Ce paramètre est obsolète.

<div id="allow_implicit_no_password">
  ## allow_implicit_no_password
</div>

Interdit la création d’un utilisateur sans mot de passe, sauf si &#39;IDENTIFIED WITH no&#95;password&#39; est explicitement indiqué.

```xml
<allow_implicit_no_password>1</allow_implicit_no_password>
```

<div id="allow_no_password">
  ## allow_no_password
</div>

Indique si le type de mot de passe non sécurisé no&#95;password est autorisé ou non.

```xml
<allow_no_password>1</allow_no_password>
```

<div id="allow_plaintext_password">
  ## allow_plaintext_password
</div>

Indique si les types d’authentification par mot de passe en clair (non sécurisés) sont autorisés ou non.

```xml
<allow_plaintext_password>1</allow_plaintext_password>
```
