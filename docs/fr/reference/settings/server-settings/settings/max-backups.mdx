Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_backups_io_thread_pool_free_size">
  ## max_backups_io_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Oui" />Si le nombre de threads **inactifs** dans le pool de threads d’E/S des sauvegardes dépasse `max_backup_io_thread_pool_free_size`, ClickHouse libérera les ressources utilisées par les threads inactifs et réduira la taille du pool. Des threads pourront être recréés si nécessaire.

<div id="max_backups_io_thread_pool_size">
  ## max_backups_io_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Oui" />ClickHouse utilise des threads du pool de threads d’E/S des sauvegardes pour effectuer les opérations d’E/S des sauvegardes sur S3. `max_backups_io_thread_pool_size` limite le nombre maximal de threads dans ce pool.
