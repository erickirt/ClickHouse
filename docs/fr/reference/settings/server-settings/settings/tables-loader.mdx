Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir des sources de ClickHouse.

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Définit le nombre de threads qui exécutent des tâches de chargement asynchrones dans le pool d’arrière-plan. Le pool d’arrière-plan sert à charger les tables de manière asynchrone après le démarrage du serveur lorsqu’aucune requête n’est en attente de la table. Il peut être utile de conserver un nombre réduit de threads dans le pool d’arrière-plan s’il y a beaucoup de tables. Cela permet de réserver des ressources CPU pour l’exécution concurrente des requêtes.

<Note>
  Une valeur de `0` signifie que tous les CPU disponibles seront utilisés.
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Définit le nombre de threads qui exécutent les tâches de chargement dans le pool au premier plan. Le pool au premier plan est utilisé pour charger les tables de manière synchrone avant que le serveur ne commence à écouter sur un port, ainsi que pour charger les tables dont le chargement est attendu. Le pool au premier plan a une priorité plus élevée que le pool d’arrière-plan. Cela signifie qu’aucune tâche ne démarre dans le pool d’arrière-plan tant que des tâches sont en cours d’exécution dans le pool au premier plan.

<Note>
  Une valeur de `0` signifie que tous les CPU disponibles seront utilisés.
</Note>
