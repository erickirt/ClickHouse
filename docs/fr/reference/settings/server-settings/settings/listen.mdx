Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="listen_backlog">
  ## listen_backlog
</div>

<SettingsInfoBlock type="UInt32" default_value="4096" changeable_without_restart="No" />

Backlog (taille de la file d’attente des connexions en attente) du socket d’écoute. La valeur par défaut de `<4096>` est la même que sous Linux 5.4+).

En général, il n’est pas nécessaire de modifier cette valeur, car :

* La valeur par défaut est suffisamment élevée,
* Le serveur dispose d’un thread distinct pour accepter les connexions des clients.

Ainsi, même si `<TcpExtListenOverflows>` (de `<nstat>`) est non nul et que ce compteur augmente pour le serveur ClickHouse, cela ne signifie pas que cette valeur doit être augmentée, car :

* En général, si `<4096>` ne suffit pas, cela indique plutôt un problème interne de scaling de ClickHouse ; il est donc préférable de signaler un issue.
* Cela ne signifie pas que le serveur pourra gérer davantage de connexions plus tard (et même si c’était le cas, les clients pourraient d’ici là être partis ou déconnectés).

**Exemple**

```xml
<listen_backlog>4096</listen_backlog>
```

<div id="listen_host">
  ## listen_host
</div>

Restriction des hôtes à partir desquels les requêtes peuvent provenir. Si vous voulez que le serveur réponde à tous, indiquez `::`.

Exemples :

```xml
<listen_host>::1</listen_host>
<listen_host>127.0.0.1</listen_host>
```

<div id="listen_reuse_port">
  ## listen_reuse_port
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Permet à plusieurs serveurs d’écouter sur la même adresse:port. Les requêtes seront dirigées vers un serveur choisi aléatoirement par le système d’exploitation. Il n’est pas recommandé d’activer ce paramètre.

**Exemple**

```xml
<listen_reuse_port>0</listen_reuse_port>
```

<div id="listen_try">
  ## listen_try
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Le serveur ne s’arrêtera pas si les réseaux IPv6 ou IPv4 ne sont pas disponibles au moment de l’écoute.

**Exemple**

```xml
<listen_try>0</listen_try>
```
