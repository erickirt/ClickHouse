Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="distributed_ddl">
  ## distributed_ddl
</div>

Gère l&#39;exécution des [requêtes DDL distribuées](/fr/reference/statements/distributed-ddl) (`CREATE`, `DROP`, `ALTER`, `RENAME`) dans le cluster.
Fonctionne uniquement si [ZooKeeper](/fr/reference/settings/server-settings/settings/zookeeper#zookeeper) est activé.

Les paramètres configurables de `<distributed_ddl>` incluent :

| Paramètre              | Description                                                                                                                                                            | Valeur par défaut                            |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| `cleanup_delay_period` | le nettoyage démarre après la réception d&#39;un nouvel événement de nœud si le dernier nettoyage n&#39;a pas eu lieu il y a moins de `cleanup_delay_period` secondes. | `60` secondes                                |
| `max_tasks_in_queue`   | le nombre maximal de tâches pouvant se trouver dans la file d&#39;attente.                                                                                             | `1,000`                                      |
| `path`                 | le chemin dans Keeper pour la `task_queue` des requêtes DDL                                                                                                            |                                              |
| `pool_size`            | le nombre de requêtes `ON CLUSTER` pouvant être exécutées simultanément                                                                                                |                                              |
| `profile`              | le profil utilisé pour exécuter les requêtes DDL                                                                                                                       |                                              |
| `task_max_lifetime`    | supprime le nœud si son ancienneté dépasse cette valeur.                                                                                                               | `7 * 24 * 60 * 60` (une semaine en secondes) |

**Exemple**

```xml
<distributed_ddl>
    <!-- Path in ZooKeeper to queue with DDL queries -->
    <path>/clickhouse/task_queue/ddl</path>

    <!-- Settings from this profile will be used to execute DDL queries -->
    <profile>default</profile>

    <!-- Controls how much ON CLUSTER queries can be run simultaneously. -->
    <pool_size>1</pool_size>

    <!--
         Cleanup settings (active tasks will not be removed)
    -->

    <!-- Controls task TTL (default 1 week) -->
    <task_max_lifetime>604800</task_max_lifetime>

    <!-- Controls how often cleanup should be performed (in seconds) -->
    <cleanup_delay_period>60</cleanup_delay_period>

    <!-- Controls how many tasks could be in the queue -->
    <max_tasks_in_queue>1000</max_tasks_in_queue>
</distributed_ddl>
```

<div id="distributed_ddl_use_initial_user_and_roles">
  ## distributed_ddl_use_initial_user_and_roles
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="Non" />Si elle est activée, les requêtes ON CLUSTER conserveront et utiliseront, pour leur exécution sur des segments distants, l’utilisateur et les rôles de l’initiateur. Cela garantit un contrôle d’accès cohérent à l’échelle du cluster, mais nécessite que l’utilisateur et les rôles existent sur tous les nœuds.
