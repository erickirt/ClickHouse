Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="http_connections_hard_limit">
  ## http_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Une exception est levée lors d&#39;une tentative de création si cette limite est atteinte. Définissez cette valeur sur 0 pour désactiver cette limite stricte. La limite s&#39;applique aux connexions HTTP qui n&#39;appartiennent à aucun disque ni stockage.

<div id="http_connections_rcvbuf">
  ## http_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Taille de l’option SO&#95;RCVBUF pour les connexions HTTP générales. Si elle est définie sur une valeur supérieure à 0, elle remplace l’autotuning TCP du noyau pour le tampon de réception. 0 = valeur par défaut du noyau (autotuning). Remarque : remettre ce paramètre à 0 ne réactive l’autotuning que pour les connexions nouvellement créées ; les connexions existantes du pool conservent des tailles de tampon fixes jusqu’à ce qu’elles soient recréées.

<div id="http_connections_sndbuf">
  ## http_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Taille de l’option SO&#95;SNDBUF pour les connexions HTTP générales. Si elle est définie sur une valeur supérieure à 0, elle remplace l’autotuning TCP du noyau pour le tampon d’envoi. 0 = valeur par défaut du noyau (autotuning). Remarque : remettre ce paramètre à 0 ne rétablit l’autotuning que pour les connexions nouvellement créées ; les connexions existantes du pool conservent des tailles de tampon fixes jusqu’à leur recréation.

<div id="http_connections_soft_limit">
  ## http_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Les connexions au-dessus de cette limite ont une durée de vie nettement plus courte. La limite s’applique aux connexions HTTP qui n’appartiennent à aucun disque ni stockage.

<div id="http_connections_store_limit">
  ## http_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />Les connexions au-delà de cette limite sont réinitialisées après utilisation. Définissez cette valeur sur 0 pour désactiver le cache des connexions. Cette limite s&#39;applique aux connexions HTTP qui ne sont associées à aucun disque ni stockage.

<div id="http_connections_warn_limit">
  ## http_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />Des messages d’avertissement sont consignés dans les logs si le nombre de connexions en cours d’utilisation dépasse cette limite. La limite s’applique aux connexions HTTP qui n’appartiennent à aucun disque ni stockage.
