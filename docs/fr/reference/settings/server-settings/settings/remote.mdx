<div id="remote_servers">
  ## remote_servers
</div>

Configuration des clusters utilisés par le moteur de table [Distributed](/fr/reference/engines/table-engines/special/distributed) et la fonction de table `cluster`.

**Exemple**

```xml
<remote_servers incl="clickhouse_remote_servers" />
```

Pour la valeur de l’attribut `incl`, consultez la section &quot;[Fichiers de configuration](/fr/concepts/features/configuration/server-config/configuration-files)&quot;.

<div id="remote_url_allow_hosts">
  ## remote_url_allow_hosts
</div>

Liste des hôtes autorisés à être utilisés dans les moteurs de stockage et les fonctions de table liés aux URL.

Lors de l’ajout d’un hôte avec la balise XML `\<host\>` :

* il doit être spécifié exactement comme dans l’URL, car le nom est vérifié avant la résolution DNS. Par exemple : `<host>clickhouse.com</host>`
* si le port est explicitement indiqué dans l’URL, alors host:port est vérifié dans son ensemble. Par exemple : `<host>clickhouse.com:80</host>`
* si l’hôte est spécifié sans port, alors n’importe quel port de cet hôte est autorisé. Par exemple : si `<host>clickhouse.com</host>` est spécifié, alors `clickhouse.com:20` (FTP), `clickhouse.com:80` (HTTP), `clickhouse.com:443` (HTTPS), etc. sont autorisés.
* si l’hôte est spécifié sous forme d’adresse IP, alors il est vérifié tel qu’il est indiqué dans l’URL. Par exemple : `[2a02:6b8:a::a]`.
* s’il y a des redirections et que leur prise en charge est activée, alors chaque redirection (le champ location) est vérifiée.

Par exemple :

```sql
<remote_url_allow_hosts>
    <host>clickhouse.com</host>
</remote_url_allow_hosts>
```
