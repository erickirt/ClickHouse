Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="jemalloc_enable_background_threads">
  ## jemalloc_enable_background_threads
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="Non" />Active les threads en arrière-plan de jemalloc. Jemalloc utilise des threads en arrière-plan pour nettoyer les pages mémoire inutilisées. Le désactiver peut entraîner une dégradation des performances.

<div id="jemalloc_enable_global_profiler">
  ## jemalloc_enable_global_profiler
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="Non" />Active le profileur d’allocation de jemalloc pour tous les threads. Jemalloc échantillonnera les allocations ainsi que toutes les désallocations des allocations échantillonnées.
Les profils peuvent être vidés à l’aide de SYSTEM JEMALLOC FLUSH PROFILE, qui peut être utilisé pour l’analyse des allocations.
Les échantillons peuvent également être stockés dans system.trace&#95;log à l’aide de la config jemalloc&#95;collect&#95;global&#95;profile&#95;samples&#95;in&#95;trace&#95;log ou avec le paramètre de requête jemalloc&#95;collect&#95;profile&#95;samples&#95;in&#95;trace&#95;log.
Voir [Profilage des allocations](/fr/concepts/features/performance/allocation-profiling)
