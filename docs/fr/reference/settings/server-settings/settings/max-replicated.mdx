Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir des sources de ClickHouse.

<div id="max_replicated_fetches_network_bandwidth_for_server">
  ## max_replicated_fetches_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Non" />Le débit maximal des échanges de données sur le réseau, en octets par seconde, pour les fetches répliqués. Zéro signifie qu’il n’y a pas de limite.

<div id="max_replicated_sends_network_bandwidth_for_server">
  ## max_replicated_sends_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Non" />Le débit maximal d&#39;échange de données sur le réseau, en octets par seconde, pour les envois répliqués. Zéro signifie illimité.

<div id="max_replicated_table_num_to_throw">
  ## max_replicated_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Si le nombre de tables répliquées est supérieur à cette valeur, le serveur lèvera une exception.

Ne comptabilise les tables que pour les moteurs de base de données suivants :

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  Une valeur de `0` signifie qu’aucune limite n’est appliquée.
</Note>

**Exemple**

```xml
<max_replicated_table_num_to_throw>400</max_replicated_table_num_to_throw>
```
