Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_prefixes_deserialization_thread_pool_free_size">
  ## max_prefixes_deserialization_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Oui" />

Si le nombre de threads **inactifs** dans le pool de threads de désérialisation des préfixes dépasse `max_prefixes_deserialization_thread_pool_free_size`, ClickHouse libère les ressources occupées par les threads inactifs et réduit la taille du pool. Des threads peuvent être recréés si nécessaire.

<div id="max_prefixes_deserialization_thread_pool_size">
  ## max_prefixes_deserialization_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="Oui" />

ClickHouse utilise des threads du pool de threads de désérialisation des préfixes pour lire en parallèle les métadonnées des colonnes et des sous-colonnes depuis les préfixes de fichiers dans les Wide parts de MergeTree. `max_prefixes_deserialization_thread_pool_size` limite le nombre maximal de threads dans le pool.
