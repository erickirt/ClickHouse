<div id="parts_killer_max_condemned_parts_per_batch">
  ## parts_killer_max_condemned_parts_per_batch
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="100000" changeable_without_restart="No" />

Nombre maximal de parts condamnées qu’une table SharedMergeTree récupère depuis Keeper en un seul cycle de nettoyage. En limitant la taille du Batch, on maintient le listage et le multi-get dans le délai d’expiration des opérations Keeper, de sorte que le nettoyage continue d’avancer même lorsqu’une table a accumulé un très grand nombre de parts condamnées en attente. Doit être supérieur à zéro : une valeur de `0` ferait que Keeper renvoie une liste vide à chaque cycle, bloquant définitivement le nettoyage. Disponible uniquement dans ClickHouse Cloud
