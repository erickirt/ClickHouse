Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="background_buffer_flush_schedule_pool_size">
  ## background_buffer_flush_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="Augmentation uniquement" />Nombre maximal de threads utilisés pour effectuer en arrière-plan les opérations de flush des [tables utilisant le moteur Buffer](/fr/reference/engines/table-engines/special/buffer).

<div id="background_common_pool_size">
  ## background_common_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />Le nombre maximal de threads utilisés en arrière-plan pour exécuter diverses opérations (principalement le garbage collection) sur les tables [*MergeTree-engine](/fr/reference/engines/table-engines/mergetree-family/index).

<div id="background_distributed_schedule_pool_size">
  ## background_distributed_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="Augmentation uniquement" />Le nombre maximal de threads utilisés pour exécuter les envois distribués.

<div id="background_fetches_pool_size">
  ## background_fetches_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />Le nombre maximal de threads utilisés en arrière-plan pour récupérer des parties de données à partir d’une autre réplique pour les tables [*MergeTree-engine](/fr/reference/engines/table-engines/mergetree-family/index).

<div id="background_message_broker_schedule_pool_size">
  ## background_message_broker_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="Augmentation uniquement" />Nombre maximal de threads utilisés pour exécuter les opérations en arrière-plan du streaming de messages.

<div id="background_move_pool_size">
  ## background_move_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />Nombre maximal de threads utilisés en arrière-plan pour déplacer des parties de données vers un autre disk ou volume pour les tables de type *MergeTree-engine.

<div id="background_pool_size">
  ## background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />

Définit le nombre de threads qui effectuent les merges et les mutations en arrière-plan pour les tables utilisant des engines *MergeTree-engine.

<Note>
  * Ce paramètre peut également être appliqué au démarrage du serveur à partir de la configuration du profil `default`, pour assurer la rétrocompatibilité au démarrage du serveur ClickHouse.
  * Vous pouvez uniquement augmenter le nombre de threads à l’exécution.
  * Pour réduire le nombre de threads, vous devez redémarrer le serveur.
  * En ajustant ce paramètre, vous gérez la charge du CPU et du disque.
</Note>

<Danger>
  Une taille de pool plus petite utilise moins de ressources CPU et disque, mais les processus en arrière-plan progressent plus lentement, ce qui peut à terme affecter les performances des requêtes.
</Danger>

Avant de le modifier, veuillez également consulter les paramètres *MergeTree-engine associés, tels que :

* [`number_of_free_entries_in_pool_to_lower_max_size_of_merge`](/fr/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_lower_max_size_of_merge).
* [`number_of_free_entries_in_pool_to_execute_mutation`](/fr/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_mutation).
* [`number_of_free_entries_in_pool_to_execute_optimize_entire_partition`](/fr/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_optimize_entire_partition)

**Exemple**

```xml
<background_pool_size>16</background_pool_size>
```
