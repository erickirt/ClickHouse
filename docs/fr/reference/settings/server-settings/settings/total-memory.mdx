Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir des sources de ClickHouse.

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Non" />Collecte les allocations aléatoires d’une taille inférieure ou égale à la valeur spécifiée, avec une probabilité égale à `total_memory_profiler_sample_probability`. 0 signifie que ce paramètre est désactivé. Il peut être utile de définir &#39;max&#95;untracked&#95;memory&#39; sur 0 pour que ce seuil fonctionne comme prévu.

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Collecte aléatoirement les allocations d&#39;une taille supérieure ou égale à la valeur spécifiée, avec une probabilité de `total_memory_profiler_sample_probability`. 0 signifie désactivé. Vous pouvez définir `max_untracked_memory` sur 0 afin que ce seuil fonctionne comme prévu.

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Chaque fois que l’utilisation mémoire du serveur dépasse le seuil suivant, exprimé en nombre d’octets, le profiler mémoire collecte la pile d’appels de l’allocation. Zéro signifie que le profiler mémoire est désactivé. Des valeurs inférieures à quelques mégaoctets ralentiront le serveur.

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="Non" />

Permet de collecter des allocations et désallocations aléatoires et de les consigner dans la table système [system.trace&#95;log](/fr/reference/system-tables/trace_log), avec `trace_type` défini sur `MemorySample`, selon la probabilité spécifiée. La probabilité s’applique à chaque allocation ou désallocation, quelle que soit la taille de l’allocation. Notez que l’échantillonnage n’a lieu que lorsque la quantité de mémoire non suivie dépasse la limite de mémoire non suivie (la valeur par défaut est de `4` MiB). Cette limite peut être abaissée si [total&#95;memory&#95;profiler&#95;step](/fr/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step) est réduit. Vous pouvez définir `total_memory_profiler_step` sur `1` pour un échantillonnage encore plus fin.

Valeurs possibles :

* Double positif.
* `0` — L’écriture des allocations et désallocations aléatoires dans la table système `system.trace_log` est désactivée.
