Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="mlock_executable">
  ## mlock_executable
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Exécutez `<mlockall>` après le démarrage afin de réduire la latence des premières requêtes et d’éviter que l’exécutable ClickHouse ne soit paginé en cas de forte charge d’IO.

<Note>
  Il est recommandé d’activer cette option, mais cela augmentera le temps de démarrage de quelques secondes. Gardez à l’esprit que ce paramètre ne fonctionnera pas sans la capacité &quot;CAP&#95;IPC&#95;LOCK&quot;.
</Note>

**Exemple**

```xml
<mlock_executable>false</mlock_executable>
```
