Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_io_thread_pool_free_size">
  ## max_io_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Oui" />

Si le nombre de threads **inactifs** dans le pool de threads d’E/S dépasse `max_io_thread_pool_free_size`, ClickHouse libère les ressources occupées par les threads inactifs et réduit la taille du pool. Les threads pourront être recréés si nécessaire.

<div id="max_io_thread_pool_size">
  ## max_io_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="Oui" />

ClickHouse utilise des threads du pool de threads d’E/S pour effectuer certaines opérations d’E/S (par ex. pour interagir avec S3). `max_io_thread_pool_size` limite le nombre maximal de threads du pool.
