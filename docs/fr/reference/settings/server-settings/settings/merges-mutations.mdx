Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="merges_mutations_memory_usage_soft_limit">
  ## merges_mutations_memory_usage_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Définit la limite de RAM pouvant être utilisée pour les opérations de fusion et de mutation.
Si ClickHouse atteint la limite définie, il ne planifiera plus aucune nouvelle opération de fusion ou de mutation en arrière-plan, mais continuera d’exécuter les tâches déjà planifiées.

<Note>
  Une valeur de `0` signifie qu’il n’y a pas de limite.
</Note>

**Exemple**

```xml
<merges_mutations_memory_usage_soft_limit>0</merges_mutations_memory_usage_soft_limit>
```

<div id="merges_mutations_memory_usage_to_ram_ratio">
  ## merges_mutations_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.5" changeable_without_restart="No" />

La valeur par défaut de `merges_mutations_memory_usage_soft_limit` est calculée selon la formule `memory_amount * merges_mutations_memory_usage_to_ram_ratio`.
