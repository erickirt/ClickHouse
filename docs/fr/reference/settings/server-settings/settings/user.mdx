Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="user_directories">
  ## user_directories
</div>

Section du fichier de configuration contenant les paramètres suivants :

* Chemin vers le fichier de configuration contenant les utilisateurs prédéfinis.
* Chemin vers le dossier où sont stockés les utilisateurs créés par des commandes SQL.
* Chemin du nœud ZooKeeper où sont stockés et répliqués les utilisateurs créés par des commandes SQL.

Si cette section est spécifiée, les chemins de [users&#95;config](/fr/reference/settings/server-settings/settings/users#users_config) et [access&#95;control&#95;path](/fr/reference/settings/server-settings/settings/access-control#access_control_path) ne seront pas utilisés.

La section `user_directories` peut contenir un nombre quelconque d’éléments ; l’ordre des éléments détermine leur préséance (plus un élément est placé haut, plus sa préséance est élevée).

**Exemples**

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <local_directory>
        <path>/var/lib/clickhouse/access/</path>
    </local_directory>
</user_directories>
```

Les utilisateurs, les rôles, les politiques de lignes, les quotas et les profils peuvent également être stockés dans ZooKeeper :

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <replicated>
        <zookeeper_path>/clickhouse/access/</zookeeper_path>
    </replicated>
</user_directories>
```

Vous pouvez également définir les sections `memory` — qui signifie que les informations sont stockées uniquement en mémoire, sans être écrites sur disque — et `ldap` — qui signifie que les informations sont stockées sur un serveur LDAP.

Pour ajouter un serveur LDAP en tant que répertoire distant pour des utilisateurs qui ne sont pas définis localement, définissez une seule section `ldap` avec les paramètres suivants :

| Paramètre | Description                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `roles`   | section contenant une liste de rôles définis localement qui seront attribués à chaque utilisateur récupéré sur le serveur LDAP. Si aucun rôle n&#39;est spécifié, l&#39;utilisateur ne pourra effectuer aucune action après l&#39;authentification. Si l&#39;un des rôles listés n&#39;est pas défini localement au moment de l&#39;authentification, la tentative d&#39;authentification échouera comme si le mot de passe fourni était incorrect. |
| `server`  | l&#39;un des noms de serveurs LDAP définis dans la section de config `ldap_servers`. Ce paramètre est obligatoire et ne peut pas être vide.                                                                                                                                                                                                                                                                                                         |

**Exemple**

```xml
<ldap>
    <server>my_ldap_server</server>
        <roles>
            <my_local_role1 />
            <my_local_role2 />
        </roles>
</ldap>
```

<div id="user_files_path">
  ## user_files_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_files/" changeable_without_restart="No" />

Le répertoire contenant les fichiers utilisateur. Utilisé dans la fonction de table [file()](/fr/reference/functions/table-functions/file), [fileCluster()](/fr/reference/functions/table-functions/fileCluster).

**Exemple**

```xml
<user_files_path>/var/lib/clickhouse/user_files/</user_files_path>
```

<div id="user_profile_events_per_cpu">
  ## user_profile_events_per_cpu
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Répartit les compteurs `ProfileEvents` par utilisateur par CPU afin d’éviter la contention entre CPU sur les lignes de cache lorsque le nombre de threads est élevé. Chaque utilisateur consomme `getNumCPUs() * num_events * sizeof(atomic)` de mémoire (des centaines de Kio sur les machines dotées de nombreux cœurs). Désactivez cette option pour revenir à un seul atomic partagé par événement et par utilisateur, au prix du débit pour économiser de la mémoire.

<div id="user_scripts_path">
  ## user_scripts_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_scripts/" changeable_without_restart="No" />

Le répertoire contenant les fichiers de script utilisateur. Utilisé pour les fonctions définies par l’utilisateur exécutables [Fonctions définies par l’utilisateur exécutables](/fr/reference/functions/regular-functions/udf#executable-user-defined-functions).

**Exemple**

```xml
<user_scripts_path>/var/lib/clickhouse/user_scripts/</user_scripts_path>
```
