Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="No" />

Chemin sur le système de fichiers local où sont stockées les données temporaires utilisées pour le traitement des requêtes volumineuses.

<Note>
  * Une seule option peut être utilisée pour configurer le stockage des données temporaires : tmp&#95;path, tmp&#95;policy, temporary&#95;data&#95;in&#95;cache.
  * Le slash de fin est obligatoire.
</Note>

**Exemple**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

Politique de stockage des données temporaires. Tous les fichiers portant le préfixe `tmp` seront supprimés au démarrage.

<Note>
  Recommandations pour l’utilisation du stockage objet pour `tmp_policy` :

  * Utilisez un `bucket:path` distinct sur chaque serveur
  * Utilisez `metadata_type=plain`
  * Vous pouvez également définir un TTL pour ce bucket
</Note>

<Note>
  * Une seule option peut être utilisée pour configurer le stockage des données temporaires : `tmp_path` ,`tmp_policy`, `temporary_data_in_cache`.
  * `move_factor`, `keep_free_space_bytes`,`max_data_part_size_bytes` sont ignorés.
  * La politique doit comporter exactement *un volume*

  Pour plus d’informations, consultez la documentation du [moteur de table MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree).
</Note>

**Exemple**

Lorsque `/disk1` est plein, les données temporaires seront stockées sur `/disk2`.

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
