Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="temporary_data_in_cache">
  ## temporary_data_in_cache
</div>

Avec cette option, les données temporaires seront stockées dans le cache du disque concerné.
Dans cette section, vous devez indiquer le nom du disque de type `cache`.
Dans ce cas, le cache et les données temporaires partageront le même espace, et le cache du disque pourra être évincé pour libérer de l’espace au profit des données temporaires.

<Note>
  Une seule option peut être utilisée pour configurer le stockage des données temporaires : `tmp_path`, `tmp_policy`, `temporary_data_in_cache`.
</Note>

**Exemple**

Le cache de `local_disk` et les données temporaires seront tous deux stockés dans `/tiny_local_cache` sur le système de fichiers, géré par `tiny_local_cache`.

```xml highlight={9-16,20}
<clickhouse>
<storage_configuration>
<disks>
<local_disk>
<type>local</type>
<path>/local_disk/</path>
</local_disk>

<tiny_local_cache>
<type>cache</type>
<disk>local_disk</disk>
<path>/tiny_local_cache/</path>
<max_size_rows>10M</max_size_rows>
<max_file_segment_size>1M</max_file_segment_size>
<cache_on_write_operations>1</cache_on_write_operations>
</tiny_local_cache>
</disks>
</storage_configuration>

<temporary_data_in_cache>tiny_local_cache</temporary_data_in_cache>
</clickhouse>
```
