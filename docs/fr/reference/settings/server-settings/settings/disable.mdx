Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="disable_insertion_and_mutation">
  ## disable_insertion_and_mutation
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Désactive les requêtes insert/alter/delete. Ce paramètre peut être activé si l’on a besoin de nœuds en lecture seule, afin d’éviter que les insertions et les mutations n’affectent les performances en lecture. Les insertions dans des moteurs externes (S3, DataLake, MySQL, PostgreSQL, Kafka, etc.) restent autorisées malgré ce paramètre.

<div id="disable_internal_dns_cache">
  ## disable_internal_dns_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Désactive le cache DNS interne. Recommandé pour exécuter ClickHouse dans des environnements dont l’infrastructure change fréquemment, comme Kubernetes.

<div id="disable_tunneling_for_https_requests_over_http_proxy">
  ## disable_tunneling_for_https_requests_over_http_proxy
</div>

Par défaut, le tunneling (c.-à-d. `HTTP CONNECT`) est utilisé pour effectuer des requêtes `HTTPS` via un proxy `HTTP`. Ce paramètre permet de le désactiver.

**no&#95;proxy**

Par défaut, toutes les requêtes passent par le proxy. Pour le contourner pour des hôtes spécifiques, la variable `no_proxy` doit être définie.
Elle peut être définie dans la clause `<proxy>` pour les resolvers de liste et distants, ou comme variable d&#39;environnement pour le resolver d&#39;environnement.
Elle prend en charge les adresses IP, les domaines, les sous-domaines et le caractère générique `'*'` pour un contournement complet. Les points en tête sont supprimés, comme le fait curl.

**Exemple**

La configuration ci-dessous contourne le proxy pour les requêtes vers `clickhouse.cloud` et tous ses sous-domaines (par ex., `auth.clickhouse.cloud`).
Il en va de même pour GitLab, même s&#39;il comporte un point en tête. `gitlab.com` et `about.gitlab.com` contourneraient tous deux le proxy.

```xml
<proxy>
    <no_proxy>clickhouse.cloud,.gitlab.com</no_proxy>
    <http>
        <uri>http://proxy1</uri>
        <uri>http://proxy2:3128</uri>
    </http>
    <https>
        <uri>http://proxy1:3128</uri>
    </https>
</proxy>
```
