Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="storage_configuration">
  ## storage_configuration
</div>

Permet une configuration du stockage sur plusieurs disques.

La configuration de stockage suit la structure suivante :

```xml
<storage_configuration>
    <disks>
        <!-- configuration -->
    </disks>
    <policies>
        <!-- configuration -->
    </policies>
</storage_configuration>
```

<div id="configuration-of-disks">
  ### Configuration de `disks`
</div>

La configuration de `disks` suit la structure ci-dessous :

```xml
<storage_configuration>
    <disks>
        <disk_name_1>
            <path>/mnt/fast_ssd/clickhouse/</path>
        </disk_name_1>
        <disk_name_2>
            <path>/mnt/hdd1/clickhouse/</path>
            <keep_free_space_bytes>10485760</keep_free_space_bytes>
        </disk_name_2>
        <disk_name_3>
            <path>/mnt/hdd2/clickhouse/</path>
            <keep_free_space_bytes>10485760</keep_free_space_bytes>
        </disk_name_3>
        ...
    </disks>
</storage_configuration>
```

Les sous-balises ci-dessus définissent les paramètres suivants pour `disks` :

| Paramètre               | Description                                                                                                             |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `<disk_name_N>`         | Nom du disque, qui doit être unique.                                                                                    |
| `path`                  | Chemin dans lequel les données du serveur seront stockées (répertoires `data` et `shadow`). Il doit se terminer par `/` |
| `keep_free_space_bytes` | Taille de l’espace libre réservé sur le disque.                                                                         |

<Note>
  L’ordre des disques n’a pas d’importance.
</Note>

<div id="configuration-of-policies">
  ### Configuration des politiques
</div>

Les sous-balises ci-dessus définissent les paramètres suivants pour `policies` :

| Paramètre                    | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `policy_name_N`              | Nom de la politique. Les noms de politique doivent être uniques.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| `volume_name_N`              | Nom du volume. Les noms de volume doivent être uniques.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `disk`                       | Le disque situé dans le volume.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `max_data_part_size_bytes`   | Taille maximale d&#39;un fragment de données pouvant résider sur l&#39;un des disques de ce volume. Si l&#39;opération de merge produit un fragment dont la taille attendue dépasse max&#95;data&#95;part&#95;size&#95;bytes, le fragment sera écrit sur le volume suivant. En pratique, cette fonctionnalité permet de stocker les nouveaux fragments de petite taille sur un volume rapide (SSD), puis de les déplacer vers un volume lent (HDD) lorsqu&#39;ils deviennent volumineux. N&#39;utilisez pas cette option si la politique ne comporte qu&#39;un seul volume.                                                                     |
| `move_factor`                | La part d&#39;espace libre disponible sur le volume. Si l&#39;espace libre passe en dessous de cette valeur, les données commenceront à être transférées vers le volume suivant, s&#39;il existe. Pour le transfert, les fragments sont triés par taille, du plus grand au plus petit (ordre décroissant), et les fragments dont la taille totale suffit à satisfaire la condition `move_factor` sont sélectionnés. Si la taille totale de tous les fragments est insuffisante, tous les fragments seront déplacés.                                                                                                                             |
| `perform_ttl_move_on_insert` | Désactive le déplacement, lors de l&#39;insertion, des données dont le TTL a expiré. Par défaut (si cette option est activée), si des données insérées ont déjà expiré selon la règle de déplacement basée sur la durée de vie, elles sont immédiatement déplacées vers le volume / disque spécifié dans la règle de déplacement. Cela peut considérablement ralentir l&#39;insertion si le volume / disque cible est lent (par exemple, S3). Si cette option est désactivée, la partie expirée des données est écrite sur le volume par défaut, puis immédiatement déplacée vers le volume spécifié dans la règle correspondant au TTL expiré. |
| `load_balancing`             | Politique d&#39;équilibrage des disques : `round_robin` ou `least_used`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| `least_used_ttl_ms`          | Définit le timeout (en millisecondes) de mise à jour de l&#39;espace disponible sur tous les disques (`0` - toujours mettre à jour, `-1` - ne jamais mettre à jour, valeur par défaut : `60000`). Notez que si le disque est utilisé uniquement par ClickHouse et ne fera pas l&#39;objet d&#39;un redimensionnement du système de fichiers à la volée, vous pouvez utiliser la valeur `-1`. Dans tous les autres cas, ce n&#39;est pas recommandé, car cela finira par entraîner une allocation incorrecte de l&#39;espace.                                                                                                                    |
| `prefer_not_to_merge`        | Désactive la fusion des parties de données sur ce volume. Remarque : ce paramètre est potentiellement nuisible et peut entraîner des ralentissements. Lorsqu&#39;il est activé (ce que nous déconseillons), la fusion des données sur ce volume est interdite (ce qui est défavorable). Cela permet de contrôler la façon dont ClickHouse interagit avec les disques lents. Nous recommandons de ne pas l&#39;utiliser du tout.                                                                                                                                                                                                                 |
| `volume_priority`            | Définit la priorité (l&#39;ordre) dans laquelle les volumes sont remplis. Plus la valeur est petite, plus la priorité est élevée. Les valeurs du paramètre doivent être des entiers naturels et couvrir la plage de 1 à N (N étant la plus grande valeur de paramètre spécifiée) sans lacunes.                                                                                                                                                                                                                                                                                                                                                  |

Pour `volume_priority` :

* Si tous les volumes ont ce paramètre, ils sont priorisés dans l&#39;ordre spécifié.
* Si seuls *certains* volumes l&#39;ont, les volumes qui ne l&#39;ont pas ont la priorité la plus basse. Ceux qui l&#39;ont sont priorisés selon la valeur de la balise, et la priorité des autres est déterminée par leur ordre de description dans le fichier de configuration.
* Si *aucun* volume ne reçoit ce paramètre, leur ordre est déterminé par l&#39;ordre de description dans le fichier de configuration.
* La priorité des volumes ne peut pas être identique.
