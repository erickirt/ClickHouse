Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="zookeeper">
  ## zookeeper
</div>

Contient des paramètres qui permettent à ClickHouse d&#39;interagir avec un cluster [ZooKeeper](http://zookeeper.apache.org/). ClickHouse utilise ZooKeeper pour stocker les métadonnées des répliques lors de l&#39;utilisation de tables répliquées. Si les tables répliquées ne sont pas utilisées, cette section de paramètres peut être omise.

Les paramètres suivants peuvent être configurés à l&#39;aide de sous-balises :

| Setting                                           | Description                                                                                                                                                                                                                                                                                                                                                                                                                        |
| ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `node`                                            | Endpoint ZooKeeper. Vous pouvez définir plusieurs endpoints. Par ex. `<node index="1"><host>example_host</host><port>2181</port></node>`. L&#39;attribut `index` spécifie l&#39;ordre des nœuds lors des tentatives de connexion au cluster ZooKeeper.                                                                                                                                                                             |
| `operation_timeout_ms`                            | Délai d&#39;expiration maximal pour une opération, en millisecondes.                                                                                                                                                                                                                                                                                                                                                               |
| `session_timeout_ms`                              | Délai d&#39;expiration maximal pour la session client, en millisecondes.                                                                                                                                                                                                                                                                                                                                                           |
| `root` (facultatif)                               | Le znode utilisé comme racine pour les znodes utilisés par le serveur ClickHouse.                                                                                                                                                                                                                                                                                                                                                  |
| `fallback_session_lifetime.min` (facultatif)      | Limite minimale de la durée de vie d&#39;une session ZooKeeper vers le nœud de fallback lorsque le primaire n&#39;est pas disponible (load-balancing). Définie en secondes. Par défaut : 3 heures.                                                                                                                                                                                                                                 |
| `fallback_session_lifetime.max` (facultatif)      | Limite maximale de la durée de vie d&#39;une session ZooKeeper vers le nœud de fallback lorsque le primaire n&#39;est pas disponible (load-balancing). Définie en secondes. Par défaut : 6 heures.                                                                                                                                                                                                                                 |
| `identity` (facultatif)                           | Nom d&#39;utilisateur et mot de passe requis par ZooKeeper pour accéder aux znodes demandés.                                                                                                                                                                                                                                                                                                                                       |
| `use_compression` (facultatif)                    | Active la compression dans le protocole Keeper si défini sur `true`.                                                                                                                                                                                                                                                                                                                                                               |
| `use_xid_64` (facultatif)                         | Active les ID de transaction sur 64 bits. Définissez cette valeur sur `true` pour activer le format étendu d&#39;ID de transaction. Par défaut : `false`.                                                                                                                                                                                                                                                                          |
| `pass_opentelemetry_tracing_context` (facultatif) | Active la propagation du contexte de traçage OpenTelemetry vers les requêtes Keeper. Lorsqu&#39;elle est activée, des spans de traçage sont créés pour les opérations Keeper, ce qui permet un traçage distribué entre ClickHouse et Keeper. Voir [Tracing ClickHouse Keeper Requests](/fr/guides/oss/deployment-and-scaling/monitoring/opentelemetry#tracing-clickhouse-keeper-requests) pour plus de détails. Par défaut : `false`. |

Il existe également le paramètre `zookeeper_load_balancing` (facultatif), qui vous permet de sélectionner l&#39;algorithme de sélection des nœuds ZooKeeper :

| Algorithm Name                  | Description                                                                                                                                     |
| ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| `random`                        | sélectionne aléatoirement l&#39;un des nœuds ZooKeeper.                                                                                         |
| `in_order`                      | sélectionne le premier nœud ZooKeeper ; s&#39;il n&#39;est pas disponible, le deuxième, et ainsi de suite.                                      |
| `nearest_hostname`              | sélectionne un nœud ZooKeeper dont le nom d&#39;hôte est le plus similaire à celui du serveur, la comparaison se faisant sur le préfixe du nom. |
| `hostname_levenshtein_distance` | comme nearest&#95;hostname, mais compare les noms d&#39;hôte à l&#39;aide de la distance de Levenshtein.                                        |
| `first_or_random`               | sélectionne le premier nœud ZooKeeper ; s&#39;il n&#39;est pas disponible, sélectionne aléatoirement l&#39;un des nœuds ZooKeeper restants.     |
| `round_robin`                   | sélectionne le premier nœud ZooKeeper, puis sélectionne le suivant en cas de reconnexion.                                                       |

**Exemple de configuration**

```xml
<zookeeper>
    <node>
        <host>example1</host>
        <port>2181</port>
    </node>
    <node>
        <host>example2</host>
        <port>2181</port>
    </node>
    <session_timeout_ms>30000</session_timeout_ms>
    <operation_timeout_ms>10000</operation_timeout_ms>
    <!-- Optional. Chroot suffix. Should exist. -->
    <root>/path/to/zookeeper/node</root>
    <!-- Optional. Zookeeper digest ACL string. -->
    <identity>user:password</identity>
    <!--<zookeeper_load_balancing>random / in_order / nearest_hostname / hostname_levenshtein_distance / first_or_random / round_robin</zookeeper_load_balancing>-->
    <zookeeper_load_balancing>random</zookeeper_load_balancing>
    <!-- Optional. Enable 64-bit transaction IDs. -->
    <use_xid_64>false</use_xid_64>
    <!-- Optional. Enable OpenTelemetry tracing context propagation. -->
    <pass_opentelemetry_tracing_context>false</pass_opentelemetry_tracing_context>
</zookeeper>
```
