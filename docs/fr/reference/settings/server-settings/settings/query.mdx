Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="query_cache">
  ## query_cache
</div>

Configuration du [cache de requêtes](/fr/concepts/features/performance/caches/query-cache).

Les paramètres suivants sont disponibles :

| Paramètre                 | Description                                                                                                              | Valeur par défaut |
| ------------------------- | ------------------------------------------------------------------------------------------------------------------------ | ----------------- |
| `max_entries`             | Le nombre maximal de résultats de requêtes `SELECT` stockés dans le cache.                                               | `1024`            |
| `max_entry_size_in_bytes` | La taille maximale, en octets, des résultats de requêtes `SELECT` pouvant être enregistrés dans le cache.                | `1048576`         |
| `max_entry_size_in_rows`  | Le nombre maximal de lignes que les résultats de requêtes `SELECT` peuvent contenir pour être enregistrés dans le cache. | `30000000`        |
| `max_size_in_bytes`       | La taille maximale du cache, en octets. `0` signifie que le cache de requêtes est désactivé.                             | `1073741824`      |

<Note>
  * Les paramètres modifiés prennent effet immédiatement.
  * Les données du cache de requêtes sont allouées en DRAM. Si la mémoire est limitée, veillez à définir une faible valeur pour `max_size_in_bytes` ou à désactiver complètement le cache de requêtes.
</Note>

**Exemple**

```xml
<query_cache>
    <max_size_in_bytes>1073741824</max_size_in_bytes>
    <max_entries>1024</max_entries>
    <max_entry_size_in_bytes>1048576</max_entry_size_in_bytes>
    <max_entry_size_in_rows>30000000</max_entry_size_in_rows>
</query_cache>
```

<div id="query_log">
  ## query_log
</div>

Paramètre permettant de journaliser les requêtes reçues avec le paramètre [log&#95;queries=1](/fr/reference/settings/session-settings).

Les requêtes sont journalisées dans la table [system.query&#95;log](/fr/reference/system-tables/query_log), et non dans un fichier distinct. Vous pouvez modifier le nom de la table via le paramètre `table` (voir ci-dessous).

<SystemLogParameters />

Si la table n&#39;existe pas, ClickHouse la créera. Si la structure du journal des requêtes a changé lors de la mise à jour du serveur ClickHouse, la table ayant l&#39;ancienne structure est renommée, et une nouvelle table est créée automatiquement.

**Exemple**

```xml
<query_log>
    <database>system</database>
    <table>query_log</table>
    <engine>Engine = MergeTree PARTITION BY event_date ORDER BY event_time TTL event_date + INTERVAL 30 day</engine>
    <flush_interval_milliseconds>7500</flush_interval_milliseconds>
    <max_size_rows>1048576</max_size_rows>
    <reserved_size_rows>8192</reserved_size_rows>
    <buffer_size_rows_flush_threshold>524288</buffer_size_rows_flush_threshold>
    <flush_on_crash>false</flush_on_crash>
</query_log>
```

<div id="query_masking_rules">
  ## query_masking_rules
</div>

Règles basées sur des expressions régulières, appliquées aux requêtes ainsi qu’à tous les messages de journalisation avant leur stockage dans les journaux du serveur,
les tables [`system.query_log`](/fr/reference/system-tables/query_log), [`system.text_log`](/fr/reference/system-tables/text_log), [`system.processes`](/fr/reference/system-tables/processes), ainsi que dans les journaux envoyés au client. Cela permet d’éviter
la fuite de données sensibles issues des requêtes SQL, telles que des noms, des adresses e-mail, des identifiants personnels ou des numéros de carte de crédit, dans les journaux.

**Exemple**

```xml
<query_masking_rules>
    <rule>
        <name>hide SSN</name>
        <regexp>(^|\D)\d{3}-\d{2}-\d{4}($|\D)</regexp>
        <replace>000-00-0000</replace>
    </rule>
</query_masking_rules>
```

**Champs de configuration**:

| paramètre | Description                                                                                  |
| --------- | -------------------------------------------------------------------------------------------- |
| `name`    | nom de la règle (facultatif)                                                                 |
| `regexp`  | expression régulière compatible RE2 (obligatoire)                                            |
| `replace` | chaîne de substitution pour les données sensibles (facultatif, par défaut : six astérisques) |

Les règles de masquage sont appliquées à l’ensemble de la requête (afin d’éviter toute fuite de données sensibles provenant de requêtes mal formées / impossibles à analyser).

La table [`system.events`](/fr/reference/system-tables/events) contient le compteur `QueryMaskingRulesMatch`, qui indique le nombre total de correspondances avec les règles de masquage des requêtes.

Pour les requêtes distribuées, chaque serveur doit être configuré séparément, sinon les sous-requêtes transmises à d’autres
nœuds seront stockées sans masquage.

<div id="query_metric_log">
  ## query_metric_log
</div>

Il est désactivé par défaut.

**Activation**

Pour activer manuellement la collecte de l’historique des Metric [`system.query_metric_log`](/fr/reference/system-tables/query_metric_log), créez `/etc/clickhouse-server/config.d/query_metric_log.xml` avec le contenu suivant :

```xml
<clickhouse>
    <query_metric_log>
        <database>system</database>
        <table>query_metric_log</table>
        <flush_interval_milliseconds>7500</flush_interval_milliseconds>
        <collect_interval_milliseconds>1000</collect_interval_milliseconds>
        <max_size_rows>1048576</max_size_rows>
        <reserved_size_rows>8192</reserved_size_rows>
        <buffer_size_rows_flush_threshold>524288</buffer_size_rows_flush_threshold>
        <flush_on_crash>false</flush_on_crash>
    </query_metric_log>
</clickhouse>
```

**Désactivation**

Pour désactiver le paramètre `query_metric_log`, vous devez créer le fichier `/etc/clickhouse-server/config.d/disable_query_metric_log.xml` suivant avec le contenu ci-dessous :

```xml
<clickhouse>
    <query_metric_log remove="1" />
</clickhouse>
```
