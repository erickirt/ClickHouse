Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Taille minimale, en octets, d’une allocation C++ générique (du type de celles effectuées par les conteneurs standard, les chaînes, l’extension de `std::vector`, les pointeurs intelligents, etc.) pouvant déclencher `MEMORY_LIMIT_EXCEEDED` une fois `max_server_memory_usage` atteint. Les allocations génériques plus petites sont toujours comptabilisées par le memory tracker, mais elles restent autorisées même au-delà de la limite, ce qui réduit les échecs intempestifs pendant les phases de nettoyage et de gestion des exceptions à l’approche d’un OOM.

Les allocations effectuées par les grands buffers internes de ClickHouse — données de colonne, tables de hachage, arenas, pipelines d’exécution des requêtes, buffers d’E/S — respectent toujours la limite mémoire et peuvent lever une exception quelle que soit cette valeur. Ce paramètre n’affecte que les allocations implicites passant par `operator new`.

Une valeur de `0` (par défaut) préserve le comportement historique : les allocations implicites via `operator new` ne sont jamais refusées par le memory tracker, et seules les allocations explicites de ClickHouse peuvent déclencher `MEMORY_LIMIT_EXCEEDED`.

Remarque : pour éviter les effets de bord, il est recommandé de définir une valeur supérieure à `max_untracked_memory`.

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Oui" />

Rapport minimal entre le temps d’attente du CPU de l’OS (métrique `OSCPUWaitMicroseconds`) et le temps d’activité (métrique `OSCPUVirtualTimeMicroseconds`) à partir duquel la fermeture de connexions est envisagée. Une interpolation linéaire entre les rapports minimal et maximal est utilisée pour calculer la probabilité ; à ce niveau, cette probabilité est de 0.
Voir [Contrôle du comportement en cas de surcharge CPU du serveur](/fr/concepts/features/configuration/settings/server-overload) pour plus de détails.
