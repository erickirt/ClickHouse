Ces paramètres configurent le ClickHouse server et sont générés automatiquement à partir du code source de ClickHouse.

<div id="top_level_domains_list">
  ## top_level_domains_list
</div>

Définit la liste des domaines de premier niveau personnalisés à ajouter ; chaque entrée doit être au format `<name>/path/to/file</name>`.

Par exemple :

```xml
<top_level_domains_lists>
    <public_suffix_list>/path/to/public_suffix_list.dat</public_suffix_list>
</top_level_domains_lists>
```

Voir aussi :

* la fonction [`cutToFirstSignificantSubdomainCustom`](/fr/reference/functions/regular-functions/url-functions#cutToFirstSignificantSubdomainCustom) et ses variantes,
  qui accepte le nom d’une liste de TLD personnalisée et renvoie la partie du domaine qui inclut les sous-domaines de plus haut niveau jusqu’au premier sous-domaine significatif.

<div id="top_level_domains_path">
  ## top_level_domains_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/top_level_domains/" changeable_without_restart="No" />

Le répertoire des domaines de premier niveau.

**Exemple**

```xml
<top_level_domains_path>/var/lib/clickhouse/top_level_domains/</top_level_domains_path>
```
