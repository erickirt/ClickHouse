Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="custom_cached_disks_base_directory">
  ## custom_cached_disks_base_directory
</div>

Ce paramètre spécifie le chemin du cache pour les disques cached personnalisés (créés à partir de SQL).
`custom_cached_disks_base_directory` a une priorité plus élevée pour les disques personnalisés que `filesystem_caches_path` (figurant dans `filesystem_caches_path.xml`),
qui est utilisé si le premier est absent.
Le path du paramètre de filesystem cache doit se trouver à l&#39;intérieur de ce directory,
sinon une exception sera levée, empêchant la création du disk.

<Note>
  Cela n&#39;affectera pas les disks créés avec une version antérieure pour lesquels le server a été upgraded.
  Dans ce cas, aucune exception ne sera levée, afin de permettre au server de démarrer correctement.
</Note>

Exemple :

```xml
<custom_cached_disks_base_directory>/var/lib/clickhouse/caches/</custom_cached_disks_base_directory>
```

<div id="custom_settings_prefixes">
  ## custom_settings_prefixes
</div>

Liste des préfixes utilisés pour les [paramètres personnalisés](/fr/concepts/features/configuration/settings/settings-query-level#custom_settings).
Si plusieurs préfixes sont utilisés, séparez-les par des virgules.

**Exemple**

```xml
<custom_settings_prefixes>SQL_</custom_settings_prefixes>
```
