Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Une exception est levée lors d&#39;une tentative de création lorsque cette limite est atteinte. Définissez cette valeur sur 0 pour désactiver la limite stricte. Cette limite s&#39;applique aux connexions des disques.

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />La taille de l’option SO&#95;RCVBUF pour les connexions aux disques (S3, Azure, GCS). La valeur par défaut est 204800 (200 KB) : le stockage objet est accessible au sein de la même Region, où le produit débit-délai est bien inférieur à 100 KB. Une faible limite suffit donc à atteindre le débit maximal, tout en évitant les tampons de plusieurs MiB par socket que l’autotuning du noyau allouerait autrement. Définissez cette valeur sur 0 pour utiliser à la place l’autotuning TCP du noyau pour le tampon de réception. Remarque : rétablir ce paramètre à 0 ne réactive l’autotuning que pour les connexions nouvellement créées ; les connexions du pool existantes conservent des tailles de tampon fixes jusqu’à ce qu’elles soient recréées.

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Taille de l’option SO&#95;SNDBUF pour les connexions du disk (S3, Azure, GCS). Si elle est définie sur une valeur supérieure à 0, elle surcharge l’autotuning TCP du noyau pour le tampon d’envoi. 0 = valeur par défaut du noyau (autotuning). Remarque : remettre ce paramètre à 0 ne rétablit l’autotuning que pour les connexions nouvellement créées ; les connexions existantes du pool conservent une taille de tampon fixe jusqu’à leur recréation.

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />Les connexions au-delà de cette limite ont une durée de vie nettement plus courte. Cette limite s&#39;applique aux connexions aux disques.

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />Les connexions dépassant cette limite sont réinitialisées après utilisation. Définissez cette valeur sur 0 pour désactiver le cache de connexions. La limite s&#39;applique aux connexions des disques.

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />Des messages d’avertissement sont consignés dans les logs si le nombre de connexions utilisées dépasse cette limite. La limite s’applique aux connexions des disques.
