Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="distributed_ddl.cleanup_delay_period">
  ## distributed_ddl.cleanup_delay_period
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />le nettoyage commence après la réception d’un événement de nouveau nœud si le dernier nettoyage n’a pas eu lieu il y a moins de `<cleanup_delay_period>` secondes.

<div id="distributed_ddl.max_tasks_in_queue">
  ## distributed_ddl.max_tasks_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />le nombre maximal de tâches pouvant se trouver dans la queue.

<div id="distributed_ddl.path">
  ## distributed_ddl.path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/ddl/" changeable_without_restart="No" />chemin dans Keeper vers la `<task_queue>` des requêtes DDL

<div id="distributed_ddl.pool_size">
  ## distributed_ddl.pool_size
</div>

<SettingsInfoBlock type="Int32" default_value="1" changeable_without_restart="No" />nombre de requêtes `<ON CLUSTER>` pouvant être exécutées en parallèle

<div id="distributed_ddl.profile">
  ## distributed_ddl.profile
</div>

profil utilisé pour exécuter les requêtes DDL

<div id="distributed_ddl.replicas_path">
  ## distributed_ddl.replicas_path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/replicas/" changeable_without_restart="No" />le chemin dans Keeper du `<task_queue>` pour les répliques

<div id="distributed_ddl.task_max_lifetime">
  ## distributed_ddl.task_max_lifetime
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" changeable_without_restart="No" />supprime le nœud si sa durée de vie dépasse cette valeur.
