Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_thread_pool_free_size">
  ## max_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />

Si le nombre de threads **inactifs** dans le pool global de threads dépasse [`max_thread_pool_free_size`](/fr/reference/settings/server-settings/settings/max-thread#max_thread_pool_free_size), ClickHouse libère les ressources occupées par certains threads et réduit la taille du pool. Des threads peuvent être recréés si nécessaire.

**Exemple**

```xml
<max_thread_pool_free_size>1200</max_thread_pool_free_size>
```

<div id="max_thread_pool_size">
  ## max_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />

ClickHouse utilise des threads du pool global de threads pour traiter les requêtes. S&#39;il n&#39;y a pas de thread inactif disponible pour traiter une requête, un nouveau thread est créé dans le pool. `max_thread_pool_size` limite le nombre maximal de threads du pool.

**Exemple**

```xml
<max_thread_pool_size>12000</max_thread_pool_size>
```
