Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_database_num_to_throw">
  ## max_database_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Si le nombre de bases de données dépasse cette valeur, le serveur lèvera une exception. 0 signifie qu’il n’y a pas de limite.

<div id="max_database_num_to_warn">
  ## max_database_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Oui" />

Si le nombre de bases de données attachées dépasse la valeur spécifiée, le serveur ClickHouse ajoute des messages d’avertissement à la table `system.warnings`.

**Exemple**

```xml
<max_database_num_to_warn>50</max_database_num_to_warn>
```

<div id="max_database_replicated_create_table_thread_pool_size">
  ## max_database_replicated_create_table_thread_pool_size
</div>

<SettingsInfoBlock type="UInt32" default_value="1" changeable_without_restart="No" />Le nombre de threads utilisés pour créer des tables lors de la récupération d’une replica dans DatabaseReplicated. Zéro signifie que le nombre de threads est égal au nombre de cœurs.
