Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_dictionary_num_to_throw">
  ## max_dictionary_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Oui" />

Si le nombre de dictionnaires dépasse cette valeur, le serveur lèvera une exception.

Ne compte les tables que pour les moteurs de base de données suivants :

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  Une valeur de `0` signifie qu&#39;il n&#39;y a aucune limite.
</Note>

**Exemple**

```xml
<max_dictionary_num_to_throw>400</max_dictionary_num_to_throw>
```

<div id="max_dictionary_num_to_warn">
  ## max_dictionary_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Oui" />

Si le nombre de dictionnaires attachés dépasse la valeur spécifiée, le serveur ClickHouse ajoutera des messages d’avertissement à la table `system.warnings`.

**Exemple**

```xml
<max_dictionary_num_to_warn>400</max_dictionary_num_to_warn>
```
