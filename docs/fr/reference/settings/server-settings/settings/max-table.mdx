Ces paramètres configurent serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Si le nombre de tables dépasse cette valeur, le serveur lèvera une exception.

Les tables suivantes ne sont pas comptabilisées :

* view
* remote
* dictionary
* system

Seules les tables des moteurs de base de données suivants sont comptabilisées :

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  Une valeur de `0` signifie qu’il n’y a aucune limite.
</Note>

**Exemple**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Oui" />

Si le nombre de tables attachées dépasse la valeur spécifiée, le serveur ClickHouse ajoute des messages d&#39;avertissement à la table `system.warnings`.

**Exemple**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

Restriction concernant la suppression des tables.

Si la taille d’une table [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree) dépasse `max_table_size_to_drop` (en octets), vous ne pouvez pas la supprimer à l’aide d’une requête [`DROP`](/fr/reference/statements/drop) ou [`TRUNCATE`](/fr/reference/statements/truncate).

<Note>
  Une valeur de `0` signifie que vous pouvez supprimer toutes les tables sans aucune restriction.

  Ce paramètre ne nécessite pas de redémarrer le serveur ClickHouse pour être appliqué. Une autre façon de désactiver cette restriction consiste à créer le fichier `<clickhouse-path>/flags/force_drop_table`.
</Note>

**Exemple**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
