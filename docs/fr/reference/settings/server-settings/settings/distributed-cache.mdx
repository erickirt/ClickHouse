Ces paramètres configurent le serveur ClickHouse et sont générés automatiquement à partir du code source de ClickHouse.

<div id="distributed_cache_apply_throttling_settings_from_client">
  ## distributed_cache_apply_throttling_settings_from_client
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Indique si le serveur de cache doit appliquer les paramètres de limitation du débit reçus du client.

<div id="distributed_cache_keep_up_free_connections_ratio">
  ## distributed_cache_keep_up_free_connections_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.1" changeable_without_restart="No" />Limite souple du nombre de connexions actives que le distributed cache tentera de maintenir libres. Lorsque le nombre de connexions libres passe sous distributed&#95;cache&#95;keep&#95;up&#95;free&#95;connections&#95;ratio * max&#95;connections, les connexions dont l&#39;activité est la plus ancienne sont fermées jusqu&#39;à ce que ce nombre repasse au-dessus de la limite.

<div id="distributed_cache_write_pool_size">
  ## distributed_cache_write_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Le nombre maximal de requêtes d’écriture du distributed cache pouvant s’exécuter simultanément sur un thread d’arrière-plan (toutes requêtes confondues). Lorsque cette limite est atteinte, l’écriture passe par le cache en mode intégré (sur le thread appelant) plutôt que sur un thread d’arrière-plan ; elle n’est donc ni bloquée dans l’attente d’un slot ni à l’origine d’un nombre illimité de threads.
