<div id="inactive_parts_to_delay_insert">
  ## inactive_parts_to_delay_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Si le nombre de parties inactives dans une partition donnée de la table dépasse
la valeur `inactive_parts_to_delay_insert`, un `INSERT` est artificiellement
ralenti.

<Tip>
  Ceci est utile lorsque le serveur ne parvient pas à nettoyer les parties assez rapidement.
</Tip>

Valeurs possibles :

* Tout entier positif.

<div id="inactive_parts_to_throw_insert">
  ## inactive_parts_to_throw_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Si le nombre de parties inactives dans une partition dépasse la valeur
`inactive_parts_to_throw_insert`, `INSERT` est interrompu avec
l’exception suivante :

> &quot;Trop de parties inactives (N). Le nettoyage des parties est nettement
> plus lent que les insertions&quot;.

Valeurs possibles :

* Tout entier positif.
