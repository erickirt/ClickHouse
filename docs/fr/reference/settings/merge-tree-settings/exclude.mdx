  ## exclude_deleted_rows_for_part_size_in_merge
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Si ce paramètre est activé, la taille réelle estimée des parties de données (c.-à-d. en excluant les lignes
supprimées via `DELETE FROM`) sera utilisée lors de la sélection des
parties à fusionner. Notez que ce comportement n&#39;est déclenché que pour les parties de données
affectées par un `DELETE FROM` exécuté après l&#39;activation de ce paramètre.

Valeurs possibles :

* `true`
* `false`

<div id="exclude_materialize_skip_indexes_on_merge">
  ## exclude_materialize_skip_indexes_on_merge
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.10"},{"label": ""},{"label": "Nouveau paramètre."}]}]} />

Exclut de la construction et du stockage lors des fusions la liste d&#39;index de saut fournie, séparée par des virgules. N&#39;a aucun effet si
[materialize&#95;skip&#95;indexes&#95;on&#95;merge](/fr/reference/settings/merge-tree-settings/materialize#materialize_skip_indexes_on_merge) est false.

Les index de saut exclus seront néanmoins toujours construits et stockés par une requête
[MATERIALIZE INDEX](/fr/reference/statements/alter/skipping-index#materialize-index) explicite ou lors des INSERTs, selon le
paramètre de [session](/fr/reference/settings/session-settings#materialize_skip_indexes_on_insert) [materialize&#95;skip&#95;indexes&#95;on&#95;insert].

Exemple :

```sql
CREATE TABLE tab
(
a UInt64,
b UInt64,
INDEX idx_a a TYPE minmax,
INDEX idx_b b TYPE set(3)
)
ENGINE = MergeTree ORDER BY tuple() SETTINGS exclude_materialize_skip_indexes_on_merge = 'idx_a';

INSERT INTO tab SELECT number, number / 50 FROM numbers(100); -- setting has no effect on INSERTs

-- idx_a will be excluded from update during background or explicit merge via OPTIMIZE TABLE FINAL

-- can exclude multiple indexes by providing a list
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = 'idx_a, idx_b';

-- default setting, no indexes excluded from being updated during merge
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = '';
```
