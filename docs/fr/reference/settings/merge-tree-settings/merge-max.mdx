<div id="merge_max_block_size">
  ## merge_max_block_size
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="8192" />

Le nombre de lignes lues en mémoire à partir des parts fusionnées.

Valeurs possibles :

* Tout entier positif.

L&#39;opération de fusion lit les lignes des parts par blocs de `merge_max_block_size` lignes, puis
fusionne et écrit le résultat dans une nouvelle part. Le bloc lu est placé en RAM,
donc `merge_max_block_size` influe sur la quantité de RAM nécessaire à la fusion.
Ainsi, les fusions peuvent consommer une grande quantité de RAM pour les tables dont les lignes sont très larges
(si la taille moyenne d&#39;une ligne est de 100kb, alors lors de la fusion de 10 parts,
(100kb * 10 * 8192) = ~ 8GB de RAM). En diminuant `merge_max_block_size`,
vous pouvez réduire la quantité de RAM requise pour une fusion, mais au prix d&#39;une fusion plus lente.

<div id="merge_max_block_size_bytes">
  ## merge_max_block_size_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="10485760" />

Nombre d’octets que doivent contenir les blocs formés pour les opérations de fusion. Par défaut,
a la même valeur que `index_granularity_bytes`.

<div id="merge_max_dynamic_subcolumns_in_compact_part">
  ## merge_max_dynamic_subcolumns_in_compact_part
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="auto" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "auto"},{"label": "Ajout d'un nouveau paramètre pour limiter le nombre de sous-colonnes dynamiques dans une part Compact après fusion, quels que soient les paramètres spécifiés dans le type de données"}]}]} />

Le nombre maximal de sous-colonnes dynamiques pouvant être créées dans chaque colonne d&#39;une part de données Compact après fusion.
Il permet de contrôler le nombre de sous-colonnes dynamiques dans une part Compact, quels que soient les paramètres dynamiques spécifiés dans le type de données.

Par exemple, si la table comporte une colonne de type JSON(max&#95;dynamic&#95;paths=1024) et que le paramètre merge&#95;max&#95;dynamic&#95;subcolumns&#95;in&#95;compact&#95;part est défini sur 128,
après fusion dans une part de données Compact, le nombre de chemins dynamiques sera ramené à 128 dans cette part et seuls 128 chemins seront écrits sous forme de sous-colonnes dynamiques.

  ## merge_max_dynamic_subcolumns_in_wide_part
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="auto" />

après fusion dans la Wide part, le nombre de chemins dynamiques sera ramené à 128 dans cette part et seuls 128 chemins seront écrits sous forme de sous-colonnes dynamiques.
