<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "16"},{"label": "Nouveau paramètre. Identique à `concurrent_part_removal_threshold`, mais utilisé lorsqu’au moins une
part à supprimer est stockée sur un disque distant. La valeur par défaut est plus basse
car chaque suppression de part sur un stockage distant nécessite généralement un
aller-retour réseau (par ex. une requête HTTP `DELETE` par part sur le stockage objet), de sorte qu’une
suppression en série, même de 100 parts, peut bloquer un `DROP TABLE` pendant des dizaines de
secondes.
