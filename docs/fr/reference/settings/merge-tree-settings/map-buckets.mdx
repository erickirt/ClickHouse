  ## map_buckets_coefficient
</div>

<SettingsInfoBlock type="Float" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "1"},{"label": "Ajout d'un paramètre permettant de contrôler le coefficient utilisé dans les stratégies `sqrt` et `linear` pour calculer le nombre de buckets pour la sérialisation Map `with_buckets`"}]}]} />

Le coefficient utilisé dans les stratégies [map&#95;buckets&#95;strategy](/fr/reference/settings/merge-tree-settings/map-buckets#map_buckets_strategy) `sqrt` et `linear` pour calculer le nombre de buckets à partir de la taille moyenne de la map.
Pour la stratégie `sqrt` : `round(map_buckets_coefficient * sqrt(avg_map_size))`.
Pour la stratégie `linear` : `round(map_buckets_coefficient * avg_map_size)`.
Ignoré lorsque `map_buckets_strategy` est défini sur `constant`.

  ## map_buckets_min_avg_size
</div>

<SettingsInfoBlock type="UInt64" default_value="32" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "32"},{"label": "Ajoute un paramètre pour contrôler la taille moyenne minimale des maps (nombre de clés par ligne) requise pour appliquer la sérialisation `with_buckets`"}]}]} />

Taille moyenne minimale des maps (nombre de clés par ligne) requise pour appliquer la sérialisation `with_buckets`.
Si la taille moyenne des maps est inférieure à cette valeur, un seul bucket est utilisé, quels que soient les autres paramètres de bucket.
Une valeur de `0` désactive ce seuil et applique toujours la stratégie de bucketing.
Ce paramètre est utile pour éviter le surcoût de la sérialisation en buckets pour les petites maps, lorsque le gain est négligeable.

<div id="map_buckets_strategy">
  ## map_buckets_strategy
</div>

<SettingsInfoBlock type="MergeTreeMapBucketsStrategy" default_value="sqrt" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "sqrt"},{"label": "Ajout d'un paramètre permettant de contrôler la stratégie de sélection du nombre de buckets pour la sérialisation Map `with_buckets`"}]}]} />

Contrôle la stratégie utilisée pour déterminer le nombre de buckets dans la sérialisation Map `with_buckets`, en fonction de la taille moyenne des maps.

Valeurs possibles :

* constant — Utilise toujours [max&#95;buckets&#95;in&#95;map](/fr/reference/settings/merge-tree-settings/max#max_buckets_in_map) comme nombre de buckets, quelle que soit la taille moyenne des maps.
* sqrt — Utilise `round(map_buckets_coefficient * sqrt(avg_map_size))` comme nombre de buckets, limité à l&#39;intervalle `[1, max_buckets_in_map]`.
* linear — Utilise `round(map_buckets_coefficient * avg_map_size)` comme nombre de buckets, limité à l&#39;intervalle `[1, max_buckets_in_map]`.
