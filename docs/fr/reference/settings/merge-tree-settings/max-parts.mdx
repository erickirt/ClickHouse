<div id="max_parts_in_total">
  ## max_parts_in_total
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

Si le nombre total de parts actives dans toutes les partitions d&#39;une table dépasse la
valeur `max_parts_in_total`, l&#39;opération `INSERT` est interrompue avec l&#39;exception `Too many parts
(N)`.

Valeurs possibles :

* Tout entier positif.

Un grand nombre de parts dans une table réduit les performances des requêtes ClickHouse
et augmente le temps de démarrage de ClickHouse. Le plus souvent, cela résulte d&#39;une
mauvaise conception (erreurs dans le choix d&#39;une stratégie de partitionnement : partitions
trop petites).

<div id="max_parts_to_merge_at_once">
  ## max_parts_to_merge_at_once
</div>

<SettingsInfoBlock type="UInt64" default_value="100" />

Nombre maximal de parts pouvant être fusionnées simultanément (0 = désactivé). N&#39;affecte pas la
requête OPTIMIZE FINAL.
