  ## map_serialization_version

Contrôle la méthode de sérialisation utilisée pour les colonnes `Map`.

Valeurs possibles :

* basic — Utilise la sérialisation standard pour `Map`.
* with&#95;buckets — Répartit les clés dans des buckets lors de la sérialisation. L’utilisation de buckets améliore la lecture de clés individuelles dans le Map.

Le nombre de buckets dans la sérialisation `with_buckets` est déterminé par [max&#95;buckets&#95;in&#95;map](/fr/reference/settings/merge-tree-settings/max#max_buckets_in_map) et [map&#95;buckets&#95;strategy](/fr/reference/settings/merge-tree-settings/map-buckets#map_buckets_strategy).

  ## map_serialization_version_for_zero_level_parts
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "Ajout d’un paramètre permettant de contrôler la version de sérialisation de Map pour les parts de niveau zéro"}]}]} />

Ce paramètre permet de définir une version de sérialisation différente pour les
colonnes `Map` des parts de niveau zéro créées lors des insertions.
Il peut être utile de conserver la sérialisation `basic` pour les parts de niveau zéro afin d’éviter
une baisse des performances lors des insertions, tout en utilisant `with_buckets` pour les parts fusionnées.
