<div id="min_compress_block_size">
  ## min_compress_block_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Taille minimale des blocs de données non compressées requise pour la compression lors de

<div id="min_marks_to_honor_max_concurrent_queries">
  ## min_marks_to_honor_max_concurrent_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Le nombre minimal de marks lus par la requête pour appliquer le paramètre [max&#95;concurrent&#95;queries](/fr/reference/settings/merge-tree-settings/max#max_concurrent_queries).

  Les requêtes resteront limitées par d&#39;autres paramètres `max_concurrent_queries`.

* Entier positif.

<div id="min_merge_bytes_to_use_direct_io">
  ## min_merge_bytes_to_use_direct_io
</div>

<SettingsInfoBlock type="UInt64" default_value="10737418240" />

Le volume minimal de données d’une opération de fusion requis pour utiliser
l’accès en E/S directes au disque de stockage. Lors de la fusion de parties de données, ClickHouse calcule
le volume de stockage total de toutes les données à fusionner. Si ce volume dépasse
`min_merge_bytes_to_use_direct_io` octets, ClickHouse lit et écrit les
données sur le disque de stockage à l’aide de l’interface d’E/S directes (option `O_DIRECT`).
Si `min_merge_bytes_to_use_direct_io = 0`, les E/S directes sont désactivées.

<div id="min_parts_to_merge_at_once">
  ## min_parts_to_merge_at_once
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Nombre minimal de parties de données que le sélecteur de fusion peut retenir pour une fusion en une seule fois
(paramètre de niveau expert, ne le modifiez pas si vous ne comprenez pas ce qu&#39;il fait).
0 - désactivé. Fonctionne avec les sélecteurs de fusion Simple et StochasticSimple.

<div id="min_replicated_logs_to_keep">
  ## min_replicated_logs_to_keep
</div>

<SettingsInfoBlock type="UInt64" default_value="10" />

Conserve environ ce nombre des derniers enregistrements dans le journal de ZooKeeper, même s’ils sont obsolètes. Cela n’affecte pas le fonctionnement des tables : ce paramètre est utilisé uniquement pour diagnostiquer le journal de ZooKeeper avant son nettoyage.

Valeurs possibles :

* Tout entier positif.
