  ## escape_index_filenames
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "1"},{"label": "Échapper les caractères non ASCII dans les noms de fichiers créés pour les index"}]}]} />

Avant la version 26.1, nous n&#39;échappions pas les caractères spéciaux dans les noms de fichiers créés pour les index secondaires, ce qui pouvait poser problème lorsque certains
caractères dans les noms d&#39;index entraînaient la création de parts corrompues. Ce paramètre a été ajouté uniquement pour des raisons de compatibilité. Il ne doit pas être modifié, sauf si vous
lisez d&#39;anciennes parts avec des index dont le nom contient des caractères non ASCII.
