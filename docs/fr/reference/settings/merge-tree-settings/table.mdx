<div id="table_readonly">
  ## table_readonly
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nouveau paramètre permettant de marquer la table en lecture seule, afin d’empêcher les insertions et les modifications"}]}]} />

Si elle est définie sur true, la table est en mode lecture seule et n’effectue aucune modification sur le disque.

Toutes les opérations au premier plan qui modifieraient la table sont rejetées : insertions, mutations, `OPTIMIZE` et les commandes de partition qui modifient les données
(`ATTACH`/`MOVE`/`DROP`/`DROP DETACHED`/`FETCH`/`REPLACE PARTITION`, ainsi que `MOVE PARTITION ... TO TABLE` ciblant cette table). Les opérations
qui ne modifient pas les données de la table, telles que `FREEZE`/`UNFREEZE` et `FORGET PARTITION`, restent autorisées.

Aucun travail en arrière-plan n’est non plus planifié : les merges réguliers, les merges TTL (`DELETE`/`MOVE`/recompression), les merges de recompression, les mutations en arrière-plan,
et les déplacements de parts en arrière-plan sont tous désactivés. Par conséquent, une table avec un TTL ne récupère plus et ne déplace plus ses données expirées tant que ce paramètre
est activé.

Le paramètre peut toujours être rétabli avec `ALTER TABLE ... MODIFY SETTING table_readonly = 0` (ou `RESET SETTING`). Il n’est pas pris en charge pour `ReplicatedMergeTree`.
