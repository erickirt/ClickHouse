<div id="max_delay_to_insert">
  ## max_delay_to_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1" />

Valeur en secondes utilisée pour calculer le délai de `INSERT` si le
nombre de parties actives au sein d&#39;une même partition dépasse la
valeur [parts&#95;to&#95;delay&#95;insert](/fr/reference/settings/merge-tree-settings/parts-to#parts_to_delay_insert).

Valeurs possibles :

* Tout entier positif.

Le délai (en millisecondes) de `INSERT` est calculé à l&#39;aide de la formule :

```code
max_k = parts_to_throw_insert - parts_to_delay_insert
k = 1 + parts_count_in_partition - parts_to_delay_insert
delay_milliseconds = pow(max_delay_to_insert * 1000, k / max_k)
```

Par exemple, si une partition comporte 299 parties actives et que parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1, l’opération `INSERT` est
retardée de `pow( 1 * 1000, (1 + 299 - 150) / (300 - 150) ) = 1000`
millisecondes.

À partir de la version 23.1, la formule devient :

```code
allowed_parts_over_threshold = parts_to_throw_insert - parts_to_delay_insert
parts_over_threshold = parts_count_in_partition - parts_to_delay_insert + 1
delay_milliseconds = max(min_delay_to_insert_ms, (max_delay_to_insert * 1000)
* parts_over_threshold / allowed_parts_over_threshold)
```

Par exemple, si une partition comporte 224 parts actives et que parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1,
min&#95;delay&#95;to&#95;insert&#95;ms = 10, `INSERT` est différé de `max( 10, 1 * 1000 *
(224 - 150 + 1) / (300 - 150) ) = 500` millisecondes.
