<div id="dead_blobs_to_delay_insert">
  ## dead_blobs_to_delay_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "100000"},{"label": "Nouveau paramètre permettant de ralentir artificiellement les insertions lorsque les files d’attente de blobs morts des disques de la table accumulent trop de blobs en attente de suppression."}]}]} />

Si le nombre de blobs en attente de suppression dans les files d’attente de blobs morts des disques de la table dépasse la
valeur `dead_blobs_to_delay_insert`, un `INSERT` est artificiellement ralenti (jusqu&#39;à `max_delay_to_insert`).

La file d’attente de blobs morts appartient au disque et est partagée par toutes les tables qu’il contient (y compris les blobs de tables déjà
supprimées) ; dimensionnez donc le seuil pour l’ensemble du disque plutôt que pour une seule table.

<Tip>
  Utile lorsqu’un serveur ne parvient pas à nettoyer les blobs assez rapidement.
</Tip>

Valeurs possibles :

* Tout entier positif.
* 0 — désactivé.

<div id="dead_blobs_to_throw_insert">
  ## dead_blobs_to_throw_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1000000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1000000"},{"label": "Nouveau paramètre permettant de rejeter les insertions lorsque les files d’attente de blobs morts des disques de la table accumulent trop de blobs en attente de suppression."}]}]} />

Si le nombre de blobs en attente de suppression dans les files d’attente de blobs morts des disques de la table dépasse la
valeur `dead_blobs_to_throw_insert`, l’opération `INSERT` est interrompue avec l’erreur suivante :

> &quot;Trop de dead blobs dans la file d’attente (N) sur les disques de la table. Le nettoyage des blobs est nettement
> plus lent que les insertions&quot;

La file d’attente de blobs morts appartient au disque et est partagée par toutes les tables qui s’y trouvent (y compris les blobs de tables déjà
supprimées). Dimensionnez donc le seuil pour l’ensemble du disque plutôt que pour une seule table.

Valeurs possibles :

* Tout entier positif.
* 0 — désactivé.
