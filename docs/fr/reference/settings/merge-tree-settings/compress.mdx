<div id="compress_per_column_in_compact_parts">
  ## compress_per_column_in_compact_parts
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "1"},{"label": "Nouveau paramètre"}]}]} />

Contrôle l&#39;organisation physique des parties compactes. Si true (par défaut), chaque colonne d&#39;une granule
commence dans un nouveau bloc compressé, ce qui permet à ClickHouse d&#39;éviter de lire des colonnes inutiles
depuis le disque. Si false, toutes les colonnes d&#39;une granule sont regroupées dans le même bloc compressé,
ce qui améliore le taux de compression, mais nécessite de décompresser davantage de données lors des lectures.
Ce paramètre est avantageux pour les charges de travail qui lisent toujours toutes les colonnes (par ex. les projections).

  ## compress_primary_key
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

La compression de la clé primaire est prise en charge, ce qui réduit la taille du fichier de clé primaire et accélère
la transmission sur le réseau.
