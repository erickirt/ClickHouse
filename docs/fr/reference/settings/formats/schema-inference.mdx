<div id="schema_inference_hints">
  ## schema_inference_hints
</div>

La liste des noms de colonnes et des types à utiliser comme indications pour l’inférence du schéma dans les formats sans schéma.

Exemple :

Requête :

```sql
desc format(JSONEachRow, '{"x" : 1, "y" : "String", "z" : "0.0.0.0" }') settings schema_inference_hints='x UInt8, z IPv4';
```

Résultat :

```sql
x   UInt8
y   Nullable(String)
z   IPv4
```

<Note>
  Si `schema_inference_hints` n’est pas correctement formaté, ou s’il contient une faute de frappe, un type de données erroné, etc., l’intégralité de `schema&#95;inference&#95;hints` sera ignorée.
</Note>

<div id="schema_inference_make_columns_nullable">
  ## schema_inference_make_columns_nullable
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="3" />

Détermine si les types inférés doivent être rendus `Nullable` lors de l&#39;inférence du schéma.
Valeurs possibles :

* 0 - le type inféré ne sera jamais `Nullable` (utilisez input&#95;format&#95;null&#95;as&#95;default pour définir le comportement à adopter avec les valeurs nulles dans ce cas),
* 1 - tous les types inférés seront `Nullable`,
* 2 ou `auto` - le type inféré sera `Nullable` uniquement si la colonne contient `NULL` dans un échantillon analysé lors de l&#39;inférence du schéma, ou si les métadonnées du fichier contiennent des informations sur la nullabilité de la colonne,
* 3 - la nullabilité du type inféré correspondra aux métadonnées du fichier si le format la prend en charge (par exemple Parquet), et sera sinon toujours `Nullable` (par exemple CSV).

<div id="schema_inference_make_json_columns_nullable">
  ## schema_inference_make_json_columns_nullable
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Contrôle si les types JSON inférés sont rendus `Nullable` lors de l&#39;inférence du schéma.
Si ce paramètre est activé en même temps que schema&#95;inference&#95;make&#95;columns&#95;nullable, le type JSON inféré sera `Nullable`.

<div id="schema_inference_mode">
  ## schema_inference_mode
</div>

<SettingsInfoBlock type="SchemaInferenceMode" default_value="default" />

Mode d’inférence du schéma. &#39;default&#39; : considère que tous les fichiers ont le même schéma et que celui-ci peut être inféré à partir de n’importe quel fichier ; &#39;union&#39; : les fichiers peuvent avoir des schémas différents et le schéma résultant doit être l’union des schémas de tous les fichiers
