<div id="check_conversion_from_numbers_to_enum">
  ## check_conversion_from_numbers_to_enum
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Lève une exception lors de la conversion de Numbers en Enum si la valeur n&#39;existe pas dans l&#39;Enum.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

**Exemple**

```text
CREATE TABLE tab (
  val Enum('first' = 1, 'second' = 2, 'third' = 3)
) ENGINE = Memory;

INSERT INTO tab SETTINGS check_conversion_from_numbers_to_enum = 1 VALUES (4); -- returns an error
```

<div id="insert_distributed_one_random_shard">
  ## insert_distributed_one_random_shard
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive l&#39;insertion sur un shard aléatoire dans une table [Distributed](/fr/reference/engines/table-engines/special/distributed) lorsqu&#39;aucune clé de distribution n&#39;est définie.

Par défaut, lors de l&#39;insertion de données dans une table `Distributed` comportant plus d&#39;un shard, le serveur ClickHouse rejette toute requête d&#39;insertion si aucune clé de distribution n&#39;est définie. Lorsque `insert_distributed_one_random_shard = 1`, les insertions sont autorisées et les données sont réparties aléatoirement entre tous les shards.

Valeurs possibles :

* 0 — L&#39;insertion est rejetée s&#39;il y a plusieurs shards et qu&#39;aucune clé de distribution n&#39;est définie.
* 1 — L&#39;insertion est effectuée aléatoirement parmi tous les shards disponibles lorsqu&#39;aucune clé de distribution n&#39;est définie.

<div id="interval_output_format">
  ## interval_output_format
</div>

<SettingsInfoBlock type="IntervalOutputFormat" default_value="numeric" />

Permet de choisir différents formats de sortie pour la représentation textuelle des types `Interval`.

Valeurs possibles :

* `kusto` - Format de sortie de style KQL.

  ClickHouse affiche les intervalles au [format KQL](https://learn.microsoft.com/en-us/dotnet/standard/base-types/standard-timespan-format-strings#the-constant-c-format-specifier). Par exemple, `toIntervalDay(2)` serait affiché sous la forme `2.00:00:00`. Notez que, pour les types `Interval` de longueur variable (c.-à-d. `IntervalMonth` et `IntervalYear`), le nombre moyen de secondes par intervalle est pris en compte.

* `numeric` - Format de sortie numérique.

  ClickHouse affiche les intervalles selon leur représentation numérique sous-jacente. Par exemple, `toIntervalDay(2)` serait affiché sous la forme `2`.

Voir aussi :

* [Interval](/fr/reference/data-types/special-data-types/interval)

<div id="precise_float_parsing">
  ## precise_float_parsing
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Utilise l’algorithme d’analyse précise des nombres à virgule flottante, qui renvoie toujours la valeur représentable la plus proche de l’entrée. Lorsqu’il est désactivé, un algorithme plus rapide mais moins précis est utilisé, et le résultat peut alors différer du résultat précis sur les bits de poids faible.
