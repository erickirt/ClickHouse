  ## deduplicate_insert
</div>

<SettingsInfoBlock type="DeduplicateInsertMode" default_value="enable" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "enable"},{"label": "Active la déduplication pour toutes les insertions synchrones et asynchrones par défaut."}]}]} />

Active ou désactive la déduplication des blocs pour `INSERT INTO` (pour les tables Replicated*).
Ce paramètre remplace les paramètres `insert_deduplicate` et `async_insert_deduplicate`.
Ce paramètre a trois valeurs possibles :

* disable — La déduplication est désactivée pour la requête `INSERT INTO`.
* enable — La déduplication est activée pour la requête `INSERT INTO`.
* backward&#95;compatible&#95;choice — La déduplication est activée si `insert_deduplicate` ou `async_insert_deduplicate` sont activés pour le type d&#39;insertion concerné.

<SettingsInfoBlock type="DeduplicateInsertSelectMode" default_value="enable_when_possible" />

Active ou désactive la déduplication des blocs de `INSERT SELECT` (pour les tables Replicated*).
Ce paramètre remplace `insert_deduplicate` et `deduplicate_insert` pour les requêtes `INSERT SELECT`.
Ce paramètre a quatre valeurs possibles :

* disable — La déduplication est désactivée pour les requêtes `INSERT SELECT`.
* force&#95;enable — La déduplication est activée pour les requêtes `INSERT SELECT`. Si le résultat du SELECT n&#39;est pas stable, une exception est levée.
* enable&#95;when&#95;possible — La déduplication est activée si `insert_deduplicate` est activé et que le résultat du SELECT est stable, sinon elle est désactivée.
* enable&#95;even&#95;for&#95;bad&#95;queries - La déduplication est activée si `insert_deduplicate` est activé. Si le résultat du SELECT n&#39;est pas stable, un avertissement est consigné, mais la requête est exécutée avec déduplication. Cette option est conservée pour assurer la compatibilité ascendante. Envisagez plutôt d&#39;utiliser d&#39;autres options, car celle-ci peut entraîner des résultats inattendus.
