<div id="query_cache_compress_entries">
  ## query_cache_compress_entries
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Compresse les entrées du [cache de requêtes](/fr/concepts/features/performance/caches/query-cache). Réduit la consommation de mémoire du cache de requêtes au prix d’insertions et de lectures plus lentes.

Valeurs possibles :

* 0 - Désactivé
* 1 - Activé

<div id="query_cache_max_entries">
  ## query_cache_max_entries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Le nombre maximal de résultats de requête que l’utilisateur courant peut stocker dans le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache). 0 signifie qu’il n’y a pas de limite.

Valeurs possibles :

* Entier positif &gt;= 0.

Quantité maximale de mémoire (en octets) que l’utilisateur courant peut allouer dans le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache). 0 signifie illimité.

Valeurs possibles :

* Entier &gt;= 0.

Nombre minimal d’exécutions d’une requête `SELECT` avant que son résultat soit stocké dans le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache).

<div id="query_cache_nondeterministic_function_handling">
  ## query_cache_nondeterministic_function_handling
</div>

<SettingsInfoBlock type="QueryResultCacheNondeterministicFunctionHandling" default_value="throw" />

Contrôle la façon dont le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache) gère les requêtes `SELECT` comportant des fonctions non déterministes comme `rand()` ou `now()`.

Valeurs possibles :

* `'throw'` - Lever une exception et ne pas mettre en cache le résultat de la requête.
* `'save'` - Mettre en cache le résultat de la requête.
* `'ignore'` - Ne pas mettre en cache le résultat de la requête et ne pas lever d&#39;exception.

<div id="query_cache_share_between_users">
  ## query_cache_share_between_users
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Si ce paramètre est activé, le résultat des requêtes `SELECT` mises en cache dans le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache) peut être lu par d&#39;autres utilisateurs.
Il n&#39;est pas recommandé d&#39;activer ce paramètre pour des raisons de sécurité.

Valeurs possibles :

* 0 - Désactivé
* 1 - Activé

<div id="query_cache_squash_partial_results">
  ## query_cache_squash_partial_results
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Fusionne les blocs de résultats partiels en blocs de taille [max&#95;block&#95;size](/fr/reference/settings/session-settings/max#max_block_size). Réduit les performances lors des insertions dans le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache), mais améliore la compressibilité des entrées du cache (voir [query&#95;cache&#95;compress-entries](/fr/reference/settings/session-settings/query-cache#query_cache_compress_entries)).

Valeurs possibles :

* 0 - Désactivé
* 1 - Activé

<div id="query_cache_system_table_handling">
  ## query_cache_system_table_handling
</div>

<SettingsInfoBlock type="QueryResultCacheSystemTableHandling" default_value="throw" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.4"},{"label": "throw"},{"label": "Le cache de requêtes ne met plus en cache les résultats des requêtes effectuées sur les tables système"}]}]} />

Contrôle la façon dont le [cache de requêtes](/fr/concepts/features/performance/caches/query-cache) gère les requêtes `SELECT` sur les tables système, c.-à-d. les tables des bases de données `system.*` et `information_schema.*`.

<div id="query_cache_tag">
  ## query_cache_tag
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.8"},{"label": ""},{"label": "Nouveau paramètre pour ajouter un libellé aux paramètres du cache de requêtes."}]}]} />
