  ## cluster_function_process_archive_on_multiple_nodes
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.7"},{"label": "1"},{"label": "Nouveau paramètre"}]}]} />

Si cette option est définie sur `true`, elle améliore les performances du traitement des archives dans les fonctions de cluster. Elle doit être définie sur `false` pour assurer la compatibilité et éviter les erreurs lors d’une mise à niveau vers la version 25.7 ou ultérieure si vous utilisez des fonctions de cluster avec des archives sur des versions antérieures.
