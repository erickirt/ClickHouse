<div id="aggregate_function_input_format">
  ## aggregate_function_input_format
</div>

<SettingsInfoBlock type="AggregateFunctionInputFormat" default_value="state" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.12"},{"label": "state"},{"label": "Nouveau paramètre permettant de contrôler le format d'entrée d'AggregateFunction lors des opérations INSERT. La valeur du paramètre est définie sur state par défaut"}]}]} />

Format de l&#39;entrée d&#39;AggregateFunction lors des opérations INSERT.

Valeurs possibles :

* `state` — Chaîne binaire contenant l&#39;état sérialisé (par défaut). Il s&#39;agit du comportement par défaut, dans lequel les valeurs AggregateFunction sont attendues sous forme de données binaires.
* `value` — Le format attend une seule valeur pour l&#39;argument de la fonction d&#39;agrégation ou, en cas d&#39;arguments multiples, un tuple de ces valeurs. Elles seront désérialisées à l&#39;aide de l&#39;IDataType ou du DataTypeTuple correspondant, puis agrégées pour former l&#39;état.
* `array` — Le format attend un Array de valeurs, comme décrit ci-dessus pour l&#39;option `value`. Tous les éléments du tableau seront agrégés pour former l&#39;état.

**Exemples**

Pour une table de structure :

```sql
CREATE TABLE example (
    user_id UInt64,
    avg_session_length AggregateFunction(avg, UInt32)
);
```

Avec `aggregate_function_input_format = 'value'` :

```sql
INSERT INTO example FORMAT CSV
123,456
```

Avec `aggregate_function_input_format = 'array'` :

```sql
INSERT INTO example FORMAT CSV
123,"[456,789,101]"
```

Remarque : les formats `value` et `array` sont plus lents que le format `state`, qui est le format par défaut, car ils nécessitent de créer et d’agréger des valeurs lors de l’insertion.

<div id="aggregate_functions_null_for_empty">
  ## aggregate_functions_null_for_empty
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive la réécriture de toutes les fonctions d’agrégation d’une requête, en leur ajoutant le suffixe [-OrNull](/fr/reference/functions/aggregate-functions/combinators#-ornull). Activez ce paramètre pour assurer la compatibilité avec la norme SQL.
Cette fonctionnalité est implémentée via une réécriture de requête (semblable au paramètre [count&#95;distinct&#95;implementation](/fr/reference/settings/session-settings/count-distinct#count_distinct_implementation)) afin d’obtenir des résultats cohérents pour les requêtes distribuées.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

**Exemple**

Considérez la requête suivante avec des fonctions d’agrégation :

```sql
SELECT SUM(-1), MAX(0) FROM system.one WHERE 0;
```

Avec `aggregate_functions_null_for_empty = 0`, on obtiendrait :

```text
┌─SUM(-1)─┬─MAX(0)─┐
│       0 │      0 │
└─────────┴────────┘
```

Avec `aggregate_functions_null_for_empty = 1`, le résultat serait :

```text
┌─SUMOrNull(-1)─┬─MAXOrNull(0)─┐
│          NULL │         NULL │
└───────────────┴──────────────┘
```
