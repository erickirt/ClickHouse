Définit comment les types MySQL sont convertis en types ClickHouse correspondants. Il s&#39;agit d&#39;une liste séparée par des virgules, contenant n&#39;importe quelle combinaison de `decimal`, `datetime64`, `date2Date32`, `date2String` ou `geometry`. Tous les mappages modernes (`decimal`, `datetime64`, `date2Date32`, `geometry`) sont activés par défaut.

* `decimal` : convertit les types `NUMERIC` et `DECIMAL` en `Decimal` lorsque la précision le permet.
* `datetime64` : convertit les types `DATETIME` et `TIMESTAMP` en `DateTime64` au lieu de `DateTime` lorsque la précision n&#39;est pas `0`.
* `date2Date32` : convertit `DATE` en `Date32` au lieu de `Date`. Prend préséance sur `date2String`.
* `date2String` : convertit `DATE` en `String` au lieu de `Date`. Est remplacé par `datetime64`.
* `geometry` : convertit les types spatiaux de MySQL (`LINESTRING`, `POLYGON`, `MULTILINESTRING`, `MULTIPOLYGON`) en types géométriques ClickHouse correspondants, et le type générique `GEOMETRY` en type englobant `Geometry`. `POINT` est toujours converti en `Point`, indépendamment de cette option. Comme une colonne `GEOMETRY` générique peut contenir n&#39;importe quel sous-type, la lecture d&#39;une valeur dont le sous-type n&#39;a pas d&#39;équivalent dans ClickHouse (`MULTIPOINT`, `GEOMETRYCOLLECTION`) déclenche une exception à la lecture ; les colonnes déclarées comme `MULTIPOINT` ou `GEOMETRYCOLLECTION` sont mappées à `String`.
