  ## jemalloc_collect_profile_samples_in_trace_log

<div id="jemalloc_enable_profiler">
  ## jemalloc_enable_profiler
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "0"},{"label": "Nouveau paramètre"}]}]} />

Active le profiler jemalloc pour la requête. Jemalloc échantillonne les allocations ainsi que toutes les désallocations associées aux allocations échantillonnées.
Les profils peuvent être vidés à l’aide de SYSTEM JEMALLOC FLUSH PROFILE afin d’analyser les allocations.
Les échantillons peuvent également être stockés dans system.trace&#95;log à l’aide de la config jemalloc&#95;collect&#95;global&#95;profile&#95;samples&#95;in&#95;trace&#95;log ou du paramètre de requête jemalloc&#95;collect&#95;profile&#95;samples&#95;in&#95;trace&#95;log.
Voir [Profilage des allocations](/fr/concepts/features/performance/allocation-profiling)
