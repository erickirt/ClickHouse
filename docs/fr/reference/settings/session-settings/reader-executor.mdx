  ## reader_executor_max_tail_for_drain
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="1048576" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1048576"},{"label": "Nouveau paramètre expérimental de ReaderExecutor : limite de vidage pour achever une connexion longue interrompue."}]}]} />

Limite de vidage pour le `ReaderExecutor` expérimental : une connexion source longue interrompue à moins de ce nombre d’octets de sa borne droite est d’abord lue jusqu’à cette borne, afin qu’elle puisse se terminer et revenir dans le pool de connexions en étant réutilisable, au lieu d’être comptabilisée comme connexion incomplète.

<div id="reader_executor_min_bytes_for_seek">
  ## reader_executor_min_bytes_for_seek
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="2097152" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "2097152"},{"label": "Nouveau paramètre expérimental de ReaderExecutor : seuil d’écart vers l’avant pour le franchissement sur une connexion source maintenue ouverte."}]}]} />

Seuil d’écart vers l’avant pour le `ReaderExecutor` expérimental : un écart jusqu’à cette valeur est ignoré sur la connexion source ouverte (franchi / lu en continu) au lieu d’effectuer une lecture distincte sur la source ou de rouvrir la connexion. Définissez-le près du point d’équilibre entre le coût en débit et le coût des requêtes afin que ce franchissement reste rentable.

<div id="reader_executor_use_long_connections">
  ## reader_executor_use_long_connections
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Nouveau paramètre expérimental de ReaderExecutor (désactivé par défaut) : réutilise une connexion source conservée sur plusieurs fenêtres séquentielles."}]}]} />

Réutilise, dans le `ReaderExecutor` expérimental, une connexion source longue et bornée d&#39;une fenêtre à l&#39;autre. Une connexion longue est une connexion dont la plage dépasse la fenêtre de lecture actuelle ; lorsqu&#39;elle est désactivée, l&#39;exécuteur ne consomme aucun budget du pool de connexions et chaque fenêtre ouvre une connexion éphémère à usage unique (mode sans état).
