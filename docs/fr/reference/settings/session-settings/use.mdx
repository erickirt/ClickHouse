<div id="use_constant_folding_in_index_analysis">
  ## use_constant_folding_in_index_analysis
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Nouveau paramètre permettant de substituer, pour chaque part, les constantes au niveau des partitions dans le prédicat de filtre lors de l'analyse des index MergeTree, afin d'améliorer l'élagage des filtres dont les branches dépendent des valeurs de partition."}]}]} />

Substitue les constantes au niveau des partitions dans le prédicat de filtre lors de l&#39;analyse, pour chaque part, de la clé primaire et des skip indexes.

Lorsque la clé de partition apparaît dans le filtre avec des colonnes de clé primaire ou de skip index, cela permet à l&#39;analyse des index de substituer séparément la valeur de partition dans chaque part. C&#39;est particulièrement utile pour les filtres disjonctifs dont les branches ciblent des partitions différentes. Par exemple, avec `PARTITION BY a` et `ORDER BY b` :

```sql
SELECT * FROM t WHERE (a = 1 AND b >= 1) OR (a = 2 AND b > 10) OR (a = 3 AND b > 10)
```

Pour la part de la partition `a = 1`, la condition se simplifie en `b >= 1`, tandis que pour les partitions `a = 2` et `a = 3`, elle se simplifie en `b > 10`, de sorte que chaque part est analysée avec le prédicat qui lui correspond réellement.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

<div id="use_strict_insert_block_limits">
  ## use_strict_insert_block_limits
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "Nouveau paramètre permettant d’appliquer strictement les bornes min et max du bloc d’insertion lors des insertions. Lorsque min < max, les limites max ont préséance."}]}]} />

Lorsqu’il est activé, ce paramètre applique strictement les limites minimale et maximale de taille du bloc d’insertion.

Un bloc est émis lorsque :

* Seuils min (AND) : min&#95;insert&#95;block&#95;size&#95;rows ET min&#95;insert&#95;block&#95;size&#95;bytes sont tous deux atteints.
* Seuils max (OR) : max&#95;insert&#95;block&#95;size&#95;rows OU max&#95;insert&#95;block&#95;size&#95;bytes est atteint.

Lorsqu’il est désactivé, un bloc est émis lorsque :

* Seuils min (OR) : min&#95;insert&#95;block&#95;size&#95;rows OU min&#95;insert&#95;block&#95;size&#95;bytes est atteint.

**Remarque** : si les paramètres max sont inférieurs aux paramètres min, les limites max ont préséance et les blocs seront émis avant que les seuils min ne soient atteints.

**Remarque** : ce paramètre est automatiquement désactivé pour les inserts asynchrones, car les inserts asynchrones associent un token de déduplication à chaque entrée, ce qui est incompatible avec le fractionnement des blocs nécessaire à l’application de limites strictes.

Désactivé par défaut.

<div id="use_uncompressed_cache">
  ## use_uncompressed_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Indique s’il faut utiliser un cache de blocs non compressés. Accepte 0 ou 1. Par défaut, 0 (désactivé).
L’utilisation du cache non compressé (uniquement pour les tables de la famille MergeTree) peut réduire considérablement la latence et augmenter le débit lorsqu’un grand nombre de requêtes courtes sont exécutées. Activez ce paramètre pour les utilisateurs qui envoient fréquemment des requêtes courtes. Tenez également compte du paramètre de configuration [uncompressed&#95;cache&#95;size](/fr/reference/settings/server-settings/settings#uncompressed_cache_size) (défini uniquement dans le fichier de configuration) — la taille des blocs du cache non compressé. Par défaut, elle est de 8 Gio. Le cache non compressé se remplit selon les besoins et les données les moins utilisées sont automatiquement supprimées.

Pour les requêtes qui lisent un volume de données déjà important (un million de lignes ou plus), le cache non compressé est automatiquement désactivé afin de réserver l’espace aux requêtes véritablement petites. Cela signifie que vous pouvez laisser le paramètre &#39;use&#95;uncompressed&#95;cache&#39; toujours défini sur 1.
