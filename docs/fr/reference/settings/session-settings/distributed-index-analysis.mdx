  ## distributed_index_analysis
</div>

<div id="distributed_index_analysis_only_on_coordinator">
  ## distributed_index_analysis_only_on_coordinator
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "Nouveau paramètre."}]}]} />

S&#39;il est activé, l&#39;analyse distribuée des index s&#39;exécute uniquement sur le coordinateur.
Cela évite de générer des requêtes en O(N^2) lorsque le prédicat contient des sous-requêtes (par ex. `IN (SELECT ...)`),
car chaque réplique suiveuse déclencherait sinon de manière indépendante sa propre analyse distribuée des index,
mais cela rend l&#39;analyse distribuée des index moins efficace lorsque de grandes tables sont utilisées dans les sous-requêtes.
