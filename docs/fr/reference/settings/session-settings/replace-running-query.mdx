<div id="replace_running_query">
  ## replace_running_query
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Lorsque vous utilisez l&#39;interface HTTP, vous pouvez transmettre le paramètre &#39;query&#95;id&#39;. Il s&#39;agit d&#39;une chaîne quelconque servant d&#39;identifiant de requête.
Si une requête du même utilisateur avec le même &#39;query&#95;id&#39; existe déjà à ce moment-là, le comportement dépend du paramètre &#39;replace&#95;running&#95;query&#39;.

`0` (par défaut) – Lever une exception (ne pas autoriser l&#39;exécution de la requête si une requête avec le même &#39;query&#95;id&#39; est déjà en cours d&#39;exécution).

`1` – Annuler l&#39;ancienne requête et lancer la nouvelle.

Définissez ce paramètre sur 1 pour implémenter des suggestions pour les conditions de segmentation. Après la saisie du caractère suivant, si l&#39;ancienne requête n&#39;est pas encore terminée, elle doit être annulée.

<div id="replace_running_query_max_wait_ms">
  ## replace_running_query_max_wait_ms
</div>

<SettingsInfoBlock type="Millisecondes" default_value="5000" />

Temps d&#39;attente avant la fin d&#39;exécution de la requête ayant le même `query_id`, lorsque le paramètre [replace&#95;running&#95;query](/fr/reference/settings/session-settings/replace-running-query#replace_running_query) est actif.

Valeurs possibles :

* Entier positif.
* 0 — Déclenche une exception, ce qui empêche d&#39;exécuter une nouvelle requête si le serveur exécute déjà une requête avec le même `query_id`.
