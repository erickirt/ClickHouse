  ## jemalloc_profile_text_collapsed_use_count
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "Nouveau paramètre permettant d’agréger par nombre d’allocations plutôt que par octets dans le format collapsed du profil du tas jemalloc"}]}]} />

Lors de l’utilisation du format de sortie &#39;collapsed&#39; pour le profil du tas jemalloc, agrégez par nombre d’allocations plutôt que par octets. Lorsque false (par défaut), chaque pile d’appels est pondérée par les octets actifs ; lorsque true, par le nombre d’allocations actives.

  ## jemalloc_profile_text_symbolize_with_inline
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "1"},{"label": "Nouveau paramètre permettant de contrôler l’inclusion des frames inline lors de la symbolisation du profil du tas jemalloc. Détermine s’il faut inclure les frames inline lors de la symbolisation du profil du tas jemalloc. Lorsqu’il est activé, les frames inline sont incluses, ce qui peut considérablement ralentir le processus de symbolisation ; lorsqu’il est désactivé, elles sont ignorées. N’affecte que les formats de sortie &#39;symbolized&#39; et &#39;collapsed&#39;.
