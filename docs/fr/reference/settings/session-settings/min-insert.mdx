<div id="min_insert_block_size_bytes">
  ## min_insert_block_size_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="268402944" />

Taille minimale des blocs (en octets) à former pour l’insertion dans une table.

Ce paramètre fonctionne avec min&#95;insert&#95;block&#95;size&#95;rows et contrôle la formation des blocs dans les mêmes contextes (analyse du format et opérations INSERT). Voir min&#95;insert&#95;block&#95;size&#95;rows pour plus de détails sur le moment et la manière dont ces paramètres s’appliquent.

Valeurs possibles :

* Entier positif.
* 0 — le paramètre n’intervient pas dans la formation des blocs.

<div id="min_insert_block_size_bytes_for_materialized_views">
  ## min_insert_block_size_bytes_for_materialized_views
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Définit le nombre minimal d’octets dans le bloc pouvant être inséré dans une table par une requête `INSERT`. Les blocs plus petits sont regroupés en blocs plus grands. Ce paramètre s’applique uniquement aux blocs insérés dans une [vue matérialisée](/fr/reference/statements/create/view). En ajustant ce paramètre, vous contrôlez le regroupement des blocs lors de l’insertion dans une vue matérialisée et évitez une utilisation excessive de la mémoire.

Valeurs possibles :

* Tout entier positif.
* 0 — Regroupement désactivé.

<div id="min_insert_block_size_rows">
  ## min_insert_block_size_rows
</div>

<SettingsInfoBlock type="UInt64" default_value="1048449" />

Taille minimale des blocs (en lignes) à former pour l&#39;insertion dans une table.

Ce paramètre contrôle la formation des blocs dans deux contextes :

1. Analyse des formats : lorsque le serveur analyse des formats d&#39;entrée orientés lignes (CSV, TSV, JSONEachRow, etc.) depuis n&#39;importe quelle interface (HTTP, clickhouse-client avec des données intégrées, gRPC, PostgreSQL wire protocol), les blocs sont générés lorsque :

   * Les deux seuils min&#95;insert&#95;block&#95;size&#95;rows AND min&#95;insert&#95;block&#95;size&#95;bytes sont atteints, OR
   * L&#39;un des deux seuils max&#95;insert&#95;block&#95;size&#95;rows OR max&#95;insert&#95;block&#95;size&#95;bytes est atteint

   Remarque : lorsque vous utilisez clickhouse-client ou clickhouse-local pour lire depuis un fichier, le client analyse lui-même les données et ce paramètre s&#39;applique côté client.

2. Opérations INSERT : pendant les requêtes INSERT et lorsque les données transitent par des vues matérialisées, le comportement de ce paramètre dépend de `use_strict_insert_block_limits` :

   * Lorsqu&#39;il est activé : les blocs sont générés lorsque :
     * Seuils min (AND) : les deux seuils min&#95;insert&#95;block&#95;size&#95;rows AND min&#95;insert&#95;block&#95;size&#95;bytes sont atteints
     * Seuils max (OR) : l&#39;un des deux seuils max&#95;insert&#95;block&#95;size&#95;rows OR max&#95;insert&#95;block&#95;size&#95;bytes est atteint

   * Lorsqu&#39;il est désactivé (par défaut) : les blocs sont générés lorsque min&#95;insert&#95;block&#95;size&#95;rows OR min&#95;insert&#95;block&#95;size&#95;bytes est atteint. Les paramètres max&#95;insert&#95;block&#95;size ne sont pas appliqués.

Valeurs possibles :

* Entier positif.
* 0 — le paramètre ne participe pas à la formation des blocs.

  ## min_insert_block_size_rows_for_materialized_views
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Définit le nombre minimal de lignes qu’un bloc doit contenir pour pouvoir être inséré dans une table par une requête `INSERT`. Les blocs plus petits sont regroupés en blocs plus grands. Ce paramètre s’applique uniquement aux blocs insérés dans une [vue matérialisée](/fr/reference/statements/create/view). En l’ajustant, vous contrôlez le regroupement des blocs lors de l’insertion dans la vue matérialisée et évitez une utilisation excessive de la mémoire.

Valeurs possibles :

* 0 — Regroupement désactivé.
