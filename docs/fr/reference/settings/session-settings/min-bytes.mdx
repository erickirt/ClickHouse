<div id="min_bytes_to_use_direct_io">
  ## min_bytes_to_use_direct_io
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Le volume minimal de données nécessaire pour utiliser les E/S directes sur le disque de stockage.

ClickHouse utilise ce paramètre lors de la lecture des données des tables. Si le volume total de toutes les données à lire dépasse `min_bytes_to_use_direct_io` octets, ClickHouse lit alors les données sur le disque de stockage avec l’option `O_DIRECT`.

Valeurs possibles :

* 0 — Les E/S directes sont désactivées.
* Entier positif.

<div id="min_bytes_to_use_mmap_io">
  ## min_bytes_to_use_mmap_io
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Il s’agit d’un paramètre expérimental. Définit la taille minimale à partir de laquelle les gros fichiers sont lus sans copie des données du noyau vers l’espace utilisateur. Le seuil recommandé est d’environ 64 Mo, car [mmap/munmap](https://en.wikipedia.org/wiki/Mmap) est lent. Ce paramètre n’a de sens que pour les gros fichiers et n’est utile que si les données se trouvent dans le cache de pages.

Valeurs possibles :

* Entier positif.
* 0 — Les gros fichiers sont lus uniquement avec copie des données du noyau vers l’espace utilisateur.
