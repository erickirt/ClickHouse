Ces paramètres sont disponibles dans [system.settings](/fr/reference/system-tables/settings) et sont générés automatiquement à partir du [code source](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="compatibility">
  ## compatibility
</div>

Le paramètre `compatibility` fait que ClickHouse utilise les paramètres par défaut d’une version antérieure de ClickHouse, spécifiée comme valeur du paramètre.

Si certains paramètres sont définis sur des valeurs autres que les valeurs par défaut, ils sont respectés (seuls les paramètres qui n’ont pas été modifiés sont affectés par le paramètre `compatibility`).

Ce paramètre prend comme valeur un numéro de version ClickHouse sous forme de chaîne, par exemple `22.3` ou `22.8`. Une valeur vide signifie que ce paramètre est désactivé.

Désactivé par défaut.

<Note>
  Dans ClickHouse Cloud, le paramètre `compatibility` par défaut au niveau du service doit être défini par ClickHouse Cloud Support. Veuillez [ouvrir un ticket](https://clickhouse.cloud/support) pour le faire configurer.
  Cependant, le paramètre `compatibility` peut être surchargé au niveau de l’utilisateur, du rôle, du profil, de la requête ou de la session à l’aide des mécanismes standard de configuration de ClickHouse, par exemple `SET compatibility = '22.3'` dans une session ou `SETTINGS compatibility = '22.3'` dans une requête.
</Note>

Compatibilité : lorsqu’elle est activée, les paramètres de requête d’une URL pré-signée (par ex. X-Amz-*) sont intégrés à la clé S3 (comportement legacy),
de sorte que &#39;?&#39; est interprété comme un caractère générique dans le chemin. Lorsqu’elle est désactivée (valeur par défaut), les paramètres de requête de l’URL pré-signée sont conservés dans la requête de l’URL
afin d’éviter d’interpréter &#39;?&#39; comme un caractère générique.
