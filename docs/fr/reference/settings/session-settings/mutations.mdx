<div id="mutations_max_literal_size_to_replace">
  ## mutations_max_literal_size_to_replace
</div>

<SettingsInfoBlock type="UInt64" default_value="16384" />

Taille maximale, en octets, d’un littéral sérialisé à remplacer dans les requêtes `UPDATE` et `DELETE`. Ce paramètre ne prend effet que si au moins l’un des deux paramètres ci-dessus est activé. Valeur par défaut : 16384 (16 Kio).

<div id="mutations_sync">
  ## mutations_sync
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Permet d’exécuter les requêtes `ALTER TABLE ... UPDATE|DELETE|MATERIALIZE INDEX|MATERIALIZE PROJECTION|MATERIALIZE COLUMN|MATERIALIZE STATISTICS` ([mutations](/fr/reference/statements/alter/index#mutations)) de façon synchrone.

Valeurs possibles :

| Valeur | Description                                                                                                                                                                             |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `0`    | Les mutations s’exécutent de manière asynchrone.                                                                                                                                        |
| `1`    | La requête attend que toutes les mutations soient terminées sur le serveur actuel.                                                                                                      |
| `2`    | La requête attend que toutes les mutations soient terminées sur toutes les répliques (si elles existent).                                                                               |
| `3`    | La requête attend uniquement les répliques actives. Pris en charge uniquement pour `SharedMergeTree`. Pour `ReplicatedMergeTree`, le comportement est identique à `mutations_sync = 2`. |
