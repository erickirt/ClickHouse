  ## validate_enum_literals_in_operators

Si cette option est activée, les littéraux enum utilisés dans des opérateurs tels que `IN`, `NOT IN`, `==` et `!=` sont validés par rapport au type enum, et une exception est levée si le littéral n’est pas une valeur enum valide.

  ## validate_mutation_query
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.11"},{"label": "1"},{"label": "Nouveau paramètre qui valide les requêtes de mutation par défaut."}]}]} />

Valider les requêtes de mutation avant de les accepter. Les mutations sont exécutées en arrière-plan, et l&#39;exécution d&#39;une requête invalide entraînera leur blocage, nécessitant une intervention manuelle.

Ne modifiez ce paramètre que si vous rencontrez un bogue non rétrocompatible.

<div id="validate_polygons">
  ## validate_polygons
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "20.4"},{"label": "1"},{"label": "Lever une exception si le polygone est invalide dans la fonction pointInPolygon par défaut, au lieu de renvoyer des résultats potentiellement erronés"}]}]} />

Active ou désactive le déclenchement d&#39;une exception dans la fonction [pointInPolygon](/fr/reference/functions/regular-functions/geo/coordinates#pointinpolygon) si le polygone est auto-intersecté ou présente une tangence à lui-même.

Valeurs possibles :

* 0 — Le déclenchement d&#39;une exception est désactivé. `pointInPolygon` accepte les polygones invalides et peut renvoyer des résultats incorrects.
* 1 — Le déclenchement d&#39;une exception est activé.
