<div id="log_queries">
  ## log_queries
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Active la journalisation des requêtes.

Les requêtes envoyées à ClickHouse avec ce paramètre sont journalisées conformément aux règles du paramètre de configuration du serveur [query&#95;log](/fr/reference/settings/server-settings/settings#query_log).

Exemple :

```text
log_queries=1
```

<div id="log_queries_cut_to_length">
  ## log_queries_cut_to_length
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

Si la longueur de la requête dépasse un seuil spécifié (en octets), la requête est tronquée lors de son écriture dans le query log. La longueur de la requête affichée dans le journal texte standard est également limitée.

<div id="log_queries_min_query_duration_ms">
  ## log_queries_min_query_duration_ms
</div>

<SettingsInfoBlock type="Milliseconds" default_value="0" />

Si ce paramètre est activé (non nul), les requêtes plus rapides que la valeur définie ne seront pas consignées dans le journal (vous pouvez voir cela comme un `long_query_time` pour le [journal des requêtes lentes de MySQL](https://dev.mysql.com/doc/refman/5.7/slow-query-log.html)) ; en pratique, cela signifie que vous ne les trouverez pas dans les tables suivantes :

* `system.query_log`
* `system.query_thread_log`

Seules les requêtes du type suivant seront consignées dans le journal :

* `QUERY_FINISH`

* `EXCEPTION_WHILE_PROCESSING`

* Type : millisecondes

* Valeur par défaut : 0 (toute requête)

<div id="log_queries_min_type">
  ## log_queries_min_type
</div>

<SettingsInfoBlock type="LogQueriesType" default_value="QUERY_START" />

Type minimal de `query_log` à consigner.

Valeurs possibles :

* `QUERY_START` (`=1`)
* `QUERY_FINISH` (`=2`)
* `EXCEPTION_BEFORE_START` (`=3`)
* `EXCEPTION_WHILE_PROCESSING` (`=4`)

Peut être utilisé pour limiter les entrées envoyées à `query_log` ; par exemple, si vous vous intéressez uniquement aux erreurs, vous pouvez utiliser `EXCEPTION_WHILE_PROCESSING` :

```text
log_queries_min_type='EXCEPTION_WHILE_PROCESSING'
```

<div id="log_queries_probability">
  ## log_queries_probability
</div>

<SettingsInfoBlock type="Float" default_value="1" />

Permet à un utilisateur de n’écrire dans les tables système [query&#95;log](/fr/reference/system-tables/query_log), [query&#95;thread&#95;log](/fr/reference/system-tables/query_thread_log) et [query&#95;views&#95;log](/fr/reference/system-tables/query_views_log) qu’un échantillon de requêtes sélectionnées aléatoirement avec la probabilité spécifiée. Cela permet de réduire la charge lorsqu’il y a un grand volume de requêtes par seconde.

Valeurs possibles :

* 0 — Les requêtes ne sont pas enregistrées dans les tables système.
* Nombre à virgule flottante positif dans la plage [0..1]. Par exemple, si la valeur du paramètre est `0.5`, environ la moitié des requêtes sont enregistrées dans les tables système.
* 1 — Toutes les requêtes sont enregistrées dans les tables système.
