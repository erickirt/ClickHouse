Valeurs possibles :

* N’importe quel entier positif.
* 0 - Désactivé (délai d’expiration infini).

<div id="http_make_head_request">
  ## http_make_head_request
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Le paramètre `http_make_head_request` permet d&#39;exécuter une requête `HEAD` lors de la lecture de données via HTTP afin de récupérer des informations sur le fichier à lire, telles que sa taille. Comme il est activé par défaut, il peut être utile de désactiver ce paramètre lorsque le server ne prend pas en charge les requêtes `HEAD`.

<div id="http_native_compression_disable_checksumming_on_decompress">
  ## http_native_compression_disable_checksumming_on_decompress
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive la vérification de la somme de contrôle lors de la décompression des données de requête HTTP POST envoyées par le client. Utilisé uniquement avec le format de compression natif de ClickHouse (pas avec `gzip` ou `deflate`).

Pour plus d&#39;informations, consultez la [description de l&#39;interface HTTP](/fr/concepts/features/interfaces/http).

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

<div id="http_zlib_compression_level">
  ## http_zlib_compression_level
</div>

<SettingsInfoBlock type="Int64" default_value="3" />

Définit le niveau de compression des données dans la réponse à une requête HTTP si [enable&#95;http&#95;compression = 1](/fr/reference/settings/session-settings/enable#enable_http_compression).

Valeurs possibles : nombres de 1 à 12. Les niveaux supérieurs à `9` nécessitent le build par défaut avec `libdeflate` ; un build sans `libdeflate` prend en charge les niveaux de 1 à 9.
