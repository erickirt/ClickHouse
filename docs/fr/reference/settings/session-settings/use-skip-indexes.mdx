<div id="use_skip_indexes_for_disjunctions">
  ## use_skip_indexes_for_disjunctions
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.12"},{"label": "1"},{"label": "Nouveau paramètre"}]}]} />

Évalue les filtres WHERE comportant des conditions AND et OR mixtes à l’aide des index de saut. Exemple : WHERE A = 5 AND (B = 5 OR C = 5).
Si ce paramètre est désactivé, les index de saut sont toujours utilisés pour évaluer les conditions WHERE, mais celles-ci ne doivent contenir que des clauses reliées par AND.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

  ## use_skip_indexes_for_top_k

Active l&#39;utilisation des index de saut de données pour le filtrage TopK.

Lorsqu&#39;il est activé, si un index minmax de saut existe sur la colonne dans une requête `ORDER BY <column> LIMIT n`, l&#39;optimiseur tentera d&#39;utiliser l&#39;index minmax pour ignorer les granules non pertinentes pour le résultat final. Cela peut réduire la latence des requêtes.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

<div id="use_skip_indexes_if_final">
  ## use_skip_indexes_if_final
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": "1"},{"label": "Changement de la valeur par défaut du paramètre"}]}]} />

Contrôle l&#39;utilisation des index de saut lors de l&#39;exécution d&#39;une requête avec le modificateur FINAL.

Les index de saut peuvent exclure des lignes (granules) contenant les données les plus récentes, ce qui peut entraîner des résultats incorrects pour une requête avec le modificateur FINAL. Lorsque ce paramètre est activé, les index de saut sont appliqués même avec le modificateur FINAL, ce qui peut améliorer les performances, mais avec le risque d&#39;ignorer des mises à jour récentes. Ce paramètre doit être activé en même temps que le paramètre use&#95;skip&#95;indexes&#95;if&#95;final&#95;exact&#95;mode (activé par défaut).

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

<div id="use_skip_indexes_if_final_exact_mode">
  ## use_skip_indexes_if_final_exact_mode
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": "1"},{"label": "Modification de la valeur par défaut du paramètre"}]}, {"id": "row-2","items": [{"label": "25.5"},{"label": "0"},{"label": "Ce paramètre a été introduit pour aider le modificateur FINAL à renvoyer des résultats corrects avec les index de saut"}]}]} />

Contrôle si les granules renvoyées par un index de saut sont élargies dans les parts les plus récentes afin de renvoyer des résultats corrects lors de l&#39;exécution d&#39;une requête avec le modificateur FINAL.

L&#39;utilisation d&#39;index de saut peut exclure des lignes (granules) contenant les données les plus récentes, ce qui peut entraîner des résultats incorrects. Ce paramètre permet de garantir des résultats corrects en parcourant les parts les plus récentes qui chevauchent les plages renvoyées par l&#39;index de saut. Ce paramètre ne doit être désactivé que si des résultats approximatifs fondés sur la consultation de l&#39;index de saut conviennent à une application.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

<div id="use_skip_indexes_on_data_read">
  ## use_skip_indexes_on_data_read
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "1"},{"label": "Le text index est désormais GA"}]}, {"id": "row-2","items": [{"label": "26.1"},{"label": "0"},{"label": "Activé par défaut"}]}, {"id": "row-3","items": [{"label": "25.9"},{"label": "0"},{"label": "Nouveau paramètre"}]}]} />

Active l’utilisation des index de saut de données pendant la lecture des données.

Lorsqu’ils sont activés, les index de saut sont évalués dynamiquement au fur et à mesure de la lecture de chaque granule de données, au lieu d’être analysés à l’avance avant le début de l’exécution de la requête. Cela peut réduire la latence de démarrage des requêtes.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.
