  ## enable_optimize_predicate_expression
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "18.12.17"},{"label": "1"},{"label": "Active par défaut l’optimisation des prédicats dans les sous-requêtes"}]}]} />

Active le pushdown des prédicats dans les requêtes `SELECT`.

Le pushdown des prédicats peut réduire considérablement le trafic réseau pour les requêtes distribuées.

Valeurs possibles :

* 0 — Désactivé.
* 1 — Activé.

Utilisation

Considérez les requêtes suivantes :

1. `SELECT count() FROM test_table WHERE date = '2018-10-10'`
2. `SELECT count() FROM (SELECT * FROM test_table) WHERE date = '2018-10-10'`

Si `enable_optimize_predicate_expression = 1`, le temps d’exécution de ces requêtes est identique, car ClickHouse applique `WHERE` à la sous-requête pendant son traitement.

Si `enable_optimize_predicate_expression = 0`, le temps d’exécution de la deuxième requête est nettement plus long, car la clause `WHERE` s’applique à l’ensemble des données une fois la sous-requête terminée.
