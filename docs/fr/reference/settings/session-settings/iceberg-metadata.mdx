<div id="iceberg_metadata_log_level">
  ## iceberg_metadata_log_level
</div>

<SettingsInfoBlock type="IcebergMetadataLogLevel" default_value="none" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "none"},{"label": "Nouveau paramètre."}]}]} />

Contrôle le niveau de journalisation des métadonnées des tables Iceberg dans `system.iceberg_metadata_log`.
En général, ce paramètre peut être modifié à des fins de débogage.

Valeurs possibles :

* none - Aucune journalisation des métadonnées.
* metadata - Fichier racine metadata.json.
* manifest&#95;list&#95;metadata - Tout ce qui précède + les métadonnées de la liste de manifestes Avro correspondant à un snapshot.
* manifest&#95;list&#95;entry - Tout ce qui précède + les entrées de la liste de manifestes Avro.
* manifest&#95;file&#95;metadata - Tout ce qui précède + les métadonnées des fichiers manifestes Avro parcourus.
* manifest&#95;file&#95;entry - Tout ce qui précède + les entrées des fichiers manifestes Avro parcourus.

<div id="iceberg_metadata_staleness_ms">
  ## iceberg_metadata_staleness_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nouveau paramètre permettant d'utiliser une version des métadonnées mise en cache lors des opérations READ afin d'éviter une récupération depuis le catalogue distant"}]}]} />

Si cette valeur est non nulle, la récupération des métadonnées Iceberg depuis le catalogue distant est ignorée s&#39;il existe un instantané des métadonnées mis en cache plus récent que la fenêtre d&#39;obsolescence indiquée. Une valeur de zéro signifie qu&#39;il faut toujours récupérer la dernière version des métadonnées depuis le catalogue distant. Définir ce paramètre sur une valeur non nulle permet de réduire la latence des opérations de lecture au prix d&#39;un certain degré d&#39;obsolescence.
