<div id="max_skip_unavailable_shards_num">
  ## max_skip_unavailable_shards_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nouveau paramètre permettant de limiter le nombre de shards pouvant être ignorés sans erreur lorsque skip_unavailable_shards est activé."}]}]} />

Lorsque `skip_unavailable_shards` est activé, ce paramètre limite le nombre maximal de shards pouvant être ignorés sans erreur.
Si le nombre de shards indisponibles dépasse cette valeur, une exception est levée au lieu de les ignorer sans erreur.

Une valeur de 0 signifie qu&#39;il n&#39;y a pas de limite (comportement par défaut — tous les shards indisponibles peuvent être ignorés).

<div id="max_skip_unavailable_shards_ratio">
  ## max_skip_unavailable_shards_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nouveau paramètre permettant de limiter la proportion de shards pouvant être ignorés silencieusement lorsque skip_unavailable_shards est activé."}]}]} />

Lorsque `skip_unavailable_shards` est activé, limite la proportion maximale (de 0 à 1) de shards pouvant être ignorés silencieusement.
Si la proportion de shards indisponibles par rapport au nombre total de shards dépasse cette valeur, une exception est levée au lieu de les ignorer silencieusement.

Une valeur de 0 signifie qu&#39;il n&#39;y a pas de limite (comportement par défaut — tous les shards indisponibles peuvent être ignorés).
