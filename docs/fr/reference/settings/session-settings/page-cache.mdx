  ## page_cache_block_size
</div>

<SettingsInfoBlock type="UInt64" default_value="1048576" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "1048576"},{"label": "Ce paramètre peut désormais être ajusté pour chaque requête."}]}]} />

Taille, en octets, des blocs de fichier stockés dans le cache de pages en espace utilisateur. Toutes les lectures qui passent par le cache sont arrondies au multiple de cette taille supérieur.

Ce paramètre peut être ajusté pour chaque requête, mais les entrées de cache associées à des tailles de bloc différentes ne peuvent pas être réutilisées. Le modifier invalide de fait les entrées déjà présentes dans le cache.

Une valeur plus élevée, comme 1 MiB, convient bien aux requêtes à haut débit, tandis qu’une valeur plus faible, comme 64 KiB, est préférable pour les requêtes ponctuelles à faible latence.

<div id="page_cache_max_coalesced_bytes">
  ## page_cache_max_coalesced_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="16777216" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "16777216"},{"label": "Nouveau paramètre limitant la taille d’une lecture fusionnée unique utilisée pour alimenter le cache de pages en espace utilisateur en cas de défaut de cache."}]}]} />

Lorsque `readBigAt` alimente le cache de pages en espace utilisateur, les défauts de cache consécutifs sont regroupés en une seule lecture depuis le stockage sous-jacent. Ce paramètre limite la taille d’une lecture fusionnée en octets ; les séries de défauts plus longues sont réparties sur plusieurs lectures. Il limite l’utilisation transitoire de mémoire du tampon temporaire lors de lectures à froid parallèles.

Une valeur plus élevée réduit le nombre de requêtes HTTP lors de parcours à froid sur le stockage objet ; une valeur plus faible réduit le pic de mémoire transitoire.
