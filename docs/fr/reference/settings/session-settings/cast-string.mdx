  ## cast_string_to_date_time_mode
</div>

<SettingsInfoBlock type="DateTimeInputFormat" default_value="best_effort" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "best_effort"},{"label": "Utilisation améliorée"}]}, {"id": "row-2","items": [{"label": "25.6"},{"label": "basic"},{"label": "Permet d’utiliser différents modes d’analyse de DateTime lors du CAST de String en DateTime"}]}]} />

Permet de choisir un parseur pour la représentation textuelle de la date et de l’heure lors du CAST depuis `String`.

Valeurs possibles :

* `'best_effort'` — Active l’analyse étendue.

  ClickHouse peut analyser le format de base `YYYY-MM-DD HH:MM:SS` ainsi que tous les formats de date et d’heure [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601). Par exemple, `'2018-06-08T01:02:03.000Z'`.

* `'best_effort_us'` — Semblable à `best_effort` (voir la différence dans [parseDateTimeBestEffortUS](/fr/reference/functions/regular-functions/type-conversion-functions#parseDateTimeBestEffortUS)

* `'basic'` — Utilise le parseur de base.

  ClickHouse peut analyser uniquement les formats de base `YYYY-MM-DD HH:MM:SS` ou `YYYY-MM-DD`. Par exemple, `2019-08-20 10:18:56` ou `2019-08-20`.
