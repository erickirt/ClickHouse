<div id="distributed_ddl_output_mode">
  ## distributed_ddl_output_mode
</div>

<SettingsInfoBlock type="DistributedDDLOutputMode" default_value="throw" />

Définit le format du résultat de la requête DDL distribuée.

Valeurs possibles :

* `throw` — Renvoie le jeu de résultats avec le statut d&#39;exécution de la requête pour tous les hôtes où la requête est terminée. Si la requête a échoué sur certains hôtes, la première exception est relancée. Si la requête n&#39;est pas encore terminée sur certains hôtes et que [distributed&#95;ddl&#95;task&#95;timeout](/fr/reference/settings/session-settings/distributed-ddl#distributed_ddl_task_timeout) est dépassé, une exception `TIMEOUT_EXCEEDED` est levée.
* `none` — Similaire à `throw`, mais la requête DDL distribuée ne renvoie aucun jeu de résultats.
* `null_status_on_timeout` — Renvoie `NULL` comme statut d&#39;exécution dans certaines lignes du jeu de résultats au lieu de lever `TIMEOUT_EXCEEDED` si la requête n&#39;est pas terminée sur les hôtes correspondants.
* `never_throw` — Ne lève pas `TIMEOUT_EXCEEDED` et ne relance pas d&#39;exception si la requête a échoué sur certains hôtes.
* `none_only_active` - similaire à `none`, mais n&#39;attend pas les répliques inactives de la base de données `Replicated`. Remarque : avec ce mode, il est impossible de déterminer que la requête n&#39;a pas été exécutée sur une réplique et qu&#39;elle sera exécutée en arrière-plan.
* `null_status_on_timeout_only_active` — similaire à `null_status_on_timeout`, mais n&#39;attend pas les répliques inactives de la base de données `Replicated`
* `throw_only_active` — similaire à `throw`, mais n&#39;attend pas les répliques inactives de la base de données `Replicated`

Valeur par défaut dans Cloud : `none_only_active`.

<div id="distributed_ddl_task_timeout">
  ## distributed_ddl_task_timeout
</div>

<SettingsInfoBlock type="Int64" default_value="180" />

Définit le délai d&#39;attente des réponses aux requêtes DDL provenant de tous les hôtes du cluster. Si une requête DDL n&#39;a pas été exécutée sur tous les hôtes, la réponse contiendra une erreur de délai d&#39;attente et la requête sera exécutée en mode asynchrone. Une valeur négative signifie un délai d&#39;attente infini.

Valeurs possibles :

* Entier positif.
* 0 — Mode asynchrone.
* Entier négatif — délai d&#39;attente infini.
