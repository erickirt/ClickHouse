<div id="wait_for_part_commit_in_dependent_materialized_views">
  ## wait_for_part_commit_in_dependent_materialized_views
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "0"},{"label": "Nouveau paramètre"}]}]} />

Détermine si chaque sink valide la part qu’il vient d’écrire avant l’exécution de sa propre cascade de vues matérialisées dépendantes, afin qu’une cascade qui relit la source via `JOIN` voie la part écrite par ce sink.

La garantie s’applique à chaque instance de sink : les parts écrites par d’autres threads de sink du même `INSERT` peuvent ne pas être encore visibles. Ce paramètre ne garantit aucun ordre de validation entre les threads.

N’a aucun effet sur les insertions dans des tables sans vues matérialisées dépendantes.
