  ## memory_overcommit_ratio_denominator

Elle représente la limite de mémoire souple lorsque la limite stricte est atteinte au niveau global.
Cette valeur est utilisée pour calculer le ratio de surallocation de la requête.
La valeur 0 signifie que la requête est ignorée.
Pour en savoir plus, consultez la page [surallocation de mémoire](/fr/concepts/features/configuration/settings/memory-overcommit).
