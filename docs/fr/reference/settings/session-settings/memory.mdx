<div id="memory_tracker_fault_probability">
  ## memory_tracker_fault_probability
</div>

<SettingsInfoBlock type="Float" default_value="0" />

Pour tester la `exception safety`, lever une exception chaque fois que de la mémoire est allouée, selon la probabilité spécifiée.

<div id="memory_usage_overcommit_max_wait_microseconds">
  ## memory_usage_overcommit_max_wait_microseconds
</div>

<SettingsInfoBlock type="UInt64" default_value="5000000" />

Durée maximale pendant laquelle un thread attend que de la mémoire soit libérée en cas de memory overcommit au niveau de l&#39;utilisateur.
Si le timeout est atteint et que la mémoire n&#39;est pas libérée, une exception est levée.
Pour en savoir plus sur le [memory overcommit](/fr/concepts/features/configuration/settings/memory-overcommit).
