<div id="hdfs_create_new_file_on_insert">
  ## hdfs_create_new_file_on_insert
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive la création d’un nouveau fichier à chaque insertion dans les tables du moteur HDFS. Si ce paramètre est activé, un nouveau fichier HDFS est créé à chaque insertion, selon un schéma de nommage semblable à celui-ci :

initial : `data.Parquet.gz` -&gt; `data.1.Parquet.gz` -&gt; `data.2.Parquet.gz`, etc.

Valeurs possibles :

* 0 — la requête `INSERT` ajoute de nouvelles données à la fin du fichier.
* 1 — la requête `INSERT` crée un nouveau fichier.

<div id="hdfs_skip_empty_files">
  ## hdfs_skip_empty_files
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive l’ignorance des fichiers vides dans les tables utilisant le moteur [HDFS](/fr/reference/engines/table-engines/integrations/hdfs).

Valeurs possibles :

* 0 — `SELECT` lève une exception si le fichier vide n’est pas compatible avec le format demandé.
* 1 — `SELECT` renvoie un résultat vide pour un fichier vide.

<div id="hdfs_truncate_on_insert">
  ## hdfs_truncate_on_insert
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive la troncature avant une opération d’insertion dans les tables du moteur HDFS. Si cette option est désactivée, une exception est levée en cas de tentative d’insertion si un fichier existe déjà dans HDFS.

Valeurs possibles :

* 0 — la requête `INSERT` ajoute de nouvelles données à la fin du fichier.
* 1 — la requête `INSERT` remplace le contenu existant du fichier par les nouvelles données.
