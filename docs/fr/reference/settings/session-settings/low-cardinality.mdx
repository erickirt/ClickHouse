<div id="low_cardinality_allow_in_native_format">
  ## low_cardinality_allow_in_native_format
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Autorise ou limite l&#39;utilisation du type de données [LowCardinality](/fr/reference/data-types/lowcardinality) avec le format [Native](/fr/reference/formats/Native).

Si l&#39;utilisation de `LowCardinality` est limitée, le serveur ClickHouse convertit les colonnes `LowCardinality` en colonnes ordinaires pour les requêtes `SELECT`, et les colonnes ordinaires en colonnes `LowCardinality` pour les requêtes `INSERT`.

Ce paramètre est principalement nécessaire pour les clients tiers qui ne prennent pas en charge le type de données `LowCardinality`.

Valeurs possibles :

* 1 — L&#39;utilisation de `LowCardinality` n&#39;est pas limitée.
* 0 — L&#39;utilisation de `LowCardinality` est limitée.

<div id="low_cardinality_max_dictionary_size">
  ## low_cardinality_max_dictionary_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8192" />

Définit la taille maximale, en lignes, d’un dictionnaire global partagé pour le type de données [LowCardinality](/fr/reference/data-types/lowcardinality), qui peut être écrit dans un système de fichiers de stockage. Ce paramètre évite les problèmes de RAM en cas de croissance illimitée du dictionnaire. Toutes les données qui ne peuvent pas être encodées en raison de cette limite de taille du dictionnaire sont écrites par ClickHouse de manière ordinaire.

Valeurs possibles :

* Tout entier positif.

<div id="low_cardinality_use_single_dictionary_for_part">
  ## low_cardinality_use_single_dictionary_for_part
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive l&#39;utilisation d&#39;un dictionnaire unique pour la data part.

Par défaut, le serveur ClickHouse surveille la taille des dictionnaires et, si l&#39;un d&#39;eux déborde, le server commence à écrire le suivant. Pour empêcher la création de plusieurs dictionnaires, définissez `low_cardinality_use_single_dictionary_for_part = 1`.

Valeurs possibles :

* 1 — La création de plusieurs dictionnaires pour la data part est interdite.
* 0 — La création de plusieurs dictionnaires pour la data part n&#39;est pas interdite.
