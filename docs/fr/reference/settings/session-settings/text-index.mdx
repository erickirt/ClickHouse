  ## text_index_lazy_intersection_density_threshold

**Alias** : `text_index_density_threshold`

<SettingsInfoBlock type="Float" default_value="0.2" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0.2"},{"label": "Renommé depuis `text_index_density_threshold` (conservé comme alias) ; sélectionne l’algorithme d’intersection des posting lists en mode lazy d’application des posting lists."}]}]} />

Seuil de densité des posting lists qui détermine l’algorithme d’intersection en mode lazy d’application des posting lists (`text_index_posting_list_apply_mode = 'lazy'`).
En dessous du seuil : intersection leapfrog (privilégie les posting lists clairsemées). À partir du seuil : intersection bitmap en brute-force (privilégie les posting lists denses).

  ## text_index_like_max_postings_to_read
</div>

<div id="text_index_like_min_pattern_length">
  ## text_index_like_min_pattern_length
</div>

<SettingsInfoBlock type="UInt64" default_value="4" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "4"},{"label": "Nouveau paramètre"}]}]} />

Longueur minimale de la sous-chaîne alphanumérique recherchée dans un motif LIKE/ILIKE pour que l’évaluation LIKE de l’index de texte par parcours du dictionnaire soit utilisée.
Les motifs plus courts que ce seuil correspondent à trop de tokens du dictionnaire et sont ignorés afin d’éviter des parcours coûteux.

Nécessite que `use_text_index_like_evaluation_by_dictionary_scan` soit activé.

<div id="text_index_posting_list_apply_mode">
  ## text_index_posting_list_apply_mode
</div>

<SettingsInfoBlock type="TextIndexPostingListApplyMode" default_value="materialize" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "materialize"},{"label": "Nouveau paramètre pour le mode d’application lazy des posting lists"}]}]} />

Contrôle la façon dont les posting lists sont appliquées lors des requêtes utilisant un index de texte.
&#39;materialize&#39; (par défaut) décode immédiatement les posting lists en bitmaps Roaring.
&#39;lazy&#39; utilise un décodage à la demande basé sur des curseurs (nécessite un format d’index avec un codec sérialisé).
