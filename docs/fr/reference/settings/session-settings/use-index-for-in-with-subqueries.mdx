  ## use_index_for_in_with_subqueries
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Tente d’utiliser un index lorsqu’il y a une sous-requête ou une expression de table à droite de l’opérateur IN.

<div id="use_index_for_in_with_subqueries_max_values">
  ## use_index_for_in_with_subqueries_max_values
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Taille maximale de l’ensemble situé dans la partie droite de l’opérateur IN pour utiliser l’index de table pour le filtrage. Cela permet d’éviter une dégradation des performances et une consommation mémoire plus élevée dues à la préparation de structures de données supplémentaires pour les requêtes volumineuses. Zéro signifie qu’il n’y a pas de limite.
