<div id="force_data_skipping_indices">
  ## force_data_skipping_indices
</div>

Désactive l’exécution de la requête si les indices de saut de données spécifiés n’ont pas été utilisés.

Prenons l’exemple suivant :

```sql
CREATE TABLE data
(
    key Int,
    d1 Int,
    d1_null Nullable(Int),
    INDEX d1_idx d1 TYPE minmax GRANULARITY 1,
    INDEX d1_null_idx assumeNotNull(d1_null) TYPE minmax GRANULARITY 1
)
Engine=MergeTree()
ORDER BY key;

SELECT * FROM data_01515;
SELECT * FROM data_01515 SETTINGS force_data_skipping_indices=''; -- query will produce CANNOT_PARSE_TEXT error.
SELECT * FROM data_01515 SETTINGS force_data_skipping_indices='d1_idx'; -- query will produce INDEX_NOT_USED error.
SELECT * FROM data_01515 WHERE d1 = 0 SETTINGS force_data_skipping_indices='d1_idx'; -- Ok.
SELECT * FROM data_01515 WHERE d1 = 0 SETTINGS force_data_skipping_indices='`d1_idx`'; -- Ok (example of full featured parser).
SELECT * FROM data_01515 WHERE d1 = 0 SETTINGS force_data_skipping_indices='`d1_idx`, d1_null_idx'; -- query will produce INDEX_NOT_USED error, since d1_null_idx is not used.
SELECT * FROM data_01515 WHERE d1 = 0 AND assumeNotNull(d1_null) = 0 SETTINGS force_data_skipping_indices='`d1_idx`, d1_null_idx'; -- Ok.
```

<div id="force_index_by_date">
  ## force_index_by_date
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Désactive l’exécution de la requête si l’index ne peut pas être utilisé avec une date.

Fonctionne avec les tables de la famille MergeTree.

Si `force_index_by_date=1`, ClickHouse vérifie si la requête comporte une condition sur la clé de date pouvant être utilisée pour restreindre les plages de données. S’il n’existe aucune condition appropriée, une exception est levée. Cependant, il ne vérifie pas si cette condition réduit la quantité de données à lire. Par exemple, la condition `Date != ' 2000-01-01 '` est acceptable même lorsqu’elle correspond à toutes les données de la table (c’est-à-dire que l’exécution de la requête nécessite un parcours complet). Pour plus d’informations sur les plages de données dans les tables MergeTree, consultez [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree).

<div id="force_primary_key">
  ## force_primary_key
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Désactive l’exécution de la requête si l’utilisation de l’index de clé primaire n’est pas possible.

Fonctionne avec les tables de la famille MergeTree.

Si `force_primary_key=1`, ClickHouse vérifie si la requête comporte une condition sur la clé primaire qui peut être utilisée pour restreindre les plages de données. S’il n’existe pas de condition appropriée, une exception est levée. Cependant, ClickHouse ne vérifie pas si cette condition réduit la quantité de données à lire. Pour plus d’informations sur les plages de données dans les tables MergeTree, voir [MergeTree](/fr/reference/engines/table-engines/mergetree-family/mergetree).
