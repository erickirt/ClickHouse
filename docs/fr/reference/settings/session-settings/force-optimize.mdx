<div id="force_optimize_projection">
  ## force_optimize_projection
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Active ou désactive l&#39;utilisation obligatoire des [projections](/fr/reference/engines/table-engines/mergetree-family/mergetree#projections) dans les requêtes `SELECT` lorsque l&#39;optimisation des projections est activée (voir le paramètre [optimize&#95;use&#95;projections](/fr/reference/settings/session-settings/optimize-use#optimize_use_projections)).

Valeurs possibles :

* 0 — L&#39;optimisation des projections n&#39;est pas obligatoire.
* 1 — L&#39;optimisation des projections est obligatoire.

<div id="force_optimize_projection_name">
  ## force_optimize_projection_name
</div>

S’il est défini sur une chaîne non vide, vérifiez que cette projection est utilisée au moins une fois dans la requête.

Valeurs possibles :

* chaîne : nom de la projection utilisée dans une requête

<div id="force_optimize_skip_unused_shards">
  ## force_optimize_skip_unused_shards
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Active ou désactive l’exécution des requêtes lorsque [optimize&#95;skip&#95;unused&#95;shards](/fr/reference/settings/session-settings/optimize-skip#optimize_skip_unused_shards) est activé et qu’il n’est pas possible d’ignorer les segments inutilisés. Si ce n’est pas possible et que ce paramètre est activé, une exception est levée.

Valeurs possibles :

* 0 — Désactivé. ClickHouse ne lève pas d’exception.
* 1 — Activé. L’exécution des requêtes est désactivée uniquement si la table possède une clé de sharding.
* 2 — Activé. L’exécution des requêtes est désactivée, qu’une clé de sharding soit ou non définie pour la table.

<div id="force_optimize_skip_unused_shards_nesting">
  ## force_optimize_skip_unused_shards_nesting
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Contrôle le niveau d’imbrication de la [requête distribuée](/fr/reference/settings/session-settings/force-optimize#force_optimize_skip_unused_shards) dont dépend [`force_optimize_skip_unused_shards`](/fr/reference/settings/session-settings/force-optimize#force_optimize_skip_unused_shards) (et requiert donc toujours [`force_optimize_skip_unused_shards`](/fr/reference/settings/session-settings/force-optimize#force_optimize_skip_unused_shards)), dans le cas où vous avez une table `Distributed` qui interroge une autre table `Distributed`.

Valeurs possibles :

* 0 - Désactivé, `force_optimize_skip_unused_shards` fonctionne toujours.
* 1 — Active `force_optimize_skip_unused_shards` uniquement pour le premier niveau.
* 2 — Active `force_optimize_skip_unused_shards` jusqu’au deuxième niveau.
