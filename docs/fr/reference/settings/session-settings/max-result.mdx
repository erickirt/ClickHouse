<div id="max_result_bytes">
  ## max_result_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Limite la taille du résultat en octets (non compressés). La requête s&#39;arrête après le traitement d&#39;un bloc de données si le seuil est atteint,
mais elle ne tronque pas le dernier bloc du résultat ; la taille du résultat peut donc dépasser le seuil.

**Points de vigilance**

La taille du résultat en mémoire est prise en compte pour ce seuil.
Même si la taille du résultat est faible, celui-ci peut pointer vers des structures de données plus volumineuses en mémoire,
comme les dictionnaires des colonnes LowCardinality et les Arenas des colonnes AggregateFunction ;
le seuil peut donc être dépassé malgré la faible taille du résultat.

<Warning>
  Ce paramètre est de bas niveau et doit être utilisé avec prudence
</Warning>

<div id="max_result_rows">
  ## max_result_rows
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Limite le nombre de lignes dans le résultat. Cette limite est également vérifiée pour les sous-requêtes et sur les serveurs distants lors de l&#39;exécution de parties d&#39;une requête distribuée.
Aucune limite n&#39;est appliquée lorsque la valeur est `0`.

La requête s&#39;arrête après le traitement d&#39;un bloc de données si le seuil est atteint, mais
elle ne tronque pas le dernier bloc du résultat ; la taille du résultat peut donc être
supérieure au seuil.
