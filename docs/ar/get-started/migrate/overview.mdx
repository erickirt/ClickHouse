توجد عدة مسارات لترحيل البيانات إلى ClickHouse Cloud، وذلك بحسب مكان وجود بياناتك حاليًا:

| مسار الترحيل                                                             | الوصف                                                                               |
| ------------------------------------------------------------------------ | ----------------------------------------------------------------------------------- |
| [PostgreSQL](/ar/get-started/migrate/postgres/index)                        | رحّل بيانات PostgreSQL واستعلاماته إلى ClickHouse.                                  |
| [BigQuery](/ar/get-started/migrate/bigquery/index)                          | رحّل مجموعات البيانات وأعباء العمل من BigQuery إلى ClickHouse Cloud.                |
| [Snowflake](/ar/get-started/migrate/snowflake/overview)                     | رحّل بيانات Snowflake وأعباء عمل SQL إلى ClickHouse Cloud.                          |
| [Elasticsearch](/ar/get-started/migrate/elastic/overview)                   | رحّل بيانات Elasticsearch واستعلاماته إلى ClickHouse.                               |
| [Redshift](/ar/get-started/migrate/redshift/overview)                       | رحّل بيانات Redshift وأعباء العمل إلى ClickHouse Cloud.                             |
| [OSS to Cloud](/ar/get-started/migrate/oss-to-cloud/clickhouse-to-cloud)    | انقل البيانات من نشر ClickHouse مُدار ذاتيًا إلى ClickHouse Cloud.                  |
| [Other methods](/ar/get-started/migrate/other-methods/clickhouse-local-etl) | استخدم clickhouse-local أو أدوات ETL أو التخزين الكائني لسيناريوهات الترحيل الأخرى. |
