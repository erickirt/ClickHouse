<div id="trace_profile_events">
  ## trace_profile_events
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

يُفعّل أو يُعطّل جمع stacktraces عند كل تحديث لـ profile events، إلى جانب اسم حدث profile وقيمة الزيادة، ثم إرسالها إلى [trace&#95;log](/ar/reference/system-tables/trace_log).

القيم الممكنة:

* 1 — tracing لـ profile events مفعّل.
* 0 — tracing لـ profile events معطّل.

<div id="trace_profile_events_list">
  ## trace_profile_events_list
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": ""},{"label": "إعداد جديد"}]}]} />

عند تمكين الإعداد `trace_profile_events`، اقتصر الأحداث المتتبَّعة على القائمة المحددة من الأسماء المفصولة بفواصل.
إذا كانت قيمة `trace_profile_events_list` سلسلة فارغة (وهو الوضع الافتراضي)، فتتبَّع جميع profile events.

قيمة مثال: &#39;DiskS3ReadMicroseconds,DiskS3ReadRequestsCount,SelectQueryTimeMicroseconds,ReadBufferFromS3Bytes&#39;

يتيح استخدام هذا الإعداد جمع البيانات بدقة أكبر عند التعامل مع عدد كبير من الاستعلامات، لأن الكم الكبير جدًا من الأحداث قد يؤدي، بخلاف ذلك، إلى امتلاء طابور سجل النظام الداخلي، مما يتسبب في إسقاط جزء منها.
