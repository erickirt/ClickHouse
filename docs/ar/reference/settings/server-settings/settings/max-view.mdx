تضبط هذه الإعدادات خادم ClickHouse، وقد أُنشئت تلقائيًا من الشيفرة المصدرية لـ ClickHouse.

<div id="max_view_num_to_throw">
  ## max_view_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="نعم" />

إذا كان عدد طرق العرض أكبر من هذه القيمة، فسيُثير الخادم استثناءً.

لا يُحتسب إلا عدد الجداول لمحركات قواعد البيانات التالية:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  تعني القيمة `0` عدم وجود أي قيد.
</Note>

**مثال**

```xml
<max_view_num_to_throw>400</max_view_num_to_throw>
```

<div id="max_view_num_to_warn">
  ## max_view_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="Yes" />

إذا تجاوز عدد طرق العرض المرفقة القيمة المحددة، فسيضيف خادم ClickHouse رسائل تحذير إلى جدول `system.warnings`.

**مثال**

```xml
<max_view_num_to_warn>400</max_view_num_to_warn>
```
