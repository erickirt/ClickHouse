تُهيّئ هذه الإعدادات خادم ClickHouse، وقد وُلِّدت تلقائيًا من الشيفرة المصدرية لـ ClickHouse.

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="No" />

المسار على نظام الملفات المحلي لتخزين البيانات المؤقتة المستخدمة في معالجة الاستعلامات الكبيرة.

<Note>
  * لا يمكن استخدام إلا خيار واحد لتكوين تخزين البيانات المؤقتة: tmp&#95;path, tmp&#95;policy, temporary&#95;data&#95;in&#95;cache.
  * الشرطة المائلة الأخيرة إلزامية.
</Note>

**مثال**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

سياسة لتخزين البيانات المؤقتة. ستُزال جميع الملفات ذات البادئة `tmp` عند بدء التشغيل.

<Note>
  توصيات لاستخدام تخزين الكائنات مع `tmp_policy`:

  * استخدم `bucket:path` منفصلًا على كل خادم
  * استخدم `metadata_type=plain`
  * قد ترغب أيضًا في تعيين TTL لهذا الـ `bucket`
</Note>

<Note>
  * لا يمكن استخدام سوى خيار واحد لتهيئة تخزين البيانات المؤقتة: `tmp_path` ,`tmp_policy`, `temporary_data_in_cache`.
  * يتم تجاهل `move_factor` و`keep_free_space_bytes` و`max_data_part_size_bytes`.
  * يجب أن تحتوي السياسة على *وحدة تخزين* واحدة فقط

  لمزيد من المعلومات، راجع توثيق [محرك جدول MergeTree](/ar/reference/engines/table-engines/mergetree-family/mergetree).
</Note>

**مثال**

عندما يمتلئ `/disk1`، ستُخزَّن البيانات المؤقتة على `/disk2`.

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
