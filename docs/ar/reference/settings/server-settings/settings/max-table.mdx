تُهيِّئ هذه الإعدادات خادم ClickHouse، وقد أُنشئت تلقائيًا من الشيفرة المصدرية لـ ClickHouse.

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

إذا كان عدد الجداول أكبر من هذه القيمة، فسيرفع الخادم استثناءً.

لا تُحتسب الجداول التالية:

* view
* remote
* dictionary
* system

لا يُحتسب سوى الجداول التابعة لمحركات قواعد البيانات التالية:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  تعني القيمة `0` عدم وجود أي حد.
</Note>

**مثال**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

إذا تجاوز عدد الجداول المرفقة القيمة المحددة، فسيضيف خادم ClickHouse رسائل تحذير إلى جدول `system.warnings`.

**مثال**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

قيد على حذف الجداول.

إذا تجاوز حجم جدول [MergeTree](/ar/reference/engines/table-engines/mergetree-family/mergetree) قيمة `max_table_size_to_drop` (بالبايت)، فلن تتمكن من حذفه باستخدام استعلام [`DROP`](/ar/reference/statements/drop) أو استعلام [`TRUNCATE`](/ar/reference/statements/truncate).

<Note>
  تعني القيمة `0` أنه يمكنك حذف جميع الجداول من دون أي قيود.

  لا يتطلب تطبيق هذا الإعداد إعادة تشغيل خادم ClickHouse. وهناك طريقة أخرى لتعطيل هذا القيد، وهي إنشاء الملف `<clickhouse-path>/flags/force_drop_table`.
</Note>

**مثال**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
