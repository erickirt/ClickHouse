تُهيِّئ هذه الإعدادات خادم ClickHouse، وهي مُولَّدة تلقائيًا من الشفرة المصدرية لـ ClickHouse.

<div id="max_database_num_to_throw">
  ## max_database_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />إذا تجاوز عدد قواعد البيانات هذه القيمة، فسيرفع الخادم استثناءً. تعني القيمة 0 عدم وجود حد.

<div id="max_database_num_to_warn">
  ## max_database_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Yes" />

إذا تجاوز عدد قواعد البيانات المرفقة القيمة المحددة، فسيضيف خادم ClickHouse رسائل تحذيرية إلى جدول `system.warnings`.

**مثال**

```xml
<max_database_num_to_warn>50</max_database_num_to_warn>
```

<div id="max_database_replicated_create_table_thread_pool_size">
  ## max_database_replicated_create_table_thread_pool_size
</div>

<SettingsInfoBlock type="UInt32" default_value="1" changeable_without_restart="No" />عدد خيوط التنفيذ المستخدمة لإنشاء الجداول أثناء استعادة النسخة المتماثلة في DatabaseReplicated. تعني القيمة صفر أن عدد خيوط التنفيذ يساوي عدد الأنوية.
