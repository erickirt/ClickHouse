تضبط هذه الإعدادات خادم ClickHouse، وهي مُولَّدة تلقائيًا من الشيفرة المصدرية لـ ClickHouse.

<div id="jemalloc_enable_background_threads">
  ## jemalloc_enable_background_threads
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />يُفعّل الخيوط الخلفية في jemalloc. يستخدم Jemalloc خيوطًا خلفية لتنظيف صفحات الذاكرة غير المستخدمة. وقد يؤدي تعطيله إلى تدهور الأداء.

<div id="jemalloc_enable_global_profiler">
  ## jemalloc_enable_global_profiler
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />يُفعِّل أداة تنميط التخصيص في jemalloc لجميع الخيوط. سيأخذ Jemalloc عينات من عمليات تخصيص الذاكرة ومن جميع عمليات تحرير الذاكرة لعمليات التخصيص التي شملتها العيّنة.
يمكن تفريغ ملفات التعريف باستخدام SYSTEM JEMALLOC FLUSH PROFILE، ويمكن استخدام ذلك لتحليل التخصيص.
يمكن أيضًا تخزين العينات في system.trace&#95;log باستخدام config ‏jemalloc&#95;collect&#95;global&#95;profile&#95;samples&#95;in&#95;trace&#95;log أو باستخدام إعداد الاستعلام jemalloc&#95;collect&#95;profile&#95;samples&#95;in&#95;trace&#95;log.
راجع [تنميط التخصيص](/ar/concepts/features/performance/allocation-profiling)
