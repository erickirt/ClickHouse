تُعِدّ هذه الإعدادات خادم ClickHouse، وقد أُنشئت تلقائيًا من الشيفرة المصدرية لـ ClickHouse.

<div id="max_dictionary_num_to_throw">
  ## max_dictionary_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

إذا كان عدد القواميس أكبر من هذه القيمة، فسيرفع الخادم استثناءً.

يُحتسب فقط عدد الجداول الخاصة بمحركات قواعد البيانات التالية:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  تعني القيمة `0` عدم وجود أي حد.
</Note>

**مثال**

```xml
<max_dictionary_num_to_throw>400</max_dictionary_num_to_throw>
```

<div id="max_dictionary_num_to_warn">
  ## max_dictionary_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Yes" />

إذا تجاوز عدد القواميس المرفقة القيمة المحددة، فسيضيف خادم ClickHouse رسائل تحذير إلى جدول `system.warnings`.

**مثال**

```xml
<max_dictionary_num_to_warn>400</max_dictionary_num_to_warn>
```
