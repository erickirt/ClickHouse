<div id="description">
  ## الوصف
</div>

يجمع هذا الجدول الوثائق المضمّنة للمكوّنات الموحّدة في النظام في جدول واحد. ويمثّل كل صف كيانًا واحدًا (مثل دالة، أو محرك جدول، أو نوع بيانات، وما إلى ذلك)، ويحتوي على الوثائق المرجعية لذلك الكيان معروضة بصيغة Markdown — أي المحتوى نفسه المنشور على الموقع الإلكتروني والمُتاح عبر جداول `system.*` الخاصة بكل نوع.

يُجمَّع `description` من الأجزاء المنظَّمة للوثائق المضمّنة (`description` و`syntax` والوسيطات والأمثلة وما إلى ذلك)، بحيث يحتوي عمود واحد على الوثائق الكاملة للكيان. وعندما تتضمّن الوثائق المضمّنة الصفحة المرجعية كاملةً داخل `description` الخاص بها (كما هو الحال بالنسبة إلى المكوّنات التي تُولَّد صفحات موقعها الإلكتروني منها تلقائيًا: محركات الجداول، ومحركات قواعد البيانات، وأنواع البيانات، والتنسيقات، ودوال الجداول)، تُنشر الصفحة كما هي، من دون إلحاق أقسام مُنشأة من حقول البيانات الوصفية المنظَّمة — لأن متن الصفحة يغطي هذه المادة بالفعل. وتُعرض الأسماء المستعارة كإحالة مرجعية قصيرة إلى الكيان المرجعي، مثل: ``Alias of `trunc`.``

يجمع هذا الجدول، إلى حدٍّ ما، المعلومات المتاحة في جداول الوثائق الخاصة بكل نوع ([`system.functions`](/ar/reference/system-tables/functions)، و[`system.table_engines`](/ar/reference/system-tables/table_engines)، و[`system.data_type_families`](/ar/reference/system-tables/data_type_families)، وغيرها). وهو مخصّص، على وجه الخصوص، لدعم أمر `help` تفاعلي في العميل، لكنه مفيد أيضًا بحد ذاته.

تُجمع الأنواع التالية من الكيانات (وتظهر قيمة العمود `type` بين قوسين):

* الدوال (`Function`)
* الدوال التجميعية (`Aggregate Function`)
* دوال الجداول (`Table Function`)
* محركات الجداول (`Table Engine`)
* محركات قواعد البيانات (`Database Engine`)
* أنواع البيانات (`Data Type`)
* تخطيطات القواميس (`Dictionary Layout`)
* مصادر القواميس (`Dictionary Source`)
* مُركِّبات الدوال التجميعية (`Aggregate Function Combinator`)
* أنواع فهارس تخطي البيانات (`Data Skipping Index`)
* أنواع الأقراص (`Disk Type`)
* الإعدادات (`Setting`)
* إعدادات MergeTree (`MergeTree Setting`)
* إعدادات الخادم (`Server Setting`)
* التنسيقات (`Format`)
* ترميزات الضغط (`Compression Codec`)
* أحداث الملف الشخصي (`Profile Event`)
* المقاييس الحالية (`Current Metric`)
* المقاييس غير المتزامنة (`Asynchronous Metric`)
* جداول النظام (`System Table`)

بالنسبة إلى الإعدادات (من أي نوع)، تكون الوثائق هي وصف الإعداد، إلى جانب نوعه وقيمته الافتراضية؛ ولا تُعرض الإعدادات المتقادمة. وبالنسبة إلى جداول النظام، تكون الوثائق هي تعليق الجدول، إلى جانب قائمة أعمدته (اسم كل عمود ونوعه ووصفه).

يحتوي العمود `source` على مسار الملف المصدري الذي تُعرَّف فيه وثائق الكيان، نسبةً إلى جذر المستودع. وبالنسبة إلى معظم الكيانات، يُلتقط هذا تلقائيًا في الموضع الذي يُنشأ فيه كائن الوثائق (موضع تسجيل المكوّن)؛ أما الأنواع التي يُوثَّق كلٌّ منها في ملف مصدري واحد (مثل الإعدادات وأحداث الملف الشخصي والمقاييس الحالية)، فيكون ذلك الملف.

<div id="columns">
  ## الأعمدة
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ar/reference/data-types/index)) — اسم الكيان، مثل `domainWithoutWWW` أو `MergeTree`.
* `type` ([Enum8(&#39;Function&#39; = 1, &#39;Aggregate Function&#39; = 2, &#39;Table Function&#39; = 3, &#39;Table Engine&#39; = 4, &#39;Database Engine&#39; = 5, &#39;Data Type&#39; = 6, &#39;Dictionary Layout&#39; = 7, &#39;Dictionary Source&#39; = 8, &#39;Aggregate Function Combinator&#39; = 9, &#39;Data Skipping Index&#39; = 10, &#39;Disk Type&#39; = 11, &#39;Setting&#39; = 12, &#39;MergeTree Setting&#39; = 13, &#39;Server Setting&#39; = 14, &#39;Format&#39; = 15, &#39;Compression Codec&#39; = 16, &#39;Profile Event&#39; = 17, &#39;Current Metric&#39; = 18, &#39;Asynchronous Metric&#39; = 19, &#39;System Table&#39; = 20)](/ar/reference/data-types/index)) — نوع الكيان، مثل `Function` أو `Table Engine`.
* `description` ([String](/ar/reference/data-types/index)) — الوثائق المرجعية للكيان معروضة بتنسيق Markdown، ومجمّعة من الوثائق المضمّنة (وهي نفس المحتوى المنشور على الموقع الإلكتروني)، بما في ذلك syntax والأمثلة والأجزاء المنظَّمة الأخرى، إن وُجدت.
* `source` ([String](/ar/reference/data-types/index)) — المسار إلى الملف المصدري الذي تُعرَّف فيه وثائق الكيان، نسبةً إلى جذر المستودع. يكون فارغًا إذا كان موقع المصدر غير معروف.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## مثال
</div>

راجع وثائق كيان معيّن:

```sql title="Query"
SELECT description
FROM system.documentation
WHERE type = 'Table Engine' AND name = 'MergeTree'
FORMAT TSVRaw;
```

يمكن أن يشير الاسم نفسه إلى عدة أنواع من الكيانات (على سبيل المثال، توجد `file` كدالة جدول وكمصدر قاموس)، لذا من المفيد البحث عن الاسم عبر جميع الأنواع:

```sql title="Query"
SELECT type, name
FROM system.documentation
WHERE name = 'file'
ORDER BY type;
```

احسب الكيانات الموثقة لكل نوع:

```sql title="Query"
SELECT type, count()
FROM system.documentation
GROUP BY type
ORDER BY count() DESC;
```
