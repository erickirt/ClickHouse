<div id="description">
  ## Descrição
</div>

Reúne em uma única tabela a documentação embutida dos componentes uniformes do sistema. Cada linha corresponde a uma entidade (uma função, um motor de tabela, um tipo de dado e assim por diante) e contém a documentação de referência dessa entidade em Markdown — o mesmo conteúdo publicado no site e exposto pelas tabelas `system.*` específicas por tipo.

A `description` é montada a partir das partes estruturadas da documentação embutida (`description`, `syntax`, argumentos, exemplos e assim por diante), de modo que uma única coluna contenha a documentação completa de uma entidade. Quando a documentação embutida traz a página de referência inteira em sua `description` (como acontece com os componentes cujas páginas no site são geradas automaticamente a partir dela: motores de tabela, motores de banco de dados, tipos de dados, formatos e funções de tabela), a página é publicada como está, sem acrescentar seções compostas a partir dos campos de metadados estruturados — o corpo da página já cobre esse material. Os aliases são exibidos como uma referência curta à entidade canônica, por exemplo ``Alias of `trunc`.``

De certa forma, esta tabela reúne as informações disponíveis nas tabelas de documentação específicas por tipo ([`system.functions`](/pt-BR/reference/system-tables/functions), [`system.table_engines`](/pt-BR/reference/system-tables/table_engines), [`system.data_type_families`](/pt-BR/reference/system-tables/data_type_families) e outras). Ela foi concebida, em particular, para dar suporte a um comando `help` interativo no client, mas também é útil por si só.

Os seguintes tipos de entidades são reunidos (o valor da coluna `type` é mostrado entre parênteses):

* Funções (`Function`)
* Funções de agregação (`Aggregate Function`)
* Funções de tabela (`Table Function`)
* Motores de tabela (`Table Engine`)
* Motores de banco de dados (`Database Engine`)
* Tipos de dados (`Data Type`)
* Layouts de dicionário (`Dictionary Layout`)
* Fontes de dicionário (`Dictionary Source`)
* Combinadores de funções de agregação (`Aggregate Function Combinator`)
* Tipos de índice data skipping (`Data Skipping Index`)
* Tipos de disco (`Disk Type`)
* Configurações (`Setting`)
* Configurações do MergeTree (`MergeTree Setting`)
* Configurações do servidor (`Server Setting`)
* Formatos (`Format`)
* Codecs de compressão (`Compression Codec`)
* Eventos de perfil (`Profile Event`)
* Métricas atuais (`Current Metric`)
* Métricas assíncronas (`Asynchronous Metric`)
* Tabelas de sistema (`System Table`)

No caso das configurações (de qualquer tipo), a documentação corresponde à descrição da configuração, juntamente com seu tipo e valor padrão; configurações obsoletas não são expostas. No caso das tabelas de sistema, a documentação é o comentário da tabela junto com a lista de suas colunas (o nome, o tipo e a descrição de cada uma).

A coluna `source` contém o caminho do arquivo-fonte em que a documentação da entidade é definida, relativo à raiz do repositório. Para a maioria das entidades, ela é capturada automaticamente no local em que o objeto de documentação é construído (o local de registro do componente); para os tipos documentados, cada um, em um único arquivo-fonte (como configurações, eventos de perfil e métricas atuais), é esse arquivo.

<div id="columns">
  ## Colunas
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/pt-BR/reference/data-types/index)) — O nome da entidade, por exemplo `domainWithoutWWW` ou `MergeTree`.
* `type` ([Enum8(&#39;Function&#39; = 1, &#39;Aggregate Function&#39; = 2, &#39;Table Function&#39; = 3, &#39;Table Engine&#39; = 4, &#39;Database Engine&#39; = 5, &#39;Data Type&#39; = 6, &#39;Dictionary Layout&#39; = 7, &#39;Dictionary Source&#39; = 8, &#39;Aggregate Function Combinator&#39; = 9, &#39;Data Skipping Index&#39; = 10, &#39;Disk Type&#39; = 11, &#39;Setting&#39; = 12, &#39;MergeTree Setting&#39; = 13, &#39;Server Setting&#39; = 14, &#39;Format&#39; = 15, &#39;Compression Codec&#39; = 16, &#39;Profile Event&#39; = 17, &#39;Current Metric&#39; = 18, &#39;Asynchronous Metric&#39; = 19, &#39;System Table&#39; = 20)](/pt-BR/reference/data-types/index)) — O tipo da entidade, por exemplo `Function` ou `Table Engine`.
* `description` ([String](/pt-BR/reference/data-types/index)) — A documentação de referência da entidade, renderizada em Markdown e compilada a partir da documentação embutida (o mesmo conteúdo publicado no site), incluindo sintaxe, exemplos e outras partes estruturadas, se houver.
* `source` ([String](/pt-BR/reference/data-types/index)) — O caminho para o arquivo-fonte em que a documentação da entidade é definida, relativo à raiz do repositório. Vazio se a localização da origem for desconhecida.

<div id="example">
  ## Exemplo
</div>

Consulte a documentação de uma entidade específica:

```sql title="Query"
SELECT description
FROM system.documentation
WHERE type = 'Table Engine' AND name = 'MergeTree'
FORMAT TSVRaw;
```

O mesmo nome pode se referir a vários tipos de entidades (por exemplo, existe tanto a função de tabela `file` quanto a origem de dicionário `file`), por isso é útil procurar esse nome em todos os tipos:

```sql title="Query"
SELECT type, name
FROM system.documentation
WHERE name = 'file'
ORDER BY type;
```

Conte as entidades documentadas de cada tipo:

```sql title="Query"
SELECT type, count()
FROM system.documentation
GROUP BY type
ORDER BY count() DESC;
```
