  ## write_marks_for_substreams_in_compact_parts
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.8"},{"label": "1"},{"label": "Habilita, por padrão, a gravação de marcas para subfluxos em partes compactas"}]}, {"id": "row-2","items": [{"label": "25.5"},{"label": "0"},{"label": "Nova configuração"}]}]} />

Habilita a gravação de marcas para cada subfluxo em vez de para cada coluna em partes compactas.
Isso permite ler subcolunas individuais da parte de dados com eficiência.

Por exemplo, a coluna `t Tuple(a String, b UInt32, c Array(Nullable(UInt32)))` é serializada nos seguintes subfluxos:

* `t.a` para dados String do elemento da tupla `a`
* `t.b` para dados UInt32 do elemento da tupla `b`
* `t.c.size0` para os tamanhos do array do elemento da tupla `c`
* `t.c.null` para o mapa de NULL dos elementos do array aninhado do elemento da tupla `c`
* `t.c` para dados UInt32 dos elementos do array aninhado do elemento da tupla `c`

Quando essa configuração está habilitada, gravaremos uma marca para cada um desses 5 subfluxos, o que significa que poderemos ler
separadamente os dados de cada subfluxo individual de cada grânulo, se necessário. Por exemplo, se quisermos ler a subcoluna `t.c`, leremos apenas os dados dos
subfluxos `t.c.size0`, `t.c.null` e `t.c` e não leremos dados dos subfluxos `t.a` e `t.b`. Quando essa configuração está desativada,
gravaremos uma marca apenas para a coluna de nível superior `t`, o que significa que sempre leremos todos os dados da coluna do grânulo, mesmo que precisemos apenas dos dados de alguns subfluxos.
