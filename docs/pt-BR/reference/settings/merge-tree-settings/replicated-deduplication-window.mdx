<div id="replicated_deduplication_window">
  ## replicated_deduplication_window
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "10000"},{"label": "aumentar o valor padrão"}]}]} />

O número de blocos inseridos mais recentemente para os quais o ClickHouse Keeper armazena
somas de hash para verificar duplicatas.

Valores possíveis:

* Qualquer inteiro positivo.
* 0 (desabilita a desduplicação)

Para [desduplicação de inserção](/pt-BR/reference/engines/table-engines/mergetree-family/replication),
ao gravar em tabelas replicadas, o ClickHouse grava somas de hash de desduplicação no
ClickHouse Keeper. As somas de hash são armazenadas apenas para os
`replicated_deduplication_window` blocos mais recentes. As somas de hash mais antigas são removidas do
ClickHouse Keeper.

A soma de hash cobre todo o bloco inserido, portanto uma inserção é desduplicada apenas quando todos os seus
dados correspondem a uma inserção anterior (uma nova tentativa), e não por parte individual.

Um valor alto para `replicated_deduplication_window` torna as `Inserts` mais lentas porque mais
entradas precisam ser comparadas.

<div id="replicated_deduplication_window_for_async_inserts">
  ## replicated_deduplication_window_for_async_inserts
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" />

Configuração legada mantida para atualização gradual com versões mistas. Novas inserções usam desduplicação com o
hash unificado controlado por `replicated_deduplication_window`; esta configuração agora apenas limita quantos
hashes legados de async-insert são mantidos no diretório `async_blocks` no ClickHouse Keeper, que
ainda é gravado por réplicas mais antigas durante uma atualização gradual e limpo pelo líder atual.
Um valor de `0` não retém nenhum deles — a próxima limpeza remove todas as entradas — portanto, mantenha o
padrão (ou um valor maior) para preservar a janela legada de desduplicação async durante o upgrade.

<div id="replicated_deduplication_window_seconds">
  ## replicated_deduplication_window_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="3600" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.10"},{"label": "3600"},{"label": "reduzir o valor padrão"}]}]} />

O número de segundos após os quais as somas de hash dos blocos inseridos são
removidas do ClickHouse Keeper.

Possíveis valores:

* Qualquer número inteiro positivo.

Semelhante a [replicated&#95;deduplication&#95;window](/pt-BR/reference/settings/merge-tree-settings/replicated-deduplication-window#replicated_deduplication_window),
`replicated_deduplication_window_seconds` especifica por quanto tempo armazenar as somas de hash
dos blocos para desduplicação de inserção. As somas de hash mais antigas que
`replicated_deduplication_window_seconds` são removidas do ClickHouse Keeper,
mesmo que sejam menores que ` replicated_deduplication_window`.

O tempo é relativo ao momento do registro mais recente, e não ao tempo
corrido. Se for o único registro, ele será armazenado para sempre.

<div id="replicated_deduplication_window_seconds_for_async_inserts">
  ## replicated_deduplication_window_seconds_for_async_inserts
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" />

Configuração legada mantida para atualização gradual com versões mistas. Junto com
`replicated_deduplication_window_for_async_inserts`, ela delimita por quanto tempo os hashes legados de async-insert
são mantidos no diretório `async_blocks` no ClickHouse Keeper (gravados por réplicas mais antigas durante uma
atualização gradual e removidos pelo líder atual). Novos inserts usam
`replicated_deduplication_window_seconds`.
