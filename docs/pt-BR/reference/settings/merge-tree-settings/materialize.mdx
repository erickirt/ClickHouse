<div id="materialize_skip_indexes_on_merge">
  ## materialize_skip_indexes_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.1"},{"label": "1"},{"label": "Nova configuração"}]}]} />

Quando habilitada, as mesclagens criam e armazenam índices de skipping para novas partes.
Caso contrário, eles podem ser criados/armazenados explicitamente com [MATERIALIZE INDEX](/pt-BR/reference/statements/alter/skipping-index#materialize-index)
ou [durante INSERTs](/pt-BR/reference/settings/session-settings#materialize_skip_indexes_on_insert).

Consulte também [exclude&#95;materialize&#95;skip&#95;indexes&#95;on&#95;merge](/pt-BR/reference/settings/merge-tree-settings/exclude#exclude_materialize_skip_indexes_on_merge) para um controle mais granular.

<div id="materialize_statistics_on_merge">
  ## materialize_statistics_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "1"},{"label": "Nova configuração"}]}]} />

Quando ativada, as mesclagens geram e armazenam estatísticas para novas partes.
Caso contrário, elas podem ser criadas/armazenadas explicitamente com [MATERIALIZE STATISTICS](/pt-BR/reference/statements/alter/statistics)
ou [durante INSERTs](/pt-BR/reference/settings/session-settings#materialize_statistics_on_insert)

<div id="materialize_ttl_recalculate_only">
  ## materialize_ttl_recalculate_only
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Recalcula as informações de TTL apenas ao executar MATERIALIZE TTL
