<div id="table_readonly">
  ## table_readonly
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nova configuração para marcar a tabela como somente leitura, impedindo inserções e modificações"}]}]} />

Se definido como true, a tabela entra em modo somente leitura e não realiza modificações no disco.

Todas as operações em primeiro plano que modificariam a tabela são rejeitadas: inserções, mutações, `OPTIMIZE` e os comandos de partição que alteram dados
(`ATTACH`/`MOVE`/`DROP`/`DROP DETACHED`/`FETCH`/`REPLACE PARTITION`, bem como `MOVE PARTITION ... TO TABLE` tendo esta tabela como destino). Operações
que não modificam os dados da tabela, como `FREEZE`/`UNFREEZE` e `FORGET PARTITION`, continuam permitidas.

Nenhuma tarefa em segundo plano também é agendada: merges regulares, merges de TTL (`DELETE`/`MOVE`/recompressão), merges de recompressão, mutações em segundo plano
e movimentações de partes em segundo plano são todos suprimidos. Como consequência, uma tabela com TTL deixa de liberar ou mover seus dados expirados enquanto esta configuração
estiver habilitada.

A configuração sempre pode ser revertida com `ALTER TABLE ... MODIFY SETTING table_readonly = 0` (ou `RESET SETTING`). Não há suporte para `ReplicatedMergeTree`.
