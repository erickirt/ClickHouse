<div id="dead_blobs_to_delay_insert">
  ## dead_blobs_to_delay_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "100000"},{"label": "Nova configuração para retardar artificialmente as inserções quando as filas de blobs mortos nos disks da tabela acumularem blobs demais pendentes de remoção."}]}]} />

Se o número de blobs pendentes de remoção nas filas de blobs mortos nos disks da tabela exceder o
valor de `dead_blobs_to_delay_insert`, um `INSERT` será artificialmente retardado (até `max_delay_to_insert`).

A fila de blobs mortos pertence ao disk e é compartilhada por todas as tabelas nele (incluindo blobs de tabelas já
removidas), portanto dimensione o limite para o disk inteiro, e não para uma única tabela.

<Tip>
  Isso é útil quando um servidor não consegue limpar os blobs com rapidez suficiente.
</Tip>

Valores possíveis:

* Qualquer número inteiro positivo.
* 0 — desabilitado.

<div id="dead_blobs_to_throw_insert">
  ## dead_blobs_to_throw_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1000000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1000000"},{"label": "Nova configuração para rejeitar inserções quando as filas de blobs mortos dos disks da tabela acumularem blobs demais pendentes de remoção."}]}]} />

Se o número de blobs pendentes de remoção nas filas de blobs mortos dos disks da tabela for maior que o
valor `dead_blobs_to_throw_insert`, o `INSERT` será interrompido com o seguinte erro:

> &quot;Blobs mortos demais na fila (N) nos disks da tabela. A limpeza de blobs está ocorrendo
> de forma significativamente mais lenta do que as inserções&quot;

A fila de blobs mortos pertence ao disk e é compartilhada por todas as tabelas nele (incluindo blobs de tabelas já
removidas), portanto dimensione o limite para o disk inteiro, e não para uma única tabela.

Valores possíveis:

* Qualquer número inteiro positivo.
* 0 — desabilitado.
