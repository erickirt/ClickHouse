<div id="map_serialization_version">
  ## map_serialization_version
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "Adicionar uma configuração para controlar a versão da serialização de Map"}]}]} />

Controla o método de serialização usado para colunas `Map`.

Valores possíveis:

* basic — Usa a serialização padrão de `Map`.
* with&#95;buckets — Divide as chaves em buckets durante a serialização. O uso de buckets melhora a leitura de chaves individuais do Map.

O número de buckets na serialização `with_buckets` é determinado por [max&#95;buckets&#95;in&#95;map](/pt-BR/reference/settings/merge-tree-settings/max#max_buckets_in_map) e [map&#95;buckets&#95;strategy](/pt-BR/reference/settings/merge-tree-settings/map-buckets#map_buckets_strategy).

<div id="map_serialization_version_for_zero_level_parts">
  ## map_serialization_version_for_zero_level_parts
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "Adiciona uma configuração para controlar a versão de serialização de Map para partes de nível zero"}]}]} />

Essa configuração permite especificar uma versão de serialização diferente para
colunas `Map` em partes de nível zero criadas durante inserções.
Pode ser útil manter a serialização `basic` para partes de nível zero para evitar
degradação de desempenho durante inserções, enquanto se usa `with_buckets` para partes mescladas.
