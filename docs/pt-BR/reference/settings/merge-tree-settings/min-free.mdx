Essas configurações estão disponíveis em [system.merge&#95;tree&#95;settings](/pt-BR/reference/system-tables/merge_tree_settings) e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="min_free_disk_bytes_to_perform_insert">
  ## min_free_disk_bytes_to_perform_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O número mínimo de bytes que devem estar livres no disco para
inserir dados. Se o número de bytes livres disponíveis for menor que
`min_free_disk_bytes_to_perform_insert`, uma exceção é lançada e o
insert não é executado. Observe que esta configuração:

* leva em consideração a configuração `keep_free_space_bytes`.
* não leva em consideração a quantidade de dados que será gravada pela
  operação `INSERT`.
* só é verificada se for especificado um número positivo (diferente de zero) de bytes

Valores possíveis:

* Qualquer inteiro positivo.

<Note>
  Se `min_free_disk_bytes_to_perform_insert` e `min_free_disk_ratio_to_perform_insert`
  forem especificados, o ClickHouse considerará o valor que permitirá realizar
  inserts com uma quantidade maior de espaço livre em disco.
</Note>

<div id="min_free_disk_ratio_to_perform_insert">
  ## min_free_disk_ratio_to_perform_insert
</div>

<SettingsInfoBlock type="Float" default_value="0" />

A razão mínima entre o espaço livre e o espaço total em disco para executar um `INSERT`. Deve ser um
valor de ponto flutuante entre 0 e 1. Observe que esta configuração:

* leva em conta a configuração `keep_free_space_bytes`.
* não leva em conta a quantidade de dados que será gravada pela
  operação `INSERT`.
* só é verificada se uma razão positiva (não zero) for especificada

Valores possíveis:

* Float, 0.0 - 1.0

Observe que, se tanto `min_free_disk_ratio_to_perform_insert` quanto
`min_free_disk_bytes_to_perform_insert` forem especificados, o ClickHouse considerará
o valor que permitir realizar inserts com uma quantidade maior de
espaço livre.
