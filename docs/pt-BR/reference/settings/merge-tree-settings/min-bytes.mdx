  ## min_bytes_for_compact_part

<SettingsInfoBlock type="UInt64" default_value="0" />

<div id="min_bytes_for_full_part_storage">
  ## min_bytes_for_full_part_storage
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Disponível apenas no ClickHouse Cloud. Tamanho mínimo não comprimido, em bytes, para
usar o tipo Full de armazenamento para a parte de dados em vez de packed

<div id="min_bytes_for_wide_part">
  ## min_bytes_for_wide_part
</div>

<SettingsInfoBlock type="UInt64" default_value="10485760" />

Número mínimo de bytes/linhas em uma parte de dados que pode ser armazenada no formato `Wide`.
Você pode definir uma, ambas ou nenhuma dessas configurações.

<div id="min_bytes_to_rebalance_partition_over_jbod">
  ## min_bytes_to_rebalance_partition_over_jbod
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Define a quantidade mínima de bytes necessária para habilitar o balanceamento ao distribuir novas
partes grandes pelos disks do volume [JBOD](https://en.wikipedia.org/wiki/Non-RAID_drive_architectures).

Valores possíveis:

* Inteiro positivo.
* `0` — O balanceamento é desabilitado.

**Uso**

O valor da configuração `min_bytes_to_rebalance_partition_over_jbod` não deve
ser inferior ao valor de
[max&#95;bytes&#95;to&#95;merge&#95;at&#95;max&#95;space&#95;in&#95;pool](/pt-BR/reference/settings/merge-tree-settings/max-bytes#max_bytes_to_merge_at_max_space_in_pool)
/ 1024. Caso contrário, o ClickHouse lança uma exceção.
