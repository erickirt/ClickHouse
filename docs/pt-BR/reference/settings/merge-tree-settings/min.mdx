Essas configurações estão disponíveis em [system.merge&#95;tree&#95;settings](/pt-BR/reference/system-tables/merge_tree_settings) e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="min_absolute_delay_to_close">
  ## min_absolute_delay_to_close
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Atraso absoluto mínimo para encerrar, parar de atender requisições e não
retornar Ok durante a verificação de status.

<div id="min_columns_to_activate_adaptive_write_buffer">
  ## min_columns_to_activate_adaptive_write_buffer
</div>

<SettingsInfoBlock type="UInt64" default_value="500" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "500"},{"label": "Nova configuração"}]}]} />

Permite reduzir o uso de memória em tabelas com muitas colunas ao usar buffers de gravação adaptativos.

Valores possíveis:

* 0 - ilimitado
* 1 - sempre habilitado

<div id="min_compress_block_size">
  ## min_compress_block_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Tamanho mínimo dos blocos de dados não comprimidos necessário para a compressão ao
gravar a próxima mark. Você também pode especificar essa configuração nas configurações globais
(consulte a configuração [min&#95;compress&#95;block&#95;size](/pt-BR/reference/settings/merge-tree-settings/min#min_compress_block_size)
). O valor especificado quando a tabela é criada substitui o valor global
dessa configuração.

<div id="min_index_granularity_bytes">
  ## min_index_granularity_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="1024" />

Tamanho mínimo permitido dos grânulos de dados, em bytes.

Serve como proteção contra a criação acidental de tabelas com
`index_granularity_bytes` muito baixo.

<div id="min_marks_to_honor_max_concurrent_queries">
  ## min_marks_to_honor_max_concurrent_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O número mínimo de marcas lidas pela consulta para que a configuração [max&#95;concurrent&#95;queries](/pt-BR/reference/settings/merge-tree-settings/max#max_concurrent_queries) seja aplicada.

<Note>
  As consultas ainda serão limitadas por outras configurações de `max_concurrent_queries`.
</Note>

Valores possíveis:

* Inteiro positivo.
* `0` — Desabilitado (o limite de `max_concurrent_queries` não se aplica a nenhuma consulta).

**Exemplo**

```xml
<min_marks_to_honor_max_concurrent_queries>10</min_marks_to_honor_max_concurrent_queries>
```

<div id="min_merge_bytes_to_use_direct_io">
  ## min_merge_bytes_to_use_direct_io
</div>

<SettingsInfoBlock type="UInt64" default_value="10737418240" />

O volume mínimo de dados da operação de mesclagem necessário para usar acesso
de E/S direta ao disco de armazenamento. Ao fazer a mesclagem de partes de dados, o ClickHouse calcula
o volume total de armazenamento de todos os dados que serão mesclados. Se o volume exceder
`min_merge_bytes_to_use_direct_io` bytes, o ClickHouse lê e grava os
dados no disco de armazenamento usando a interface de E/S direta (opção `O_DIRECT`).
Se `min_merge_bytes_to_use_direct_io = 0`, a E/S direta será desabilitada.

<div id="min_parts_to_merge_at_once">
  ## min_parts_to_merge_at_once
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Quantidade mínima de partes de dados que o selector de mesclagem pode selecionar para mesclar de uma só vez
(configuração de nível avançado; não altere se você não entende o que ela faz).
0 - desabilitado. Funciona para os selectors de mesclagem Simple e StochasticSimple.

<div id="min_replicated_logs_to_keep">
  ## min_replicated_logs_to_keep
</div>

<SettingsInfoBlock type="UInt64" default_value="10" />

Mantém aproximadamente esse número dos registros mais recentes no log do ZooKeeper, mesmo que estejam
obsoletos. Isso não afeta o funcionamento das tabelas: é usado apenas para diagnosticar o
log do ZooKeeper antes da limpeza.

Valores possíveis:

* Qualquer inteiro positivo.
