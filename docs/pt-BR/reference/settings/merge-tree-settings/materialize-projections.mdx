<div id="materialize_projections_on_insert">
  ## materialize_projections_on_insert
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "1"},{"label": "Nova configuração"}]}]} />

Quando habilitada, as instruções INSERT criam novas partes com projeções.
Caso contrário, elas podem ser criadas explicitamente com [MATERIALIZE PROJECTION](/pt-BR/reference/statements/alter/projection#materialize-projection)
ou durante as mesclagens com [materialize&#95;projections&#95;on&#95;merge](/pt-BR/reference/settings/merge-tree-settings/materialize-projections#materialize_projections_on_merge).

<div id="materialize_projections_on_merge">
  ## materialize_projections_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "0"},{"label": "Nova configuração"}]}]} />

Quando habilitada, uma mesclagem recria uma projeção ausente em todas as partes de origem (por exemplo, porque elas foram
inseridas com `materialize_projections_on_insert = 0`), fazendo com que a parte mesclada passe a ter a projeção.

As mesclagens ainda combinam apenas partes que compartilham o mesmo conjunto de projeções. Para aplicar retrospectivamente uma projeção a todas as partes existentes,
use [MATERIALIZE PROJECTION](/pt-BR/reference/statements/alter/projection#materialize-projection) de forma explícita. As projeções
também são criadas durante instruções INSERT com [materialize&#95;projections&#95;on&#95;insert](/pt-BR/reference/settings/merge-tree-settings/materialize-projections#materialize_projections_on_insert).
