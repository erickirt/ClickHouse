<div id="merge_max_block_size">
  ## merge_max_block_size
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="8192" />

O número de linhas lidas das partes mescladas para a memória.

Valores possíveis:

* Qualquer número inteiro positivo.

A mesclagem lê linhas das partes em blocos de `merge_max_block_size` linhas e, em seguida,
mescla e grava o resultado em uma nova parte. O bloco lido é colocado na RAM,
portanto `merge_max_block_size` afeta a quantidade de RAM necessária para a mesclagem.
Assim, mesclagens podem consumir uma grande quantidade de RAM em tabelas com linhas muito largas
(se o tamanho médio da linha for 100kb, então, ao mesclar 10 partes,
(100kb * 10 * 8192) = ~ 8GB de RAM). Ao reduzir `merge_max_block_size`,
você pode diminuir a quantidade de RAM necessária para uma mesclagem, mas torná-la mais lenta.

<div id="merge_max_block_size_bytes">
  ## merge_max_block_size_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="10485760" />

Quantos bytes devem ser formados em blocos para operações de mesclagem. Por padrão,
tem o mesmo valor de `index_granularity_bytes`.

<div id="merge_max_dynamic_subcolumns_in_compact_part">
  ## merge_max_dynamic_subcolumns_in_compact_part
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="auto" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "auto"},{"label": "Adicionar uma nova configuração para limitar o número de subcolunas dinâmicas na parte Compact após a mesclagem, independentemente dos parâmetros especificados no tipo de dados"}]}]} />

O número máximo de subcolunas dinâmicas que podem ser criadas em cada coluna na parte de dados Compact após a mesclagem.
Isso permite controlar o número de subcolunas dinâmicas na parte Compact, independentemente dos parâmetros dinâmicos especificados no tipo de dados.

Por exemplo, se a tabela tiver uma coluna com o tipo JSON(max&#95;dynamic&#95;paths=1024) e a configuração merge&#95;max&#95;dynamic&#95;subcolumns&#95;in&#95;compact&#95;part estiver definida como 128,
após a mesclagem na parte de dados Compact, o número de caminhos dinâmicos será reduzido para 128 nessa parte, e apenas 128 caminhos serão gravados como subcolunas dinâmicas.

<div id="merge_max_dynamic_subcolumns_in_wide_part">
  ## merge_max_dynamic_subcolumns_in_wide_part
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="auto" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.11"},{"label": "auto"},{"label": "Adicionar uma nova configuração para limitar o número de subcolunas dinâmicas na parte Wide após a mesclagem, independentemente dos parâmetros especificados no tipo de dados"}]}]} />

O número máximo de subcolunas dinâmicas que podem ser criadas em cada coluna na parte de dados Wide após a mesclagem.
Isso permite reduzir o número de arquivos criados na parte de dados Wide, independentemente dos parâmetros dinâmicos especificados no tipo de dados.

Por exemplo, se a tabela tiver uma coluna com o tipo JSON(max&#95;dynamic&#95;paths=1024) e a configuração merge&#95;max&#95;dynamic&#95;subcolumns&#95;in&#95;wide&#95;part estiver definida como 128,
após a mesclagem para a parte de dados Wide, o número de caminhos dinâmicos será reduzido para 128 nessa parte, e apenas 128 caminhos serão gravados como subcolunas dinâmicas.
