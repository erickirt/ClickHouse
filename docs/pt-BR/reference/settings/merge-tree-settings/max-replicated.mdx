Estas configurações estão disponíveis em [system.merge&#95;tree&#95;settings](/pt-BR/reference/system-tables/merge_tree_settings) e são geradas automaticamente com base no código-fonte do ClickHouse.

<div id="max_replicated_fetches_network_bandwidth">
  ## max_replicated_fetches_network_bandwidth
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Limita a velocidade máxima da transferência de dados pela rede, em bytes por
segundo, para fetches [replicados](/pt-BR/reference/engines/table-engines/mergetree-family/replication).
Essa configuração é aplicada a uma tabela específica, ao contrário da
configuração [`max_replicated_fetches_network_bandwidth_for_server`](/pt-BR/reference/settings/merge-tree-settings/max-replicated#max_replicated_fetches_network_bandwidth),
que é aplicada ao servidor.

Você pode limitar tanto a largura de banda de rede do servidor quanto a de uma
tabela específica, mas, para isso, o valor da configuração no nível da tabela
deve ser menor que o do nível do servidor. Caso contrário, o servidor considera
apenas a configuração `max_replicated_fetches_network_bandwidth_for_server`.

A configuração não é aplicada com total precisão.

Valores possíveis:

* Inteiro positivo.
* `0` — Ilimitado.

Valor padrão: `0`.

**Uso**

Pode ser usada para limitar a velocidade ao replicar dados para adicionar ou
substituir novos nós.

<div id="max_replicated_logs_to_keep">
  ## max_replicated_logs_to_keep
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" />

Quantos registros podem haver no log do ClickHouse Keeper se houver uma
réplica inativa. Uma réplica inativa é considerada perdida quando esse número é excedido.

Valores possíveis:

* Qualquer número inteiro positivo.

<div id="max_replicated_merges_in_queue">
  ## max_replicated_merges_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" />

Quantas tarefas de mesclagem e mutação de partes são permitidas simultaneamente na
fila do ReplicatedMergeTree.

<div id="max_replicated_merges_with_ttl_in_queue">
  ## max_replicated_merges_with_ttl_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1" />

Quantas tarefas de mesclagem de partes com TTL são permitidas simultaneamente na
fila do ReplicatedMergeTree.

<div id="max_replicated_mutations_in_queue">
  ## max_replicated_mutations_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="8" />

Quantas tarefas de mutação de partes são permitidas simultaneamente na
fila do ReplicatedMergeTree.

<div id="max_replicated_sends_network_bandwidth">
  ## max_replicated_sends_network_bandwidth
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Limita a velocidade máxima de transferência de dados pela rede, em bytes por
segundo, para envios [replicados](/pt-BR/reference/engines/table-engines/mergetree-family/replacingmergetree).
Essa configuração é aplicada a uma tabela específica, diferentemente da
configuração [`max_replicated_sends_network_bandwidth_for_server`](/pt-BR/reference/settings/merge-tree-settings/max-replicated#max_replicated_sends_network_bandwidth),
que é aplicada ao servidor.

Você pode limitar tanto a rede do servidor quanto a rede de uma tabela
específica, mas, para isso, o valor da configuração no nível da tabela
deve ser menor que o valor no nível do servidor. Caso contrário, o servidor
considera apenas a configuração
`max_replicated_sends_network_bandwidth_for_server`.

A configuração não é aplicada com total precisão.

Valores possíveis:

* Inteiro positivo.
* `0` — Ilimitado.

**Uso**

Pode ser usada para limitar a velocidade ao replicar dados ao adicionar ou substituir
novos nós.
