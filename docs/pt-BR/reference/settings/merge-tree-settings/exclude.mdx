<div id="exclude_deleted_rows_for_part_size_in_merge">
  ## exclude_deleted_rows_for_part_size_in_merge
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Se habilitada, a estimativa do tamanho real das partes de dados (ou seja, excluindo as linhas
que foram removidas por `DELETE FROM`) será usada ao selecionar
partes para mesclar. Observe que esse comportamento só é aplicado a partes de dados
afetadas por `DELETE FROM` executado depois que essa configuração for habilitada.

Valores possíveis:

* `true`
* `false`

<div id="exclude_materialize_skip_indexes_on_merge">
  ## exclude_materialize_skip_indexes_on_merge
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.10"},{"label": ""},{"label": "Nova configuração."}]}]} />

Exclui da criação e do armazenamento durante as mesclagens a lista fornecida de skip indexes, delimitada por vírgulas. Não tem efeito se
[materialize&#95;skip&#95;indexes&#95;on&#95;merge](/pt-BR/reference/settings/merge-tree-settings/materialize#materialize_skip_indexes_on_merge) estiver definida como false.

Os skip indexes excluídos ainda serão criados e armazenados por uma
consulta [MATERIALIZE INDEX](/pt-BR/reference/statements/alter/skipping-index#materialize-index) explícita ou durante INSERTs, dependendo da
configuração de sessão [materialize&#95;skip&#95;indexes&#95;on&#95;insert](/pt-BR/reference/settings/session-settings#materialize_skip_indexes_on_insert).

Exemplo:

```sql
CREATE TABLE tab
(
a UInt64,
b UInt64,
INDEX idx_a a TYPE minmax,
INDEX idx_b b TYPE set(3)
)
ENGINE = MergeTree ORDER BY tuple() SETTINGS exclude_materialize_skip_indexes_on_merge = 'idx_a';

INSERT INTO tab SELECT number, number / 50 FROM numbers(100); -- setting has no effect on INSERTs

-- idx_a will be excluded from update during background or explicit merge via OPTIMIZE TABLE FINAL

-- can exclude multiple indexes by providing a list
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = 'idx_a, idx_b';

-- default setting, no indexes excluded from being updated during merge
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = '';
```
