Essas configurações estão disponíveis em [system.merge&#95;tree&#95;settings](/pt-BR/reference/system-tables/merge_tree_settings) e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="detach_not_byte_identical_parts">
  ## detach_not_byte_identical_parts
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Ativa ou desativa o desanexamento de uma parte de dados em uma réplica após um merge ou uma
mutação, caso ela não seja idêntica byte a byte às partes de dados em outras réplicas. Se
desativada, a parte de dados é removida. Ative essa configuração se quiser
analisar essas partes depois.

A configuração se aplica a tabelas `MergeTree` com
[a replicação de dados](/pt-BR/reference/engines/table-engines/mergetree-family/replacingmergetree) habilitada.

Valores possíveis:

* `0` — As partes são removidas.
* `1` — As partes são desanexadas.

<div id="detach_old_local_parts_when_cloning_replica">
  ## detach_old_local_parts_when_cloning_replica
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Não remove partes locais antigas ao reparar uma réplica perdida.

Valores possíveis:

* `true`
* `false`
