Estas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="temporary_data_in_cache">
  ## temporary_data_in_cache
</div>

Com esta opção, os dados temporários serão armazenados no cache do disco em questão.
Nesta seção, você deve especificar o nome do disco com o tipo `cache`.
Nesse caso, o cache e os dados temporários compartilharão o mesmo espaço, e o cache do disco poderá ser removido para criar espaço para os dados temporários.

<Note>
  Apenas uma opção pode ser usada para configurar o armazenamento de dados temporários: `tmp_path` ,`tmp_policy`, `temporary_data_in_cache`.
</Note>

**Exemplo**

Tanto o cache de `local_disk` quanto os dados temporários serão armazenados em `/tiny_local_cache` no sistema de arquivos, gerenciado por `tiny_local_cache`.

```xml highlight={9-16,20}
<clickhouse>
<storage_configuration>
<disks>
<local_disk>
<type>local</type>
<path>/local_disk/</path>
</local_disk>

<tiny_local_cache>
<type>cache</type>
<disk>local_disk</disk>
<path>/tiny_local_cache/</path>
<max_size_rows>10M</max_size_rows>
<max_file_segment_size>1M</max_file_segment_size>
<cache_on_write_operations>1</cache_on_write_operations>
</tiny_local_cache>
</disks>
</storage_configuration>

<temporary_data_in_cache>tiny_local_cache</temporary_data_in_cache>
</clickhouse>
```
