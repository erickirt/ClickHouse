Essas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Sim" />

Tamanho mínimo, em bytes, de uma alocação genérica em C++ (do tipo feito por contêineres padrão, strings, crescimento de `std::vector`, smart pointers etc.) que pode gerar `MEMORY_LIMIT_EXCEEDED` quando `max_server_memory_usage` for atingido. Alocações genéricas menores ainda são contabilizadas pelo rastreador de memória, mas podem ser concluídas com sucesso mesmo depois de ultrapassar o limite, o que reduz falhas espúrias durante rotinas de limpeza e tratamento de exceções próximas de OOM.

As alocações feitas pelos próprios buffers grandes do ClickHouse — dados de coluna, tabelas hash, arenas, pipelines de consulta, buffers de E/S — sempre respeitam o limite de memória e podem gerar uma exceção independentemente desse valor. Essa configuração afeta apenas alocações implícitas que passam por `operator new`.

Um valor de `0` (padrão) preserva o comportamento legado: alocações implícitas de `operator new` nunca são recusadas pelo rastreador de memória, e apenas alocações explícitas do ClickHouse podem gerar `MEMORY_LIMIT_EXCEEDED`.

Observação: para evitar efeitos colaterais, recomenda-se definir um valor maior que `max_untracked_memory`.

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Sim" />

Razão mínima entre os tempos de espera da CPU do SO (métrica `OSCPUWaitMicroseconds`) e de atividade (métrica `OSCPUVirtualTimeMicroseconds`) para considerar encerrar conexões. A interpolação linear entre a razão mínima e a máxima é usada para calcular a probabilidade; nesse ponto, a probabilidade é 0.
Consulte [Controle do comportamento em caso de sobrecarga de CPU no servidor](/pt-BR/concepts/features/configuration/settings/server-overload) para mais detalhes.
