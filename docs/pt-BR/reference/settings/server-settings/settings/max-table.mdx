Essas configurações configuram o servidor ClickHouse e são autogeradas a partir do código-fonte do ClickHouse.

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Se o número de tabelas for maior que esse valor, o servidor lançará uma exceção.

As tabelas a seguir não são contabilizadas:

* view
* remote
* dictionary
* system

Conta apenas as tabelas dos seguintes motores de banco de dados:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  O valor `0` significa que não há limitação.
</Note>

**Exemplo**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

Se o número de tabelas anexadas exceder o valor especificado, o servidor ClickHouse adicionará mensagens de aviso na tabela `system.warnings`.

**Exemplo**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

Restrição para excluir tabelas.

Se o tamanho de uma tabela [MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree) exceder `max_table_size_to_drop` (em bytes), você não poderá excluí-la usando uma consulta [`DROP`](/pt-BR/reference/statements/drop) ou [`TRUNCATE`](/pt-BR/reference/statements/truncate).

<Note>
  Um valor de `0` significa que você pode excluir todas as tabelas sem nenhuma restrição.

  Esta configuração não requer reinicialização do servidor ClickHouse para ser aplicada. Outra forma de desabilitar a restrição é criar o arquivo `<clickhouse-path>/flags/force_drop_table`.
</Note>

**Exemplo**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
