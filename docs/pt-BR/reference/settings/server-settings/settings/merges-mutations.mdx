Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="merges_mutations_memory_usage_soft_limit">
  ## merges_mutations_memory_usage_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Define o limite de quanta RAM pode ser usada para executar operações de merge e mutação.
Se o ClickHouse atingir o limite definido, ele não agendará novas operações de merge ou mutação em segundo plano, mas continuará executando as tarefas já agendadas.

<Note>
  Um valor de `0` significa sem limite.
</Note>

**Exemplo**

```xml
<merges_mutations_memory_usage_soft_limit>0</merges_mutations_memory_usage_soft_limit>
```

<div id="merges_mutations_memory_usage_to_ram_ratio">
  ## merges_mutations_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.5" changeable_without_restart="No" />

O valor padrão de `merges_mutations_memory_usage_soft_limit` é calculado por `memory_amount * merges_mutations_memory_usage_to_ram_ratio`.
