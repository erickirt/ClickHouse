Essas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="allow_feature_tier">
  ## allow_feature_tier
</div>

<SettingsInfoBlock type="UInt32" default_value="0" changeable_without_restart="Yes" />

Controla se o usuário pode alterar configurações relacionadas aos diferentes níveis de recursos.

* `0` - São permitidas alterações em qualquer configuração (experimental, beta, produção).
* `1` - São permitidas apenas alterações em configurações de recursos beta e de produção. Alterações em configurações experimentais são rejeitadas.
* `2` - São permitidas apenas alterações em configurações de recursos de produção. Alterações em configurações experimentais ou beta são rejeitadas.

Isso equivale a definir uma restrição readonly em todos os recursos `EXPERIMENTAL` / `BETA`.

<Note>
  Um valor de `0` significa que todas as configurações podem ser alteradas.
</Note>

<div id="allow_impersonate_user">
  ## allow_impersonate_user
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Habilita/desabilita o recurso IMPERSONATE (EXECUTE AS target&#95;user). A configuração foi descontinuada.

<div id="allow_implicit_no_password">
  ## allow_implicit_no_password
</div>

Proíbe a criação de um usuário sem senha, a menos que &#39;IDENTIFIED WITH no&#95;password&#39; seja especificado explicitamente.

```xml
<allow_implicit_no_password>1</allow_implicit_no_password>
```

<div id="allow_no_password">
  ## allow_no_password
</div>

Define se é permitido usar o tipo de senha inseguro no&#95;password.

```xml
<allow_no_password>1</allow_no_password>
```

<div id="allow_plaintext_password">
  ## allow_plaintext_password
</div>

Define se tipos de senha em texto puro (inseguros) são permitidos.

```xml
<allow_plaintext_password>1</allow_plaintext_password>
```
