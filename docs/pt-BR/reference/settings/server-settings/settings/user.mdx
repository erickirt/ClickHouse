Essas configurações configuram o servidor ClickHouse e são geradas automaticamente com base no código-fonte do ClickHouse.

<div id="user_directories">
  ## user_directories
</div>

Seção do arquivo de configuração que contém as seguintes configurações:

* Caminho para o arquivo de configuração com usuários predefinidos.
* Caminho para a pasta em que os usuários criados por comandos SQL são armazenados.
* Caminho do nó do ZooKeeper em que os usuários criados por comandos SQL são armazenados e replicados.

Se esta seção estiver especificada, os caminhos de [users&#95;config](/pt-BR/reference/settings/server-settings/settings/users#users_config) e [access&#95;control&#95;path](/pt-BR/reference/settings/server-settings/settings/access-control#access_control_path) não serão usados.

A seção `user_directories` pode conter qualquer número de itens; a ordem dos itens determina sua precedência (quanto mais acima o item, maior a precedência).

**Exemplos**

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <local_directory>
        <path>/var/lib/clickhouse/access/</path>
    </local_directory>
</user_directories>
```

Usuários, roles, políticas em nível de linha, cotas e perfis também podem ser armazenados no ZooKeeper:

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <replicated>
        <zookeeper_path>/clickhouse/access/</zookeeper_path>
    </replicated>
</user_directories>
```

Você também pode definir as seções `memory` — que significa armazenar informações apenas na memória, sem gravá-las em disco — e `ldap` — que significa armazenar informações em um servidor LDAP.

Para adicionar um servidor LDAP como diretório remoto de usuários que não estão definidos localmente, defina uma única seção `ldap` com as seguintes configurações:

| Setting  | Description                                                                                                                                                                                                                                                                                                                                                                                      |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `roles`  | seção com uma lista de roles definidas localmente que serão atribuídas a cada usuário recuperado do servidor LDAP. Se nenhuma role for especificada, o usuário não poderá executar nenhuma ação após a autenticação. Se alguma das roles listadas não estiver definida localmente no momento da autenticação, a tentativa de autenticação falhará como se a senha fornecida estivesse incorreta. |
| `server` | um dos nomes de servidores LDAP definidos na seção de configuração `ldap_servers`. Este parâmetro é obrigatório e não pode estar vazio.                                                                                                                                                                                                                                                          |

**Exemplo**

```xml
<ldap>
    <server>my_ldap_server</server>
        <roles>
            <my_local_role1 />
            <my_local_role2 />
        </roles>
</ldap>
```

<div id="user_files_path">
  ## user_files_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_files/" changeable_without_restart="Não" />

O diretório que contém os arquivos do usuário. Usado na table function [file()](/pt-BR/reference/functions/table-functions/file), [fileCluster()](/pt-BR/reference/functions/table-functions/fileCluster).

**Exemplo**

```xml
<user_files_path>/var/lib/clickhouse/user_files/</user_files_path>
```

<div id="user_profile_events_per_cpu">
  ## user_profile_events_per_cpu
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="Não" />Divide os contadores `ProfileEvents` por usuário entre as CPUs para evitar contenção de linha de cache entre CPUs quando há muitas threads. Cada usuário consome `getNumCPUs() * num_events * sizeof(atomic)` de memória (centenas de KiB em máquinas com muitos núcleos). Desative para voltar a usar um único `atomic` compartilhado por evento por usuário, trocando throughput por memória.

<div id="user_scripts_path">
  ## user_scripts_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_scripts/" changeable_without_restart="Não" />

O diretório que contém os arquivos de script do usuário. Usado por funções definidas pelo usuário executáveis [Funções definidas pelo usuário executáveis](/pt-BR/reference/functions/regular-functions/udf#executable-user-defined-functions).

**Exemplo**

```xml
<user_scripts_path>/var/lib/clickhouse/user_scripts/</user_scripts_path>
```
