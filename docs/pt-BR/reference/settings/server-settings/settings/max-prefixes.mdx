Essas configurações definem o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="max_prefixes_deserialization_thread_pool_free_size">
  ## max_prefixes_deserialization_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Se o número de threads **ociosas** no pool de threads de desserialização de prefixos exceder `max_prefixes_deserialization_thread_pool_free_size`, o ClickHouse liberará os recursos usados pelas threads inativas e reduzirá o tamanho do pool. As threads poderão ser criadas novamente, se necessário.

<div id="max_prefixes_deserialization_thread_pool_size">
  ## max_prefixes_deserialization_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="Sim" />

O ClickHouse usa threads do pool de threads de desserialização de prefixos para fazer a leitura paralela dos metadados de colunas e subcolunas a partir dos prefixos de arquivo em partes Wide do MergeTree. `max_prefixes_deserialization_thread_pool_size` limita o número máximo de threads no pool.
