Estas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="logger.async">
  ## logger.async
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Quando `<true>` (padrão), o logging será assíncrono (uma thread em segundo plano por canal de saída). Caso contrário, o registro será feito na thread que chama LOG.

<div id="logger.async_queye_max_size">
  ## logger.async_queye_max_size
</div>

<SettingsInfoBlock type="UInt64" default_value="65536" changeable_without_restart="No" />Ao usar logging assíncrono, este é o número máximo de mensagens mantidas na fila enquanto aguardam o flush. As mensagens excedentes serão descartadas.
