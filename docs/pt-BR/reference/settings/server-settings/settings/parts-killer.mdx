Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="parts_killer_max_condemned_parts_per_batch">
  ## parts_killer_max_condemned_parts_per_batch
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="100000" changeable_without_restart="No" />

Número máximo de partes condenadas que uma tabela SharedMergeTree obtém do Keeper em um único ciclo de limpeza. Limitar o lote mantém a listagem e o multi-get dentro do timeout da operação do Keeper, para que a limpeza continue progredindo mesmo quando uma tabela tiver acumulado um volume muito grande de partes condenadas pendentes. Deve ser maior que zero: um valor de `0` faria o Keeper retornar uma listagem vazia em cada ciclo, interrompendo permanentemente a limpeza. Disponível apenas no ClickHouse Cloud

<div id="parts_killer_pool_size">
  ## parts_killer_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="128" changeable_without_restart="No" />

Threads para limpeza de partes pelo parts killer do shared merge tree. Disponível apenas no ClickHouse Cloud
