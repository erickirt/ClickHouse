Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="Não" />

Caminho no sistema de arquivos local para armazenar dados temporários ao processar consultas grandes.

<Note>
  * Apenas uma opção pode ser usada para configurar o armazenamento de dados temporários: tmp&#95;path, tmp&#95;policy, temporary&#95;data&#95;in&#95;cache.
  * A barra no final é obrigatória.
</Note>

**Exemplo**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

Política de armazenamento para dados temporários. Todos os arquivos com o prefixo `tmp` serão removidos na inicialização.

<Note>
  Recomendações para usar armazenamento de objetos como `tmp_policy`:

  * Use um `bucket:path` separado em cada servidor
  * Use `metadata_type=plain`
  * Você também pode querer definir TTL para esse bucket
</Note>

<Note>
  * Apenas uma opção pode ser usada para configurar o armazenamento de dados temporários: `tmp_path`, `tmp_policy`, `temporary_data_in_cache`.
  * `move_factor`, `keep_free_space_bytes` e `max_data_part_size_bytes` são ignorados.
  * A política deve ter exatamente *um volume*

  Para mais informações, consulte a documentação de [motor de tabela MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree).
</Note>

**Exemplo**

Quando `/disk1` estiver cheio, os dados temporários serão armazenados em `/disk2`.

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
