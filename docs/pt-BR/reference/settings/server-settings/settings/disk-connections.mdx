Estas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Uma exceção é lançada ao tentar criar uma conexão quando esse limite é atingido. Defina como 0 para desativar o limite rígido. O limite se aplica às conexões dos disks.

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />O tamanho da opção SO&#95;RCVBUF para conexões de disco (S3, Azure, GCS). O valor padrão é 204800 (200 KB): o armazenamento de objetos é acessado dentro da mesma região, onde o produto largura de banda-latência fica bem abaixo de 100 KB, portanto um limite pequeno oferece vazão máxima e evita os buffers de vários MiB por socket que, de outra forma, o ajuste automático do kernel alocaria. Defina como 0 para usar o ajuste automático TCP do kernel para o buffer de recepção. Observação: alterar essa configuração de volta para 0 restaura o ajuste automático apenas para conexões recém-criadas; as conexões existentes no pool mantêm tamanhos de buffer fixos até serem recriadas.

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />O tamanho da opção `SO_SNDBUF` para conexões de disco (S3, Azure, GCS). Se definido com um valor maior que 0, substitui o ajuste automático de TCP do kernel para o buffer de envio. 0 = padrão do kernel (ajuste automático). Nota: alterar essa configuração de volta para 0 restaura o ajuste automático apenas para conexões criadas posteriormente; as conexões em pool existentes mantêm tamanhos de buffer fixos até serem recriadas.

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Não" />As conexões acima desse limite têm um tempo de vida útil significativamente menor. O limite se aplica às conexões com os disks.

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="Não" />Conexões acima desse limite são encerradas após o uso. Defina como 0 para desativar o cache de conexões. O limite se aplica às conexões dos disks.

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />Mensagens de aviso são gravadas nos logs se o número de conexões em uso for maior que esse limite. O limite se aplica às conexões de disks.
