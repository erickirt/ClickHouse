Essas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="storage_connections_hard_limit">
  ## storage_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Uma exceção é lançada ao tentar criar uma conexão quando esse limite é atingido. Defina como 0 para desativar a limitação rígida. O limite se aplica às conexões de armazenamento.

<div id="storage_connections_rcvbuf">
  ## storage_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />O tamanho da opção SO&#95;RCVBUF para conexões de armazenamento (replicação, consultas distribuídas). Se definido como um valor maior que 0, substitui o autotuning de TCP do kernel para o buffer de recepção. 0 = padrão do kernel (autotuning). Observação: alterar essa configuração de volta para 0 restaura o autotuning apenas para conexões recém-criadas; as conexões em pool existentes mantêm tamanhos de buffer fixos até serem recriadas.

<div id="storage_connections_sndbuf">
  ## storage_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />O tamanho da opção SO&#95;SNDBUF para conexões de armazenamento (replicação, consultas distribuídas). Se for definido como um valor maior que 0, substitui o autotuning do TCP do kernel para o buffer de envio. 0 = padrão do kernel (autotuning). Observação: alterar essa configuração de volta para 0 restaura o autotuning apenas para conexões recém-criadas; conexões em pool existentes mantêm tamanhos de buffer fixos até serem recriadas.

<div id="storage_connections_soft_limit">
  ## storage_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Conexões acima desse limite têm um tempo de vida útil significativamente menor. O limite se aplica às conexões de armazenamento.

<div id="storage_connections_store_limit">
  ## storage_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />As conexões acima desse limite são reiniciadas após o uso. Defina como 0 para desativar o cache de conexões. O limite se aplica às conexões de armazenamento.

<div id="storage_connections_warn_limit">
  ## storage_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />Mensagens de aviso são registradas nos logs se o número de conexões em uso for maior que esse limite. O limite se aplica às conexões de armazenamento.
