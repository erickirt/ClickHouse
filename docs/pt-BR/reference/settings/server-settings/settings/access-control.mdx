Estas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="access_control_improvements">
  ## access_control_improvements
</div>

Configurações para melhorias opcionais no sistema de controle de acesso.

| Setting                                         | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            | Default |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------- |
| `on_cluster_queries_require_cluster_grant`      | Define se consultas `ON CLUSTER` exigem o privilégio `CLUSTER`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | `true`  |
| `role_cache_expiration_time_seconds`            | Define o número de segundos, desde o último acesso, durante os quais uma função é armazenada no cache de funções.                                                                                                                                                                                                                                                                                                                                                                                                                                                     | `600`   |
| `select_from_information_schema_requires_grant` | Define se `SELECT * FROM information_schema.<table>` exige algum privilégio ou pode ser executada por qualquer usuário. Se definido como true, essa consulta exigirá `GRANT SELECT ON information_schema.<table>`, assim como no caso de tabelas comuns.                                                                                                                                                                                                                                                                                                               | `true`  |
| `select_from_system_db_requires_grant`          | Define se `SELECT * FROM system.<table>` exige algum privilégio ou pode ser executada por qualquer usuário. Se definido como true, essa consulta exigirá `GRANT SELECT ON system.<table>`, assim como no caso de tabelas que não são do sistema. Exceções: algumas system tables (`tables`, `columns`, `databases` e algumas tabelas constantes, como `one` e `contributors`) continuam acessíveis a todos; além disso, se houver um privilégio `SHOW` concedido (por exemplo, `SHOW USERS`), a system table correspondente (isto é, `system.users`) ficará acessível. | `true`  |
| `settings_constraints_replace_previous`         | Define se uma constraint em um profile de settings para determinada configuração cancelará as actions da constraint anterior (definida em outros profiles) para essa configuração, incluindo campos que não forem definidos pela nova constraint. Isso também habilita o tipo de constraint `changeable_in_readonly`.                                                                                                                                                                                                                                                  | `true`  |
| `table_engines_require_grant`                   | Define se a criação de uma tabela com um table engine específico exige um privilégio.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  | `false` |
| `throw_on_unmatched_row_policies`               | Define se a leitura de uma tabela deve lançar uma exceção caso a tabela tenha row policies, mas nenhuma delas se aplique ao current user                                                                                                                                                                                                                                                                                                                                                                                                                               | `false` |
| `users_without_row_policies_can_read_rows`      | Define se usuários sem row policies permissivas ainda podem ler linhas usando uma consulta `SELECT`. Por exemplo, se houver dois usuários, A e B, e uma row policy estiver definida apenas para A, então, se essa configuração for true, o usuário B verá todas as linhas. Se essa configuração for false, o usuário B não verá nenhuma linha.                                                                                                                                                                                                                         | `true`  |

Exemplo:

```xml
<access_control_improvements>
    <throw_on_unmatched_row_policies>true</throw_on_unmatched_row_policies>
    <users_without_row_policies_can_read_rows>true</users_without_row_policies_can_read_rows>
    <on_cluster_queries_require_cluster_grant>true</on_cluster_queries_require_cluster_grant>
    <select_from_system_db_requires_grant>true</select_from_system_db_requires_grant>
    <select_from_information_schema_requires_grant>true</select_from_information_schema_requires_grant>
    <settings_constraints_replace_previous>true</settings_constraints_replace_previous>
    <table_engines_require_grant>false</table_engines_require_grant>
    <role_cache_expiration_time_seconds>600</role_cache_expiration_time_seconds>
</access_control_improvements>
```

<div id="access_control_path">
  ## access_control_path
</div>

Caminho para uma pasta onde um servidor ClickHouse armazena as configurações de usuários e funções criadas por comandos SQL.
