Estas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="disable_insertion_and_mutation">
  ## disable_insertion_and_mutation
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Desabilita consultas insert, alter e delete. Esta configuração deve ser habilitada quando forem necessários nós somente leitura, para evitar que inserções e mutações afetem o desempenho de leitura. Inserções em motores externos (S3, DataLake, MySQL, PostrgeSQL, Kafka, etc) são permitidas apesar desta configuração.

<div id="disable_internal_dns_cache">
  ## disable_internal_dns_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Desabilita o cache DNS interno. Recomendado para executar o ClickHouse em sistemas com infraestrutura que muda com frequência, como o Kubernetes.

<div id="disable_tunneling_for_https_requests_over_http_proxy">
  ## disable_tunneling_for_https_requests_over_http_proxy
</div>

Por padrão, o tunelamento (ou seja, `HTTP CONNECT`) é usado para fazer solicitações `HTTPS` por meio de um proxy `HTTP`. Essa configuração pode ser usada para desativá-lo.

**no&#95;proxy**

Por padrão, todas as solicitações passarão pelo proxy. Para desativá-lo para hosts específicos, a variável `no_proxy` deve ser definida.
Ela pode ser definida dentro da cláusula `<proxy>` para resolvers de lista e remotos e como variável de ambiente para o resolver de ambiente.
Ela oferece suporte a endereços IP, domínios, subdomínios e ao caractere curinga `'*'` para ignorar totalmente o proxy. Pontos no início são removidos, assim como no curl.

**Exemplo**

A configuração abaixo ignora o proxy para solicitações destinadas a `clickhouse.cloud` e todos os seus subdomínios (por exemplo, `auth.clickhouse.cloud`).
O mesmo vale para o GitLab, mesmo com um ponto no início. Tanto `gitlab.com` quanto `about.gitlab.com` ignoram o proxy.

```xml
<proxy>
    <no_proxy>clickhouse.cloud,.gitlab.com</no_proxy>
    <http>
        <uri>http://proxy1</uri>
        <uri>http://proxy2:3128</uri>
    </http>
    <https>
        <uri>http://proxy1:3128</uri>
    </https>
</proxy>
```
