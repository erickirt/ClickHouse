Estas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="max_thread_pool_free_size">
  ## max_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />

Se o número de threads **ociosas** no Global Thread pool for maior que [`max_thread_pool_free_size`](/pt-BR/reference/settings/server-settings/settings/max-thread#max_thread_pool_free_size), o ClickHouse libera os recursos ocupados por algumas threads, e o tamanho do pool é reduzido. As threads podem ser criadas novamente, se necessário.

**Exemplo**

```xml
<max_thread_pool_free_size>1200</max_thread_pool_free_size>
```

<div id="max_thread_pool_size">
  ## max_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />

O ClickHouse usa threads do Global Thread pool para processar consultas. Se não houver nenhuma thread ociosa para processar uma consulta, uma nova thread será criada no pool. `max_thread_pool_size` limita o número máximo de threads no pool.

**Exemplo**

```xml
<max_thread_pool_size>12000</max_thread_pool_size>
```
