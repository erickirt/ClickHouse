Essas configurações definem o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="jemalloc_enable_background_threads">
  ## jemalloc_enable_background_threads
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Habilita as threads em segundo plano do jemalloc. O jemalloc usa threads em segundo plano para liberar páginas de memória não utilizadas. Desativá-las pode causar degradação no desempenho.

<div id="jemalloc_enable_global_profiler">
  ## jemalloc_enable_global_profiler
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Habilita o profiler de alocação do jemalloc para todas as threads. O jemalloc fará a amostragem das alocações e de todas as desalocações das alocações amostradas.
Os perfis podem ser descarregados usando SYSTEM JEMALLOC FLUSH PROFILE, que pode ser usado para análise de alocação.
As amostras também podem ser armazenadas em system.trace&#95;log usando a config jemalloc&#95;collect&#95;global&#95;profile&#95;samples&#95;in&#95;trace&#95;log ou com o setting de consulta jemalloc&#95;collect&#95;profile&#95;samples&#95;in&#95;trace&#95;log.
Veja [Profiling de alocações](/pt-BR/concepts/features/performance/allocation-profiling)
