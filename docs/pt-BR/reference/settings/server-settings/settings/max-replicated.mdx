Essas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="max_replicated_fetches_network_bandwidth_for_server">
  ## max_replicated_fetches_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />A velocidade máxima de transferência de dados pela rede, em bytes por segundo, para fetches replicados. Zero significa sem limite.

<div id="max_replicated_sends_network_bandwidth_for_server">
  ## max_replicated_sends_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />A velocidade máxima de transferência de dados pela rede, em bytes por segundo, para envios replicados. Zero significa ilimitado.

<div id="max_replicated_table_num_to_throw">
  ## max_replicated_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Se o número de tabelas replicadas for maior que esse valor, o servidor lançará uma exceção.

Conta apenas as tabelas dos motores de banco de dados:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  O valor `0` significa que não há limitação.
</Note>

**Exemplo**

```xml
<max_replicated_table_num_to_throw>400</max_replicated_table_num_to_throw>
```
