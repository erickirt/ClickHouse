Essas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="distributed_cache_apply_throttling_settings_from_client">
  ## distributed_cache_apply_throttling_settings_from_client
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Indica se o servidor de cache deve aplicar as configurações de throttling recebidas do cliente.

<div id="distributed_cache_keep_up_free_connections_ratio">
  ## distributed_cache_keep_up_free_connections_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.1" changeable_without_restart="No" />Limite flexível para o número de conexões ativas livres que o cache distribuído tentará manter. Quando o número de conexões livres ficar abaixo de distributed&#95;cache&#95;keep&#95;up&#95;free&#95;connections&#95;ratio * max&#95;connections, as conexões com a atividade mais antiga serão fechadas até que esse número volte a ficar acima do limite.

<div id="distributed_cache_write_pool_size">
  ## distributed_cache_write_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />O número máximo de solicitações de gravação do cache distribuído executadas simultaneamente em uma background thread (em todas as queries). Quando o limite é atingido, uma gravação passa pelo cache inline (na thread que fez a chamada), em vez de por uma background thread, de modo que não fica bloqueada aguardando um slot nem cria um número ilimitado de threads.
