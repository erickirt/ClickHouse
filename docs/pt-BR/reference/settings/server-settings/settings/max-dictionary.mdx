Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="max_dictionary_num_to_throw">
  ## max_dictionary_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Se o número de dicionários for maior que esse valor, o servidor gerará uma exceção.

Conta apenas as tabelas dos motores de banco de dados:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  Um valor de `0` significa que não há limitação.
</Note>

**Exemplo**

```xml
<max_dictionary_num_to_throw>400</max_dictionary_num_to_throw>
```

<div id="max_dictionary_num_to_warn">
  ## max_dictionary_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Yes" />

Se o número de dicionários anexados exceder o valor especificado, o servidor ClickHouse adicionará mensagens de aviso à tabela `system.warnings`.

**Exemplo**

```xml
<max_dictionary_num_to_warn>400</max_dictionary_num_to_warn>
```
