Essas configurações definem o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="custom_cached_disks_base_directory">
  ## custom_cached_disks_base_directory
</div>

Essa configuração especifica o caminho do cache para discos em cache personalizados (criados via SQL).
`custom_cached_disks_base_directory` tem prioridade mais alta para discos personalizados em relação a `filesystem_caches_path` (encontrado em `filesystem_caches_path.xml`),
que é usado se o primeiro não estiver presente.
O caminho da configuração do filesystem cache deve estar dentro desse diretório,
caso contrário, será lançada uma exceção, impedindo a criação do disco.

<Note>
  Isso não afeta discos criados em uma versão mais antiga para os quais o servidor passou por upgrade.
  Nesse caso, nenhuma exceção será lançada, para permitir que o servidor seja iniciado com sucesso.
</Note>

Exemplo:

```xml
<custom_cached_disks_base_directory>/var/lib/clickhouse/caches/</custom_cached_disks_base_directory>
```

<div id="custom_settings_prefixes">
  ## custom_settings_prefixes
</div>

Lista de prefixos usados em [configurações personalizadas](/pt-BR/concepts/features/configuration/settings/settings-query-level#custom_settings).
Vários prefixos devem ser separados por vírgulas.

**Exemplo**

```xml
<custom_settings_prefixes>SQL_</custom_settings_prefixes>
```
