Estas configurações definem o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Não" />

Define o número de threads que executam tarefas de carregamento assíncronas no pool em segundo plano. O pool em segundo plano é usado para carregar tabelas de forma assíncrona após a inicialização do servidor, caso não haja consultas aguardando a tabela. Pode ser vantajoso manter um número baixo de threads no pool em segundo plano se houver muitas tabelas. Isso reservará recursos de CPU para a execução concorrente de consultas.

<Note>
  Um valor de `0` significa que todas as CPUs disponíveis serão usadas.
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Define o número de threads que executam tarefas de carregamento no pool em primeiro plano. O pool em primeiro plano é usado para carregar tabelas de forma síncrona antes de o servidor começar a escutar em uma porta e para carregar tabelas que estão sendo aguardadas. O pool em primeiro plano tem prioridade mais alta que o pool em segundo plano. Isso significa que nenhuma tarefa é iniciada no pool em segundo plano enquanto houver tarefas em execução no pool em primeiro plano.

<Note>
  Um valor de `0` significa que todas as CPUs disponíveis serão usadas.
</Note>
