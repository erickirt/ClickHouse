Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="backup_log">
  ## backup_log
</div>

Configurações da tabela do sistema [backup&#95;log](/pt-BR/reference/system-tables/backup_log) para registrar operações `BACKUP` e `RESTORE`.

<SystemLogParameters />

**Exemplo**

```xml
<clickhouse>
    <backup_log>
        <database>system</database>
        <table>backup_log</table>
        <flush_interval_milliseconds>1000</flush_interval_milliseconds>
        <partition_by>toYYYYMM(event_date)</partition_by>
        <max_size_rows>1048576</max_size_rows>
        <reserved_size_rows>8192</reserved_size_rows>
        <buffer_size_rows_flush_threshold>524288</buffer_size_rows_flush_threshold>
        <flush_on_crash>false</flush_on_crash>
        <!-- <engine>Engine = MergeTree PARTITION BY event_date ORDER BY event_time TTL event_date + INTERVAL 30 day</engine> -->
    </backup_log>
</clickhouse>
```

<div id="backup_threads">
  ## backup_threads
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="16" changeable_without_restart="No" />O número máximo de threads para executar solicitações `BACKUP`.
