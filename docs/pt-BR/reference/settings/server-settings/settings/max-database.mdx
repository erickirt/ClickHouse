Essas configurações definem o servidor ClickHouse e são geradas automaticamente com base no código-fonte do ClickHouse.

<div id="max_database_num_to_throw">
  ## max_database_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Se o número de bancos de dados for maior que esse valor, o servidor gerará uma exceção. 0 significa sem limite.

<div id="max_database_num_to_warn">
  ## max_database_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Yes" />

Se o número de bancos de dados anexados exceder o valor especificado, o servidor ClickHouse adicionará mensagens de aviso à tabela `system.warnings`.

**Exemplo**

```xml
<max_database_num_to_warn>50</max_database_num_to_warn>
```

<div id="max_database_replicated_create_table_thread_pool_size">
  ## max_database_replicated_create_table_thread_pool_size
</div>

<SettingsInfoBlock type="UInt32" default_value="1" changeable_without_restart="No" />Número de threads para criar tabelas durante a recuperação de réplicas no DatabaseReplicated. Zero significa um número de threads igual ao número de núcleos.
