Essas configurações definem o servidor do ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="listen_backlog">
  ## listen_backlog
</div>

<SettingsInfoBlock type="UInt32" default_value="4096" changeable_without_restart="No" />

Backlog (tamanho da fila de conexões pendentes) do socket de escuta. O valor padrão de `<4096>` é o mesmo que o do Linux 5.4+).

Normalmente, esse valor não precisa ser alterado, porque:

* O valor padrão é grande o suficiente,
* Para aceitar conexões de clientes, o servidor tem uma thread separada.

Portanto, mesmo que você tenha `<TcpExtListenOverflows>` (de `<nstat>`) diferente de zero e esse contador aumente no servidor ClickHouse, isso não significa que esse valor precise ser aumentado, porque:

* Normalmente, se `<4096>` não for suficiente, isso indica algum problema interno de scaling do ClickHouse, então é melhor relatar um issue.
* Isso não significa que o servidor conseguirá lidar com mais conexões depois (e, mesmo que conseguisse, nesse momento os clientes podem já ter ido embora ou sido desconectados).

**Exemplo**

```xml
<listen_backlog>4096</listen_backlog>
```

<div id="listen_host">
  ## listen_host
</div>

Restrição aos hosts de onde as solicitações podem vir. Se você quiser que o servidor responda em todos eles, especifique `::`.

Exemplos:

```xml
<listen_host>::1</listen_host>
<listen_host>127.0.0.1</listen_host>
```

<div id="listen_reuse_port">
  ## listen_reuse_port
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Permite que vários servidores escutem no mesmo endereço:porta. As solicitações serão encaminhadas pelo sistema operacional para um servidor escolhido aleatoriamente. Não é recomendável habilitar essa configuração.

**Exemplo**

```xml
<listen_reuse_port>0</listen_reuse_port>
```

<div id="listen_try">
  ## listen_try
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

O servidor não será encerrado se as redes IPv6 ou IPv4 estiverem indisponíveis ao tentar escutar.

**Exemplo**

```xml
<listen_try>0</listen_try>
```
