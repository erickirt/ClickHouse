Essas configurações são usadas para configurar o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="distributed_ddl.cleanup_delay_period">
  ## distributed_ddl.cleanup_delay_period
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />a limpeza começa após o recebimento de um evento de novo nó, se a última limpeza não tiver sido feita há menos de `<cleanup_delay_period>` segundos.

<div id="distributed_ddl.max_tasks_in_queue">
  ## distributed_ddl.max_tasks_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />o número máximo de tarefas que podem ficar na fila.

<div id="distributed_ddl.path">
  ## distributed_ddl.path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/ddl/" changeable_without_restart="No" />o caminho no Keeper da `<task_queue>` para consultas DDL

<div id="distributed_ddl.pool_size">
  ## distributed_ddl.pool_size
</div>

<SettingsInfoBlock type="Int32" default_value="1" changeable_without_restart="No" />número de consultas `<ON CLUSTER>` que podem ser executadas simultaneamente

<div id="distributed_ddl.profile">
  ## distributed_ddl.profile
</div>

o perfil usado para executar consultas DDL

<div id="distributed_ddl.replicas_path">
  ## distributed_ddl.replicas_path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/replicas/" changeable_without_restart="No" />o caminho no Keeper para a `<task_queue>` de réplicas

<div id="distributed_ddl.task_max_lifetime">
  ## distributed_ddl.task_max_lifetime
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" changeable_without_restart="No" />remove o nó se sua idade for maior que este valor.
