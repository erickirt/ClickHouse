Estas configurações configuram o servidor ClickHouse e são geradas automaticamente a partir do código-fonte do ClickHouse.

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Coleta alocações aleatórias com tamanho menor ou igual ao valor especificado, com probabilidade igual a `total_memory_profiler_sample_probability`. 0 significa desabilitado. Talvez seja interessante definir &#39;max&#95;untracked&#95;memory&#39; como 0 para que esse limiar funcione como esperado.

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Coleta alocações aleatórias com tamanho maior ou igual ao valor especificado, com probabilidade igual a `total_memory_profiler_sample_probability`. 0 significa que está desabilitado. Talvez seja necessário definir &#39;max&#95;untracked&#95;memory&#39; como 0 para que esse limiar funcione conforme o esperado.

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Sempre que o uso de memória do servidor ultrapassar cada próximo passo em número de bytes, o profiler de memória coletará o stack trace de alocação. Zero desabilita o profiler de memória. Valores abaixo de alguns megabytes deixarão o servidor mais lento.

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="No" />

Permite coletar alocações e desalocações aleatórias e gravá-las na tabela de sistema [system.trace&#95;log](/pt-BR/reference/system-tables/trace_log), com `trace_type` igual a `MemorySample`, com a probabilidade especificada. A probabilidade se aplica a cada alocação ou desalocação, independentemente do tamanho da alocação. Observe que a amostragem ocorre somente quando a quantidade de memória não rastreada excede o limite de memória não rastreada (o valor padrão é `4` MiB). Esse limite pode ser reduzido se [total&#95;memory&#95;profiler&#95;step](/pt-BR/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step) for reduzido. Você pode definir `total_memory_profiler_step` como `1` para uma amostragem extra granular.

Valores possíveis:

* Double positivo.
* `0` — A gravação de alocações e desalocações aleatórias na tabela de sistema `system.trace_log` fica desabilitada.
