Essas configurações estão disponíveis em [system.settings](/pt-BR/reference/system-tables/settings) e são geradas automaticamente com base no [código-fonte](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="log_query_settings">
  ## log_query_settings
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Registra as configurações da consulta no query&#95;log e no log de spans do OpenTelemetry.

<div id="log_query_threads">
  ## log_query_threads
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Configura o registro em log das threads de consulta.

As threads de consulta são registradas na tabela [system.query&#95;thread&#95;log](/pt-BR/reference/system-tables/query_thread_log). Esta configuração só tem efeito quando [log&#95;queries](/pt-BR/reference/settings/session-settings/log-queries#log_queries) é true. As threads de consulta executadas pelo ClickHouse com essa configuração são registradas de acordo com as regras definidas no parâmetro de configuração do servidor [query&#95;thread&#95;log](/pt-BR/reference/settings/server-settings/settings#query_thread_log).

Valores possíveis:

* 0 — Desabilitado.
* 1 — Habilitado.

**Exemplo**

```text
log_query_threads=1
```

<div id="log_query_views">
  ## log_query_views
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Configura o registro das visualizações de consultas.

Quando uma consulta executada pelo ClickHouse com essa configuração habilitada tiver visualizações associadas (materializadas ou visualizações em tempo real), elas serão registradas no parâmetro de configuração do servidor [query&#95;views&#95;log](/pt-BR/reference/settings/server-settings/settings#query_views_log).

Exemplo:

```text
log_query_views=1
```
