  ## jemalloc_profile_text_collapsed_use_count

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "Nova configuração para agregar pela contagem de alocações em vez de bytes no formato collapsed do perfil de heap do jemalloc"}]}]} />

Ao usar o formato de saída &#39;collapsed&#39; para o perfil de heap do jemalloc, agregue pela contagem de alocações em vez de bytes. Quando false (padrão), cada stack é ponderada por bytes ativos; quando true, pela contagem de alocações ativas.

  ## jemalloc_profile_text_symbolize_with_inline
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "1"},{"label": "Nova configuração para controlar se os frames inline devem ser incluídos ao simbolizar o perfil de heap do jemalloc. Quando habilitada, os frames inline são incluídos, ao custo de uma simbolização mais lenta; quando desabilitada, eles são ignorados para gerar uma saída mais rápida"}]}]} />

Define se os frames inline devem ser incluídos ao simbolizar o perfil de heap do jemalloc. Quando habilitada, os frames inline são incluídos, o que pode tornar o processo de simbolização muito mais lento; quando desabilitada, eles são ignorados. Afeta apenas os formatos de saída &#39;symbolized&#39; e &#39;collapsed&#39;.
