<div id="s3_create_new_file_on_insert">
  ## s3_create_new_file_on_insert
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Habilita ou desabilita a criação de um novo arquivo a cada inserção em tabelas com o motor S3. Se estiver habilitado, a cada inserção será criado um novo objeto no S3 com a respectiva chave, seguindo um padrão como este:

inicial: `data.Parquet.gz` -&gt; `data.1.Parquet.gz` -&gt; `data.2.Parquet.gz`, etc.

Valores possíveis:

* 0 — a consulta `INSERT` cria um novo arquivo ou falha se o arquivo já existir e `s3_truncate_on_insert` não estiver definido.
* 1 — a consulta `INSERT` cria um novo arquivo a cada inserção, usando um sufixo (a partir do segundo), se `s3_truncate_on_insert` não estiver definido.

Veja mais detalhes [aqui](/pt-BR/integrations/connectors/data-ingestion/AWS/integrating-s3-with-clickhouse#inserting-data).

<div id="s3_disable_checksum">
  ## s3_disable_checksum
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Não calcule um checksum ao enviar um arquivo para o S3. Isso acelera as gravações ao evitar processamento excessivo no arquivo. Em geral, isso é seguro, pois os dados das tabelas MergeTree já recebem checksums do ClickHouse, e, quando o S3 é acessado via HTTPS, a camada TLS já fornece integridade durante a transferência pela rede. Ainda assim, checksums adicionais no S3 oferecem uma camada extra de proteção.

<div id="s3_slow_all_threads_after_network_error">
  ## s3_slow_all_threads_after_network_error
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "1"},{"label": "Nova configuração"}]}]} />

Quando definido como `true`, todas as threads que executam requisições ao S3 para o mesmo endpoint de backup ficam mais lentas
depois que uma única requisição ao S3 encontra um erro de rede passível de nova tentativa, como timeout de socket.
Quando definido como `false`, cada thread lida com o backoff das requisições ao S3 de forma independente das demais.

<div id="s3_truncate_on_insert">
  ## s3_truncate_on_insert
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Ativa ou desativa o truncamento antes de inserções em tabelas com o motor S3. Se estiver desativado, uma exceção será lançada ao tentar inserir caso um objeto S3 já exista.

Valores possíveis:

* 0 — a consulta `INSERT` cria um novo arquivo ou falha se o arquivo já existir e `s3_create_new_file_on_insert` não estiver definido.
* 1 — a consulta `INSERT` substitui o conteúdo existente do arquivo pelos novos dados.

Veja mais detalhes [aqui](/pt-BR/integrations/connectors/data-ingestion/AWS/integrating-s3-with-clickhouse#inserting-data).

<div id="s3_use_adaptive_timeouts">
  ## s3_use_adaptive_timeouts
</div>

<SettingsInfoBlock type="Bool" default_value="1" />
