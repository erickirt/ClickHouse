<div id="page_cache_block_size">
  ## page_cache_block_size
</div>

<SettingsInfoBlock type="UInt64" default_value="1048576" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "1048576"},{"label": "Tornou essa configuração ajustável por consulta."}]}]} />

Tamanho dos fragmentos de arquivo a serem armazenados no cache de páginas em espaço de usuário, em bytes. Todas as leituras que passam pelo cache serão arredondadas para cima para um múltiplo desse tamanho.

Essa configuração pode ser ajustada por consulta, mas entradas de cache com tamanhos de bloco diferentes não podem ser reutilizadas. Alterar essa configuração invalida, na prática, as entradas existentes no cache.

Um valor mais alto, como 1 MiB, é bom para consultas de alta taxa de transferência, enquanto um valor mais baixo, como 64 KiB, é bom para consultas pontuais de baixa latência.

<div id="page_cache_lookahead_blocks">
  ## page_cache_lookahead_blocks
</div>

<SettingsInfoBlock type="UInt64" default_value="16" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "16"},{"label": "Tornou essa configuração ajustável por consulta."}]}]} />

Em caso de falha de cache no cache de páginas em espaço de usuário, leia de uma só vez até esta quantidade de blocos consecutivos do armazenamento subjacente, caso eles também não estejam no cache. Cada bloco tem `page_cache_block_size` bytes.

Um valor mais alto é bom para consulta de alta taxa de transferência, enquanto consultas pontuais de baixa latência funcionam melhor sem pré-leitura.

<div id="page_cache_max_coalesced_bytes">
  ## page_cache_max_coalesced_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="16777216" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "16777216"},{"label": "Nova configuração para limitar o tamanho de uma única leitura coalescida usada para preencher o cache de páginas em espaço de usuário após uma falha de cache."}]}]} />

Quando `readBigAt` preenche o cache de páginas em espaço de usuário, falhas de cache consecutivas são agrupadas em uma única leitura do armazenamento subjacente. Essa configuração limita o tamanho, em bytes, de uma leitura coalescida; sequências maiores de falhas são divididas em várias leituras. Ela limita o uso transitório de memória do buffer temporário durante leituras frias paralelas.

Um valor mais alto reduz o número de requisições HTTP em varreduras frias no armazenamento de objetos; um valor mais baixo reduz o pico de memória transitória.
