  ## use_skip_indexes

  ## use_skip_indexes_for_top_k

Habilita o uso de data skipping indexes para filtragem TopK.

Quando habilitada, se existir um índice minmax na coluna da consulta `ORDER BY <column> LIMIT n`, o otimizador tentará usar o índice minmax para ignorar grânulos que não sejam relevantes para o resultado final. Isso pode reduzir a latência da consulta.

Valores possíveis:

* 0 — Desabilitado.
* 1 — Habilitado.

<div id="use_skip_indexes_if_final">
  ## use_skip_indexes_if_final
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": "1"},{"label": "Mudança no valor padrão da configuração"}]}]} />

Controla se os skip indexes são usados ao executar uma consulta com o modificador FINAL.

Os skip indexes podem excluir linhas (grânulos) que contêm os dados mais recentes, o que pode levar a resultados incorretos em uma consulta com o modificador FINAL. Quando essa configuração está habilitada, os skip indexes são aplicados mesmo com o modificador FINAL, o que pode melhorar o desempenho, mas com o risco de ignorar atualizações recentes. Essa configuração deve ser habilitada em conjunto com a configuração use&#95;skip&#95;indexes&#95;if&#95;final&#95;exact&#95;mode (o padrão é habilitado).

Valores possíveis:

* 0 — Desabilitado.
* 1 — Habilitado.

  ## use_skip_indexes_if_final_exact_mode
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": "1"},{"label": "Alteração no valor padrão da configuração"}]}, {"id": "row-2","items": [{"label": "25.5"},{"label": "0"},{"label": "Esta configuração foi introduzida para ajudar a consulta FINAL a retornar resultados corretos com skip indexes"}]}]} />

Controla se os grânulos retornados por um skip index são expandidos em partes mais recentes para garantir resultados corretos ao executar uma consulta com o modificador FINAL.

O uso de skip indexes pode excluir linhas (grânulos) que contêm os dados mais recentes, o que pode levar a resultados incorretos. Esta configuração pode garantir o retorno de resultados corretos ao examinar partes mais recentes que se sobrepõem aos intervalos retornados pelo skip index. Esta configuração deve ser desabilitada apenas se resultados aproximados com base na consulta ao skip index forem aceitáveis para uma aplicação.

Valores possíveis:

* 0 — Desabilitado.
* 1 — Habilitado.

Habilita o uso de data skipping indexes durante a leitura de dados.

Quando habilitado, esses índices são avaliados dinamicamente no momento em que cada grânulo de dados é lido, em vez de serem analisados antecipadamente antes do início da execução da consulta. Isso pode reduzir a latência inicial da consulta.

Valores possíveis:

* 0 — Desabilitado.
* 1 — Habilitado.
