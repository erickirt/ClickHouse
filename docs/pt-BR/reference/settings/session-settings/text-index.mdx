  ## text_index_lazy_intersection_density_threshold
</div>

**Aliases**: `text_index_density_threshold`

<SettingsInfoBlock type="Float" default_value="0.2" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0.2"},{"label": "Renomeado de `text_index_density_threshold` (mantido como alias); seleciona o algoritmo de interseção de posting lists no modo lazy de aplicação de posting lists."}]}]} />

Limite de densidade de posting lists que seleciona o algoritmo de interseção no modo lazy de aplicação de posting lists (`text_index_posting_list_apply_mode = 'lazy'`).
Abaixo do limite: interseção leapfrog (favorece posting lists esparsas). No limite ou acima dele: interseção de bitmap por força bruta (favorece posting lists densas).

<div id="text_index_like_max_postings_to_read">
  ## text_index_like_max_postings_to_read
</div>

<SettingsInfoBlock type="UInt64" default_value="50" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "50"},{"label": "Nova configuração"}]}]} />

Requer que `use_text_index_like_evaluation_by_dictionary_scan` esteja habilitado.

<div id="text_index_like_min_pattern_length">
  ## text_index_like_min_pattern_length
</div>

<SettingsInfoBlock type="UInt64" default_value="4" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "4"},{"label": "Nova configuração"}]}]} />

Comprimento mínimo da substring alfanumérica em um padrão LIKE/ILIKE necessário para usar a avaliação de LIKE do índice de texto por varredura do dicionário.
Padrões menores que esse limite correspondem a tokens demais no dicionário e são ignorados para evitar varreduras custosas.

Requer que `use_text_index_like_evaluation_by_dictionary_scan` esteja habilitado.

<div id="text_index_posting_list_apply_mode">
  ## text_index_posting_list_apply_mode
</div>

<SettingsInfoBlock type="TextIndexPostingListApplyMode" default_value="materialize" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "materialize"},{"label": "Nova configuração para o modo lazy de aplicação de posting list"}]}]} />

Controla como as posting lists são aplicadas durante consultas com índice de texto.
&#39;materialize&#39; (padrão) decodifica imediatamente as posting lists em Roaring Bitmaps.
&#39;lazy&#39; usa decodificação sob demanda baseada em cursor (requer um formato de índice com codec serializado).
