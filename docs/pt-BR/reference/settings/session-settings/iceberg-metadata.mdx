<div id="iceberg_metadata_log_level">
  ## iceberg_metadata_log_level
</div>

<SettingsInfoBlock type="IcebergMetadataLogLevel" default_value="none" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "none"},{"label": "Nova configuração."}]}]} />

Controla o nível de registro de metadados de tabelas Iceberg em system.iceberg&#95;metadata&#95;log.
Normalmente, essa configuração pode ser alterada para fins de depuração.

Valores possíveis:

* none - Nenhum registro de metadados.
* metadata - Arquivo raiz metadata.json.
* manifest&#95;list&#95;metadata - Tudo o que está acima + metadados da lista de manifestos Avro correspondente a um snapshot.
* manifest&#95;list&#95;entry - Tudo o que está acima + entradas da lista de manifestos Avro.
* manifest&#95;file&#95;metadata - Tudo o que está acima + metadados dos arquivos de manifesto Avro percorridos.
* manifest&#95;file&#95;entry - Tudo o que está acima + entradas dos arquivos de manifesto Avro percorridos.

<div id="iceberg_metadata_staleness_ms">
  ## iceberg_metadata_staleness_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Nova configuração que permite usar a versão de metadado em cache nas operações READ para evitar a busca no catálogo remoto"}]}]} />

Se não for zero, ignora a busca de metadado do Iceberg no catálogo remoto se houver um snapshot de metadado em cache mais recente do que a janela de obsolescência especificada. Zero significa sempre buscar a versão mais recente de metadado no catálogo remoto. Definir isso com um valor diferente de zero troca atualização por menor latência nas operações de leitura.
