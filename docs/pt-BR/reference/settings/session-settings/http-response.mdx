<div id="http_response_buffer_size">
  ## http_response_buffer_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O número de bytes a serem armazenados em buffer na memória do servidor antes de enviar uma resposta HTTP ao cliente ou gravar no disco (quando http&#95;wait&#95;end&#95;of&#95;query está habilitado).

<div id="http_response_headers">
  ## http_response_headers
</div>

<SettingsInfoBlock type="Map" default_value="{}" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.12"},{"label": ""},{"label": "Nova configuração."}]}]} />

Permite adicionar ou sobrescrever cabeçalhos HTTP que o servidor retornará na resposta com um resultado de consulta bem-sucedido.
Isso afeta apenas a interface HTTP.

Se o cabeçalho já estiver definido por padrão, o valor fornecido o sobrescreverá.
Se o cabeçalho não estiver definido por padrão, ele será adicionado à lista de cabeçalhos.
Os cabeçalhos definidos por padrão pelo servidor e não sobrescritos por esta configuração serão mantidos.

A configuração permite definir um cabeçalho com um valor constante. Atualmente, não há como definir um cabeçalho com um valor calculado dinamicamente.

Nem nomes nem valores podem conter caracteres de controle ASCII.

Se você implementar um aplicativo de UI que permita aos usuários modificar configurações, mas ao mesmo tempo tome decisões com base nos cabeçalhos retornados, recomenda-se restringir esta configuração a readonly.

Exemplo: `SET http_response_headers = $${'Content-Type': 'image/png'}$$`
