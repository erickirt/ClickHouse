<div id="max_temporary_columns">
  ## max_temporary_columns
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O número máximo de colunas temporárias que devem ser mantidas simultaneamente na RAM
durante a execução de uma consulta, incluindo colunas constantes. Se uma consulta gerar
um número de colunas temporárias em memória maior que o especificado como resultado de
um cálculo intermediário, uma exceção será lançada.

<Tip>
  Essa configuração é útil para evitar consultas excessivamente complexas.
</Tip>

O valor `0` significa sem limite.

<div id="max_temporary_data_on_disk_size_for_query">
  ## max_temporary_data_on_disk_size_for_query
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

A quantidade máxima de dados consumidos por arquivos temporários em disco, em bytes, por todas as
consultas em execução simultânea.

Valores possíveis:

* Inteiro positivo.
* `0` — ilimitado (padrão)

<div id="max_temporary_data_on_disk_size_for_user">
  ## max_temporary_data_on_disk_size_for_user
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

A quantidade máxima de dados consumidos por arquivos temporários em disco, em bytes, para todas as
consultas de usuário em execução simultânea.

Valores possíveis:

* Inteiro positivo.
* `0` — ilimitado (padrão)

<div id="max_temporary_non_const_columns">
  ## max_temporary_non_const_columns
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Assim como `max_temporary_columns`, este é o número máximo de colunas temporárias que devem
ser mantidas na RAM simultaneamente ao executar uma consulta, sem contar as
colunas constantes.

<Note>
  Colunas constantes são formadas com bastante frequência ao executar uma consulta, mas exigem
  praticamente zero recursos computacionais.
</Note>
