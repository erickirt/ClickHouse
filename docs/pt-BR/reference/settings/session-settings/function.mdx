<div id="function_base58_max_input_size">
  ## function_base58_max_input_size
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "10000"},{"label": "Nova configuração que limita o tamanho da entrada de `base58Encode`, `base58Decode` e `tryBase58Decode` (cuja conversão é quadrática em relação ao comprimento da entrada) a 10 KB por padrão. O valor de compatibilidade `0` desabilita o limite, restaurando o comportamento anterior de aceitar entradas arbitrariamente grandes."}]}]} />

Tamanho máximo, em bytes, de um único valor de entrada para as funções `base58Encode`, `base58Decode` e `tryBase58Decode`. A conversão genérica `base58` é quadrática em relação ao comprimento da entrada, portanto um único valor grande pode levar muito tempo para ser executado. `base58` foi projetado para dados curtos (chaves, hashes, endereços), portanto o padrão de 10 KB é um limite de segurança generoso. `base58Encode` e `base58Decode` geram `TOO_LARGE_STRING_SIZE` para entradas maiores, enquanto `tryBase58Decode` retorna uma string vazia. Um valor de `0` desabilita o limite (comportamento anterior à introdução desta configuração). As funções lineares `base32` e `base64` não são afetadas.

<div id="function_date_trunc_return_type_behavior">
  ## function_date_trunc_return_type_behavior
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.7"},{"label": "0"},{"label": "Adiciona uma nova configuração para preservar o comportamento antigo da função dateTrunc"}]}, {"id": "row-2","items": [{"label": "25.4"},{"label": "0"},{"label": "Altera o tipo de resultado da função dateTrunc para argumentos DateTime64\/Date32 para DateTime64\/Date32, independentemente da unidade de tempo, para obter o resultado correto para valores negativos"}]}]} />

Permite alterar o comportamento do tipo de retorno da função `dateTrunc`.

Valores possíveis:

* 0 - Quando o segundo argumento é `DateTime64/Date32`, o tipo de retorno será `DateTime64/Date32`, independentemente da unidade de tempo no primeiro argumento.
* 1 - Para `Date32`, o resultado é sempre `Date`. Para `DateTime64`, o resultado é `DateTime` para unidades de tempo `second` e maiores.
