<div id="system_events_show_zero_values">
  ## system_events_show_zero_values
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Permite selecionar eventos com valor zero da tabela [`system.events`](/pt-BR/reference/system-tables/events).

Alguns sistemas de monitoramento exigem que todos os valores de métricas sejam enviados a eles em cada checkpoint, mesmo que o valor da métrica seja zero.

Valores possíveis:

* 0 — Desativado.
* 1 — Ativado.

**Exemplos**

Consulta

```sql
SELECT * FROM system.events WHERE event='QueryMemoryLimitExceeded';
```

Resultado

```text
Ok.
```

Consulta

```sql
SET system_events_show_zero_values = 1;
SELECT * FROM system.events WHERE event='QueryMemoryLimitExceeded';
```

Resultado

```text
┌─event────────────────────┬─value─┬─description───────────────────────────────────────────┐
│ QueryMemoryLimitExceeded │     0 │ Number of times when memory limit exceeded for query. │
└──────────────────────────┴───────┴───────────────────────────────────────────────────────┘
```

<div id="system_metric_log_show_zero_values_in_histograms">
  ## system_metric_log_show_zero_values_in_histograms
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "0"},{"label": "Nova configuração que controla se dados de histogramas com valor zero são gravados na coluna aninhada histograms de system.metric_log."}]}]} />

Controla se dados de histogramas com valor zero são gravados na coluna aninhada `histograms` de `system.metric_log`.

Por padrão, histogramas cujo `count` total de observações é zero são ignorados e, dentro de cada histograma emitido, entradas de bucket sem observações também são omitidas do map `histogram`. Ative esta opção para gravar todos os histogramas e todos os buckets, independentemente da contagem — útil para sistemas de monitoramento que exigem que toda métrica apareça em cada checkpoint.

Valores possíveis:

* 0 — Desabilitado. Histogramas com `count = 0` não são emitidos; os histogramas emitidos incluem apenas buckets que receberam pelo menos uma observação.
* 1 — Habilitado. Todos os histogramas são gravados, e todo limite de bucket aparece em `histogram`.
