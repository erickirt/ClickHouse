Essas configurações estão disponíveis em [system.settings](/pt-BR/reference/system-tables/settings) e são geradas automaticamente a partir da [fonte](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="enable_filesystem_cache">
  ## enable_filesystem_cache
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Usa cache para o remote filesystem. Esta configuração não ativa/desativa o cache para disks (isso deve ser feito via config do disk), mas permite ignorar o cache em algumas queries, se desejado

<div id="enable_filesystem_cache_log">
  ## enable_filesystem_cache_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Permite registrar, para cada consulta, o log de cache do sistema de arquivos

<div id="enable_filesystem_cache_on_write_operations">
  ## enable_filesystem_cache_on_write_operations
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Ativa ou desativa o cache `write-through`. Se definido como `false`, o cache `write-through` é desativado para operações de escrita. Se definido como `true`, o cache `write-through` é ativado desde que `cache_on_write_operations` esteja habilitado na seção de configuração do disco de cache da configuração do servidor.
Consulte [&quot;Usando cache local&quot;](/pt-BR/concepts/features/configuration/server-config/storing-data#using-local-cache) para mais detalhes.

Valor padrão no Cloud: `1`.

<div id="enable_filesystem_read_prefetches_log">
  ## enable_filesystem_read_prefetches_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Registra no system.filesystem prefetch&#95;log durante a consulta. Deve ser usado apenas para testes ou depuração; não é recomendável ativá-lo por padrão
