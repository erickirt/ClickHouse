  ## s3_validate_etag_on_read
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "Nova configuração para detectar sobrescritas simultâneas in-place de objetos S3\/GCS durante uma leitura, validando o ETag da resposta GET em relação ao ETag listado. previous_value=false, portanto a `compatibility` com versões anteriores à 26.7 restaura o comportamento preexistente (sem validação)."}]}]} />

Ao ler um objeto do S3 (ou de um armazenamento compatível com S3, como o GCS), verifica se cada solicitação GET retorna o mesmo ETag observado quando o objeto foi listado. A leitura de um único arquivo emite muitas solicitações GET por faixa; se o objeto for sobrescrito in-place entre elas (por exemplo, por um processo externo reescrevendo uma chave fixa), os dados lidos podem acabar sendo montados a partir de duas gerações diferentes do objeto e se manifestar como um checksum corrompido ou um erro de análise. Quando uma divergência é detectada, a leitura falha com `S3_OBJECT_CHANGED_DURING_READ` em vez de retornar dados inconsistentes. Desative apenas para cargas de trabalho que leem intencionalmente objetos que estão sendo sobrescritos e podem tolerar leituras inconsistentes.
