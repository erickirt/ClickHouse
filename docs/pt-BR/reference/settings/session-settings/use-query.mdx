<div id="use_query_cache">
  ## use_query_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Se ativado, consultas `SELECT` podem usar o [cache de consultas](/pt-BR/concepts/features/performance/caches/query-cache). Os parâmetros [enable&#95;reads&#95;from&#95;query&#95;cache](/pt-BR/reference/settings/session-settings/enable#enable_reads_from_query_cache)
e [enable&#95;writes&#95;to&#95;query&#95;cache](/pt-BR/reference/settings/session-settings/enable#enable_writes_to_query_cache) controlam com mais detalhes como o cache é usado.

Valores possíveis:

* 0 - Desativado
* 1 - Ativado

<div id="use_query_condition_cache">
  ## use_query_condition_cache
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.4"},{"label": "1"},{"label": "Uma nova otimização"}]}, {"id": "row-2","items": [{"label": "25.3"},{"label": "0"},{"label": "Nova configuração."}]}]} />

Ativa o [cache de condição de consulta](/pt-BR/concepts/features/performance/caches/query-condition-cache). O cache armazena intervalos de grânulos em partes de dados que não satisfazem a condição da cláusula `WHERE`
e reutiliza essas informações como um índice efêmero em consultas subsequentes.

Valores possíveis:

* 0 - Desativado
* 1 - Ativado
