<div id="merge_tree_coarse_index_granularity">
  ## merge_tree_coarse_index_granularity
</div>

<SettingsInfoBlock type="UInt64" default_value="8" />

Ao procurar dados, o ClickHouse verifica as marcas dos dados no arquivo de índice. Se o ClickHouse identificar que as chaves necessárias estão em determinado intervalo, ele divide esse intervalo em `merge_tree_coarse_index_granularity` subintervalos e busca essas chaves neles, recursivamente.

Valores possíveis:

* Qualquer número inteiro par positivo.

<div id="merge_tree_compact_parts_min_granules_to_multibuffer_read">
  ## merge_tree_compact_parts_min_granules_to_multibuffer_read
</div>

<CloudOnlyBadge />

<SettingsInfoBlock type="UInt64" default_value="16" />

Só tem efeito no ClickHouse Cloud. Número de grânulos em uma stripe de uma parte compacta de tabelas MergeTree para usar o leitor multibuffer, que oferece suporte a leitura paralela e prefetch. No caso de leitura a partir de um sistema de arquivos remoto, o uso do leitor multibuffer aumenta o número de requisições de leitura. Uma parte é lida com um único buffer quando contém menos grânulos do que esse limite e também menos grânulos do que o número de colunas que estão sendo lidas, porque, nesse caso, o leitor de buffer único emite menos requisições de leitura do que o leitor multibuffer.

<div id="merge_tree_generic_exclusion_search_max_steps">
  ## merge_tree_generic_exclusion_search_max_steps
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Nova configuração para limitar o número de passos da busca genérica por exclusão no índice da chave primária."}]}]} />

Quando um filter não pode ser avaliado como um único intervalo contínuo da chave primária, por exemplo, quando usa colunas-chave diferentes da primeira, o ClickHouse executa um algoritmo iterativo de busca genérica por exclusão sobre as marcas do índice. O mesmo algoritmo é usado na análise do índice de texto. Esta configuração limita o número de passos (verificações do índice) que o algoritmo realiza em cada parte de dados.

O orçamento é consumido primeiro nos maiores intervalos de marcas restantes. Quando ele se esgota, os intervalos que não foram totalmente analisados são aceitos como um todo, de modo que a consulta continua correta, mas pode ler mais grânulos do que uma busca sem limite selecionaria. Um orçamento menor acelera a análise do índice, ao custo de ler mais dados. O limite é aproximado, e não um teto estrito para o custo da análise: a busca pode ultrapassá-lo em aproximadamente uma rodada de divisão e, quando a parte já está dividida em muitos intervalos (por exemplo, pelo cache de condições da consulta), cada um deles é verificado pelo menos uma vez, independentemente do limite.

O número de passos executados pela busca em cada parte de dados é informado nas mensagens de log da consulta no nível trace, e os profile events `IndexGenericExclusionSearchStepLimitReached` e `TextIndexGenericExclusionSearchStepLimitReached` contam quantas vezes o orçamento foi esgotado.

O valor padrão 0 significa passos ilimitados.

Possible values:

* 0 para passos ilimitados ou qualquer número inteiro positivo.

<div id="merge_tree_max_bytes_to_use_cache">
  ## merge_tree_max_bytes_to_use_cache
</div>

<SettingsInfoBlock type="UInt64" default_value="2013265920" />

Se o ClickHouse precisar ler mais de `merge_tree_max_bytes_to_use_cache` bytes em uma consulta, ele não usará o cache de blocos não comprimidos.

O cache de blocos não comprimidos armazena dados extraídos para consultas. O ClickHouse usa esse cache para acelerar as respostas a consultas pequenas repetidas. Essa configuração evita que o cache seja prejudicado por consultas que leem uma grande quantidade de dados. A configuração do servidor [uncompressed&#95;cache&#95;size](/pt-BR/reference/settings/server-settings/settings#uncompressed_cache_size) define o tamanho do cache de blocos não comprimidos.

Valores possíveis:

* Qualquer inteiro positivo.

<div id="merge_tree_max_rows_to_use_cache">
  ## merge_tree_max_rows_to_use_cache
</div>

<SettingsInfoBlock type="UInt64" default_value="1048576" />

Se o ClickHouse precisar ler mais de `merge_tree_max_rows_to_use_cache` linhas em uma consulta, ele não usará o cache de blocos não comprimidos.

O cache de blocos não comprimidos armazena dados extraídos durante a execução de consultas. O ClickHouse usa esse cache para acelerar a resposta a consultas pequenas executadas repetidamente. Essa configuração protege o cache contra a degradação causada por consultas que leem uma grande quantidade de dados. A configuração de servidor [uncompressed&#95;cache&#95;size](/pt-BR/reference/settings/server-settings/settings#uncompressed_cache_size) define o tamanho do cache de blocos não comprimidos.

Valores possíveis:

* Qualquer número inteiro positivo.
