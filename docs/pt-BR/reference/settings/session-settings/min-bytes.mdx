Essas configurações estão disponíveis em [system.settings](/pt-BR/reference/system-tables/settings) e são geradas automaticamente a partir do [código-fonte](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="min_bytes_to_use_direct_io">
  ## min_bytes_to_use_direct_io
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O volume mínimo de dados necessário para usar acesso de E/S direct ao disco de armazenamento.

O ClickHouse usa essa configuração ao ler dados de tabelas. Se o volume total de armazenamento de todos os dados a serem lidos exceder `min_bytes_to_use_direct_io` bytes, o ClickHouse lê os dados do disco de armazenamento com a opção `O_DIRECT`.

Valores possíveis:

* 0 — A E/S direct está desativada.
* Inteiro positivo.

<div id="min_bytes_to_use_mmap_io">
  ## min_bytes_to_use_mmap_io
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Esta é uma configuração experimental. Define a quantidade mínima de memória para ler arquivos grandes sem copiar dados do kernel para o espaço do usuário. O valor recomendado é de cerca de 64 MB, porque [mmap/munmap](https://en.wikipedia.org/wiki/Mmap) é lento. Isso só faz sentido para arquivos grandes e só ajuda se os dados estiverem no cache de páginas.

Valores possíveis:

* Inteiro positivo.
* 0 — Arquivos grandes são lidos apenas com a cópia de dados do kernel para o espaço do usuário.
