  ## implicit_select
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.10"},{"label": "0"},{"label": "Uma nova configuração."}]}]} />

Permite escrever consultas `SELECT` simples sem a palavra-chave `SELECT` no início, o que facilita o uso como calculadora; por exemplo, `1 + 2` se torna uma consulta válida.

No `clickhouse-local`, isso vem habilitado por padrão e pode ser desativado explicitamente.

<div id="implicit_table_at_top_level">
  ## implicit_table_at_top_level
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": ""},{"label": "Uma nova configuração, usada no clickhouse-local"}]}]} />

Se não estiver vazia, consultas sem `FROM` no nível superior lerão dessa tabela em vez de system.one.

Essa configuração é usada no clickhouse-local para o processamento de dados de entrada.
Ela pode ser definida explicitamente pelo usuário, mas não se destina a esse tipo de uso.

As subconsultas não são afetadas por esta configuração (nem subconsultas escalares, `FROM` nem `IN`).
`SELECT`s no nível superior de cadeias `UNION`, `INTERSECT` e `EXCEPT` são tratados de maneira uniforme e afetados por esta configuração, independentemente do agrupamento entre parênteses.
Não está especificado como esta configuração afeta views e consultas distribuídas.

A configuração aceita um nome de tabela (nesse caso, a tabela é resolvida a partir do banco de dados atual) ou um nome qualificado no formato `database.table`.
Tanto o nome do banco de dados quanto o da tabela não devem estar entre aspas — apenas identificadores simples são permitidos.
