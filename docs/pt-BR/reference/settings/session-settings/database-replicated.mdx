<div id="database_replicated_allow_explicit_uuid">
  ## database_replicated_allow_explicit_uuid
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "Adicionada uma nova configuração para impedir a especificação explícita do UUID da tabela"}]}]} />

0 - Não permitir especificar explicitamente UUIDs para tabelas em bancos de dados Replicated. 1 - Permitir. 2 - Permitir, mas ignorar o UUID especificado e gerar um UUID aleatório.

<div id="database_replicated_allow_heavy_create">
  ## database_replicated_allow_heavy_create
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.7"},{"label": "0"},{"label": "Consultas DDL de longa duração (CREATE AS SELECT e POPULATE) para o engine de banco de dados Replicated eram proibidas"}]}]} />

Permite consultas DDL de longa duração (CREATE AS SELECT e POPULATE) no engine de banco de dados Replicated. Observe que isso pode bloquear a fila de DDL por muito tempo.

<div id="database_replicated_allow_only_replicated_engine">
  ## database_replicated_allow_only_replicated_engine
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Permite criar apenas tabelas Replicated em um banco de dados com engine Replicated

Valor padrão no Cloud: `1`.

<div id="database_replicated_allow_replicated_engine_arguments">
  ## database_replicated_allow_replicated_engine_arguments
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "Não permitir argumentos explícitos por padrão"}]}]} />

0 - Não permitir a especificação explícita do caminho do ZooKeeper e do nome da réplica para tabelas *MergeTree em bancos de dados Replicated. 1 - Permitir. 2 - Permitir, mas ignorar o caminho especificado e usar o caminho padrão. 3 - Permitir e não registrar um aviso.

<div id="database_replicated_always_detach_permanently">
  ## database_replicated_always_detach_permanently
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Executa DETACH TABLE como DETACH TABLE PERMANENTLY se o engine do banco de dados for Replicated

<div id="database_replicated_enforce_synchronous_settings">
  ## database_replicated_enforce_synchronous_settings
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Exige espera síncrona para algumas consultas (consulte também database&#95;atomic&#95;wait&#95;for&#95;drop&#95;and&#95;detach&#95;synchronously, mutations&#95;sync, alter&#95;sync). Não é recomendável habilitar essas configurações.

<div id="database_replicated_initial_query_timeout_sec">
  ## database_replicated_initial_query_timeout_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="300" />

Define por quanto tempo a consulta DDL inicial deve aguardar o database Replicated processar as entradas anteriores da fila DDL, em segundos.

Valores possíveis:

* Inteiro positivo.
* 0 — Ilimitado.
