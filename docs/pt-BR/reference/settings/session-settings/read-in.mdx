<div id="read_in_order_use_virtual_row_per_block">
  ## read_in_order_use_virtual_row_per_block
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "Emite uma linha virtual após cada bloco durante a leitura em ordem para permitir a repriorização mais frequente das fontes no MergingSortedTransform."}]}]} />

Quando ativado junto com `read_in_order_use_virtual_row`, emite uma linha virtual após cada `bloco` lido (não apenas no início de cada `part`).
Isso permite que `MergingSortedTransform` repriorize as fontes com mais frequência, o que é útil quando os filtros downstream descartam muitas linhas e os dados estão distribuídos de forma desigual entre as partes.
Observe que isso desativa a otimização `read_in_order_use_buffering` e o merge preliminar (`read_in_order_two_level_merge_threshold`) para leitura.
