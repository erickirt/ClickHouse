<div id="optimize_aggregation_in_order">
  ## optimize_aggregation_in_order
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Habilita a otimização de [GROUP BY](/pt-BR/reference/statements/select/group-by) em consultas [SELECT](/pt-BR/reference/statements/select/index) para agregar dados seguindo a ordem correspondente em tabelas [MergeTree](/pt-BR/reference/engines/table-engines/mergetree-family/mergetree).

Valores possíveis:

* 0 — a otimização de `GROUP BY` está desabilitada.
* 1 — a otimização de `GROUP BY` está habilitada.

**Veja também**

* [Otimização de GROUP BY](/pt-BR/reference/statements/select/group-by#group-by-optimization-depending-on-table-sorting-key)

<div id="optimize_aggregation_in_order_limit">
  ## optimize_aggregation_in_order_limit
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "Nova configuração para aplicar o `LIMIT` à agregação em ordem, permitindo interrupção antecipada quando o `ORDER BY` for um prefixo da descrição de ordenação do `GROUP BY`."}]}]} />

Quando habilitado e a agregação em ordem está ativa, aplica o LIMIT à etapa de agregação para permitir a interrupção antecipada após produzir grupos suficientes. Isso reduz a quantidade de dados lidos quando o ORDER BY corresponde ao prefixo da chave do GROUP BY. Pode reduzir o valor relatado por `rows_before_limit_at_least`; use `exact_rows_before_limit` se forem necessárias contagens exatas.
