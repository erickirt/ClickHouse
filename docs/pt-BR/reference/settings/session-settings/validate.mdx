<div id="validate_mutation_query">
  ## validate_mutation_query
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.11"},{"label": "1"},{"label": "Nova configuração para validar consultas de mutação por padrão."}]}]} />

Valide as consultas de mutação antes de aceitá-las. As mutações são executadas em segundo plano, e executar uma consulta inválida fará com que elas fiquem travadas, exigindo intervenção manual.

Altere essa configuração apenas se encontrar um bug incompatível com versões anteriores.

<div id="validate_polygons">
  ## validate_polygons
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "20.4"},{"label": "1"},{"label": "Lança uma exceção por padrão se o polígono for inválido na função pointInPolygon, em vez de retornar resultados possivelmente incorretos"}]}]} />

Habilita ou desabilita o lançamento de uma exceção na função [pointInPolygon](/pt-BR/reference/functions/regular-functions/geo/coordinates#pointinpolygon) se o polígono se intersectar consigo mesmo ou for autotangente.

Valores possíveis:

* 0 — O lançamento de exceção está desabilitado. `pointInPolygon` aceita polígonos inválidos e pode retornar resultados incorretos para eles.
* 1 — O lançamento de exceção está habilitado.
