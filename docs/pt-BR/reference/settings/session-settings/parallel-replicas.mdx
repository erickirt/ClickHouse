<div id="parallel_replicas_custom_key">
  ## parallel_replicas_custom_key
</div>

<BetaBadge />

Uma expressão inteira arbitrária que pode ser usada para dividir o trabalho entre réplicas de uma tabela específica.
O valor pode ser qualquer expressão inteira.

Prefira expressões simples que usem chaves primárias.

Se a configuração for usada em um cluster composto por um único shard com várias réplicas, essas réplicas serão convertidas em shards virtuais.
Caso contrário, ela se comportará da mesma forma que a chave `SAMPLE`, usando várias réplicas de cada shard.

Expressa a fronteira local/remota das réplicas paralelas por meio de uma transformação do plano. O planner cria um plano local simples; em seguida, uma etapa de divisão é inserida acima da etapa de leitura e substituída por uma `UNION` entre uma leitura local e uma leitura remota da parte do plano abaixo da etapa de divisão, para que esse fragmento seja executado nas réplicas enquanto o restante é executado no coordenador. <div id="parallel_replicas_prefer_local_replica">
  ## parallel_replicas_prefer_local_replica
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "1"},{"label": "Nova configuração. Quando desativada, as réplicas para leitura paralela são selecionadas exclusivamente pelo algoritmo de balanceamento de carga, sem forçar a inclusão da réplica local no conjunto."}]}]} />

Quando ativada (padrão), a réplica local é sempre incluída no conjunto de réplicas usado para leitura paralela.
Quando desativada, a réplica local não recebe nenhuma preferência, e as réplicas são selecionadas exclusivamente pelo algoritmo de balanceamento de carga.
Isso permite que consultas com `max_parallel_replicas = 1` sejam direcionadas para outro host, o que pode melhorar a localidade de cache quando muitas consultas curtas são distribuídas em um cluster.
