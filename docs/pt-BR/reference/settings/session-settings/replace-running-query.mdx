<div id="replace_running_query">
  ## replace_running_query
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Ao usar a interface HTTP, o parâmetro &#39;query&#95;id&#39; pode ser informado. Ele pode ser qualquer string que sirva como identificador da consulta.
Se já existir, nesse momento, uma consulta do mesmo usuário com o mesmo &#39;query&#95;id&#39;, o comportamento dependerá do parâmetro &#39;replace&#95;running&#95;query&#39;.

`0` (padrão) – Lança uma exceção (não permite que a consulta seja executada se já houver uma consulta com o mesmo &#39;query&#95;id&#39; em execução).

`1` – Cancela a consulta antiga e começa a executar a nova.

Defina esse parâmetro como 1 para implementar sugestões de condições de segmentação. Após inserir o próximo caractere, se a consulta antiga ainda não tiver terminado, ela deverá ser cancelada.

<div id="replace_running_query_max_wait_ms">
  ## replace_running_query_max_wait_ms
</div>

<SettingsInfoBlock type="Milissegundos" default_value="5000" />

O tempo de espera para a consulta em execução com o mesmo `query_id` terminar, quando a configuração [replace&#95;running&#95;query](/pt-BR/reference/settings/session-settings/replace-running-query#replace_running_query) está ativa.

Valores possíveis:

* Inteiro positivo.
* 0 — Gera uma exceção que não permite executar uma nova consulta se o servidor já estiver executando uma consulta com o mesmo `query_id`.
