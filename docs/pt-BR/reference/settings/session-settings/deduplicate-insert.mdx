  ## deduplicate_insert
</div>

<SettingsInfoBlock type="DeduplicateInsertMode" default_value="enable" />

Habilita ou desabilita a desduplicação de blocos de `INSERT INTO` (para tabelas Replicated*).
Essa configuração substitui as configurações `insert_deduplicate` e `async_insert_deduplicate`.
Essa configuração tem três valores possíveis:

* disable — A desduplicação é desabilitada para a consulta `INSERT INTO`.
* enable — A desduplicação é habilitada para a consulta `INSERT INTO`.
* backward&#95;compatible&#95;choice — A desduplicação é habilitada se `insert_deduplicate` ou `async_insert_deduplicate` estiverem habilitados para o tipo específico de insert.

Habilita ou desabilita a desduplicação de blocos em `INSERT SELECT` (para tabelas Replicated*).
A configuração substitui `insert_deduplicate` e `deduplicate_insert` para consultas `INSERT SELECT`.
Essa configuração tem quatro valores possíveis:

* disable — A desduplicação é desabilitada para a consulta `INSERT SELECT`.
* force&#95;enable — A desduplicação é habilitada para a consulta `INSERT SELECT`. Se o resultado de `select` não for estável, uma exceção será lançada.
* enable&#95;when&#95;possible — A desduplicação é habilitada se `insert_deduplicate` estiver habilitada e o resultado de `select` for estável; caso contrário, será desabilitada.
* enable&#95;even&#95;for&#95;bad&#95;queries - A desduplicação é habilitada se `insert_deduplicate` estiver habilitada. Se o resultado de `select` não for estável, um aviso será registrado, mas a consulta será executada com desduplicação. Esta opção existe por compatibilidade retroativa. Considere usar outras opções, pois ela pode levar a resultados inesperados.
