  ## stream_like_engine_allow_direct_select
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "21.12"},{"label": "0"},{"label": "Não permitir consulta SELECT direta para Kafka/RabbitMQ/FileLog por padrão"}]}]} />

Permite consulta SELECT direta para os motores Kafka, RabbitMQ, FileLog, Redis Streams, S3Queue, AzureQueue e NATS. Caso haja visões materializadas anexadas, a consulta SELECT não será permitida, mesmo que essa configuração esteja habilitada.
Se não houver visões materializadas anexadas, habilitar essa configuração permite ler dados. Observe que, normalmente, os dados lidos são removidos da fila. Para evitar a remoção dos dados lidos, as configurações relacionadas do motor devem ser definidas corretamente.

<div id="stream_like_engine_insert_queue">
  ## stream_like_engine_insert_queue
</div>

Quando um stream-like motor lê de várias filas, o usuário precisará selecionar em qual fila inserir ao escrever. Usado por Redis Streams e NATS.
