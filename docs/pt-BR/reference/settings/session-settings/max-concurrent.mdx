<div id="max_concurrent_queries_for_all_users">
  ## max_concurrent_queries_for_all_users
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Gera uma exceção se o valor desta configuração for menor ou igual ao número atual de consultas processadas simultaneamente.

Exemplo: `max_concurrent_queries_for_all_users` pode ser definido como 99 para todos os usuários, e o administrador do banco de dados pode defini-lo como 100 para si mesmo, a fim de executar consultas para investigação mesmo quando o servidor estiver sobrecarregado.

Modificar a configuração para uma consulta ou um usuário não afeta outras consultas.

Valores possíveis:

* Inteiro positivo.
* 0 — Sem limite.

**Exemplo**

```xml
<max_concurrent_queries_for_all_users>99</max_concurrent_queries_for_all_users>
```

**Veja também**

* [max&#95;concurrent&#95;queries](/pt-BR/reference/settings/server-settings/settings#max_concurrent_queries)

Valor padrão no Cloud: `1000`.

<div id="max_concurrent_queries_for_user">
  ## max_concurrent_queries_for_user
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

O número máximo de consultas processadas simultaneamente por usuário.

Valores possíveis:

* Inteiro positivo.
* 0 — Sem limite.

**Exemplo**

```xml
<max_concurrent_queries_for_user>5</max_concurrent_queries_for_user>
```
