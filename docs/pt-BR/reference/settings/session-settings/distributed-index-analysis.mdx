<div id="distributed_index_analysis_only_on_coordinator">
  ## distributed_index_analysis_only_on_coordinator
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "Nova configuração."}]}]} />

Se ativada, a análise de índice distribuída é executada apenas no coordenador.
Isso evita consultas disparadas em O(N^2) quando o predicado contém subconsultas (por exemplo, `IN (SELECT ...)`),
porque, caso contrário, cada réplica seguidora acionaria de forma independente sua própria análise de índice distribuída,
mas torna a análise de índice distribuída menos eficiente se tabelas grandes forem usadas nas subconsultas.
