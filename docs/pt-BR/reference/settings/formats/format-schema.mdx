Estas configurações são geradas automaticamente a partir do [código-fonte](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/FormatFactorySettings.h).

<div id="format_schema">
  ## format_schema
</div>

Este parâmetro é útil quando você estiver usando formatos que exigem uma definição de esquema, como [Cap&#39;n Proto](https://capnproto.org/) ou [Protobuf](https://developers.google.com/protocol-buffers/). O valor depende do formato.

<div id="format_schema_message_name">
  ## format_schema_message_name
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": ""},{"label": "Nova configuração"}]}]} />

Define o nome da mensagem necessária no esquema definido em `format_schema`.
Para manter a compatibilidade com o formato legado de `format_schema` (`file_name:message_name`):

* Se `format_schema_message_name` não for especificado, o nome da mensagem será inferido a partir da parte `message_name` do valor legado de `format_schema`.
* Se `format_schema_message_name` for especificado ao usar o formato legado, ocorrerá um erro.

<div id="format_schema_source">
  ## format_schema_source
</div>

<SettingsInfoBlock type="String" default_value="file" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.6"},{"label": "file"},{"label": "Nova configuração"}]}]} />

Define a origem de `format_schema`.
Valores possíveis:

* &#39;file&#39; (padrão): `format_schema` é o nome de um arquivo de esquema localizado no diretório `format_schemas`.
* &#39;string&#39;: `format_schema` é o conteúdo literal do esquema.
* &#39;query&#39;: `format_schema` é uma consulta para obter o esquema.
  Quando `format_schema_source` é definido como &#39;query&#39;, as seguintes condições se aplicam:
* A consulta deve retornar exatamente um valor: uma única linha com uma única coluna do tipo string.
* O resultado da consulta é tratado como o conteúdo do esquema.
* Esse resultado é armazenado em cache localmente no diretório `format_schemas`.
* Você pode limpar o cache local usando o comando: `SYSTEM DROP FORMAT SCHEMA CACHE FOR Files`.
* Depois de armazenadas em cache, consultas idênticas não são executadas novamente para buscar o esquema até que o cache seja limpo explicitamente
* Além dos arquivos de cache locais, as mensagens Protobuf também são armazenadas em cache na memória. Mesmo após limpar os arquivos de cache locais, o cache em memória deve ser limpo usando `SYSTEM DROP FORMAT SCHEMA CACHE [FOR Protobuf]` para atualizar completamente o esquema.
* Execute a consulta `SYSTEM DROP FORMAT SCHEMA CACHE` para limpar de uma só vez o cache dos arquivos e dos esquemas de mensagens Protobuf.
