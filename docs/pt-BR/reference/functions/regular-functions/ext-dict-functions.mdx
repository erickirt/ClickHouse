<div id="passing-keys">
  ## Passando chaves para funções de dicionário
</div>

O argumento de chave (`id_expr`) de funções como `dictGet`, `dictGetOrDefault`, `dictGetOrNull` e `dictHas` depende da chave do dicionário:

* Para um dicionário com uma **chave simples** (`UInt64`), passe o valor da chave diretamente:

```sql
SELECT dictGet('simple_key_dictionary', 'attr_name', toUInt64(1));
```

* Para um dicionário com uma **chave composta (complexa)** de mais de um atributo, passe os valores da chave como uma tupla:

```sql
SELECT dictGet('complex_key_dictionary', 'attr_name', ('value_for_field1', 42));
```

* Quando a **chave composta consiste em um único atributo**, o valor da chave pode ser passado diretamente, sem precisar envolvê-lo em `tuple`. Ambos os exemplos a seguir são válidos e equivalentes:

```sql
SELECT dictGet('complex_key_dictionary', 'attr_name', 'key');
SELECT dictGet('complex_key_dictionary', 'attr_name', tuple('key'));
```

Isso também se aplica a dicionários `ip_trie`, cuja chave é um único atributo. O endereço IP a ser pesquisado pode ser passado diretamente:

```sql
SELECT dictGet('ip_trie_dictionary', 'attr_name', toIPv4('202.79.32.10'));
```

<div id="dictGet">
  ## dictGet
</div>

Introduzido em: v18.16.0

Recupera valores de um dicionário.

<Note>
  Esta função não é determinística: pode retornar resultados diferentes para os mesmos argumentos.
</Note>

**Sintaxe**

```sql
dictGet('dict_name', attr_names, id_expr)
```

**Argumentos**

* `dict_name` — Nome do dicionário. [`String`](/pt-BR/reference/data-types/string)
* `attr_names` — Nome da coluna do dicionário ou tupla de nomes de colunas. [`String`](/pt-BR/reference/data-types/string) ou [`Tuple(String)`](/pt-BR/reference/data-types/tuple)
* `id_expr` — Valor da chave. Para um dicionário com chave simples, uma expressão que retorna um valor `UInt64`. Para um dicionário com chave composta (complexa), uma expressão que retorna uma tupla dos valores da chave. Se a chave composta consistir em um único atributo, seu valor pode ser passado diretamente, sem envolvê-lo em `tuple`. [`UInt64`](/pt-BR/reference/data-types/int-uint) ou [`Tuple(T)`](/pt-BR/reference/data-types/tuple)

**Valor retornado**

Retorna o valor do atributo do dicionário correspondente a id&#95;expr, se a chave for encontrada.
Se a chave não for encontrada, retorna o conteúdo do elemento `<null_value>` especificado para o atributo na configuração do dicionário.

**Exemplos**

**Obter um único atributo**

```sql title=Query
CREATE TABLE ext_dict_test_source (id UInt32, c1 UInt32, c2 String) ENGINE = Memory;
INSERT INTO ext_dict_test_source VALUES (1, 1, '1'), (2, 2, '2'), (3, 3, '3');
CREATE DICTIONARY ext_dict_test (id UInt32, c1 UInt32, c2 String) PRIMARY KEY id SOURCE(CLICKHOUSE(TABLE 'ext_dict_test_source' DB currentDatabase())) LAYOUT(FLAT()) LIFETIME(MIN 0 MAX 0);
SELECT dictGet('ext_dict_test', 'c1', toUInt64(1)) AS val
```

```response title=Response
1
```

**Vários atributos**

```sql title=Query
CREATE TABLE dict_mult_source (id UInt32, c1 UInt32, c2 String) ENGINE = Memory;
INSERT INTO dict_mult_source VALUES (1, 1, '1'), (2, 2, '2'), (3, 3, '3');
CREATE DICTIONARY ext_dict_mult (id UInt32, c1 UInt32, c2 String) PRIMARY KEY id SOURCE(CLICKHOUSE(TABLE 'dict_mult_source' DB currentDatabase())) LAYOUT(FLAT()) LIFETIME(MIN 0 MAX 0);
SELECT
    dictGet('ext_dict_mult', ('c1','c2'), number + 1) AS val,
    toTypeName(val) AS type
FROM system.numbers
LIMIT 3;
```

```response title=Response

│ (1,'1') │ Tuple(        ↴│
│         │↳    c1 UInt32,↴│
│         │↳    c2 String) │
│ (2,'2') │ Tuple(        ↴│
│         │↳    c1 UInt32,↴│
│         │↳    c2 String) │
│ (3,'3') │ Tuple(        ↴│
│         │↳    c1 UInt32,↴│
```

<div id="dictGetAll">
  ## dictGetAll
</div>

Introduzido em: v23.5.0

Recupera os valores do atributo de todos os nós que corresponderam a cada chave em um [dicionário de árvore de expressões regulares](/pt-BR/reference/statements/create/dictionary/layouts/regexp-tree).

Além de retornar valores do tipo `Array(T)` em vez de `T`, esta função se comporta de forma semelhante a [`dictGet`](#dictGet).

<Note>
  Esta função não é determinística: pode retornar resultados diferentes para os mesmos argumentos.
</Note>

**Sintaxe**

```sql
dictGetAll(dict_name, attr_names, id_expr[, limit])
```

**Argumentos**

* `dict_name` — Nome do dicionário. [`String`](/pt-BR/reference/data-types/string)
* `attr_names` — Nome da coluna do dicionário ou tupla de nomes de colunas. [`String`](/pt-BR/reference/data-types/string) ou [`Tuple(String)`](/pt-BR/reference/data-types/tuple)
* `id_expr` — Valor da chave. Uma expressão que retorna um valor do tipo de chave do dicionário ou um valor de tupla (dependendo da configuração do dicionário). [`Expression`](/pt-BR/reference/data-types/special-data-types/expression) ou [`Tuple(T)`](/pt-BR/reference/data-types/tuple)
* `limit` — Opcional. Comprimento máximo de cada array de valores retornado. Ao truncar, os nós filhos têm precedência sobre os nós pais e, caso contrário, a ordem definida da lista para o dicionário regexp tree é respeitada. Se não for especificado, o comprimento do array é ilimitado. [`UInt*`](/pt-BR/reference/data-types/int-uint)

**Valor retornado**

Retorna um array com os valores dos atributos do dicionário que correspondem a `id_expr` para cada atributo especificado por `attr_names`.
Se não houver nenhuma chave correspondente a `id_expr` no dicionário, um array vazio será retornado.

<Note>
  O ClickHouse lança uma exceção se não conseguir interpretar o valor do atributo ou se o valor não corresponder ao tipo de dado do atributo.
</Note>

[`Array(T)`](/pt-BR/reference/data-types/array)

**Exemplos**

**Exemplo de uso**

```sql title=Query
CREATE TABLE regexp_os (id UInt64, parent_id UInt64, regexp String, keys Array(String), values Array(String)) ENGINE = Memory;
INSERT INTO regexp_os VALUES
    (1, 0, 'Linux',      ['os_replacement'], ['Linux']),
    (2, 1, 'Android',    ['os_replacement'], ['Android']),
    (3, 2, 'Android 1',  ['os_replacement'], ['Android']),
    (4, 3, 'Android 12', ['os_replacement'], ['Android']);
CREATE DICTIONARY regexp_tree (regexp String, os_replacement String DEFAULT 'Other') PRIMARY KEY regexp SOURCE(CLICKHOUSE(TABLE 'regexp_os' DB currentDatabase())) LIFETIME(MIN 0 MAX 0) LAYOUT(REGEXP_TREE);
SELECT
    'Mozilla/5.0 (Linux; Android 12; SM-G998B) Mobile Safari/537.36' AS user_agent,

    -- This will match ALL applicable patterns
    dictGetAll('regexp_tree', 'os_replacement', 'Mozilla/5.0 (Linux; Android 12; SM-G998B) Mobile Safari/537.36') AS all_matches,

    -- This returns only the first match
    dictGet('regexp_tree', 'os_replacement', 'Mozilla/5.0 (Linux; Android 12; SM-G998B) Mobile Safari/537.36') AS first_match;
```

```response title=Response
┌─user_agent─────────────────────────────────────────────────────┬─all_matches─────────────────────────────┬─first_match─┐
│ Mozilla/5.0 (Linux; Android 12; SM-G998B) Mobile Safari/537.36 │ ['Android','Android','Android','Linux'] │ Android     │
└────────────────────────────────────────────────────────────────┴─────────────────────────────────────────┴─────────────┘
```

<div id="dictGetKeys">
  ## dictGetKeys
</div>

Introduzido em: v25.12.0

Retorna a(s) chave(s) do dicionário cujo atributo é igual ao valor especificado. Esta é a função inversa de `dictGet` para um único atributo.

Use a configuração `max_reverse_dictionary_lookup_cache_size_bytes` para limitar o tamanho do cache de busca reversa por consulta usado por `dictGetKeys`.
O cache armazena tuplas de chaves serializadas para cada valor de atributo, evitando varrer o dicionário novamente dentro da mesma consulta.
O cache não é persistente entre consultas. Quando o limite é atingido, as entradas são removidas com LRU.
Isso é mais eficaz com dicionários grandes quando a entrada tem baixa cardinalidade e o conjunto de trabalho cabe no cache. Defina como `0` para desabilitar o cache.

<Note>
  Esta função não é determinística: ela pode retornar resultados diferentes para os mesmos argumentos.
</Note>

**Sintaxe**

```sql
dictGetKeys('dict_name', 'attr_name', value_expr)
```

**Argumentos**

* `dict_name` — Nome do dicionário. [`String`](/pt-BR/reference/data-types/string)
* `attr_name` — Atributo a ser comparado. [`String`](/pt-BR/reference/data-types/string)
* `value_expr` — Valor a ser comparado com o atributo. [`Expression`](/pt-BR/reference/data-types/special-data-types/expression)

**Valor retornado**

Para dicionários de chave única: um array de chaves cujo atributo é igual a `value_expr`. Para dicionários com múltiplas chaves: um array de tuplas de chaves cujo atributo é igual a `value_expr`. Se não houver nenhum atributo correspondente a `value_expr` no dicionário, será retornado um array vazio. O ClickHouse lança uma exceção se não conseguir analisar o valor do atributo ou se o valor não puder ser convertido para o tipo de dado do atributo.

**Exemplos**

**Exemplo de uso**

```sql title=Query
CREATE TABLE task_priority_source (task_id UInt64, priority_level String) ENGINE = Memory;
INSERT INTO task_priority_source VALUES (1, 'low'), (2, 'high'), (3, 'medium'), (4, 'high');
CREATE DICTIONARY task_id_to_priority_dictionary (task_id UInt64, priority_level String) PRIMARY KEY task_id SOURCE(CLICKHOUSE(TABLE 'task_priority_source' DB currentDatabase())) LAYOUT(HASHED()) LIFETIME(MIN 0 MAX 0);
SELECT arraySort(dictGetKeys('task_id_to_priority_dictionary', 'priority_level', 'high')) AS ids;
```

```response title=Response
┌─ids───┐
│ [2,4] │
└───────┘
```
