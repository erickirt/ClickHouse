  **Escrevendo Arrays**

  Use `java.sql.Connection#createArrayOf` para instanciar o objeto `java.sql.Array`. Esse objeto foi projetado para unificar o tratamento de arrays em diferentes bancos de dados.
  A conexГЈo Г© necessГЎria para passar a configuraГ§ГЈo ao mГ©todo de fГЎbrica do Array.

  O mГ©todo aceita dois argumentos:

  * `elements` - elementos do array propriamente ditos. Por exemplo `[[1, 2, 3], [4, 5, 6]]` -&gt; `new Integer[][] {{1, 2, 3}, {4, 5, 6}}`.

  Tuple pode ser representado como `Object[]` ou como `java.sql.Struct` (veja como escrever tuples abaixo).

  **Exemplo**

  ```java

      try (PreparedStatement ps = conn.prepareStatement("INSERT INTO mytable (arr) VALUES (?)")) {
          ps.setArray(1, array);
          ps.executeUpdate();
      }
  }
  ```

  ### Resolvendo Timeout JDBC em Inserts de Grande Volume

  Ao realizar grandes inserГ§Гµes no ClickHouse com longos tempos de execuГ§ГЈo, vocГЄ pode se deparar com erros de timeout JDBC como:


  Esses erros podem interromper o processo de inserГ§ГЈo de dados e afetar a estabilidade do sistema. Para resolver esse problema, pode ser necessГЎrio ajustar algumas configuraГ§Гµes de timeout no SO do cliente.

  #### Mac OS

  No macOS, as seguintes configuraГ§Гµes podem ser ajustadas para resolver o problema:

  * `net.inet.tcp.keepidle`: 60000

  * `net.inet.tcp.keepinit`: 45000
  * `net.inet.tcp.keepcnt`: 8
  * `net.inet.tcp.always_keepalive`: 1

  #### Linux

  No Linux, as configuraГ§Гµes equivalentes por si sГі podem nГЈo resolver o problema. Etapas adicionais sГЈo necessГЎrias devido Г s diferenГ§as na forma como o Linux trata as configuraГ§Гµes de keep-alive de socket. Siga estas etapas:

  1. Ajuste os seguintes parГўmetros do kernel do Linux em `/etc/sysctl.conf` ou em um arquivo de configuraГ§ГЈo relacionado:

  * `net.inet.tcp.keepidle`: 60000
  * `net.inet.tcp.keepintvl`: 45000
  * `net.inet.tcp.keepinit`: 45000
  * `net.inet.tcp.keepcnt`: 8
  * `net.inet.tcp.always_keepalive`: 1
  * `net.ipv4.tcp_keepalive_intvl`: 75
  * `net.ipv4.tcp_keepalive_probes`: 9
  * `net.ipv4.tcp_keepalive_time`: 60 (VocГЄ pode considerar reduzir esse valor em relaГ§ГЈo ao padrГЈo de 300 segundos)

  2. ApГіs modificar os parГўmetros do kernel, aplique as alteraГ§Гµes executando o seguinte comando:

  ```shell

  ```

  ApГіs definir essas configuraГ§Гµes, vocГЄ precisa garantir que seu cliente habilite a opГ§ГЈo Keep Alive no socket:

