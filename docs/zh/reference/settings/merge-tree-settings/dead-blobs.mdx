<div id="dead_blobs_to_delay_insert">
  ## dead_blobs_to_delay_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "100000"},{"label": "新增设置：当表所在磁盘的 dead blobs 队列 中积累了过多待删除 blob 时，人为降低插入速度。"}]}]} />

如果表所在磁盘的 dead blobs 队列 中待删除的 blob 数量超过
`dead_blobs_to_delay_insert` 的值，则会人为减慢一次 `INSERT` (最多减慢至 `max_delay_to_insert`) 。

dead blobs 队列 归属于磁盘，并由其上的所有表共享 (包括已删除表的 blob) ，因此应按整个磁盘而非单个表来设置该阈值。

<Tip>
  当 server 无法足够快地清理 blob 时，这项设置会很有用。
</Tip>

可能的值：

* 任意正整数。
* 0 — 禁用。

<div id="dead_blobs_to_throw_insert">
  ## dead_blobs_to_throw_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1000000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1000000"},{"label": "新增设置：当表所在磁盘的 dead blobs 队列中等待删除的 blob 积累过多时，拒绝插入。"}]}]} />

如果表所在磁盘的 dead blobs 队列中等待删除的 blob 数量超过
`dead_blobs_to_throw_insert` 的值，`INSERT` 会被中断，并报出以下错误：

> &quot;表所在磁盘的队列中 dead blobs 过多 (N)。Blob 清理速度明显
> 低于插入速度&quot;

dead blobs 队列 归属于磁盘，并由其上的所有表共享 (包括已删除表的 blob) ，
因此应针对整个磁盘而非单个表来设置该阈值。

可能的值：

* 任意正整数。
* 0 — 禁用。
