<div id="map_serialization_version">
  ## map_serialization_version
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "新增用于控制 Map 序列化版本的设置"}]}]} />

控制 `Map` 列使用的序列化方法。

可能的值：

* basic — 对 `Map` 使用标准序列化。
* with&#95;buckets — 在序列化过程中将键拆分到桶中。使用桶可提升从 Map 中读取单个键的效率。

`with_buckets` 序列化中的桶数量由 [max&#95;buckets&#95;in&#95;map](/zh/reference/settings/merge-tree-settings/max#max_buckets_in_map) 和 [map&#95;buckets&#95;strategy](/zh/reference/settings/merge-tree-settings/map-buckets#map_buckets_strategy) 决定。

<div id="map_serialization_version_for_zero_level_parts">
  ## map_serialization_version_for_zero_level_parts
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "新增了一个用于控制零级 parts 的 Map 序列化版本的设置"}]}]} />

此设置允许为插入时创建的零级 parts 的
`Map` 列指定不同的序列化版本。
如果希望在插入时避免性能下降，可以对零级 parts 保持使用 `basic`
序列化，同时对合并后的 parts 使用 `with_buckets`。
