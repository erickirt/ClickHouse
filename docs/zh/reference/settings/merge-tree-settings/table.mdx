<div id="table_readonly">
  ## table_readonly
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "用于将表标记为只读、防止插入和修改的新设置"}]}]} />

如果设置为 true，则该表将处于只读模式，不会在磁盘上执行任何修改。

所有会修改表的前台操作都会被拒绝：插入、变更、`OPTIMIZE`，以及会修改数据的分区命令
(`ATTACH`/`MOVE`/`DROP`/`DROP DETACHED`/`FETCH`/`REPLACE PARTITION`，以及以该表为目标的 `MOVE PARTITION ... TO TABLE`) 。不会修改表数据的操作，
例如 `FREEZE`/`UNFREEZE` 和 `FORGET PARTITION`，仍然允许执行。

同样也不会调度任何后台任务：常规合并、TTL 合并 (`DELETE`/`MOVE`/重新压缩) 、重新压缩合并、后台变更
以及后台分片移动都会被禁止。因此，启用此设置后，带有 TTL 的表将不再回收或移动其已过期数据。

始终可以通过 `ALTER TABLE ... MODIFY SETTING table_readonly = 0` (或 `RESET SETTING`) 将此设置切换回来。`ReplicatedMergeTree` 不支持此设置。
