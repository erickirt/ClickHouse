  ## escape_index_filenames
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "1"},{"label": "对为索引创建的文件名中的非 ASCII 字符进行转义"}]}]} />

在 26.1 之前，我们不会对为次级索引创建的文件名中的特殊符号进行转义，这可能会导致某些
索引名称中的字符生成损坏的 parts。添加此设置纯粹是出于兼容性考虑。除非你
需要读取名称中包含非 ASCII 字符的索引所对应的旧 parts，否则不应更改此设置。

对 MergeTree 表的 Wide parts 中为 Variant 数据类型子列创建的文件名中的特殊符号进行转义。此设置用于兼容性。
