<div id="prefer_fetch_merged_part_size_threshold">
  ## prefer_fetch_merged_part_size_threshold
</div>

<SettingsInfoBlock type="UInt64" default_value="10737418240" />

如果 parts 的大小总和超过此阈值，并且自复制日志条目创建以来的时间大于
`prefer_fetch_merged_part_time_threshold`，则优先从副本拉取已合并的 part，
而不是在本地执行合并。这样做是为了加快非常耗时的合并操作。

可能的值：

* 任意正整数。

<div id="prefer_fetch_merged_part_time_threshold">
  ## prefer_fetch_merged_part_time_threshold
</div>

<SettingsInfoBlock type="Seconds" default_value="3600" />

如果复制日志 (ClickHouse Keeper 或 ZooKeeper) 条目自创建以来经过的时间
超过此阈值，且 parts 的总大小
大于 `prefer_fetch_merged_part_size_threshold`，则优先从副本拉取
已合并的 part，而不是在本地执行合并。这样做是为了加快
耗时很长的合并。

可能的值：

* 任意正整数。
