<div id="object_shared_data_serialization_version">
  ## object_shared_data_serialization_version
</div>

<SettingsInfoBlock type="MergeTreeObjectSharedDataSerializationVersion" default_value="advanced" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.8"},{"label": "map"},{"label": "新增用于控制 JSON 序列化版本的设置"}]}, {"id": "row-2","items": [{"label": "25.12"},{"label": "advanced"},{"label": "默认启用高级共享数据序列化版本"}]}]} />

JSON 数据类型内共享数据的序列化版本。

可能的值：

* `map` - 将共享数据存储为 `Map(String, String)`
* `map_with_buckets` - 将共享数据存储为多个独立的 `Map(String, String)` 列。使用桶可以提升从共享数据中读取单个路径的性能。
* `advanced` - 一种专为共享数据设计的特殊序列化方式，可显著提升从共享数据中读取单个路径的性能。
  请注意，这种序列化会增加磁盘上共享数据的存储大小，因为会存储大量额外信息。

`map_with_buckets` 和 `advanced` 序列化的桶数量由以下设置决定：
[object&#95;shared&#95;data&#95;buckets&#95;for&#95;compact&#95;part](/zh/reference/settings/merge-tree-settings/object-shared#object_shared_data_buckets_for_compact_part)/[object&#95;shared&#95;data&#95;buckets&#95;for&#95;wide&#95;part](/zh/reference/settings/merge-tree-settings/object-shared#object_shared_data_buckets_for_wide_part)。

建议不要对零级 parts 使用 `advanced` 共享数据序列化，因为它可能会显著增加
插入时间。
