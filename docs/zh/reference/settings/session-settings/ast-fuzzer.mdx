  ## ast_fuzzer_any_query
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "新增设置：允许对所有查询类型进行 fuzzing，而不仅限于只读查询。"}]}]} />

当为 false (默认值) 时，服务器端 AST fuzzer (由 `ast_fuzzer_runs` 控制) 仅对只读查询 (SELECT、EXPLAIN、SHOW、DESCRIBE、EXISTS) 进行 fuzzing。当为 true 时，则会对所有查询类型进行 fuzzing，包括 DDL 和 INSERT。

<div id="ast_fuzzer_runs">
  ## ast_fuzzer_runs
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Float" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "用于启用服务器端 AST fuzzer 的新设置。"}]}]} />

启用服务器端 AST fuzzer。它会在每次普通查询后运行随机查询，并丢弃其结果。

* 0：禁用 (默认) 。
* 大于 0 且小于 1 的值 (exclusive) ：运行单个模糊测试查询的概率。
* 值 &gt;= 1：每次普通查询要运行的模糊测试查询数量。

fuzzer 会跨所有会话累积来自全部查询的 AST 片段，并随着时间推移生成越来越有意思的变体。执行失败的模糊测试查询会被静默丢弃；结果绝不会返回给客户端。
