<div id="system_events_show_zero_values">
  ## system_events_show_zero_values
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

允许从 [`system.events`](/zh/reference/system-tables/events) 中选取值为 0 的事件。

某些监控系统要求在每个检查点都传递所有指标值，即使指标值为 0 也是如此。

可能的值：

* 0 — 已禁用。
* 1 — 已启用。

**示例**

查询

```sql
SELECT * FROM system.events WHERE event='QueryMemoryLimitExceeded';
```

结果

```text
Ok.
```

查询

```sql
SET system_events_show_zero_values = 1;
SELECT * FROM system.events WHERE event='QueryMemoryLimitExceeded';
```

结果

```text
┌─event────────────────────┬─value─┬─description───────────────────────────────────────────┐
│ QueryMemoryLimitExceeded │     0 │ Number of times when memory limit exceeded for query. │
└──────────────────────────┴───────┴───────────────────────────────────────────────────────┘
```

<div id="system_metric_log_show_zero_values_in_histograms">
  ## system_metric_log_show_zero_values_in_histograms
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "0"},{"label": "用于控制是否将零值直方图数据写入 system.metric_log 的 histograms 嵌套列的新设置。"}]}]} />

控制是否将零值直方图数据写入 `system.metric_log` 的 `histograms` 嵌套列。

默认情况下，总观测 `count` 为零的直方图会被跳过；并且在每个已输出的直方图中，没有任何观测的桶条目也会从 `histogram` 映射中省略。启用此设置后，无论 `count` 是否为零，都会写入每个直方图和每个桶——这对于要求每个指标在每个 checkpoint 都出现的监控系统很有用。

可能的值：

* 0 — 已禁用。`count = 0` 的直方图不会输出；已输出的直方图仅包含至少有一次观测的桶。
* 1 — 已启用。所有直方图都会被写入，并且每个桶边界都会出现在 `histogram` 中。
