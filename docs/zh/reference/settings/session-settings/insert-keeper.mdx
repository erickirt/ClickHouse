<div id="insert_keeper_max_retries">
  ## insert_keeper_max_retries
</div>

<SettingsInfoBlock type="UInt64" default_value="20" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "23.2"},{"label": "20"},{"label": "在向 replicated MergeTree 执行 INSERT 时启用与 Keeper 的重新连接，提高可靠性"}]}]} />

该设置用于指定在向 replicated MergeTree 执行 INSERT 期间，ClickHouse Keeper (或 ZooKeeper) 请求的最大重试次数。只有因网络错误、Keeper 会话超时或请求超时而失败的 Keeper 请求才会被重试。

可能的值：

* 正整数。
* 0 — 禁用重试

Keeper 请求会在经过一段超时时间后重试。该超时时间由以下设置控制：`insert_keeper_retry_initial_backoff_ms`、`insert_keeper_retry_max_backoff_ms`。
第一次重试会在 `insert_keeper_retry_initial_backoff_ms` 指定的超时时间后进行。后续超时时间将按如下方式计算：

```
timeout = min(insert_keeper_retry_max_backoff_ms, latest_timeout * 2)
```

例如，如果 `insert_keeper_retry_initial_backoff_ms=100`、`insert_keeper_retry_max_backoff_ms=10000` 且 `insert_keeper_max_retries=8`，那么超时时间将分别为 `100, 200, 400, 800, 1600, 3200, 6400, 10000`。

除了容错之外，这些重试也是为了提供更好的用户体验——例如，如果 Keeper 因升级而重启，它们可以避免在执行 INSERT 期间返回错误。
