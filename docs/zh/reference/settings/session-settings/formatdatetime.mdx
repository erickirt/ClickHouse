这些设置可在 [system.settings](/zh/reference/system-tables/settings) 中找到，并由 [源文件](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp) 自动生成。

<div id="formatdatetime_e_with_space_padding">
  ## formatdatetime_e_with_space_padding
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "0"},{"label": "改进了与 MySQL DATE_FORMAT\/STR_TO_DATE 的兼容性"}]}]} />

函数 &#39;formatDateTime&#39; 中的格式说明符 &#39;%e&#39; 会在个位数日期前添加一个空格，例如输出 &#39; 2&#39; 而不是 &#39;2&#39;。

<div id="formatdatetime_format_without_leading_zeros">
  ## formatdatetime_format_without_leading_zeros
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

函数 `formatDateTime` 中的格式说明符 &#39;%c&#39;、&#39;%l&#39; 和 &#39;%k&#39; 会输出不带前导零的月份和小时。

<div id="formatdatetime_parsedatetime_m_is_month_name">
  ## formatdatetime_parsedatetime_m_is_month_name
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "23.4"},{"label": "1"},{"label": "改进了与 MySQL DATE_FORMAT\/STR_TO_DATE 的兼容性"}]}]} />

函数 &#39;formatDateTime&#39; 和 &#39;parseDateTime&#39; 中的格式说明符 &#39;%M&#39; 会输出/解析月份名称，而不是分钟。
