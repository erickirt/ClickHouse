  ## use_paimon_metadata_files_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "新增设置，用于启用对已解析的 Paimon metadata files（manifest 列表和 manifest 文件）的内存缓存。对于持久化 Paimon 表引擎，必须在 metadata 初始化前启用；对于表函数，则按每次查询评估该设置。可避免后续查询重复从对象存储下载并反序列化 metadata files。"}]}]} />

启用后，paimon 表函数和 paimon 存储可能会使用 paimon metadata files cache。

Paimon 表函数会按每次查询评估此设置，而持久化 Paimon 表引擎会在 metadata 初始化时固定该设置：若要更改已创建表的这一决定，必须先将其 drop，再重新创建。

可能的值：

* 0 - 禁用
* 1 - 启用
