<div id="merge_tree_coarse_index_granularity">
  ## merge_tree_coarse_index_granularity
</div>

<SettingsInfoBlock type="UInt64" default_value="8" />

搜索数据时，ClickHouse 会检查索引文件中的数据标记。如果 ClickHouse 发现所需的键位于某个范围内，就会将该范围划分为 `merge_tree_coarse_index_granularity` 个子范围，并在这些子范围内递归搜索所需的键。

可选值：

* 任意正偶整数。

<div id="merge_tree_compact_parts_min_granules_to_multibuffer_read">
  ## merge_tree_compact_parts_min_granules_to_multibuffer_read
</div>

<CloudOnlyBadge />

<SettingsInfoBlock type="UInt64" default_value="16" />

仅在 ClickHouse Cloud 中生效。此设置指定 MergeTree 表的 compact part 中单个 stripe 的粒度数达到多少时使用 multibuffer reader；该读取器支持并行读取和预取。从远程文件系统读取时，使用 multibuffer reader 会增加读取请求的数量。当某个分片包含的粒度数既小于此阈值，又小于正在读取的列数时，会使用 single buffer reader 读取该分片，因为在这种情况下，single buffer reader 发出的读取请求少于 multibuffer reader。

<div id="merge_tree_generic_exclusion_search_max_steps">
  ## merge_tree_generic_exclusion_search_max_steps
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "用于限制主键索引上 generic exclusion search 步数的新设置。"}]}]} />

当某个过滤器无法被评估为主键上的单个连续范围时，例如它使用了除第一个键列之外的其他键列，ClickHouse 会在索引标记上运行迭代式的 generic exclusion search algorithm。分析 text index 时也会使用同样的算法。此设置会限制该算法在每个数据分区片段上消耗的步数 (索引检查次数) 。

预算会优先用于当前剩余的最大标记范围。当预算耗尽时，尚未完全分析的范围会被整体接受，因此查询结果仍然正确，但读取的粒度可能会比无限制搜索选出的更多。预算越低，索引分析越快，但代价是需要读取更多数据。该限制是近似限制，而不是分析成本的严格上限：搜索大致可能会多出一轮拆分；并且当该分区片段已经被划分为许多范围时 (例如由 query condition cache 导致) ，无论限制值是多少，每个范围至少都会被检查一次。

搜索在每个数据分区片段上执行的步数会记录在该查询的 trace 级别日志消息中，而 `IndexGenericExclusionSearchStepLimitReached` 和 `TextIndexGenericExclusionSearchStepLimitReached` 这两个 profile events 会统计预算耗尽的次数。

(默认) 值 0 表示步数不受限制。

可选值：

* 0 表示步数不受限制，或任意正整数。

<div id="merge_tree_max_bytes_to_use_cache">
  ## merge_tree_max_bytes_to_use_cache
</div>

<SettingsInfoBlock type="UInt64" default_value="2013265920" />

如果 ClickHouse 在一次查询中需要读取超过 `merge_tree_max_bytes_to_use_cache` 字节的数据，则不会使用未压缩块缓存。

未压缩块缓存用于存储为查询提取的数据。ClickHouse 使用此缓存来加快重复的小型查询的响应速度。此设置可防止读取大量数据的查询冲刷缓存。[uncompressed&#95;cache&#95;size](/zh/reference/settings/server-settings/settings#uncompressed_cache_size) 这一服务器设置定义了未压缩块缓存的大小。

可能的值：

* 任意正整数。

<div id="merge_tree_max_rows_to_use_cache">
  ## merge_tree_max_rows_to_use_cache
</div>

<SettingsInfoBlock type="UInt64" default_value="1048576" />

如果 ClickHouse 在一次查询中需要读取的行数超过 `merge_tree_max_rows_to_use_cache`，则不会使用未压缩块缓存。

未压缩块缓存用于存储为查询提取出的数据。ClickHouse 使用此缓存来加快重复执行的小型查询的响应速度。此设置可防止读取大量数据的查询污染缓存。[uncompressed&#95;cache&#95;size](/zh/reference/settings/server-settings/settings#uncompressed_cache_size) 服务器设置定义了未压缩块缓存的大小。

可选值：

* 任意正整数。
