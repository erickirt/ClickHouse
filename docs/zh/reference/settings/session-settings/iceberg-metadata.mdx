<div id="iceberg_metadata_log_level">
  ## iceberg_metadata_log_level
</div>

<SettingsInfoBlock type="IcebergMetadataLogLevel" default_value="none" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "none"},{"label": "新增设置。"}]}]} />

控制 Iceberg 表元数据写入 system.iceberg&#95;metadata&#95;log 时的日志级别。
通常仅在调试时修改此设置。

可能的值：

* none - 不记录元数据日志。
* metadata - 根 metadata.json 文件。
* manifest&#95;list&#95;metadata - 以上所有内容 + 与快照对应的 avro manifest 列表中的元数据。
* manifest&#95;list&#95;entry - 以上所有内容 + avro manifest 列表条目。
* manifest&#95;file&#95;metadata - 以上所有内容 + 已遍历的 avro manifest 文件中的元数据。
* manifest&#95;file&#95;entry - 以上所有内容 + 已遍历的 avro manifest 文件条目。

<div id="iceberg_metadata_staleness_ms">
  ## iceberg_metadata_staleness_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "新增设置：允许在 READ 操作中使用缓存的元数据版本，以避免从远程 catalog 拉取"}]}]} />

如果该值非零，且存在比给定 staleness 窗口更新的缓存元数据快照，则跳过从远程 catalog 拉取 iceberg 元数据。值为 0 表示始终从远程 catalog 拉取最新的元数据版本。将此设置为非零，可以用一定的 staleness 换取更低的读取操作延迟。
