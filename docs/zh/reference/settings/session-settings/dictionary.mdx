这些设置可在 [system.settings](/zh/reference/system-tables/settings) 中查看，并由 [源代码](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp) 自动生成。

<div id="dictionary_lazy_load">
  ## dictionary_lazy_load
</div>

<SettingsInfoBlock type="BoolAuto" default_value="auto" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "auto"},{"label": "用于为单个字典覆盖服务器设置 `dictionaries_lazy_load` 的新设置。"}]}]} />

控制在 `CREATE DICTIONARY` 的 `SETTINGS` 子句中指定时字典的加载行为：`1` 表示延迟到首次使用时再加载，`0` 表示在创建时加载字典，`'auto'` 表示遵循服务器设置 `dictionaries_lazy_load`。在会话或查询级别设置时，此设置无效。

<div id="dictionary_use_async_executor">
  ## dictionary_use_async_executor
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

使用多个线程执行读取字典源的管道。仅支持使用本地 CLICKHOUSE 源的字典。
