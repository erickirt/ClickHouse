<div id="optimize_aggregation_in_order">
  ## optimize_aggregation_in_order
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

在 [MergeTree](/zh/reference/engines/table-engines/mergetree-family/mergetree) 表中，对按相应顺序聚合数据的 [SELECT](/zh/reference/statements/select/index) 查询启用 [GROUP BY](/zh/reference/statements/select/group-by) 优化。

可能的值：

* 0 — `GROUP BY` 优化已禁用。
* 1 — `GROUP BY` 优化已启用。

**另请参见**

* [GROUP BY 优化](/zh/reference/statements/select/group-by#group-by-optimization-depending-on-table-sorting-key)

<div id="optimize_aggregation_in_order_limit">
  ## optimize_aggregation_in_order_limit
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "新增设置：当 `ORDER BY` 是 `GROUP BY` 排序描述的前缀时，将 `LIMIT` 下推到按顺序聚合中，以便在满足条件后提前终止。"}]}]} />

启用后，如果按顺序聚合处于活动状态，会将 LIMIT 下推到聚合步骤中，以便在生成足够数量的分组后提前终止。当 ORDER BY 与 GROUP BY 键前缀匹配时，这样可以减少读取的数据量。它可能会降低 `rows_before_limit_at_least` 报告的值；如果需要精确计数，请使用 `exact_rows_before_limit`。
