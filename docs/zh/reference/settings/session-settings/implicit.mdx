  ## implicit_select
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.10"},{"label": "0"},{"label": "新增设置。"}]}]} />

允许编写省略开头 `SELECT` 关键字的简单 SELECT 查询，便于计算器式用法，例如 `1 + 2` 也可作为有效查询。

在 `clickhouse-local` 中，此设置默认启用，也可以显式禁用。

<div id="implicit_table_at_top_level">
  ## implicit_table_at_top_level
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": ""},{"label": "新增设置，用于 clickhouse-local"}]}]} />

如果该设置非空，顶层不带 FROM 的查询将从此表读取，而不是从 system.one 读取。

这用于 clickhouse-local 中的输入数据处理。
用户也可以显式设置该设置，但其设计目的并非用于这种场景。

子查询不受该设置影响 (包括标量子查询、FROM 子查询或 IN 子查询) 。
在 UNION、INTERSECT、EXCEPT 链中，顶层的 SELECT 会被统一处理，并受该设置影响，而不受括号分组方式的影响。
该设置会如何影响视图和分布式查询，当前未作规定。

该设置接受一个表名 (此时会从当前数据库解析该表) ，或者接受形如 &#39;database.table&#39; 的限定名。
database 和 table 名都不能加引号——只允许简单标识符。
