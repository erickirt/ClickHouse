这些设置用于配置 ClickHouse 服务器，由 ClickHouse 源代码自动生成。

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

如果表的数量大于此值，服务器将抛出异常。

以下表不计入统计：

* view
* remote
* dictionary
* system

仅统计使用以下数据库引擎的表：

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  值为 `0` 表示不受限制。
</Note>

**示例**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

如果已附加的表数量超过指定值，ClickHouse server 会向 `system.warnings` 表写入警告消息。

**示例**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

对删除表的限制。

如果 [MergeTree](/zh/reference/engines/table-engines/mergetree-family/mergetree) 表的大小超过 `max_table_size_to_drop` (以字节为单位) ，则无法使用 [`DROP`](/zh/reference/statements/drop) 查询或 [`TRUNCATE`](/zh/reference/statements/truncate) 查询将其删除。

<Note>
  值为 `0` 表示你可以不受任何限制地删除所有表。

  此设置生效无需重启 ClickHouse 服务器。另一种禁用该限制的方法是创建 `<clickhouse-path>/flags/force_drop_table` 文件。
</Note>

**示例**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
