这些设置用于配置 ClickHouse 服务器，由 ClickHouse 源代码自动生成。

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />以 `total_memory_profiler_sample_probability` 的概率收集大小小于或等于指定值的随机内存分配。0 表示禁用。你可能需要将 `max_untracked_memory` 设置为 0，以使该阈值按预期生效。

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />以 `total_memory_profiler_sample_probability` 指定的概率，收集大小大于或等于指定值的随机内存分配。0 表示禁用。你可能需要将 &#39;max&#95;untracked&#95;memory&#39; 设置为 0，才能使此阈值按预期生效。

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />每当 server 的内存使用量按字节计超过下一个步长时，内存分析器都会采集分配时的堆栈跟踪。0 表示禁用内存分析器。将该值设得低于几兆字节会拖慢 server。

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="No" />

允许按指定概率收集随机内存分配和释放，并将其作为 `trace_type` 等于 `MemorySample` 的记录写入 [system.trace&#95;log](/zh/reference/system-tables/trace_log) 系统表。该概率适用于每一次内存分配或释放，与内存分配大小无关。请注意，只有当未跟踪内存量超过未跟踪内存限制时才会发生采样 (默认值为 `4` MiB) 。如果降低 [total&#95;memory&#95;profiler&#95;step](/zh/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step)，这个限制也可以相应降低。你可以将 `total_memory_profiler_step` 设置为 `1`，以获得更细粒度的采样。

可能的值：

* 正的 Double 值。
* `0` — 禁止将随机内存分配和释放写入 `system.trace_log` 系统表。
