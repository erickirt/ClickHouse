这些设置用于配置 ClickHouse 服务器，并由 ClickHouse 源代码自动生成。

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

达到 `max_server_memory_usage` 后，允许触发 `MEMORY_LIMIT_EXCEEDED` 的通用 C++ 内存分配的最小大小 (以字节为单位) 。这类通用分配包括标准容器、字符串、`std::vector` 扩容、智能指针等产生的分配。更小的通用分配仍会计入内存跟踪器，但即使超过限制也会被允许成功，从而减少在接近 OOM 时清理和异常处理路径中出现的误报失败。

由 ClickHouse 自身的大型缓冲区发起的分配——列数据、哈希表、arenas、查询管道、IO 缓冲区——始终遵守内存限制，并且无论此值如何都可能抛出异常。此设置仅影响通过 `operator new` 进行的隐式分配。

值为 `0` (默认) 时，将保留旧行为：内存跟踪器永远不会拒绝隐式的 `operator new` 分配，只有显式的 ClickHouse 分配才会触发 `MEMORY_LIMIT_EXCEEDED`。

注意：为避免副作用，建议将该值设置为大于 `max_untracked_memory`。

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

用于判断是否断开连接的 OS CPU 等待时间 (`OSCPUWaitMicroseconds` 指标) 与忙碌时间 (`OSCPUVirtualTimeMicroseconds` 指标) 之间的最小比率。系统会使用最小比率与最大比率之间的线性插值来计算概率，在该点处概率为 0。
更多详情请参见[控制服务器 CPU 过载时的行为](/zh/concepts/features/configuration/settings/server-overload)。
