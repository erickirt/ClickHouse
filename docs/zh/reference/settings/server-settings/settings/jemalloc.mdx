这些设置用于配置 ClickHouse server，且由 ClickHouse 源代码自动生成。

<div id="jemalloc_collect_global_profile_samples_in_trace_log">
  ## jemalloc_collect_global_profile_samples_in_trace_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />将 jemalloc 的采样内存分配记录存储到 system.trace&#95;log 中

<div id="jemalloc_max_background_threads_num">
  ## jemalloc_max_background_threads_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />可创建的 jemalloc 后台线程最大数量。设为 0 时，使用 jemalloc 的默认值

<div id="jemalloc_profiler_sampling_rate">
  ## jemalloc_profiler_sampling_rate
</div>

<SettingsInfoBlock type="UInt64" default_value="19" changeable_without_restart="No" />

控制 jemalloc 的 `lg_prof_sample` —— 即分配样本之间平均间隔 (以字节为单位) 的以 2 为底对数。
默认值 19 对应 512 KiB。将其设置为较小的值会提高采样频率 (开销更大，但细节更多) ；设置为较大的值则会降低采样频率。
更改此值会调用 `prof.reset`，从而重置所有已累积的性能分析统计信息。要求先启用性能分析 (`MALLOC_CONF=prof:true`) 。
