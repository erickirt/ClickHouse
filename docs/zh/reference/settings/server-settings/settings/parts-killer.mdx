  ## parts_killer_max_condemned_parts_per_batch
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="100000" changeable_without_restart="No" />

SharedMergeTree 表在单次清理周期内从 Keeper 拉取的最大废弃 parts 数量。限制批次大小可以确保列表查询和 multi-get 操作不会超出 Keeper 的操作超时时限，因此即使表中积压了大量废弃 parts，清理也能持续推进。该值必须大于零：如果设为 `0`，Keeper 会在每个周期都返回空列表，导致清理永久停滞。仅在 ClickHouse Cloud 中可用
