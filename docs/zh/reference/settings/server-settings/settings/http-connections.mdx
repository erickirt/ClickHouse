这些设置用于配置 ClickHouse server，由 ClickHouse 源代码自动生成。

<div id="http_connections_hard_limit">
  ## http_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />达到此限制后，如再尝试创建，将抛出异常。设为 0 可关闭硬性限制。该限制适用于不属于任何 disk 或 storage 的 HTTP 连接。

<div id="http_connections_rcvbuf">
  ## http_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />通用 HTTP 连接的 SO&#95;RCVBUF 选项大小。若设置为大于 0 的值，则会覆盖接收缓冲区的内核 TCP 自动调优。0 = 内核默认值 (自动调优) 。注意：将此设置改回 0 后，只有新创建的连接才会恢复自动调优；现有的池化连接会继续保留固定的缓冲区大小，直到被重新创建。

<div id="http_connections_sndbuf">
  ## http_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />通用 HTTP 连接的 SO&#95;SNDBUF 选项大小。如果设置为大于 0 的值，则会覆盖发送缓冲区的内核 TCP 自动调优。0 = 内核默认值 (自动调优) 。注意：将此设置改回 0 只会对新创建的连接恢复自动调优；现有的连接池中的连接会保留固定的缓冲区大小，直到重新创建。

<div id="http_connections_soft_limit">
  ## http_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />超过此限制的连接存活时间会明显缩短。该限制适用于不属于任何 disk 或 Storage 的 HTTP 连接。

<div id="http_connections_store_limit">
  ## http_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />超过此限制的连接在使用后会被重置。将其设为 0 可关闭连接缓存。该限制适用于不属于任何磁盘或存储的 HTTP 连接。

<div id="http_connections_warn_limit">
  ## http_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />如果正在使用的连接数超过此限制，则会在日志中写入 Warning 消息。该限制适用于不属于任何 disk 或 storage 的 HTTP 连接。
