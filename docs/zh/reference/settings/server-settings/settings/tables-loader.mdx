这些设置用于配置 ClickHouse 服务器，且是根据 ClickHouse 源代码自动生成的。

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

设置在 background 池 中执行异步加载作业的线程数。background 池 用于在 服务器 启动后、且没有查询在等待相关表时，异步加载表。如果表的数量很多，将 background 池 中的线程数保持在较低水平可能更有利，这样可以为并发查询执行预留 CPU 资源。

<Note>
  值为 `0` 表示将使用所有可用 CPU。
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

设置在前台池中执行加载作业的线程数。前台池用于在服务器开始监听端口之前同步加载表，也用于加载需要等待的表。前台池的优先级高于后台池。这意味着，只要前台池中还有作业在运行，后台池中就不会启动任何作业。

<Note>
  值为 `0` 表示将使用所有可用的 CPU。
</Note>
