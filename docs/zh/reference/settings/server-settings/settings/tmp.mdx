这些设置用于配置 ClickHouse server，且由 ClickHouse 源代码自动生成。

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="No" />

用于在本地文件系统中存储处理大型查询时产生的临时数据的路径。

<Note>
  * 配置临时数据存储时，只能使用以下选项之一：tmp&#95;path、tmp&#95;policy、temporary&#95;data&#95;in&#95;cache。
  * 末尾斜杠必不可少。
</Note>

**示例**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

用于存放临时数据的存储策略。所有带有 `tmp` 前缀的文件都会在启动时删除。

<Note>
  将对象存储用作 `tmp_policy` 时的建议：

  * 在每台服务器上使用独立的 `bucket:path`
  * 使用 `metadata_type=plain`
  * 你可能还需要为该存储桶设置生存时间 (TTL)
</Note>

<Note>
  * 配置临时数据存储时，只能使用以下选项之一：`tmp_path`、`tmp_policy`、`temporary_data_in_cache`。
  * `move_factor`、`keep_free_space_bytes`、`max_data_part_size_bytes` 会被忽略。
  * 该策略必须且只能包含 *一个卷*

  更多信息，请参见 [MergeTree 表引擎](/zh/reference/engines/table-engines/mergetree-family/mergetree) 文档。
</Note>

**示例**

当 `/disk1` 已满时，临时数据将存储到 `/disk2`。

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
