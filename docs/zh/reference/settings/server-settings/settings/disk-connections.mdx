这些设置用于配置 ClickHouse server，并由 ClickHouse 源代码自动生成。

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />达到此限制后，如再尝试创建连接，将抛出异常。将其设置为 0 可关闭硬限制。该限制适用于磁盘连接。

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />disk (S3、Azure、GCS) 连接的 SO&#95;RCVBUF 选项大小。默认值为 204800 (200 KB) ：对象存储通常通过同一区域内网络访问，其带宽-时延乘积远低于 100 KB，因此较小的上限就能提供完整吞吐量，同时避免内核自动调优原本会为每个套接字分配数 MiB 的缓冲区。将其设为 0，则改为对接收缓冲区使用内核 TCP 自动调优。注意：将此设置改回 0 只会对新建连接恢复自动调优；现有的池化连接会保留固定的缓冲区大小，直到被重新创建。

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="是" />disk (S3、Azure、GCS) 连接的 `SO_SNDBUF` 选项大小。若设置为大于 0 的值，将覆盖内核 TCP 发送缓冲区的自动调优。0 = 内核默认值 (自动调优) 。注意：将此设置改回 0 后，只有新创建的连接才会恢复自动调优；现有的池化连接会保留固定的缓冲区大小，直到重新创建。

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />超过此限制的连接其存活时间会显著缩短。该限制适用于磁盘连接。

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />超过此限制的连接在使用后会被重置。设为 0 可关闭连接缓存。该限制适用于磁盘连接。

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />如果正在使用的连接数超过此限制，系统会将警告消息写入日志。该限制适用于磁盘连接。
