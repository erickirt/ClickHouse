这些设置用于配置 ClickHouse 服务器，并由 ClickHouse 源代码自动生成。

<div id="distributed_cache_apply_throttling_settings_from_client">
  ## distributed_cache_apply_throttling_settings_from_client
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />缓存服务器是否应使用从客户端接收的限流设置。

<div id="distributed_cache_keep_up_free_connections_ratio">
  ## distributed_cache_keep_up_free_connections_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.1" changeable_without_restart="否" />分布式缓存会尽量保持的空闲活动连接数量的软限制。当空闲连接数低于 distributed&#95;cache&#95;keep&#95;up&#95;free&#95;connections&#95;ratio * max&#95;connections 时，将关闭活动时间最早的连接，直到该数量重新高于此限制。

<div id="distributed_cache_write_pool_size">
  ## distributed_cache_write_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />可在后台线程中同时运行的分布式缓存写请求的最大数量 (跨所有查询) 。达到该限制后，写入将改为以内联方式经过缓存 (即在调用线程上执行) ，而不是在后台线程上执行，因此既不会因等待 slot 而阻塞，也不会创建数量无限增长的线程。
