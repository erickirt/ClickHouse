这些设置用于配置 ClickHouse server，并由 ClickHouse 源代码自动生成。

<div id="distributed_ddl">
  ## distributed_ddl
</div>

管理在集群上执行[分布式 DDL 查询](/zh/reference/statements/distributed-ddl) (`CREATE`、`DROP`、`ALTER`、`RENAME`) 。
仅当启用了 [ZooKeeper](/zh/reference/settings/server-settings/settings/zookeeper#zookeeper) 时有效。

`<distributed_ddl>` 中可配置的设置包括：

| 设置                     | 描述                                                   | 默认值                            |
| ---------------------- | ---------------------------------------------------- | ------------------------------ |
| `cleanup_delay_period` | 收到新节点事件后，如果距离上次清理已超过 `cleanup_delay_period` 秒，则开始清理。 | `60` 秒                         |
| `max_tasks_in_queue`   | 队列中允许存在的最大任务数。                                       | `1,000`                        |
| `path`                 | Keeper 中用于 DDL 查询的 `task_queue` 的 path               |                                |
| `pool_size`            | 可同时运行的 `ON CLUSTER` 查询数量                             |                                |
| `profile`              | 用于执行 DDL 查询的 profile                                 |                                |
| `task_max_lifetime`    | 如果节点存在时间超过该值，则将其删除。                                  | `7 * 24 * 60 * 60` (一周，以秒为单位)  |

**示例**

```xml
<distributed_ddl>
    <!-- Path in ZooKeeper to queue with DDL queries -->
    <path>/clickhouse/task_queue/ddl</path>

    <!-- Settings from this profile will be used to execute DDL queries -->
    <profile>default</profile>

    <!-- Controls how much ON CLUSTER queries can be run simultaneously. -->
    <pool_size>1</pool_size>

    <!--
         Cleanup settings (active tasks will not be removed)
    -->

    <!-- Controls task TTL (default 1 week) -->
    <task_max_lifetime>604800</task_max_lifetime>

    <!-- Controls how often cleanup should be performed (in seconds) -->
    <cleanup_delay_period>60</cleanup_delay_period>

    <!-- Controls how many tasks could be in the queue -->
    <max_tasks_in_queue>1000</max_tasks_in_queue>
</distributed_ddl>
```

<div id="distributed_ddl_use_initial_user_and_roles">
  ## distributed_ddl_use_initial_user_and_roles
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />如果启用，ON CLUSTER 查询在远程分片上执行时将保留并使用发起方的用户和角色。这样可确保整个集群中的访问控制一致，但前提是该用户和这些角色在所有节点上都存在。
