这些设置用于配置 ClickHouse 服务器，并由 ClickHouse 源代码自动生成。

<div id="async_load_databases">
  ## async_load_databases
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

异步加载数据库和表。

* 如果为 `true`，则所有使用 `Ordinary`、`Atomic` 和 `Replicated` 引擎的非系统数据库都会在 ClickHouse 服务器 启动后异步加载。请参见 `system.asynchronous_loader` 表，以及 `tables_loader_background_pool_size` 和 `tables_loader_foreground_pool_size` 服务器设置。任何尝试访问尚未加载的表的查询，都会只等待该表完成启动。如果加载 job 失败，查询会重新抛出错误 (而不是像 `async_load_databases = false` 那样关闭整个 server) 。被至少一个查询等待的表会以更高优先级加载。针对某个数据库的 DDL queries 也会只等待该数据库完成启动。还可以考虑为等待中的查询总数设置限制 `max_waiting_queries`。
* 如果为 `false`，则所有数据库都会在 server 启动时加载。

**示例**

```xml
<async_load_databases>true</async_load_databases>
```

<div id="async_load_system_database">
  ## async_load_system_database
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

异步加载系统表。如果 `system` 数据库中有大量日志表和 parts，此设置会很有帮助。它独立于 `async_load_databases` 设置。

* 如果设置为 `true`，所有使用 `Ordinary`、`Atomic` 和 `Replicated` 引擎的系统数据库都会在 ClickHouse 服务器 启动后异步加载。请参见 `system.asynchronous_loader` 表、`tables_loader_background_pool_size` 和 `tables_loader_foreground_pool_size` 服务器级设置。任何尝试访问尚未加载的系统表的查询，都会一直等待，直到该表完成启动。只要某张表正被至少一个查询等待，就会以更高优先级加载。还可以考虑设置 `max_waiting_queries`，以限制等待中的查询总数。
* 如果设置为 `false`，系统数据库会在 server 启动前加载。

**示例**

```xml
<async_load_system_database>true</async_load_system_database>
```
