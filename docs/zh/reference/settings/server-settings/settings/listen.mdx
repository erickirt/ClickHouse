这些设置用于配置 ClickHouse 服务器，由 ClickHouse 源代码自动生成。

<div id="listen_backlog">
  ## listen_backlog
</div>

<SettingsInfoBlock type="UInt32" default_value="4096" changeable_without_restart="No" />

监听套接字的 backlog (待处理连接队列的大小) 。默认值 `<4096>` 与 Linux 5.4+ 的默认值相同。

通常不需要修改此值，因为：

* 默认值已经足够大，
* 服务器 使用单独的线程来接受 client 连接。

因此，即使你看到 `<TcpExtListenOverflows>` (来自 `<nstat>`) 非零，并且该计数器在 ClickHouse 服务器 上持续增长，也不意味着需要增大此值，因为：

* 通常，如果 `<4096>` 都不够用，说明更可能是 ClickHouse 内部存在某种扩缩容问题，因此更适合提交 issue。
* 这并不意味着 服务器 之后就能处理更多连接 (即使可以，到了那个时候，clients 也可能已经离开或断开连接) 。

**示例**

```xml
<listen_backlog>4096</listen_backlog>
```

<div id="listen_host">
  ## listen_host
</div>

限制允许发起请求的主机。如果希望服务器响应所有主机的请求，请指定 `::`。

示例：

```xml
<listen_host>::1</listen_host>
<listen_host>127.0.0.1</listen_host>
```

<div id="listen_reuse_port">
  ## listen_reuse_port
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

允许多个 服务器 监听同一个 address:port。请求将由操作系统路由到随机的 服务器。不建议启用此设置。

**示例**

```xml
<listen_reuse_port>0</listen_reuse_port>
```

<div id="listen_try">
  ## listen_try
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

如果在尝试监听时 IPv6 或 IPv4 网络不可用，服务器不会退出。

**示例**

```xml
<listen_try>0</listen_try>
```
