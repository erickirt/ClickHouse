<div id="description">
  ## 说明
</div>

此表包含可即时计算并以 Prometheus 格式导出的直方图指标，并且始终保持最新。用于替代已弃用的 `system.latency_log`。

<div id="columns">
  ## 列
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/zh/reference/data-types/string)) — 指标名称。
* `value` ([Float64](/zh/reference/data-types/float)) — 指标值。
* `description` ([String](/zh/reference/data-types/string)) — 指标说明。
* `labels` ([Map(String, String)](/zh/reference/data-types/map)) — 标记。

**别名：**

* `name` — `metric` 的别名。

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 示例
</div>

您可以使用如下查询，将所有直方图指标导出为 Prometheus 格式。

```sql
SELECT
  metric AS name,
  toFloat64(value) AS value,
  description AS help,
  labels,
  'histogram' AS type
FROM system.histogram_metrics
FORMAT Prometheus
```

<div id="metric_descriptions">
  ## 指标说明
</div>

| 指标                                                           | 说明                                                              |
| ------------------------------------------------------------ | --------------------------------------------------------------- |
| `keeper_response_time_ms_bucket`                             | Keeper 的响应时间，以毫秒为单位。                                            |
| `keeper_client_queue_duration_milliseconds_bucket`           | 请求在入队前等待以及在队列中等待、直到由 Keeper client 处理之前所花费的时间，以毫秒为单位。           |
| `keeper_receive_request_time_milliseconds_bucket`            | 在 Keeper TCP handler 中接收并解析来自 client 的请求所花费的时间，以毫秒为单位。          |
| `keeper_dispatcher_requests_queue_time_milliseconds_bucket`  | 请求在 Keeper dispatcher requests queue 中停留的时间，以毫秒为单位。             |
| `keeper_write_pre_commit_time_milliseconds_bucket`           | 写入请求在 Raft commit 之前预处理所花费的时间，以毫秒为单位。                           |
| `keeper_write_commit_time_milliseconds_bucket`               | 写入请求在 Raft commit 之后处理所花费的时间，以毫秒为单位。                            |
| `keeper_dispatcher_responses_queue_time_milliseconds_bucket` | 响应在 Keeper dispatcher 响应队列中停留的时间，以毫秒为单位。                        |
| `keeper_send_response_time_milliseconds_bucket`              | 在 Keeper TCP handler 中向 client 发送响应所花费的时间 (包括排队和写入套接字) ，以毫秒为单位。 |
| `keeper_read_wait_for_write_time_milliseconds_bucket`        | 读取请求等待其所依赖的写入请求完成所花费的时间，以毫秒为单位。                                 |
| `keeper_read_process_time_milliseconds_bucket`               | 在 Keeper 中处理读取请求所花费的时间，以毫秒为单位。                                  |
| `keeper_batch_size_elements_bucket`                          | 发送到 Raft 的批次大小，以元素数为单位。                                         |
| `keeper_batch_size_bytes_bucket`                             | 发送到 Raft 的批次大小，以字节为单位。                                          |
| `filesystem_cache_evicted_segment_hits_bucket`               | File 段在被驱逐时的缓存命中次数分布，按缓存名称标记。                                   |
| `filesystem_cache_evicted_segment_size_bytes_bucket`         | 被驱逐的 File 段的字节大小分布，按缓存名称标记。                                     |
| `filesystem_cache_evicted_segment_hits_by_user_bucket`       | 被驱逐的 File 段的缓存命中次数分布，按缓存名称和用户 ID 标记。                            |
| `filesystem_cache_evicted_segment_size_bytes_by_user_bucket` | 被驱逐的 File 段的字节大小分布，按缓存名称和用户 ID 标记。                              |
