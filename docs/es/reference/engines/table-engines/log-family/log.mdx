El motor pertenece a la familia de motores `Log`. Consulte las propiedades comunes de los motores `Log` y sus diferencias en el artГӯculo [Familia de motores Log](/es/reference/engines/table-engines/log-family/index).

`Log` se diferencia de [TinyLog](/es/reference/engines/table-engines/log-family/tinylog) en que mantiene un pequeГұo archivo de &quot;marcas&quot; junto a los archivos de columna. Estas marcas se escriben en cada bloque de datos y contienen desplazamientos que indican dГіnde empezar a leer el archivo para omitir el nГәmero especificado de filas. Esto permite leer los datos de la tabla en varios hilos.
Para el acceso concurrente a los datos, las operaciones de lectura pueden realizarse simultГЎneamente, mientras que las operaciones de escritura bloquean tanto las lecturas como otras escrituras.
El motor `Log` no admite Гӯndices. Del mismo modo, si falla una escritura en la tabla, la tabla queda daГұada y al leerla se devuelve un error. El motor `Log` es adecuado para datos temporales, tablas de escritura Гәnica y pruebas o fines de demostraciГіn.

<div id="table_engines-log-writing-the-data">
  ## Escritura de los datos
</div>

El motor `Log` almacena los datos de forma eficiente escribiendo cada columna en su propio archivo. Para cada tabla, el motor `Log` escribe los siguientes archivos en la ruta de almacenamiento especificada:

* `<column>.bin`: Un archivo de datos para cada columna, que contiene los datos serializados y comprimidos.
  `__marks.mrk`: Un archivo de marcas que almacena desplazamientos y recuentos de filas para cada bloque de datos insertado. Las marcas se utilizan para facilitar una ejecuciГіn eficiente de las consultas, ya que permiten que el motor omita bloques de datos irrelevantes durante las lecturas.

<div id="writing-process">
  ### Proceso de escritura
</div>

Cuando se escriben datos en una tabla `Log`:

1. Los datos se serializan y se comprimen en bloques.
2. Para cada columna, los datos comprimidos se aГұaden a su archivo `<column>.bin` correspondiente.
3. Se aГұaden las entradas correspondientes al archivo `__marks.mrk` para registrar el desplazamiento y el nГәmero de filas de los datos reciГ©n insertados.

<div id="table_engines-log-reading-the-data">
  ## Lectura de los datos
</div>

El archivo de marcas permite a ClickHouse paralelizar la lectura de los datos. Esto significa que una consulta `SELECT` devuelve las filas en un orden impredecible. Use la clГЎusula `ORDER BY` para ordenar las filas.

<div id="table_engines-log-example-of-use">
  ## Ejemplo de uso
</div>

CreaciГіn de una tabla:

```sql

    message_type String,

```

InserciГіn de datos:

```sql

```

Usamos dos consultas `INSERT` para crear dos bloques de datos dentro de los archivos `<column>.bin`.

ClickHouse usa varios hilos al seleccionar datos. Cada hilo lee un bloque de datos distinto y devuelve las filas resultantes de forma independiente a medida que finaliza. Como resultado, el orden de los bloques de filas en la salida puede no coincidir con el orden de esos mismos bloques en la entrada. Por ejemplo:

```sql

```
