Una tupla de elementos, cada uno con un [tipo](/es/reference/data-types/index). Tuple debe contener al menos un elemento.

Las tuplas se usan para la agrupaciГіn temporal de columnas. Las columnas pueden agruparse cuando se usa una expresiГіn IN en una consulta, asГӯ como para especificar determinados parГЎmetros formales de las funciones lambda. Para mГЎs informaciГіn, consulte las secciones [operadores IN](/es/reference/statements/in) y [funciones de orden superior](/es/reference/functions/regular-functions/overview#higher-order-functions).

Las tuplas pueden ser el resultado de una consulta. En este caso, en los formatos de texto distintos de JSON, los valores aparecen separados por comas entre `()`. En los formatos JSON, las tuplas se representan como arrays (en `[]`).

<div id="creating-tuples">
  ## CreaciГіn de tuplas
</div>

Puede usar una funciГіn para crear una tupla:

```sql
tuple(T1, T2, ...)
```

Ejemplo de cГіmo crear una tupla:

```sql
SELECT tuple(1, 'a') AS x, toTypeName(x)
```

```text

в”Ӯ (1,'a') в”Ӯ Tuple(UInt8, String)      в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

Un Tuple puede contener un Гәnico elemento

Ejemplo:

```sql
SELECT tuple('a') AS x;
```

```text

в”Ӯ ('a') в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

La sintaxis `(tuple_element1, tuple_element2)` puede usarse para crear una tupla de varios elementos sin necesidad de llamar a la funciГіn `tuple()`.

Ejemplo:

```sql
SELECT (1, 'a') AS x, (today(), rand(), 'someString') AS y, ('a') AS not_a_tuple;
```

<div id="data-type-detection">
  ## DetecciГіn de tipos de datos
</div>

Al crear tuplas sobre la marcha, ClickHouse infiere el tipo de los argumentos de las tuplas como los tipos mГЎs pequeГұos capaces de contener el valor del argumento proporcionado. Si el valor es [NULL](/es/reference/settings/formats#input_format_null_as_default), el tipo inferido es [Nullable](/es/reference/data-types/nullable).

Ejemplo de detecciГіn automГЎtica de tipos de datos:

```sql
SELECT tuple(1, NULL) AS x, toTypeName(x)
```

```text
в”Ңв”Җxв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”¬в”ҖtoTypeName(tuple(1, NULL))в”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ (1, NULL) в”Ӯ Tuple(UInt8, Nullable(Nothing)) в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

<div id="referring-to-tuple-elements">
  ## Referencia a los elementos de Tuple
</div>

Se puede hacer referencia a los elementos de Tuple por nombre o por Гӯndice:

```sql title="Query"
CREATE TABLE named_tuples (`a` Tuple(s String, i Int64)) ENGINE = Memory;
INSERT INTO named_tuples VALUES (('y', 10)), (('x',-10));

SELECT a.s FROM named_tuples; -- por nombre
SELECT a.2 FROM named_tuples; -- por Гӯndice
```

<div id="comparison-operations-with-tuple">
  ## Operaciones de comparaciГіn con Tuple
</div>

Dos tuplas se comparan secuencialmente, de izquierda a derecha. Si un elemento de la primera tupla es mayor (o menor) que el elemento correspondiente de la segunda, entonces la primera tupla es mayor (o menor) que la segunda; de lo contrario, si ambos elementos son iguales, se compara el siguiente elemento.

Ejemplo:

```sql
SELECT (1, 'z') > (1, 'a') c1, (2022, 01, 02) > (2023, 04, 02) c2, (1,2,3) = (3,2,1) c3;
```

```text
в”Ңв”Җc1в”Җв”¬в”Җc2в”Җв”¬в”Җc3в”Җв”җ
в”Ӯ  1 в”Ӯ  0 в”Ӯ  0 в”Ӯ
в””в”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”ҳ
```

Ejemplos reales:

```sql
CREATE TABLE test
(
    `year` Int16,
    `month` Int8,
    `day` Int8
)
ENGINE = Memory AS
SELECT *
FROM values((2022, 12, 31), (2000, 1, 1));

SELECT * FROM test;

в”Ңв”Җyearв”Җв”¬в”Җmonthв”Җв”¬в”Җdayв”Җв”җ
в”Ӯ 2022 в”Ӯ    12 в”Ӯ  31 в”Ӯ
в”Ӯ 2000 в”Ӯ     1 в”Ӯ   1 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”ҳ

SELECT *
FROM test
WHERE (year, month, day) > (2010, 1, 1);

в”Ңв”Җyearв”Җв”¬в”Җmonthв”Җв”¬в”Җdayв”Җв”җ
в”Ӯ 2022 в”Ӯ    12 в”Ӯ  31 в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”ҳ
CREATE TABLE test
(
    `key` Int64,
    `duration` UInt32,
    `value` Float64
)
ENGINE = Memory AS
SELECT *
FROM values((1, 42, 66.5), (1, 42, 70), (2, 1, 10), (2, 2, 0));

SELECT * FROM test;

в”Ңв”Җkeyв”Җв”¬в”Җdurationв”Җв”¬в”Җvalueв”Җв”җ
в”Ӯ   1 в”Ӯ       42 в”Ӯ  66.5 в”Ӯ
в”Ӯ   1 в”Ӯ       42 в”Ӯ    70 в”Ӯ

в””в”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

<div id="nullable-tuple">
  ## Nullable(Tuple(T1, T2, ...))
</div>

<Info>
  **Funcionalidad Beta**

  Requiere `SET enable_nullable_tuple_type = 1`
  Esta es una funcionalidad Beta.
</Info>

Permite que la tupla completa sea `NULL`, a diferencia de `Tuple(Nullable(T1), Nullable(T2), ...)`, donde solo los elementos individuales pueden ser `NULL`.

| Tipo                                       | Tuple puede ser NULL | Los elementos pueden ser NULL |
| ------------------------------------------ | -------------------- | ----------------------------- |
| `Nullable(Tuple(String, Int64))`           | вң…                    | вқҢ                             |
| `Tuple(Nullable(String), Nullable(Int64))` | вқҢ                    | вң…                             |

Ejemplo:

```sql
SET enable_nullable_tuple_type = 1;

CREATE TABLE test (
    id UInt32,
    data Nullable(Tuple(String, Int64))
) ENGINE = Memory;

INSERT INTO test VALUES (1, ('hello', 42)), (2, NULL);

SELECT * FROM test WHERE data IS NULL;
```

```txt
в”Ңв”Җidв”Җв”¬в”Җdataв”Җв”җ
в”Ӯ  2 в”Ӯ бҙәбөҒбҙёбҙё в”Ӯ
в””в”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```
