<div id="working-with-data-types">
  ## Trabajar con tipos de datos
</div>

Al crear un array sobre la marcha, ClickHouse define automГЎticamente el tipo del argumento como el tipo de datos mГЎs pequeГұo capaz de almacenar todos los argumentos indicados. Si hay valores [Nullable](/es/reference/data-types/nullable) o valores literales [NULL](/es/reference/settings/formats#input_format_null_as_default), el tipo de los elementos del array tambiГ©n pasa a ser [Nullable](/es/reference/data-types/nullable).

Ejemplos de detecciГіn automГЎtica de tipos de datos:



<div id="array-size">
  ## TamaГұo de Array
</div>

Es posible conocer el tamaГұo de un array usando la subcolumna `size0` sin leer toda la columna. Para arrays multidimensionales, puedes usar `sizeN-1`, donde `N` es la dimensiГіn deseada.

**Ejemplo**

```sql title="Query"

SELECT arr.size0, arr.size1, arr.size2 FROM t_arr;
```

<div id="reading-nested-subcolumns-from-array">
  ## Lectura de subcolumnas anidadas de Array
</div>

Si el tipo anidado `T` dentro de `Array` tiene subcolumnas (por ejemplo, si es una [tupla con nombre](/es/reference/data-types/tuple)), puede leer sus subcolumnas desde un tipo `Array(T)` con los mismos nombres de subcolumna. El tipo de una subcolumna serГЎ un `Array` del tipo de la subcolumna original.

**Ejemplo**

```sql

INSERT INTO t_arr VALUES ([(1, 'Hello'), (2, 'World')]), ([(3, 'This'), (4, 'is'), (5, 'subcolumn')]);
SELECT arr.field1, toTypeName(arr.field1), arr.field2, toTypeName(arr.field2) from t_arr;
```
