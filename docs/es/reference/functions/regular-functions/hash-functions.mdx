Las funciones hash pueden usarse para el reordenamiento seudoaleatorio determinista de elementos.

Simhash es una funciГіn hash que devuelve valores hash prГіximos para argumentos prГіximos (similares).

La mayorГӯa de las funciones hash aceptan cualquier nГәmero de argumentos de cualquier tipo.

<Note>
  El hash de NULL es NULL. Para obtener un hash no NULL de una columna Nullable, envuГ©lvala en una tupla:

  ```sql
  SELECT cityHash64(tuple(NULL))
  ```
</Note>

<Note>
  Para calcular el hash de todo el contenido de una tabla, use `sum(cityHash64(tuple(*)))` (u otra funciГіn hash). `tuple` garantiza que no se omitan las filas con valores NULL. `sum` garantiza que el orden de las filas no importe.
</Note>

<div id="BLAKE3">
  ## BLAKE3
</div>

Introducido en: v22.10.0

Calcula una cadena hash BLAKE3 y devuelve el conjunto de bytes resultante como FixedString.
Esta funciГіn hash criptogrГЎfica estГЎ integrada en ClickHouse mediante la biblioteca BLAKE3 de Rust.
La funciГіn es bastante rГЎpida y ofrece un rendimiento aproximadamente dos veces superior al de SHA-2, mientras genera hashes de la misma longitud que SHA-256.


* `message` вҖ” La cadena de entrada para calcular el hash. [`String`](/es/reference/data-types/string)

**Valor devuelto**

Devuelve el hash BLAKE3 de 32 bytes de la cadena de entrada como una cadena de longitud fija. [`FixedString(32)`](/es/reference/data-types/fixedstring)

<div id="sipHash64">
  ## sipHash64
</div>

Introducido en: v1.1.0

Produce un valor hash [SipHash](https://en.wikipedia.org/wiki/SipHash) de 64 bits.

Esta es una funciГіn hash criptogrГЎfica. Funciona al menos tres veces mГЎs rГЎpido que la funciГіn hash [`MD5`](#MD5).

La funciГіn [interpreta](/es/reference/functions/regular-functions/type-conversion-functions#reinterpretAsString) todos los parГЎmetros de entrada como cadenas y calcula el valor hash de cada uno de ellos.
A continuaciГіn, combina los hashes mediante el siguiente algoritmo:

1. El primer y el segundo valor hash se concatenan en un array, del que se calcula el hash.
2. El valor hash calculado previamente y el hash del tercer parГЎmetro de entrada se procesan del mismo modo.
3. Este cГЎlculo se repite para todos los valores hash restantes de la entrada original.

<Note>
  los valores hash calculados pueden ser iguales para los mismos valores de entrada de distintos tipos de argumento.
  Esto afecta, por ejemplo, a los tipos enteros de distinto tamaГұo, `Tuple` con nombre y sin nombre con los mismos datos, `Map` y el tipo `Array(Tuple(key, value))` correspondiente con los mismos datos.
</Note>

```sql
sipHash64(arg1[, arg2, ...])
```

**Valor devuelto**

Devuelve un valor hash calculado a partir de los argumentos de entrada. [`UInt64`](/es/reference/data-types/int-uint)

**Ejemplos**

**Ejemplo de uso**

