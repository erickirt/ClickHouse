<div id="uuidv7-generation">
  ## GeneraciГіn de UUIDv7
</div>

El UUID generado contiene una marca temporal de 48 bits en milisegundos Unix, seguida de la versiГіn &quot;7&quot; (4 bits), un contador (42 bits) para distinguir los UUID dentro de un milisegundo (incluido un campo de variante &quot;2&quot;, 2 bits) y un campo aleatorio (32 bits).
Para una marca temporal dada (`unix_ts_ms`), el contador comienza con un valor aleatorio y se incrementa en 1 por cada UUID nuevo hasta que la marca temporal cambia. En caso de desbordamiento del contador, el campo de marca temporal se incrementa en 1 y el contador se restablece a un nuevo valor inicial aleatorio.
Las funciones de generaciГіn de UUID garantizan que el campo del contador dentro de una marca temporal se incremente de forma monГіtona en todas las invocaciones de la funciГіn realizadas por hilos y consultas que se ejecutan de forma concurrente.

<div id="snowflake-id-generation">
  ## GeneraciГіn de ID de Snowflake
</div>

El ID de Snowflake generado contiene la marca de tiempo Unix actual en milisegundos (41 bits mГЎs 1 bit superior en cero), seguida de un id de mГЎquina (10 bits) y un contador (12 bits) para distinguir los ID dentro de un mismo milisegundo. Para cualquier marca temporal dada (`unix_ts_ms`), el contador empieza en 0 y se incrementa en 1 para cada nuevo ID de Snowflake hasta que cambia la marca temporal. Si el contador se desborda, el campo de marca temporal se incrementa en 1 y el contador se reinicia a 0.

<Note>
  Los ID de Snowflake generados se basan en la Г©poca UNIX 1970-01-01. Aunque no existe ningГәn estГЎndar ni recomendaciГіn sobre la Г©poca de los ID de Snowflake, las implementaciones de otros sistemas pueden usar una Г©poca distinta; por ejemplo, Twitter/X (2010-11-04) o Mastodon (2015-01-01).
</Note>
