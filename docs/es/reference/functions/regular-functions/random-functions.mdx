<div id="fuzzBits">
  ## fuzzBits
</div>

Introducido en: v20.5.0

Invierte los bits de la cadena de entrada `s`, con una probabilidad `p` para cada bit.

<Note>
  Esta funciГіn no es determinista: puede devolver resultados diferentes para los mismos argumentos.
</Note>

**Sintaxis**

```sql
fuzzBits(s, p)
```

**Argumentos**

* `s` вҖ” `String` o `FixedString` sobre el que se realiza el fuzzing de bits en [`String`](/es/reference/data-types/string) o [`FixedString`](/es/reference/data-types/fixedstring)
* `p` вҖ” Probabilidad de invertir cada bit, como un nГәmero entre `0.0` y `1.0` [`Float*`](/es/reference/data-types/float)

**Valor devuelto**

Devuelve una cadena con fuzzing del mismo tipo que `s`: [`String`](/es/reference/data-types/string) o [`FixedString`](/es/reference/data-types/fixedstring)

**Ejemplos**

**Ejemplo de uso**

```sql title=Query
SELECT fuzzBits(materialize('abacaba'), 0.1)
FROM numbers(3)
```

<div id="rand">
  ## rand
</div>

Introducido en: v1.1.0

Devuelve un nГәmero `UInt32` aleatorio con distribuciГіn uniforme.

Utiliza un generador congruencial lineal con un estado inicial obtenido del sistema, lo que significa que, aunque parece aleatorio, no es verdaderamente aleatorio y puede resultar predecible si se conoce el estado inicial.
En los casos en que la aleatoriedad real sea crucial, considere usar mГ©todos alternativos, como llamadas a nivel del sistema o la integraciГіn con bibliotecas externas.

<Note>
  Esta funciГіn no es determinista: puede devolver resultados diferentes para los mismos argumentos.
</Note>

**Sintaxis**

```sql

```

**Alias**: `rand32`

**Argumentos**

* `x` вҖ” Opcional y se ignora. El Гәnico propГіsito del argumento es evitar la [eliminaciГіn de subexpresiones comunes](/es/reference/functions/regular-functions/overview#common-subexpression-elimination) cuando la misma llamada a la funciГіn se usa varias veces en una consulta. [`Any`](/es/reference/data-types/index)

**Valor devuelto**

Devuelve un nГәmero aleatorio de tipo `UInt32`. [`UInt32`](/es/reference/data-types/int-uint)

**Ejemplos**

**Ejemplo de uso**

```sql title=Query

```

```response title=Response

```

</div>

Genera un Гәnico valor aleatorio que permanece constante en todas las filas de la ejecuciГіn actual de la consulta.

* Produce valores diferentes en ejecuciones de consultas distintas

Resulta Гәtil para aplicar semillas aleatorias o identificadores consistentes en todas las filas de un conjunto de datos

<Note>
  Esta funciГіn no es determinista: puede devolver resultados diferentes para los mismos argumentos.
</Note>

**Valor devuelto**

Devuelve una columna de tipo `UInt32` que contiene el mismo valor aleatorio en cada fila. [`UInt32`](/es/reference/data-types/int-uint)

**Ejemplos**

**Uso bГЎsico**
