  ## reader_executor_max_tail_for_drain
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="1048576" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1048576"},{"label": "Nueva configuración experimental de ReaderExecutor: límite de drenaje para completar una conexión de origen larga que se ha perdido."}]}]} />

Límite de drenaje para el `ReaderExecutor` experimental: si una conexión de origen larga se pierde cuando está a esta cantidad de bytes o menos de su límite derecho, primero se termina de leer hasta ese límite, de modo que pueda completarse y volver al pool de conexiones como reutilizable, en lugar de contarse como una conexión incompleta.

  ## reader_executor_min_bytes_for_seek
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="2097152" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "2097152"},{"label": "Nueva configuración experimental de ReaderExecutor: límite de separación hacia adelante para el puenteo en una conexión de origen mantenida abierta."}]}]} />

Límite de separación hacia adelante para el `ReaderExecutor` experimental: una separación de hasta este valor se omite en la conexión de origen abierta (bridged / read through), en lugar de emitir una lectura independiente del origen o reabrir la conexión. Ajústelo cerca del punto de equilibrio entre el costo del ancho de banda y el de las solicitudes para que el puenteo siga siendo rentable.

  ## reader_executor_use_long_connections
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Nueva configuración experimental de ReaderExecutor (desactivada de forma predeterminada): reutiliza una conexión de origen retenida entre ventanas secuenciales."}]}]} />

Reutiliza una conexión de origen persistente y limitada entre ventanas en el `ReaderExecutor` experimental. Una conexión persistente es aquella cuyo rango excede la ventana de lectura actual; cuando está deshabilitada, el ejecutor no consume presupuesto del grupo de conexiones y cada ventana abre una conexión efímera de un solo uso (la ruta sin estado).
