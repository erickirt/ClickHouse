<div id="database_replicated_allow_explicit_uuid">
  ## database_replicated_allow_explicit_uuid
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "Se añadió una nueva configuración para impedir que se especifique explícitamente el UUID de la tabla"}]}]} />

0 - No permitir especificar explícitamente UUIDs para tablas en bases de datos Replicated. 1 - Permitir. 2 - Permitir, pero ignorar el UUID especificado y generar uno aleatorio en su lugar.

<div id="database_replicated_allow_heavy_create">
  ## database_replicated_allow_heavy_create
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.7"},{"label": "0"},{"label": "Las consultas DDL de larga duración (CREATE AS SELECT y POPULATE) estaban prohibidas para el motor de base de datos Replicated"}]}]} />

Permite consultas DDL de larga duración (CREATE AS SELECT y POPULATE) en el motor de base de datos Replicated. Tenga en cuenta que esto puede bloquear la cola de DDL durante mucho tiempo.

<div id="database_replicated_allow_only_replicated_engine">
  ## database_replicated_allow_only_replicated_engine
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Permite crear únicamente tablas Replicated en una base de datos con motor Replicated

Valor predeterminado en Cloud: `1`.

<div id="database_replicated_allow_replicated_engine_arguments">
  ## database_replicated_allow_replicated_engine_arguments
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "No permitir argumentos explícitos de forma predeterminada"}]}]} />

<div id="database_replicated_always_detach_permanently">
  ## database_replicated_always_detach_permanently
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Ejecutar DETACH TABLE como DETACH TABLE PERMANENTLY si el motor de base de datos es Replicated

<div id="database_replicated_enforce_synchronous_settings">
  ## database_replicated_enforce_synchronous_settings
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Hace que algunas consultas esperen de forma síncrona (véase también database&#95;atomic&#95;wait&#95;for&#95;drop&#95;and&#95;detach&#95;synchronously, mutations&#95;sync, alter&#95;sync). No se recomienda habilitar estos ajustes.

<div id="database_replicated_initial_query_timeout_sec">
  ## database_replicated_initial_query_timeout_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="300" />

Establece cuánto tiempo debe esperar la consulta DDL inicial a que la base de datos Replicated procese las entradas previas de la cola de DDL, en segundos.

Valores posibles:

* Entero positivo.
* 0 — ilimitado.
