<div id="wait_for_part_commit_in_dependent_materialized_views">
  ## wait_for_part_commit_in_dependent_materialized_views
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "0"},{"label": "Nueva configuración"}]}]} />

Controla si cada sink realiza el commit de la parte que acaba de escribir antes de que se ejecute su propia cascada de vistas materializadas dependientes, para que una cascada que vuelva a leer desde el origen mediante `JOIN` vea la parte escrita por ese sink.

La garantía se aplica por instancia de sink: es posible que las partes escritas por otros hilos de sink del mismo `INSERT` todavía no sean visibles. Esta configuración no proporciona un orden de commit entre hilos.

No tiene ningún efecto en los inserts en tablas sin vistas materializadas dependientes.
