<div id="min_insert_block_size_bytes">
  ## min_insert_block_size_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="268402944" />

El tamaño mínimo de los bloques (en bytes) que deben formarse para insertarlos en una tabla.

Esta configuración funciona junto con min&#95;insert&#95;block&#95;size&#95;rows y controla la formación de bloques en los mismos contextos (análisis del formato y operaciones INSERT). Consulta min&#95;insert&#95;block&#95;size&#95;rows para obtener información detallada sobre cuándo y cómo se aplican estas configuraciones.

Valores posibles:

* Entero positivo.
* 0 — la configuración no interviene en la formación de bloques.

<div id="min_insert_block_size_bytes_for_materialized_views">
  ## min_insert_block_size_bytes_for_materialized_views
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Establece el número mínimo de bytes que debe tener un bloque para poder insertarse en una tabla mediante una consulta `INSERT`. Los bloques más pequeños se compactan en otros más grandes. Esta configuración se aplica solo a los bloques insertados en una [vista materializada](/es/reference/statements/create/view). Al ajustar esta configuración, controlas la compactación de bloques durante el envío a la vista materializada y evitas un uso excesivo de memoria.

Posibles valores:

* Cualquier entero positivo.
* 0 — Compactación deshabilitada.

<div id="min_insert_block_size_rows">
  ## min_insert_block_size_rows
</div>

<SettingsInfoBlock type="UInt64" default_value="1048449" />

El tamaño mínimo de los bloques (en filas) que deben formarse para insertarlos en una tabla.

Esta configuración controla la formación de bloques en dos contextos:

1. Análisis de formatos: Cuando el servidor analiza formatos de entrada basados en filas (CSV, TSV, JSONEachRow, etc.) desde cualquier interfaz (HTTP, clickhouse-client con datos inline, gRPC, protocolo wire de PostgreSQL), los bloques se emiten cuando:

   * Se alcanzan tanto min&#95;insert&#95;block&#95;size&#95;rows AND min&#95;insert&#95;block&#95;size&#95;bytes, OR
   * Se alcanza max&#95;insert&#95;block&#95;size&#95;rows OR max&#95;insert&#95;block&#95;size&#95;bytes

   Nota: Cuando se usa clickhouse-client o clickhouse-local para leer desde un archivo, el propio client analiza los datos y esta configuración se aplica del lado del cliente.

2. Operaciones INSERT: Durante las consultas INSERT y cuando los datos fluyen a través de vistas materializadas, el comportamiento de esta configuración depende de `use_strict_insert_block_limits`:

   * Cuando está habilitada: Los bloques se emiten cuando:
     * Umbrales mínimos (AND): Se alcanzan tanto min&#95;insert&#95;block&#95;size&#95;rows AND min&#95;insert&#95;block&#95;size&#95;bytes
     * Umbrales máximos (OR): Se alcanza max&#95;insert&#95;block&#95;size&#95;rows OR max&#95;insert&#95;block&#95;size&#95;bytes

   * Cuando está deshabilitada (valor predeterminado): Los bloques se emiten cuando se alcanza min&#95;insert&#95;block&#95;size&#95;rows OR min&#95;insert&#95;block&#95;size&#95;bytes. No se aplican las configuraciones de max&#95;insert&#95;block&#95;size.

Valores posibles:

* Entero positivo.
* 0 — la configuración no participa en la formación de bloques.
