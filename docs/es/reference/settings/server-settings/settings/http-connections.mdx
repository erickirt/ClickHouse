Estos parámetros configuran el servidor de ClickHouse y se generan automáticamente a partir del código fuente de ClickHouse.

<div id="http_connections_hard_limit">
  ## http_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Se lanza una excepción al intentar crear una conexión cuando se alcanza este límite. Establézcalo en 0 para desactivar la limitación estricta. El límite se aplica a las conexiones HTTP que no pertenecen a ningún disco ni almacenamiento.

<div id="http_connections_rcvbuf">
  ## http_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Sí" />El tamaño de la opción SO&#95;RCVBUF para las conexiones HTTP generales. Si se establece en un valor mayor que 0, sobrescribe el autoajuste de TCP del kernel para el búfer de recepción. 0 = valor predeterminado del kernel (autoajuste). Nota: si esta configuración se vuelve a cambiar a 0, el ajuste automático se restaura solo para las conexiones recién creadas; las conexiones existentes del grupo conservan tamaños de búfer fijos hasta que se vuelvan a crear.

<div id="http_connections_sndbuf">
  ## http_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Sí" />El tamaño de la opción SO&#95;SNDBUF para las conexiones HTTP de uso general. Si se establece en un valor superior a 0, sustituye el autoajuste TCP del kernel para el búfer de envío. 0 = valor predeterminado del kernel (autoajuste). Nota: volver a establecer esta configuración en 0 restaura el autoajuste solo para las conexiones creadas recientemente; las conexiones agrupadas existentes conservan tamaños de búfer fijos hasta que se vuelvan a crear.

<div id="http_connections_soft_limit">
  ## http_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Las conexiones que superen este límite tienen un tiempo de vida útil significativamente más corto. El límite se aplica a las conexiones HTTP que no pertenecen a ningún disco ni almacenamiento.

<div id="http_connections_store_limit">
  ## http_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />Las conexiones que superen este límite se reinician después de su uso. Establézcalo en 0 para desactivar la caché de conexiones. El límite se aplica a las conexiones HTTP que no pertenecen a ningún disco ni almacenamiento.

<div id="http_connections_warn_limit">
  ## http_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />Se escriben mensajes de advertencia en los logs si el número de conexiones en uso supera este límite. El límite se aplica a las conexiones HTTP que no pertenecen a ningún disco ni almacenamiento.
