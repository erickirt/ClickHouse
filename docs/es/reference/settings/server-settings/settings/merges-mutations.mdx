Estos ajustes configuran el servidor de ClickHouse y se generan automáticamente a partir del código fuente de ClickHouse.

<div id="merges_mutations_memory_usage_soft_limit">
  ## merges_mutations_memory_usage_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Establece el límite de RAM que se puede usar para realizar operaciones de merge y mutation.
Si ClickHouse alcanza el límite establecido, no programará nuevas operaciones de merge o mutation en segundo plano, pero seguirá ejecutando las tareas ya programadas.

<Note>
  Un valor de `0` significa ilimitado.
</Note>

**Ejemplo**

```xml
<merges_mutations_memory_usage_soft_limit>0</merges_mutations_memory_usage_soft_limit>
```

<div id="merges_mutations_memory_usage_to_ram_ratio">
  ## merges_mutations_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.5" changeable_without_restart="No" />

El valor predeterminado de `merges_mutations_memory_usage_soft_limit` se calcula como `memory_amount * merges_mutations_memory_usage_to_ram_ratio`.
