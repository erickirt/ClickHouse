Estos ajustes configuran el servidor de ClickHouse y se generan automáticamente a partir del código fuente de ClickHouse.

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Recopila asignaciones aleatorias de un tamaño menor o igual que el valor especificado, con una probabilidad igual a `total_memory_profiler_sample_probability`. 0 significa que está deshabilitado. Puede que le convenga establecer &#39;max&#95;untracked&#95;memory&#39; en 0 para que este umbral funcione como se espera.

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Recopila asignaciones aleatorias de tamaño mayor o igual que el valor especificado con una probabilidad igual a `total_memory_profiler_sample_probability`. 0 significa deshabilitado. Puede que desee establecer `max_untracked_memory` en 0 para que este umbral funcione como se espera.

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Cada vez que el uso de memoria del servidor supere el siguiente intervalo de bytes, el generador de perfiles de memoria recopilará la traza de pila de la asignación. Cero significa que el generador de perfiles de memoria está deshabilitado. Los valores inferiores a unos pocos megabytes ralentizarán el servidor.

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="No" />

Permite recopilar asignaciones y liberaciones de memoria aleatorias y escribirlas en la tabla del sistema [system.trace&#95;log](/es/reference/system-tables/trace_log), con `trace_type` igual a `MemorySample`, con la probabilidad especificada. La probabilidad se aplica a cada asignación o liberación de memoria, independientemente del tamaño de la asignación. Tenga en cuenta que el muestreo solo se produce cuando la cantidad de memoria no rastreada supera el límite de memoria no rastreada (el valor predeterminado es `4` MiB). Este valor puede reducirse si se reduce [total&#95;memory&#95;profiler&#95;step](/es/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step). Puede establecer `total_memory_profiler_step` en `1` para un muestreo adicional de granularidad fina.

Valores posibles:

* Double positivo.
* `0` — La escritura de asignaciones y liberaciones de memoria aleatorias en la tabla del sistema `system.trace_log` está deshabilitada.
