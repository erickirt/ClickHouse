Estos ajustes configuran el servidor de ClickHouse y se generan automáticamente a partir del código fuente de ClickHouse.

<div id="distributed_ddl.cleanup_delay_period">
  ## distributed_ddl.cleanup_delay_period
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />la limpieza comienza después de recibir el evento de un nodo nuevo si la última limpieza no se realizó hace menos de `<cleanup_delay_period>` segundos.

<div id="distributed_ddl.max_tasks_in_queue">
  ## distributed_ddl.max_tasks_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />el número máximo de tareas que puede haber en la cola.

<div id="distributed_ddl.path">
  ## distributed_ddl.path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/ddl/" changeable_without_restart="No" />la ruta en Keeper para la `<task_queue>` de las consultas DDL

<div id="distributed_ddl.pool_size">
  ## distributed_ddl.pool_size
</div>

<SettingsInfoBlock type="Int32" default_value="1" changeable_without_restart="No" />número de consultas `<ON CLUSTER>` que se pueden ejecutar simultáneamente

<div id="distributed_ddl.profile">
  ## distributed_ddl.profile
</div>

el perfil utilizado para ejecutar las consultas DDL

<div id="distributed_ddl.replicas_path">
  ## distributed_ddl.replicas_path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/replicas/" changeable_without_restart="No" />la ruta en Keeper para la `<task_queue>` de las réplicas

<div id="distributed_ddl.task_max_lifetime">
  ## distributed_ddl.task_max_lifetime
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" changeable_without_restart="No" />elimina el nodo si su antigüedad supera este valor.
