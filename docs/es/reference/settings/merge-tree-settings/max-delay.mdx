Estas opciones de configuración están disponibles en [system.merge&#95;tree&#95;settings](/es/reference/system-tables/merge_tree_settings) y se autogeneran a partir del código fuente de ClickHouse.

<div id="max_delay_to_insert">
  ## max_delay_to_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1" />

Valor en segundos que se utiliza para calcular el retraso de `INSERT` si el
número de partes activas en una sola partición supera el
valor de [parts&#95;to&#95;delay&#95;insert](/es/reference/settings/merge-tree-settings/parts-to#parts_to_delay_insert).

Valores posibles:

* Cualquier entero positivo.

El retraso de `INSERT` (en milisegundos) se calcula con la fórmula:

```code
max_k = parts_to_throw_insert - parts_to_delay_insert
k = 1 + parts_count_in_partition - parts_to_delay_insert
delay_milliseconds = pow(max_delay_to_insert * 1000, k / max_k)
```

Por ejemplo, si una partición tiene 299 partes activas y parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1, `INSERT` se
retrasa `pow( 1 * 1000, (1 + 299 - 150) / (300 - 150) ) = 1000`
milisegundos.

A partir de la versión 23.1, la fórmula cambió a:

```code
allowed_parts_over_threshold = parts_to_throw_insert - parts_to_delay_insert
parts_over_threshold = parts_count_in_partition - parts_to_delay_insert + 1
delay_milliseconds = max(min_delay_to_insert_ms, (max_delay_to_insert * 1000)
* parts_over_threshold / allowed_parts_over_threshold)
```

Por ejemplo, si una partición tiene 224 partes activas y parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1,
min&#95;delay&#95;to&#95;insert&#95;ms = 10, `INSERT` se retrasa `max( 10, 1 * 1000 *
(224 - 150 + 1) / (300 - 150) ) = 500` milisegundos.
