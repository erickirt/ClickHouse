<div id="materialize_skip_indexes_on_merge">
  ## materialize_skip_indexes_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.1"},{"label": "1"},{"label": "Nueva configuración"}]}]} />

Si está habilitada, las fusiones crean y almacenan índices de omisión para las partes nuevas.
De lo contrario, pueden crearse/almacenarse explícitamente con [MATERIALIZE INDEX](/es/reference/statements/alter/skipping-index#materialize-index)
o [durante INSERTs](/es/reference/settings/session-settings#materialize_skip_indexes_on_insert).

Véase también [exclude&#95;materialize&#95;skip&#95;indexes&#95;on&#95;merge](/es/reference/settings/merge-tree-settings/exclude#exclude_materialize_skip_indexes_on_merge) para un control más preciso.

  ## materialize_statistics_on_merge

<SettingsInfoBlock type="Bool" default_value="1" />
