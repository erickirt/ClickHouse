<div id="quantilesExact">
  ## quantilesExact
</div>

도입 버전: v1.1.0

서로 다른 수준에서 숫자 데이터 시퀀스의 여러 [분위수](https://en.wikipedia.org/wiki/Quantile)를 동시에 정확하게 계산합니다.

이 함수는 [`quantileExact`](/ko/reference/functions/aggregate-functions/quantileExact)와 동일하지만, 여러 분위수 수준을 단일 패스로 계산할 수 있으므로 개별 분위수 함수를 호출하는 것보다 더 효율적입니다.

정확한 값을 얻기 위해 전달된 모든 값을 배열로 모은 다음 부분 정렬합니다. 따라서 이 함수는 `O(n)` 메모리를 사용하며, 여기서 `n`은 전달된 값의 개수입니다.

**구문**

```sql
quantilesExact(level1, level2, ...)(expr)
```

**매개변수**

* `level` — 분위수 수준입니다. 0부터 1 사이의 상수 부동소수점 수를 1개 이상 지정합니다. `level` 값은 `[0.01, 0.99]` 범위에서 사용하는 것을 권장합니다. [`Float*`](/ko/reference/data-types/float)

**인수**

* `expr` — 컬럼 값에 대한 표현식이며, 결과 타입은 숫자 데이터 타입, `Date`, `DateTime` 또는 `DateTime64`입니다. [`(U)Int*`](/ko/reference/data-types/int-uint) 또는 [`Int128`](/ko/reference/data-types/int-uint) 또는 [`UInt128`](/ko/reference/data-types/int-uint) 또는 [`Int256`](/ko/reference/data-types/int-uint) 또는 [`UInt256`](/ko/reference/data-types/int-uint) 또는 [`Float*`](/ko/reference/data-types/float) 또는 [`Decimal*`](/ko/reference/data-types/decimal) 또는 [`Date`](/ko/reference/data-types/date) 또는 [`DateTime`](/ko/reference/data-types/datetime) 또는 [`DateTime64`](/ko/reference/data-types/datetime64)

**반환 값**

지정한 수준의 분위수를 수준이 지정된 순서대로 담은 배열입니다. 숫자 데이터 타입의 경우 출력 형식은 입력 형식과 일치합니다. [`Array((U)Int*)`](/ko/reference/data-types/array) 또는 [`Array(Int128)`](/ko/reference/data-types/array) 또는 [`Array(UInt128)`](/ko/reference/data-types/array) 또는 [`Array(Int256)`](/ko/reference/data-types/array) 또는 [`Array(UInt256)`](/ko/reference/data-types/array) 또는 [`Array(Float*)`](/ko/reference/data-types/array) 또는 [`Array(Decimal*)`](/ko/reference/data-types/array) 또는 [`Array(Date)`](/ko/reference/data-types/array) 또는 [`Array(DateTime)`](/ko/reference/data-types/array) 또는 [`Array(DateTime64)`](/ko/reference/data-types/array)

**예시**

**여러 exact quantile 계산**

```sql title=Query
SELECT quantilesExact(0.25, 0.5, 0.75)(number) FROM numbers(10)
```

```response title=Response
┌─quantilesExact(0.25, 0.5, 0.75)(number)─┐
│ [2,5,7]                                 │
└─────────────────────────────────────────┘
```
