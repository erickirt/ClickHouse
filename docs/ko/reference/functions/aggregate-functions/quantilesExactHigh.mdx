<div id="quantilesExactHigh">
  ## quantilesExactHigh
</div>

도입 버전: v20.8.0

서로 다른 수준에서 숫자 데이터 시퀀스의 여러 [분위수](https://en.wikipedia.org/wiki/Quantile)를 동시에 정확하게 계산합니다. 수준 `0.5`에서 원소 수가 짝수이면 가운데 두 값 중 더 큰 값을 반환합니다.

이 함수는 [`quantileExactHigh`](/ko/reference/functions/aggregate-functions/quantileExactHigh)와 동일하지만, 한 번의 처리로 여러 분위수 수준을 계산할 수 있으므로 개별 분위수 함수를 각각 호출하는 것보다 더 효율적입니다.

**구문**

```sql
quantilesExactHigh(level1, level2, ...)(expr)
```

**매개변수**

* `level` — 분위수 수준입니다. 0부터 1까지의 상수 부동소수점 수를 하나 이상 지정합니다. `level` 값은 `[0.01, 0.99]` 범위에서 사용하는 것을 권장합니다. [`Float*`](/ko/reference/data-types/float)

**인수**

* `expr` — 컬럼 값을 대상으로 하는 표현식이며, 결과는 숫자 데이터 타입, `Date`, `DateTime` 또는 `DateTime64`여야 합니다. [`(U)Int*`](/ko/reference/data-types/int-uint) 또는 [`Float*`](/ko/reference/data-types/float) 또는 [`Decimal*`](/ko/reference/data-types/decimal) 또는 [`Date`](/ko/reference/data-types/date) 또는 [`DateTime`](/ko/reference/data-types/datetime) 또는 [`DateTime64`](/ko/reference/data-types/datetime64)

**반환 값**

지정한 수준의 분위수를, 지정한 수준의 순서와 동일한 순서로 담은 배열을 반환합니다. 숫자 데이터 타입의 경우 출력 형식은 입력 형식과 일치합니다. [`Array((U)Int*)`](/ko/reference/data-types/array) 또는 [`Array(Float*)`](/ko/reference/data-types/array) 또는 [`Array(Decimal*)`](/ko/reference/data-types/array) 또는 [`Array(Date)`](/ko/reference/data-types/array) 또는 [`Array(DateTime)`](/ko/reference/data-types/array) 또는 [`Array(DateTime64)`](/ko/reference/data-types/array)

**예시**

**여러 개의 정확한 상위 분위수 계산**

```sql title=Query
SELECT quantilesExactHigh(0.25, 0.5, 0.75)(number) FROM numbers(10)
```

```response title=Response
┌─quantilesExactHigh(0.25, 0.5, 0.75)(number)─┐
│ [2,5,7]                                     │
└─────────────────────────────────────────────┘
```
