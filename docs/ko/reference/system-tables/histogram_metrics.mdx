<div id="description">
  ## 설명
</div>

이 테이블(table)에는 즉시 계산되어 Prometheus 포맷으로 내보낼 수 있는 히스토그램 메트릭이 포함되어 있습니다. 항상 최신 상태입니다. 더 이상 사용이 권장되지 않는 `system.latency_log`를 대체합니다.

<div id="columns">
  ## 컬럼
</div>

{/*AUTOGENERATED_START*/}

* `metric` ([String](/ko/reference/data-types/string)) — 메트릭 이름.
* `value` ([Float64](/ko/reference/data-types/float)) — 메트릭 값.
* `description` ([String](/ko/reference/data-types/string)) — 메트릭 설명.
* `labels` ([Map(String, String)](/ko/reference/data-types/map)) — 메트릭 레이블.

**별칭:**

* `name` — `metric`의 별칭.

{/*AUTOGENERATED_END*/}

<div id="example">
  ## 예시
</div>

다음과 같은 쿼리를 사용하면 모든 히스토그램 메트릭을 Prometheus 포맷으로 내보낼 수 있습니다.

```sql
SELECT
  metric AS name,
  toFloat64(value) AS value,
  description AS help,
  labels,
  'histogram' AS type
FROM system.histogram_metrics
FORMAT Prometheus
```

<div id="metric_descriptions">
  ## 메트릭 설명
</div>

| 메트릭                                                          | 설명                                                                                |
| ------------------------------------------------------------ | --------------------------------------------------------------------------------- |
| `keeper_response_time_ms_bucket`                             | Keeper의 응답 시간입니다. 단위는 밀리초입니다.                                                     |
| `keeper_client_queue_duration_milliseconds_bucket`           | 요청이 enqueued되기를 기다리는 시간과 Keeper 클라이언트에서 처리되기 전까지 큐에서 대기하는 시간입니다. 단위는 밀리초입니다.      |
| `keeper_receive_request_time_milliseconds_bucket`            | Keeper TCP handler에서 클라이언트의 요청을 수신하고 파싱하는 데 걸리는 시간입니다. 단위는 밀리초입니다.                |
| `keeper_dispatcher_requests_queue_time_milliseconds_bucket`  | 요청이 Keeper 디스패처 요청 큐에서 대기하는 시간입니다. 단위는 밀리초입니다.                                    |
| `keeper_write_pre_commit_time_milliseconds_bucket`           | Raft commit 전에 쓰기 요청을 전처리하는 데 걸리는 시간입니다. 단위는 밀리초입니다.                              |
| `keeper_write_commit_time_milliseconds_bucket`               | Raft commit 후에 쓰기 요청을 처리하는 데 걸리는 시간입니다. 단위는 밀리초입니다.                               |
| `keeper_dispatcher_responses_queue_time_milliseconds_bucket` | 응답이 Keeper 디스패처 응답 큐에서 대기하는 시간입니다. 단위는 밀리초입니다.                                    |
| `keeper_send_response_time_milliseconds_bucket`              | Keeper TCP handler에서 클라이언트로 응답을 보내는 데 걸리는 시간입니다(큐 대기 및 socket 쓰기 포함). 단위는 밀리초입니다. |
| `keeper_read_wait_for_write_time_milliseconds_bucket`        | 읽기 요청이 의존하는 쓰기 요청이 완료될 때까지 대기하는 시간입니다. 단위는 밀리초입니다.                                |
| `keeper_read_process_time_milliseconds_bucket`               | Keeper에서 읽기 요청을 처리하는 데 걸리는 시간입니다. 단위는 밀리초입니다.                                     |
| `keeper_batch_size_elements_bucket`                          | Raft로 전송되는 Batch 크기입니다. 단위는 원소 수입니다.                                              |
| `keeper_batch_size_bytes_bucket`                             | Raft로 전송되는 Batch 크기입니다. 단위는 바이트입니다.                                               |
| `filesystem_cache_evicted_segment_hits_bucket`               | 파일 세그먼트가 eviction되는 시점의 캐시 히트 횟수 분포이며, 캐시 이름으로 레이블이 지정됩니다.                        |
| `filesystem_cache_evicted_segment_size_bytes_bucket`         | eviction된 파일 세그먼트의 바이트 크기 분포이며, 캐시 이름으로 레이블이 지정됩니다.                               |
| `filesystem_cache_evicted_segment_hits_by_user_bucket`       | eviction된 파일 세그먼트의 캐시 히트 횟수 분포이며, 캐시 이름과 사용자 ID로 레이블이 지정됩니다.                      |
| `filesystem_cache_evicted_segment_size_bytes_by_user_bucket` | eviction된 파일 세그먼트의 바이트 크기 분포이며, 캐시 이름과 사용자 ID로 레이블이 지정됩니다.                        |
