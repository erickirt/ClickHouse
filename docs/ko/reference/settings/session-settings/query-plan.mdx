projection matcher가 두 Set이 동일한지 판단할 때 콘텐츠 해시를 계산하고 비교하는 `IN` 절 Set의 최대 행 수입니다. 이보다 큰 Set은 일치하지 않는 것으로 처리되며 projection을 건너뜁니다. 0으로 설정하면 콘텐츠 해시 비교가 완전히 비활성화됩니다. 즉, `IN` 절 Set을 포함하는 노드에서는 projection match가 절대 성공하지 않습니다.

aggregate projection matcher(및 `IN` 절 Set 비교가 필요한 향후 projection matcher)에서 사용됩니다. 콘텐츠 해시 계산은 Set 원소 수에 대해 `O(N log N)` 복잡도를 가지며, 이 설정은 쿼리나 projection에 `IN` 절이 많이 포함된 경우 계획 수립 중 발생하는 비용을 제한합니다.

<ExperimentalBadge />

<SettingsInfoBlock type="JoinOrderAlgorithm" default_value="greedy" />

쿼리 계획 최적화 시 시도할 조인 순서 알고리즘을 지정합니다. 사용할 수 있는 알고리즘은 다음과 같습니다.

* &#39;greedy&#39; - 기본 탐욕 알고리즘으로, 빠르게 동작하지만 최적의 조인 순서를 찾지 못할 수 있습니다
* &#39;dpsize&#39; - 현재 Inner 조인에만 적용되는 DPsize 알고리즘을 구현합니다. 가능한 모든 조인 순서를 고려해 가장 최적의 순서를 찾지만, 테이블 수와 조인 프레디케이트가 많은 쿼리에서는 느릴 수 있습니다.
* &#39;dpsub&#39; - Inner 조인과 비-Inner 조인을 모두 지원하는 DPsub 알고리즘을 구현합니다. 가능한 모든 조인 순서를 고려해 가장 최적의 순서를 찾지만, 테이블 수와 조인 프레디케이트가 많은 쿼리에서는 느릴 수 있습니다.
