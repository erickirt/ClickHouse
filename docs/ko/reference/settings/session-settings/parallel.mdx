  ## parallel_distributed_insert_select
</div>

<SettingsInfoBlock type="UInt64" default_value="2" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.7"},{"label": "2"},{"label": "기본적으로 병렬 분산 `INSERT ... SELECT`를 활성화"}]}]} />

병렬 분산 `INSERT ... SELECT` 쿼리를 활성화합니다.

`INSERT INTO distributed_table_a SELECT ... FROM distributed_table_b` 쿼리를 실행할 때, 두 테이블이 동일한 클러스터를 사용하고 둘 다 [복제된](/ko/reference/engines/table-engines/mergetree-family/replication) 테이블이거나 복제되지 않은 테이블이면, 이 쿼리는 각 세그먼트에서 로컬로 처리됩니다.

가능한 값:

* `0` — 비활성화됩니다.
* `1` — `SELECT`는 Distributed 엔진의 기반 테이블에서 각 세그먼트별로 실행됩니다.
* `2` — `SELECT`와 `INSERT`는 Distributed 엔진의 기반 테이블에서/로 각 세그먼트별로 실행됩니다.

v25.4부터 `ReplicatedMergeTree` 또는 `SharedMergeTree` source의 `INSERT ... SELECT`도 레플리카 전반에서 병렬화할 수 있습니다. 이를 활성화하려면 다음을 설정하십시오:

* `parallel_distributed_insert_select = 2`
* `enable_parallel_replicas = 1`

오른쪽 테이블 크기가 임계값보다 작다고 확인되면 전자가 사용됩니다.

<div id="parallel_non_joined_rows_processing">
  ## parallel_non_joined_rows_processing
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "1"},{"label": "RIGHT/FULL `parallel_hash` 조인에서 조인되지 않은 행을 병렬로 처리할 수 있도록 하는 새로운 설정입니다."}]}]} />

RIGHT 및 FULL JOIN 수행 중 오른쪽 테이블의 조인되지 않은 행을 여러 스레드가 병렬로 처리할 수 있도록 합니다.
큰 테이블에 `parallel_hash` 조인 알고리즘을 사용할 때 조인되지 않은 단계의 처리 속도를 높일 수 있습니다.
비활성화하면 조인되지 않은 행은 단일 스레드에서 처리됩니다.
