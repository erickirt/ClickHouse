실험적 `ReaderExecutor`의 drain 한계값입니다. 장기 유지된 소스 연결이 우측 경계에서 이 바이트 수 이내 지점에서 끊어지면, 먼저 해당 경계까지 모두 읽어 연결을 완료하므로 불완전한 연결로 계산되지 않고 재사용 가능한 상태로 연결 풀에 반환됩니다.

실험적 `ReaderExecutor`의 전방 gap 한계값입니다. 이 값 이하의 gap은 별도의 소스 읽기를 요청하거나 다시 열지 않고, 열려 있는 소스 연결에서 건너뛰어(브리징/연속 읽기) 처리합니다. 브리징이 비용 면에서 이득이 되도록 bandwidth와 request 비용의 손익분기점에 가깝게 설정하십시오.

실험적 `ReaderExecutor`에서 윈도우 전반에 걸쳐 범위가 제한된 장기 소스 연결을 재사용합니다. 장기 연결은 범위가 현재 읽기 윈도우를 초과하는 연결을 의미합니다. 이 기능을 비활성화하면 실행기는 연결 풀 예산을 전혀 사용하지 않으며, 각 윈도우는 수명이 짧은 일회성 연결을 엽니다(상태 비저장 경로).
