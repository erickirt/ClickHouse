앞에 SELECT 키워드를 쓰지 않아도 간단한 SELECT 쿼리를 작성할 수 있으므로 계산기처럼 사용하기에 편리합니다. 예를 들어 `1 + 2`도 유효한 쿼리로 처리됩니다.

`clickhouse-local`에서는 기본적으로 활성화되어 있으며, 명시적으로 비활성화할 수 있습니다.

비어 있지 않으면 최상위 수준에서 FROM 없이 실행되는 쿼리는 system.one 대신 이 테이블에서 읽습니다.

이 설정은 clickhouse-local에서 입력 데이터를 처리할 때 사용됩니다.
사용자가 이 설정을 명시적으로 지정할 수는 있지만, 이런 방식의 사용을 염두에 두고 만든 것은 아닙니다.

서브쿼리는 이 설정의 영향을 받지 않습니다(스칼라, FROM, IN 서브쿼리 모두 해당).
UNION, INTERSECT, EXCEPT 체인에서 최상위 수준의 SELECT는 괄호로 어떻게 그룹화되었는지와 관계없이 동일하게 취급되며, 이 설정의 영향을 받습니다.
이 설정이 VIEW 및 분산 쿼리에 어떤 영향을 주는지는 정의되어 있지 않습니다.

이 설정에는 테이블 이름을 지정할 수 있으며(이 경우 현재 데이터베이스에서 테이블을 확인함), 또는 &#39;database.table&#39; 형식의 정규화된 이름을 지정할 수 있습니다.
데이터베이스 이름과 테이블 이름은 모두 따옴표 없이 작성해야 하며, 단순 식별자만 허용됩니다.
