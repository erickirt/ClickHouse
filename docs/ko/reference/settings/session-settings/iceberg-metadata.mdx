  ## iceberg_metadata_log_level
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "none"},{"label": "새 설정입니다."}]}]} />

Iceberg 테이블의 메타데이터 로깅 수준을 `system.iceberg_metadata_log`에 기록할지 제어합니다.
이 설정은 일반적으로 디버깅 목적으로 수정할 수 있습니다.

가능한 값:

* none - 메타데이터 로그를 기록하지 않습니다.
* metadata - 루트 metadata.json 파일입니다.
* manifest&#95;list&#95;metadata - 위의 모든 항목 + 스냅샷에 해당하는 avro manifest 목록의 메타데이터입니다.
* manifest&#95;list&#95;entry - 위의 모든 항목 + avro manifest 목록의 항목입니다.
* manifest&#95;file&#95;metadata - 위의 모든 항목 + 탐색한 avro manifest 파일의 메타데이터입니다.
* manifest&#95;file&#95;entry - 위의 모든 항목 + 탐색한 avro manifest 파일의 항목입니다.

  ## iceberg_metadata_staleness_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "원격 카탈로그에서 가져오는 것을 방지하기 위해 READ 작업에서 캐시된 메타데이터 버전을 사용할 수 있도록 하는 새로운 설정"}]}]} />

0이 아닌 경우, 지정된 staleness 윈도우보다 최신인 캐시된 메타데이터 스냅샷이 있으면 원격 카탈로그에서 Iceberg 메타데이터를 가져오지 않습니다. 0이면 항상 원격 카탈로그에서 최신 메타데이터 버전을 가져옵니다. 이 값을 0이 아닌 값으로 설정하면 staleness를 감수하는 대신 읽기 작업의 지연 시간을 줄일 수 있습니다.
