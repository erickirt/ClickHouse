<div id="log_comment">
  ## log_comment
</div>

[system.query&#95;log](/ko/reference/system-tables/query_log) 테이블의 `log_comment` field 값과 서버 로그의 comment 텍스트를 지정합니다.

서버 로그의 가독성을 높이는 데 사용할 수 있습니다. 또한 [clickhouse-test](/ko/resources/develop-contribute/contribute/tests)를 실행한 후 `system.query_log`에서 테스트와 관련된 쿼리를 선별하는 데도 도움이 됩니다.

가능한 값:

* [max&#95;query&#95;size](/ko/reference/settings/session-settings/max#max_query_size) 이하의 임의의 문자열입니다. max&#95;query&#95;size를 초과하면 서버가 예외를 발생시킵니다.

**예시**

쿼리:

```sql
SET log_comment = 'log_comment test', log_queries = 1;
SELECT 1;
SYSTEM FLUSH LOGS;
SELECT type, query FROM system.query_log WHERE log_comment = 'log_comment test' AND event_date >= yesterday() ORDER BY event_time DESC LIMIT 2;
```

결과:

```text
┌─type────────┬─query─────┐
│ QueryStart  │ SELECT 1; │
│ QueryFinish │ SELECT 1; │
└─────────────┴───────────┘
```

<div id="log_formatted_queries">
  ## log_formatted_queries
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

포맷된 쿼리를 [system.query&#95;log](/ko/reference/system-tables/query_log) 시스템 테이블에 기록하도록 합니다([system.query&#95;log](/ko/reference/system-tables/query_log)의 `formatted_query` 컬럼을 채웁니다).

가능한 값:

* 0 — 포맷된 쿼리는 시스템 테이블에 기록되지 않습니다.
* 1 — 포맷된 쿼리는 시스템 테이블에 기록됩니다.
