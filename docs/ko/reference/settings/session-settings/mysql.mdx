MySQL 타입을 해당하는 ClickHouse 타입으로 어떻게 변환할지 정의합니다. `decimal`, `datetime64`, `date2Date32`, `date2String`, `geometry`를 임의로 조합한 쉼표 구분 목록입니다. 모든 최신 매핑(`decimal`, `datetime64`, `date2Date32`, `geometry`)은 기본적으로 활성화됩니다.

* `decimal`: 정밀도가 허용되는 경우 `NUMERIC` 및 `DECIMAL` 타입을 `Decimal`로 변환합니다.
* `datetime64`: 정밀도가 `0`이 아닌 경우 `DATETIME` 및 `TIMESTAMP` 타입을 `DateTime` 대신 `DateTime64`로 변환합니다.
* `date2Date32`: `DATE`를 `Date` 대신 `Date32`로 변환합니다. `date2String`보다 우선 적용됩니다.
* `date2String`: `DATE`를 `Date` 대신 `String`으로 변환합니다. `datetime64`가 적용되면 이 설정은 무시됩니다.
* `geometry`: MySQL의 공간 타입(`LINESTRING`, `POLYGON`, `MULTILINESTRING`, `MULTIPOLYGON`)을 해당하는 ClickHouse 기하학 타입으로 변환하고, 일반 `GEOMETRY` 타입은 포괄적인 `Geometry` 타입으로 변환합니다. `POINT`는 이 옵션과 관계없이 항상 `Point`로 변환됩니다. 일반 `GEOMETRY` 컬럼은 모든 하위 타입을 담을 수 있으므로, 하위 타입에 해당하는 ClickHouse 타입이 없는 값(`MULTIPOINT`, `GEOMETRYCOLLECTION`)을 읽으면 읽기 시점에 예외가 발생합니다. `MULTIPOINT` 또는 `GEOMETRYCOLLECTION`으로 선언된 컬럼은 `String`으로 매핑됩니다.
