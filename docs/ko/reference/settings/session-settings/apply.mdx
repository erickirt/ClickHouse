<div id="apply_deleted_mask">
  ## apply_deleted_mask
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

경량한 DELETE로 삭제된 행을 필터링해 제외합니다. 비활성화하면 쿼리에서 해당 행을 읽을 수 있습니다. 이는 디버깅이나 &quot;삭제 취소&quot; 시나리오에 유용합니다

<div id="apply_mutations_on_fly">
  ## apply_mutations_on_fly
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

true로 설정하면 데이터 파트(data part)에 아직 구체화되지 않은 뮤테이션(UPDATE 및 DELETE)이 SELECT 시 적용됩니다.

<div id="apply_prewhere_after_final">
  ## apply_prewhere_after_final
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.12"},{"label": "0"},{"label": "새 설정입니다. 활성화하면 PREWHERE 조건이 FINAL 처리 후에 적용됩니다."}]}]} />

활성화하면 ReplacingMergeTree 및 유사한 엔진에서 PREWHERE 조건이 FINAL 처리 후에 적용됩니다.
이는 PREWHERE가 중복된 행마다 값이 다를 수 있는 컬럼을 참조하고,
필터링 전에 FINAL이 최종적으로 남을 행을 먼저 선택하도록 하려는 경우에 유용합니다. 비활성화하면 PREWHERE는 읽는 동안 적용됩니다.
참고: apply&#95;row&#95;level&#95;security&#95;after&#95;final이 활성화되어 있고 행 정책이 정렬 키가 아닌 컬럼을 사용하는 경우, PREWHERE도
올바른 실행 순서를 유지하기 위해 나중에 적용됩니다(행 정책은 PREWHERE보다 먼저 적용되어야 합니다).

<div id="apply_row_policy_after_final">
  ## apply_row_policy_after_final
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.10"},{"label": "1"},{"label": "행 정책과 PREWHERE가 FINAL 전에 적용되던 post-#87303 회귀 문제를 수정하기 위해 이 설정이 추가되었습니다. PR #91065에서 이 설정이 도입되었고, PR #97279에서 정확성을 보장하는 true가 기본값으로 설정되었습니다. compatibility가 수정 이전의 false 동작으로 되돌아가지 않도록 {true, true}로 기록되었습니다."}]}]} />

활성화하면 행 정책과 PREWHERE가 *MergeTree 테이블에서 FINAL 처리 후에 적용됩니다. (특히 ReplacingMergeTree에서 중요합니다.)
비활성화하면 행 정책이 FINAL 전에 적용되므로, 정책이 ReplacingMergeTree 또는 유사한 엔진에서
중복 제거에 사용되어야 하는 행을 필터링하는 경우 결과가 달라질 수 있습니다.

행 정책 표현식이 ORDER BY의 컬럼에만 의존하는 경우에는 최적화를 위해 여전히 FINAL 전에 적용됩니다.
이러한 필터링은 중복 제거 결과에 영향을 주지 않기 때문입니다.

가능한 값:

* 0 — 행 정책과 PREWHERE가 FINAL 전에 적용됩니다(기본값).
* 1 — 행 정책과 PREWHERE가 FINAL 후에 적용됩니다.

<div id="apply_settings_from_server">
  ## apply_settings_from_server
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.2"},{"label": "1"},{"label": "클라이언트 측 코드(예: INSERT 입력 파싱 및 쿼리 결과 출력 형식 지정)는 서버 구성의 설정을 포함해 서버와 동일한 설정을 사용합니다."}]}]} />

클라이언트가 서버의 설정을 받아들일지 여부를 지정합니다.

이 설정은 클라이언트 측에서 수행되는 작업에만 영향을 줍니다. 특히 INSERT 입력 데이터 파싱과 쿼리 결과 출력 형식 지정에 영향을 미칩니다. 쿼리 실행의 대부분은 서버에서 이루어지므로 이 설정의 영향을 받지 않습니다.

일반적으로 이 설정은 클라이언트(클라이언트 명령줄 인수, `SET` 쿼리 또는 `SELECT` 쿼리의 `SETTINGS` 섹션)가 아니라 사용자 프로필(users.xml 또는 `ALTER USER` 같은 쿼리)에서 설정해야 합니다. 클라이언트를 통해 false로 변경할 수는 있지만 true로는 변경할 수 없습니다(사용자 프로필에 `apply_settings_from_server = false`가 설정되어 있으면 서버가 설정을 보내지 않기 때문입니다).

참고로 초기에(24.12) `send_settings_to_client`라는 서버 설정이 있었지만, 이후 사용성을 개선하기 위해 이 클라이언트 설정으로 대체되었습니다.
