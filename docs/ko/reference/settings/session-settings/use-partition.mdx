  ## use_partition_minmax_for_primary_key_pruning

프라이머리 키 컬럼이 파티션 키의 입력 컬럼이기도 한 경우, `MergeTree` 테이블의 프라이머리 키 분석 중 파티션 minmax 인덱스 범위를 사용해 더 많은 그래뉼을 프루닝합니다.

예를 들어, `ORDER BY (id, event_time)` 및 `PARTITION BY toYYYYMM(event_time)`가 있는 `MergeTree` 테이블에서 ClickHouse는 프라이머리 키 인덱스 분석 중 `event_time`의 파티션 minmax 인덱스를 사용하여 그래뉼 프루닝을 더 정확하게 결정합니다.

가능한 값:

* 0 — 비활성화됩니다.
* 1 — 활성화됩니다.

* 0 — 비활성화됨.
* 1 — 활성화됨.
