경량한 삭제의 일부로 실행되는 내부 업데이트 쿼리의 모드입니다.

가능한 값:

* `alter_update` - heavyweight mutation을 생성하는 `ALTER UPDATE` 쿼리를 실행합니다.
* `lightweight_update` - 가능하면 경량 업데이트를 실행하고, 그렇지 않으면 `ALTER UPDATE`를 실행합니다.
* `lightweight_update_force` - 가능하면 경량 업데이트를 실행하고, 그렇지 않으면 예외를 발생시킵니다.

  ## lightweight_deletes_sync

[`mutations_sync`](/ko/reference/settings/session-settings/mutations#mutations_sync)와 동일하지만 경량한 삭제 실행만 제어합니다.

가능한 값:

| 값   | 설명                                                                                                              |
| --- | --------------------------------------------------------------------------------------------------------------- |
| `0` | 뮤테이션이 비동기로 실행됩니다.                                                                                               |
| `1` | 쿼리는 현재 서버에서 경량한 삭제가 완료될 때까지 대기합니다.                                                                              |
| `2` | 쿼리는 모든 레플리카(있는 경우)에서 경량한 삭제가 완료될 때까지 대기합니다.                                                                     |
| `3` | 쿼리는 활성 레플리카에 대해서만 대기합니다. `SharedMergeTree`에서만 지원됩니다. `ReplicatedMergeTree`에서는 `mutations_sync = 2`와 동일하게 동작합니다. |
