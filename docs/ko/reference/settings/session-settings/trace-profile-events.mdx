<div id="trace_profile_events">
  ## trace_profile_events
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

각 프로파일 이벤트가 UPDATE될 때마다 프로파일 이벤트 이름, 증가값, 그리고 스택트레이스를 수집하여 [trace&#95;log](/ko/reference/system-tables/trace_log)로 전송할지 여부를 활성화하거나 비활성화합니다.

가능한 값:

* 1 — 프로파일 이벤트 추적이 활성화됩니다.
* 0 — 프로파일 이벤트 추적이 비활성화됩니다.

<div id="trace_profile_events_list">
  ## trace_profile_events_list
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": ""},{"label": "새로운 설정"}]}]} />

설정 `trace_profile_events`가 활성화되면 추적할 이벤트를 쉼표로 구분된 지정된 이름 목록으로 제한합니다.
`trace_profile_events_list`가 빈 문자열이면(기본값) 모든 프로파일 이벤트를 추적합니다.

예시 값: &#39;DiskS3ReadMicroseconds,DiskS3ReadRequestsCount,SelectQueryTimeMicroseconds,ReadBufferFromS3Bytes&#39;

이 설정을 사용하면 많은 수의 쿼리에 대해 데이터를 더 정밀하게 수집할 수 있습니다. 그렇지 않으면 방대한 양의 이벤트로 인해 내부 시스템 로그 큐에 오버플로우가 발생하여 그중 일부가 삭제될 수 있습니다.
