<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "10000"},{"label": "`base58Encode`, `base58Decode`, `tryBase58Decode`의 입력 크기(변환이 입력 `length`에 대해 2차 시간 복잡도를 가짐)를 기본적으로 10 KB로 제한하는 새로운 설정입니다. compatibility 값 `0`은 제한을 비활성화하여, 임의로 큰 입력을 허용하던 이전 동작을 복원합니다."}]}]} />

함수 `base58Encode`, `base58Decode`, `tryBase58Decode`에 전달되는 단일 입력값의 최대 크기(바이트)입니다. 일반적인 `base58` 변환은 입력 `length`에 대해 2차 시간 복잡도를 가지므로, 큰 값 하나만으로도 매우 오랜 시간 실행될 수 있습니다. `base58`은 짧은 데이터(키, 해시, 주소)를 위한 것이므로, 기본값인 10 KB는 넉넉한 안전 임계값입니다. `base58Encode`와 `base58Decode`는 이보다 큰 입력에 대해 `TOO_LARGE_STRING_SIZE`를 발생시키고, `tryBase58Decode`는 빈 문자열을 반환합니다. 값 `0`은 제한을 비활성화합니다(이 설정이 도입되기 전의 동작). 선형 시간 복잡도의 `base32` 및 `base64` 함수에는 영향을 주지 않습니다.

`dateTrunc` 함수의 반환 유형 동작을 변경할 수 있습니다.

가능한 값:

* 0 - 두 번째 인수가 `DateTime64/Date32`이면, 첫 번째 인수의 시간 단위와 관계없이 반환 유형은 `DateTime64/Date32`입니다.
* 1 - `Date32`의 경우 결과는 항상 `Date`입니다. `DateTime64`의 경우 시간 단위가 `second` 이상이면 결과는 `DateTime`입니다.
