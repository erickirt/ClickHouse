<div id="merge_tree_coarse_index_granularity">
  ## merge_tree_coarse_index_granularity
</div>

<SettingsInfoBlock type="UInt64" default_value="8" />

데이터를 검색할 때 ClickHouse는 인덱스 파일의 데이터 마크를 확인합니다. 필요한 키가 특정 범위에 있는 것을 발견하면, ClickHouse는 해당 범위를 `merge_tree_coarse_index_granularity`개의 하위 범위로 나눈 다음 그 안에서 필요한 키를 재귀적으로 검색합니다.

가능한 값:

* 모든 양의 짝수 정수.

<div id="merge_tree_compact_parts_min_granules_to_multibuffer_read">
  ## merge_tree_compact_parts_min_granules_to_multibuffer_read
</div>

<CloudOnlyBadge />

<SettingsInfoBlock type="UInt64" default_value="16" />

이 설정은 ClickHouse Cloud에서만 적용됩니다. 병렬 읽기와 프리페치를 지원하는 멀티버퍼 리더를 사용하기 위한 MergeTree 테이블의 compact 파트 stripe 내 그래뉼 수를 지정합니다. remote fs에서 읽을 때 멀티버퍼 리더를 사용하면 읽기 요청 수가 증가합니다. 파트에 포함된 그래뉼 수가 이 임계값보다 적고, 읽는 컬럼 수보다도 적으면 해당 파트는 단일 버퍼로 읽습니다. 이 경우 단일 버퍼 리더가 멀티버퍼 리더보다 읽기 요청을 더 적게 보내기 때문입니다.

<div id="merge_tree_generic_exclusion_search_max_steps">
  ## merge_tree_generic_exclusion_search_max_steps
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "프라이머리 키 인덱스에 대한 일반 제외 검색의 단계 수를 제한하는 새로운 설정입니다."}]}]} />

필터를 프라이머리 키의 단일 연속 범위로 평가할 수 없는 경우, 예를 들어 첫 번째가 아닌 키 컬럼을 사용하는 경우 ClickHouse는 인덱스 마크에 대해 반복적인 일반 제외 검색 알고리즘을 실행합니다. 동일한 알고리즘은 텍스트 인덱스 분석에도 사용됩니다. 이 설정은 알고리즘이 각 데이터 파트에 대해 수행하는 단계 수(인덱스 검사 수)를 제한합니다.

예산은 먼저 가장 큰 남은 마크 범위에 사용됩니다. 예산이 소진되면 완전히 분석되지 않은 범위는 전체가 그대로 허용되므로 쿼리의 정확성은 유지되지만, 무제한 검색이 선택했을 때보다 더 많은 그래뉼을 읽을 수 있습니다. 예산을 낮추면 더 많은 데이터를 읽는 대신 인덱스 분석 속도가 빨라집니다. 이 제한은 분석 비용에 대한 엄격한 상한이라기보다 대략적인 값입니다. 검색은 대략 한 번의 분할 라운드 정도 이 한도를 초과할 수 있으며, 파트가 이미 많은 범위로 나뉘어 있는 경우(예: 쿼리 조건 캐시로 인해)에는 제한과 관계없이 각 범위를 최소 한 번씩 검사합니다.

검색이 각 데이터 파트에서 수행한 단계 수는 쿼리의 trace 수준 로그 메시지에 보고되며, `IndexGenericExclusionSearchStepLimitReached` 및 `TextIndexGenericExclusionSearchStepLimitReached` profile events는 예산이 소진된 횟수를 집계합니다.

(기본) 값 0은 단계 수가 무제한임을 의미합니다.

사용 가능한 값:

* 무제한 단계를 의미하는 0 또는 임의의 양의 정수.

<div id="merge_tree_max_bytes_to_use_cache">
  ## merge_tree_max_bytes_to_use_cache
</div>

<SettingsInfoBlock type="UInt64" default_value="2013265920" />

하나의 쿼리에서 ClickHouse가 `merge_tree_max_bytes_to_use_cache`바이트를 초과하여 읽어야 하면 비압축 블록 캐시를 사용하지 않습니다.

비압축 블록 캐시는 쿼리용으로 추출된 데이터를 저장합니다. ClickHouse는 이 캐시를 사용해 반복적인 소규모 쿼리의 응답 속도를 높입니다. 이 설정은 많은 양의 데이터를 읽는 쿼리로 인해 캐시가 불필요하게 소모되는 것을 방지합니다. [uncompressed&#95;cache&#95;size](/ko/reference/settings/server-settings/settings#uncompressed_cache_size) 서버 설정은 비압축 블록 캐시의 크기를 정의합니다.

가능한 값:

* 모든 양의 정수.

  ## merge_tree_max_rows_to_use_cache
</div>

비압축 블록 캐시는 쿼리를 위해 추출된 데이터를 저장합니다. ClickHouse는 이 캐시를 사용하여 반복적으로 실행되는 소규모 쿼리의 응답 속도를 높입니다. 이 설정은 많은 양의 데이터를 읽는 쿼리로 인해 캐시가 불필요하게 소모되는 것을 방지합니다. [uncompressed&#95;cache&#95;size](/ko/reference/settings/server-settings/settings#uncompressed_cache_size) 서버 설정은 비압축 블록 캐시의 크기를 정의합니다.

가능한 값:

* 모든 양의 정수.
