<div id="distributed_index_analysis_only_on_coordinator">
  ## distributed_index_analysis_only_on_coordinator
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "새 설정입니다."}]}]} />

활성화하면 분산 인덱스 분석이 coordinator에서만 실행됩니다.
이렇게 하면 프레디케이트에 서브쿼리(예: `IN (SELECT ...)`)가 포함될 때 O(N^2)개의 파생 쿼리가 발생하는 것을 방지할 수 있습니다.
그렇지 않으면 각 팔로워 레플리카가 자체 분산 인덱스 분석을 독립적으로 트리거하기 때문입니다.
다만 서브쿼리에서 큰 테이블을 사용하는 경우에는 분산 인덱스 분석의 효율이 떨어집니다.
