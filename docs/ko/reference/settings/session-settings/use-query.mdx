이러한 설정은 [system.settings](/ko/reference/system-tables/settings)에서 확인할 수 있으며, [소스 코드](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp)를 기반으로 자동 생성됩니다.

<div id="use_query_cache">
  ## use_query_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

이 설정을 활성화하면 `SELECT` 쿼리에서 [쿼리 캐시](/ko/concepts/features/performance/caches/query-cache)를 사용할 수 있습니다. 매개변수 [enable&#95;reads&#95;from&#95;query&#95;cache](/ko/reference/settings/session-settings/enable#enable_reads_from_query_cache)
및 [enable&#95;writes&#95;to&#95;query&#95;cache](/ko/reference/settings/session-settings/enable#enable_writes_to_query_cache)는 캐시 사용 방식을 더 세부적으로 제어합니다.

가능한 값:

* 0 - 비활성화됨
* 1 - 활성화됨

<div id="use_query_condition_cache">
  ## use_query_condition_cache
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.4"},{"label": "1"},{"label": "새로운 최적화"}]}, {"id": "row-2","items": [{"label": "25.3"},{"label": "0"},{"label": "새 설정입니다."}]}]} />

[쿼리 조건 캐시](/ko/concepts/features/performance/caches/query-condition-cache)를 활성화합니다. 이 캐시는 `WHERE` 절의 조건을 만족하지 않는 데이터 파트(data part)의 그래뉼 범위를 저장하고,
이 정보를 후속 쿼리에서 임시 인덱스로 재사용합니다.

가능한 값:

* 0 - 비활성화됨
* 1 - 활성화됨
