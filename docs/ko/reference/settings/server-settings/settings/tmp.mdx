이 설정은 ClickHouse 서버를 구성하는 데 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="No" />

대규모 쿼리를 처리할 때 사용할 임시 데이터를 저장하는 로컬 파일 시스템 경로입니다.

<Note>
  * 임시 데이터 저장소를 구성하는 옵션으로는 tmp&#95;path, tmp&#95;policy, temporary&#95;data&#95;in&#95;cache 중 하나만 사용할 수 있습니다.
  * 끝의 슬래시는 필수입니다.
</Note>

**예시**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

임시 데이터용 스토리지 정책입니다. `tmp` 접두사가 있는 모든 파일은 시작할 때 제거됩니다.

<Note>
  `tmp_policy`로 객체 스토리지를 사용할 때의 권장 사항:

  * 각 서버에서 별도의 `bucket:path`를 사용하십시오
  * `metadata_type=plain`을 사용하십시오
  * 이 버킷에 TTL을 설정하는 것도 좋습니다
</Note>

<Note>
  * 임시 데이터 저장소를 구성할 때 사용할 수 있는 옵션은 `tmp_path`, `tmp_policy`, `temporary_data_in_cache` 중 하나뿐입니다.
  * `move_factor`, `keep_free_space_bytes`, `max_data_part_size_bytes`는 무시됩니다.
  * 정책에는 *정확히 하나의 볼륨*이 있어야 합니다

  자세한 내용은 [MergeTree 테이블 엔진](/ko/reference/engines/table-engines/mergetree-family/mergetree) 문서를 참조하십시오.
</Note>

**예시**

`/disk1`이 가득 차면 임시 데이터는 `/disk2`에 저장됩니다.

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
