이 설정은 ClickHouse 서버를 구성하는 항목이며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="http_handlers">
  ## http_handlers
</div>

사용자 지정 HTTP 핸들러를 사용할 수 있습니다.
새 http 핸들러를 추가하려면 새 `<rule>`를 추가하면 됩니다.
규칙은 정의된 순서에 따라 위에서 아래로 검사되며,
가장 먼저 일치하는 규칙의 핸들러가 실행됩니다.

다음 설정은 하위 태그로 구성할 수 있습니다:

| Sub-tags             | Definition                                                                                           |
| -------------------- | ---------------------------------------------------------------------------------------------------- |
| `url`                | 요청 URL을 일치시키기 위해 &#39;regex:&#39; 접두사를 사용해 정규식 매칭을 사용할 수 있습니다(선택 사항)                                 |
| `methods`            | 요청 메서드를 일치시키기 위해 쉼표로 여러 메서드를 구분할 수 있습니다(선택 사항)                                                       |
| `headers`            | 요청 헤더를 일치시키기 위해 각 하위 요소를 매칭합니다(하위 요소 이름은 헤더 이름). &#39;regex:&#39; 접두사를 사용해 정규식 매칭을 사용할 수 있습니다(선택 사항) |
| `handler`            | 요청 핸들러                                                                                               |
| `empty_query_string` | URL에 query string이 없는지 확인합니다                                                                         |

`handler`에는 다음 설정이 포함되며, 하위 태그로 구성할 수 있습니다:

| Sub-tags           | Definition                                                                                                                         |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------- |
| `url`              | 리디렉션할 대상 URL                                                                                                                       |
| `type`             | 지원되는 타입: static, dynamic&#95;query&#95;handler, predefined&#95;query&#95;handler, redirect                                         |
| `status`           | static 타입과 함께 사용하며, 응답 status code를 지정합니다                                                                                          |
| `query_param_name` | dynamic&#95;query&#95;handler 타입과 함께 사용하며, HTTP 요청 파라미터에서 `<query_param_name>`에 해당하는 값을 추출해 실행합니다                                  |
| `query`            | predefined&#95;query&#95;handler 타입과 함께 사용하며, 핸들러가 호출되면 쿼리를 실행합니다                                                                  |
| `content_type`     | static 타입과 함께 사용하며, 응답 content-type을 지정합니다                                                                                         |
| `response_content` | static 타입과 함께 사용하며, 클라이언트에 전송할 Response 콘텐츠입니다. &#39;file://&#39; 또는 &#39;config://&#39; 접두사를 사용하면 파일 또는 구성에서 콘텐츠를 찾아 클라이언트에 전송합니다 |

규칙 목록과 함께 `<defaults/>`를 지정할 수도 있으며, 이 경우 모든 기본 핸들러가 활성화됩니다.

예시:

```xml
<http_handlers>
    <rule>
        <url>/</url>
        <methods>POST,GET</methods>
        <headers><pragma>no-cache</pragma></headers>
        <handler>
            <type>dynamic_query_handler</type>
            <query_param_name>query</query_param_name>
        </handler>
    </rule>

    <rule>
        <url>/predefined_query</url>
        <methods>POST,GET</methods>
        <handler>
            <type>predefined_query_handler</type>
            <query>SELECT * FROM system.settings</query>
        </handler>
    </rule>

    <rule>
        <handler>
            <type>static</type>
            <status>200</status>
            <content_type>text/plain; charset=UTF-8</content_type>
            <response_content>config://http_server_default_response</response_content>
        </handler>
    </rule>
</http_handlers>
```

<div id="http_options_response">
  ## http_options_response
</div>

`OPTIONS` HTTP 요청의 응답에 헤더를 추가하는 데 사용됩니다.
`OPTIONS` 메서드는 CORS 프리플라이트 요청을 보낼 때 사용됩니다.

자세한 내용은 [OPTIONS](https://developer.mozilla.org/en-US/docs/Web/HTTP/Methods/OPTIONS)를 참조하십시오.

예시:

```xml
<http_options_response>
     <header>
            <name>Access-Control-Allow-Origin</name>
            <value>*</value>
     </header>
     <header>
          <name>Access-Control-Allow-Headers</name>
          <value>origin, x-requested-with, x-clickhouse-format, x-clickhouse-user, x-clickhouse-key, Authorization</value>
     </header>
     <header>
          <name>Access-Control-Allow-Methods</name>
          <value>POST, GET, OPTIONS</value>
     </header>
     <header>
          <name>Access-Control-Max-Age</name>
          <value>86400</value>
     </header>
</http_options_response>
```

<div id="http_server_default_response">
  ## http_server_default_response
</div>

ClickHouse HTTP(s) server에 액세스할 때 기본으로 표시되는 페이지입니다.
기본값은 &quot;Ok.&quot;이며 끝에 줄 바꿈(line feed)이 포함됩니다.

**예시**

`http://localhost: http_port`에 액세스하면 `https://tabix.io/`가 열립니다.

```xml
<http_server_default_response>
  <![CDATA[<html ng-app="SMI2"><head><base href="http://ui.tabix.io/"></head><body><div ui-view="" class="content-ui"></div><script src="http://loader.tabix.io/master.js"></script></body></html>]]>
</http_server_default_response>
```
