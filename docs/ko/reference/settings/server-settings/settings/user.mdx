이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스 코드에서 자동 생성됩니다.

<div id="user_directories">
  ## user_directories
</div>

다음 설정을 포함하는 설정 파일의 섹션입니다:

* 미리 정의된 사용자가 있는 설정 파일의 경로.
* SQL 명령으로 생성된 사용자가 저장되는 폴더의 경로.
* SQL 명령으로 생성된 사용자가 저장되고 복제되는 ZooKeeper node 경로.

이 섹션이 지정되면 [users&#95;config](/ko/reference/settings/server-settings/settings/users#users_config) 및 [access&#95;control&#95;path](/ko/reference/settings/server-settings/settings/access-control#access_control_path)의 경로는 사용되지 않습니다.

`user_directories` 섹션에는 항목을 원하는 만큼 포함할 수 있으며, 항목의 순서는 precedence를 의미합니다(위에 있는 항목일수록 precedence가 높습니다).

**예시**

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <local_directory>
        <path>/var/lib/clickhouse/access/</path>
    </local_directory>
</user_directories>
```

사용자, 역할, 행 정책, 쿼터 및 프로필은 ZooKeeper에도 저장할 수 있습니다:

```xml
<user_directories>
    <users_xml>
        <path>/etc/clickhouse-server/users.xml</path>
    </users_xml>
    <replicated>
        <zookeeper_path>/clickhouse/access/</zookeeper_path>
    </replicated>
</user_directories>
```

또한 `memory` 섹션과 `ldap` 섹션을 정의할 수 있습니다. `memory`는 디스크에 기록하지 않고 정보를 메모리에만 저장하는 것을 의미하고, `ldap`은 LDAP 서버에 정보를 저장하는 것을 의미합니다.

로컬에 정의되지 않은 사용자를 위한 원격 사용자 디렉터리로 LDAP 서버를 추가하려면, 다음 설정과 함께 하나의 `ldap` 섹션을 정의하십시오:

| 설정       | 설명                                                                                                                                                                          |
| -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `roles`  | LDAP 서버에서 가져온 각 사용자에게 할당할 로컬 정의 역할 목록이 포함된 섹션입니다. 역할을 지정하지 않으면 사용자는 인증 후 어떤 작업도 수행할 수 없습니다. 나열된 역할 중 하나라도 인증 시점에 로컬에 정의되어 있지 않으면, 제공된 비밀번호가 올바르지 않은 경우와 마찬가지로 인증 시도가 실패합니다. |
| `server` | `ldap_servers` 구성 섹션에 정의된 LDAP 서버 이름 중 하나입니다. 이 매개변수는 필수이며 비워 둘 수 없습니다.                                                                                                     |

**예시**

```xml
<ldap>
    <server>my_ldap_server</server>
        <roles>
            <my_local_role1 />
            <my_local_role2 />
        </roles>
</ldap>
```

<div id="user_files_path">
  ## user_files_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_files/" changeable_without_restart="No" />

사용자 파일이 저장된 디렉터리입니다. 테이블 함수 [file()](/ko/reference/functions/table-functions/file), [fileCluster()](/ko/reference/functions/table-functions/fileCluster)에서 사용됩니다.

**예시**

```xml
<user_files_path>/var/lib/clickhouse/user_files/</user_files_path>
```

<div id="user_profile_events_per_cpu">
  ## user_profile_events_per_cpu
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />thread 수가 많을 때 CPU 간 캐시 라인 경합을 방지하기 위해 사용자별 `ProfileEvents` 카운터를 CPU별로 세그먼트합니다. 각 사용자는 `getNumCPUs() * num_events * sizeof(atomic)`만큼의 메모리(코어 수가 많은 시스템에서는 수백 KiB)를 사용합니다. 비활성화하면 사용자별 각 이벤트에 대해 공유되는 단일 atomic으로 폴백되어, 메모리 사용량을 줄이는 대신 처리량이 낮아집니다.

<div id="user_scripts_path">
  ## user_scripts_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/user_scripts/" changeable_without_restart="No" />

사용자 스크립트 파일이 저장된 디렉터리입니다. 실행형 사용자 정의 함수 [Executable User Defined Functions](/ko/reference/functions/regular-functions/udf#executable-user-defined-functions)에 사용됩니다.

**예시**

```xml
<user_scripts_path>/var/lib/clickhouse/user_scripts/</user_scripts_path>
```
