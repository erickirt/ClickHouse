이 설정은 ClickHouse 서버 구성에 사용되며 ClickHouse 소스에서 자동 생성됩니다.

<div id="distributed_ddl">
  ## distributed_ddl
</div>

클러스터에서 [분산 DDL 쿼리](/ko/reference/statements/distributed-ddl) (`CREATE`, `DROP`, `ALTER`, `RENAME`) 실행을 관리합니다.
[ZooKeeper](/ko/reference/settings/server-settings/settings/zookeeper#zookeeper)가 활성화되어 있을 때만 작동합니다.

`<distributed_ddl>` 내에서 구성할 수 있는 설정은 다음과 같습니다:

| 설정                     | 설명                                                                         | 기본값                           |
| ---------------------- | -------------------------------------------------------------------------- | ----------------------------- |
| `cleanup_delay_period` | 마지막 정리가 `cleanup_delay_period`초 이내에 수행되지 않은 경우, 새 노드 이벤트를 수신한 후 정리가 시작됩니다. | `60`초                         |
| `max_tasks_in_queue`   | 큐에 들어갈 수 있는 최대 작업 수입니다.                                                    | `1,000`                       |
| `path`                 | DDL 쿼리용 `task_queue`의 Keeper 내 경로                                          |                               |
| `pool_size`            | 동시에 실행할 수 있는 `ON CLUSTER` 쿼리 수입니다.                                         |                               |
| `profile`              | DDL 쿼리 실행에 사용하는 프로파일입니다.                                                   |                               |
| `task_max_lifetime`    | 노드의 수명이 이 값보다 크면 삭제합니다.                                                    | `7 * 24 * 60 * 60` (초 단위 1주일) |

**예시**

```xml
<distributed_ddl>
    <!-- Path in ZooKeeper to queue with DDL queries -->
    <path>/clickhouse/task_queue/ddl</path>

    <!-- Settings from this profile will be used to execute DDL queries -->
    <profile>default</profile>

    <!-- Controls how much ON CLUSTER queries can be run simultaneously. -->
    <pool_size>1</pool_size>

    <!--
         Cleanup settings (active tasks will not be removed)
    -->

    <!-- Controls task TTL (default 1 week) -->
    <task_max_lifetime>604800</task_max_lifetime>

    <!-- Controls how often cleanup should be performed (in seconds) -->
    <cleanup_delay_period>60</cleanup_delay_period>

    <!-- Controls how many tasks could be in the queue -->
    <max_tasks_in_queue>1000</max_tasks_in_queue>
</distributed_ddl>
```

<div id="distributed_ddl_use_initial_user_and_roles">
  ## distributed_ddl_use_initial_user_and_roles
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />활성화하면 ON CLUSTER 쿼리는 원격 세그먼트에서 실행될 때 initiator의 사용자와 역할을 유지한 채 사용합니다. 이렇게 하면 클러스터 전체에서 일관된 access control이 보장되지만, 해당 사용자와 역할이 모든 node에 존재해야 합니다.
