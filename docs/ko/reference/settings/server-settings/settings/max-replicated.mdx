이 설정은 ClickHouse 서버를 구성하는 데 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="max_replicated_fetches_network_bandwidth_for_server">
  ## max_replicated_fetches_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />복제된 fetch에 대한 네트워크상의 데이터 전송 최대 속도(초당 바이트)입니다. 0은 무제한을 의미합니다.

<div id="max_replicated_sends_network_bandwidth_for_server">
  ## max_replicated_sends_network_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />복제된 전송의 네트워크를 통한 데이터 교환 최대 속도를 초당 바이트 수로 설정합니다. 0은 무제한을 의미합니다.

<div id="max_replicated_table_num_to_throw">
  ## max_replicated_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

복제된 테이블의 수가 이 값보다 크면 서버가 예외를 발생시킵니다.

다음 데이터베이스 엔진의 테이블만 계산합니다:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  값이 `0`이면 제한이 없다는 의미입니다.
</Note>

**예시**

```xml
<max_replicated_table_num_to_throw>400</max_replicated_table_num_to_throw>
```
