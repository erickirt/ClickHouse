이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="page_cache_free_memory_ratio">
  ## page_cache_free_memory_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.15" changeable_without_restart="No" />메모리 제한 중 일정 비율을 사용자 공간 페이지 캐시가 사용하지 않도록 남겨 둡니다. Linux의 `min_free_kbytes` 설정과 유사합니다.

<div id="page_cache_history_window_ms">
  ## page_cache_history_window_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />해제된 메모리를 사용자 공간 페이지 캐시에서 다시 사용할 수 있을 때까지의 지연 시간입니다.

<div id="page_cache_max_size">
  ## page_cache_max_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />사용자 공간 페이지 캐시의 최대 크기입니다. 캐시를 비활성화하려면 0으로 설정하십시오. `page_cache_min_size`보다 크면, 전체 메모리 사용량이 제한값(`max_server_memory_usage[_to_ram_ratio]`) 아래로 유지되도록 하면서 사용 가능한 메모리를 최대한 활용할 수 있도록 이 범위 내에서 캐시 크기가 지속적으로 조정됩니다.

<div id="page_cache_min_size">
  ## page_cache_min_size
</div>

<SettingsInfoBlock type="UInt64" default_value="104857600" changeable_without_restart="No" />사용자 공간 페이지 캐시의 최소 크기입니다.

<div id="page_cache_policy">
  ## page_cache_policy
</div>

<SettingsInfoBlock type="String" default_value="SLRU" changeable_without_restart="No" />사용자 공간 페이지 캐시 정책의 이름입니다.

<div id="page_cache_shards">
  ## page_cache_shards
</div>

<SettingsInfoBlock type="UInt64" default_value="4" changeable_without_restart="No" />뮤텍스 경합을 줄이기 위해 사용자 공간 페이지 캐시를 이 수만큼의 세그먼트로 분산합니다. 실험적 기능이며, 성능 개선에는 도움이 되지 않을 가능성이 높습니다.

<div id="page_cache_size_ratio">
  ## page_cache_size_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.5" changeable_without_restart="No" />사용자 공간 페이지 캐시에서 보호 큐가 전체 캐시 크기에서 차지하는 비율입니다.
