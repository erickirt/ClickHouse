이 설정은 ClickHouse 서버를 구성하며, ClickHouse 소스에서 자동으로 생성되었습니다.

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />이 한도에 도달한 상태에서 생성을 시도하면 예외가 발생합니다. 하드 제한을 비활성화하려면 0으로 설정하십시오. 이 한도는 디스크 연결에 적용됩니다.

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />디스크(S3, Azure, GCS) 연결의 SO&#95;RCVBUF 옵션 크기입니다. 기본값은 204800(200 KB)입니다. 객체 스토리지는 대역폭-지연 곱이 100 KB보다 훨씬 낮은 동일 리전 내에서 접근되므로, 작은 상한값만으로도 전체 처리량을 확보하면서 커널 TCP 자동 튜닝이 그렇지 않으면 소켓마다 할당할 수 MiB 크기의 버퍼를 피할 수 있습니다. 대신 수신 버퍼에 커널 TCP 자동 튜닝을 사용하려면 0으로 설정하십시오. 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에 대해서만 자동 튜닝이 복원되며, 기존 풀의 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />디스크(S3, Azure, GCS) 연결의 `SO_SNDBUF` 옵션 크기입니다. 0보다 큰 값으로 설정하면 전송 버퍼에 대한 kernel TCP 자동 튜닝을 재정의합니다. 0 = 커널 기본값(자동 튜닝)입니다. 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에 대해서만 자동 튜닝이 복원되며, 기존 풀 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />이 제한을 초과하는 연결은 수명이 크게 짧아집니다. 이 제한은 디스크 연결에 적용됩니다.

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />이 제한을 초과하는 연결은 사용 후 재설정됩니다. 연결 캐시를 비활성화하려면 0으로 설정하십시오. 이 제한은 디스크 연결에 적용됩니다.

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />사용 중인 연결 수가 이 한도를 초과하면 로그에 경고 메시지가 기록됩니다. 이 한도는 디스크 연결에 적용됩니다.
