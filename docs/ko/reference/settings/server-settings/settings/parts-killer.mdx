이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="parts_killer_max_condemned_parts_per_batch">
  ## parts_killer_max_condemned_parts_per_batch
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="100000" changeable_without_restart="No" />

SharedMergeTree 테이블이 한 번의 정리 사이클에서 Keeper로부터 가져올 수 있는 폐기 대상으로 지정된 파트의 최대 개수입니다. 배치 크기를 제한하면 목록 조회와 multi-get이 Keeper 작업 timeout 내에 완료되도록 할 수 있으므로, 테이블에 폐기 대상으로 지정된 파트가 매우 많이 누적된 경우에도 정리가 계속 진행됩니다. 값은 반드시 0보다 커야 합니다. `0`으로 설정하면 Keeper가 매 사이클마다 빈 목록을 반환하게 되어 정리가 영구적으로 멈추게 됩니다. ClickHouse Cloud에서만 사용할 수 있습니다.

<div id="parts_killer_pool_size">
  ## parts_killer_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="128" changeable_without_restart="No" />

shared merge tree 파트를 정리하는 parts killer 스레드의 수입니다. ClickHouse Cloud에서만 사용할 수 있습니다.
