다음 설정은 ClickHouse 서버를 구성하는 데 사용되며 ClickHouse 소스에서 자동 생성됩니다.

<div id="async_load_databases">
  ## async_load_databases
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

데이터베이스(database)와 테이블(table)을 비동기로 로드합니다.

* `true`이면 `Ordinary`, `Atomic`, `Replicated` 엔진을 사용하는 모든 non-system 데이터베이스가 ClickHouse 서버 시작 후 비동기로 로드됩니다. `system.asynchronous_loader` 테이블, `tables_loader_background_pool_size`, `tables_loader_foreground_pool_size` 서버 설정을 참조하십시오. 아직 로드되지 않은 테이블에 접근하려는 쿼리는 해당 테이블의 시작이 완료될 때까지 대기합니다. 로드 작업이 실패하면 쿼리에서 오류를 다시 발생시킵니다(`async_load_databases = false`인 경우처럼 서버 전체를 종료하지 않음). 하나 이상의 쿼리가 대기 중인 테이블은 더 높은 우선순위로 로드됩니다. 데이터베이스에 대한 DDL 쿼리는 해당 데이터베이스의 시작이 완료될 때까지 대기합니다. 또한 대기 중인 전체 쿼리 수의 제한으로 `max_waiting_queries` 설정도 함께 고려하십시오.
* `false`이면 서버 시작 시 모든 데이터베이스를 로드합니다.

**예시**

```xml
<async_load_databases>true</async_load_databases>
```

<div id="async_load_system_database">
  ## async_load_system_database
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

system 테이블을 비동기로 로드합니다. `system` 데이터베이스에 로그 테이블과 파트가 많은 경우 유용합니다. `async_load_databases` 설정과는 별개로 동작합니다.

* `true`로 설정하면 `Ordinary`, `Atomic`, `Replicated` 엔진을 사용하는 모든 system 데이터베이스가 ClickHouse 서버 시작 후 비동기로 로드됩니다. `system.asynchronous_loader` 테이블, `tables_loader_background_pool_size` 및 `tables_loader_foreground_pool_size` 서버 설정을 참조하십시오. 아직 로드되지 않은 system 테이블에 접근하려는 모든 쿼리는 해당 테이블의 시작이 완료될 때까지 대기합니다. 하나 이상의 쿼리가 대기 중인 테이블은 더 높은 우선순위로 로드됩니다. 또한 대기 중인 전체 쿼리 수를 제한하려면 `max_waiting_queries` 설정도 함께 사용하는 것을 고려하십시오.
* `false`로 설정하면 system 데이터베이스는 서버 시작 전에 로드됩니다.

**예시**

```xml
<async_load_system_database>true</async_load_system_database>
```
