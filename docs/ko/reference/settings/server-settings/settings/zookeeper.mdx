이 설정은 ClickHouse 서버를 구성하는 데 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="zookeeper">
  ## zookeeper
</div>

ClickHouse가 [ZooKeeper](http://zookeeper.apache.org/) 클러스터와 상호 작용할 수 있도록 하는 설정을 포함합니다. ClickHouse는 복제된 테이블을 사용할 때 레플리카의 메타데이터를 저장하기 위해 ZooKeeper를 사용합니다. 복제된 테이블을 사용하지 않는다면 이 매개변수 섹션은 생략할 수 있습니다.

다음 설정은 하위 태그로 구성할 수 있습니다:

| Setting                                         | Description                                                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `node`                                          | ZooKeeper endpoint입니다. 여러 endpoint를 설정할 수 있습니다. 예: `<node index="1"><host>example_host</host><port>2181</port></node>`. `index` 속성은 ZooKeeper 클러스터에 연결을 시도할 때 노드의 순서를 지정합니다.                                                                                                                                                     |
| `operation_timeout_ms`                          | 단일 작업의 최대 timeout입니다. 단위는 밀리초입니다.                                                                                                                                                                                                                                                                                                |
| `session_timeout_ms`                            | 클라이언트 session의 최대 timeout입니다. 단위는 밀리초입니다.                                                                                                                                                                                                                                                                                        |
| `root` (optional)                               | ClickHouse 서버가 사용하는 znode들의 루트로 사용되는 znode입니다.                                                                                                                                                                                                                                                                                   |
| `fallback_session_lifetime.min` (optional)      | 프라이머리를 사용할 수 없을 때 폴백 노드에 대한 zookeeper session 수명의 최소 한계입니다(로드 밸런싱). 초 단위로 설정합니다. 기본값: 3시간.                                                                                                                                                                                                                                       |
| `fallback_session_lifetime.max` (optional)      | 프라이머리를 사용할 수 없을 때 폴백 노드에 대한 zookeeper session 수명의 최대 한계입니다(로드 밸런싱). 초 단위로 설정합니다. 기본값: 6시간.                                                                                                                                                                                                                                       |
| `identity` (optional)                           | 요청한 znode에 접근할 때 ZooKeeper에서 요구하는 사용자와 password입니다.                                                                                                                                                                                                                                                                              |
| `use_compression` (optional)                    | `true`로 설정하면 Keeper protocol에서 Compression을 활성화합니다.                                                                                                                                                                                                                                                                              |
| `use_xid_64` (optional)                         | 64비트 transaction ID를 활성화합니다. 확장 transaction ID 형식을 사용하려면 `true`로 설정합니다. 기본값: `false`.                                                                                                                                                                                                                                            |
| `pass_opentelemetry_tracing_context` (optional) | Keeper request로 OpenTelemetry tracing Context를 전파하도록 활성화합니다. 활성화하면 Keeper 작업에 대한 tracing span이 생성되어 ClickHouse와 Keeper 전반에서 distributed tracing을 사용할 수 있습니다. 자세한 내용은 [Tracing ClickHouse Keeper Requests](/ko/guides/oss/deployment-and-scaling/monitoring/opentelemetry#tracing-clickhouse-keeper-requests)를 참조하십시오. 기본값: `false`. |

또한 ZooKeeper node 선택 알고리즘을 지정할 수 있는 `zookeeper_load_balancing` 설정(선택 사항)도 있습니다:

| Algorithm Name                  | Description                                                                |
| ------------------------------- | -------------------------------------------------------------------------- |
| `random`                        | ZooKeeper node 중 하나를 무작위로 선택합니다.                                           |
| `in_order`                      | 첫 번째 ZooKeeper node를 선택하고, 사용할 수 없으면 두 번째 node를 선택하는 식으로 진행합니다.            |
| `nearest_hostname`              | 서버의 호스트명과 가장 유사한 호스트명을 가진 ZooKeeper node를 선택합니다. 호스트명은 이름 접두사를 기준으로 비교합니다. |
| `hostname_levenshtein_distance` | `nearest_hostname`와 비슷하지만, 호스트명을 Levenshtein distance 방식으로 비교합니다.          |
| `first_or_random`               | 첫 번째 ZooKeeper node를 선택하고, 사용할 수 없으면 나머지 ZooKeeper node 중 하나를 무작위로 선택합니다.  |
| `round_robin`                   | 첫 번째 ZooKeeper node를 선택하고, 재연결이 발생하면 다음 node를 선택합니다.                       |

**예시 구성**

```xml
<zookeeper>
    <node>
        <host>example1</host>
        <port>2181</port>
    </node>
    <node>
        <host>example2</host>
        <port>2181</port>
    </node>
    <session_timeout_ms>30000</session_timeout_ms>
    <operation_timeout_ms>10000</operation_timeout_ms>
    <!-- Optional. Chroot suffix. Should exist. -->
    <root>/path/to/zookeeper/node</root>
    <!-- Optional. Zookeeper digest ACL string. -->
    <identity>user:password</identity>
    <!--<zookeeper_load_balancing>random / in_order / nearest_hostname / hostname_levenshtein_distance / first_or_random / round_robin</zookeeper_load_balancing>-->
    <zookeeper_load_balancing>random</zookeeper_load_balancing>
    <!-- Optional. Enable 64-bit transaction IDs. -->
    <use_xid_64>false</use_xid_64>
    <!-- Optional. Enable OpenTelemetry tracing context propagation. -->
    <pass_opentelemetry_tracing_context>false</pass_opentelemetry_tracing_context>
</zookeeper>
```

<SystemLogParameters />

**예시**

```xml
<clickhouse>
    <zookeeper_log>
        <database>system</database>
        <table>zookeeper_log</table>
        <flush_interval_milliseconds>7500</flush_interval_milliseconds>
        <ttl>event_date + INTERVAL 1 WEEK DELETE</ttl>
    </zookeeper_log>
```
