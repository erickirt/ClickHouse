이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스 코드에서 자동으로 생성됩니다.

<div id="interserver_listen_host">
  ## interserver_listen_host
</div>

ClickHouse 서버 간 데이터 교환이 가능한 host를 제한합니다.
Keeper를 사용하는 경우 동일한 제한이 서로 다른 Keeper 인스턴스 간 통신에도 적용됩니다.

<Note>
  기본적으로 이 값은 [`listen_host`](/ko/reference/settings/server-settings/settings/listen#listen_host) 설정과 동일합니다.
</Note>

**예시**

```xml
<interserver_listen_host>::ffff:a00:1</interserver_listen_host>
<interserver_listen_host>10.0.0.1</interserver_listen_host>
```

<div id="interserver_tables_status_require_auth">
  ## interserver_tables_status_require_auth
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

interserver `TablesStatusRequest`가 클러스터
`<secret>`로 인증되도록 요구합니다. 시크릿 해시를 보낼 수 있을 만큼 최신인 클라이언트(프로토콜 revision
`DBMS_MIN_REVISION_WITH_INTERSERVER_SECRET_TABLES_STATUS`)는 항상 검증됩니다. 또한 이
설정은 해시를 보내지 않는 구버전 클라이언트도 거부하므로, 기본적으로 인증되지 않은
table-status 노출이 차단됩니다.

기본값은 `true`입니다(기본적으로 보안이 적용됨). rolling upgrade 중에는 아직 upgrade되지 않은
node가 이전 프로토콜을 사용하므로 해시를 보내지 않습니다. 따라서 이러한 node에서 이미
upgrade된 node를 대상으로 시작한 `Distributed` 쿼리의 `TablesStatusRequest`는
거부됩니다. 혼합 버전 cluster를 반드시 운영해야 한다면, 모든 node의 upgrade가 완료될
때까지 upgrade된 node에서 이 값을 `false`로 설정한 다음, 완료 후 재정의를 제거하십시오.
