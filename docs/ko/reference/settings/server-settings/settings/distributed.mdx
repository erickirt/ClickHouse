이 설정은 ClickHouse 서버를 구성하는 데 사용되며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="distributed_ddl.cleanup_delay_period">
  ## distributed_ddl.cleanup_delay_period
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />마지막 정리가 `<cleanup_delay_period>`초 이내에 수행되지 않은 경우, 새 노드 이벤트를 수신하면 정리가 시작됩니다.

<div id="distributed_ddl.max_tasks_in_queue">
  ## distributed_ddl.max_tasks_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />큐에 포함될 수 있는 최대 작업 수입니다.

<div id="distributed_ddl.path">
  ## distributed_ddl.path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/ddl/" changeable_without_restart="No" />DDL 쿼리용 `<task_queue>`의 Keeper 내 경로

<div id="distributed_ddl.pool_size">
  ## distributed_ddl.pool_size
</div>

<SettingsInfoBlock type="Int32" default_value="1" changeable_without_restart="No" />동시에 실행할 수 있는 `<ON CLUSTER>` 쿼리의 수

<div id="distributed_ddl.profile">
  ## distributed_ddl.profile
</div>

DDL 쿼리 실행에 사용되는 프로필

<div id="distributed_ddl.replicas_path">
  ## distributed_ddl.replicas_path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/replicas/" changeable_without_restart="No" />레플리카의 `<task_queue>`에 대한 Keeper 경로

<div id="distributed_ddl.task_max_lifetime">
  ## distributed_ddl.task_max_lifetime
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" changeable_without_restart="No" />노드의 경과 시간이 이 값보다 크면 삭제합니다.
