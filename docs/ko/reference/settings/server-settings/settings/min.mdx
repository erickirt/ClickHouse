이 설정은 ClickHouse 서버를 구성하기 위한 것이며 ClickHouse 소스 코드에서 자동 생성됩니다.

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

`max_server_memory_usage`에 도달한 후 `MEMORY_LIMIT_EXCEEDED`를 발생시킬 수 있는 일반적인 C++ 할당(표준 컨테이너, 문자열, `std::vector` 확장, 스마트 포인터 등에서 이루어지는 종류)의 최소 크기(바이트)입니다. 이보다 작은 일반 할당도 메모리 추적기에 계속 반영되지만, 한도를 초과한 뒤에도 성공이 허용되므로 OOM 직전의 정리 및 예외 처리 경로에서 불필요한 실패를 줄일 수 있습니다.

ClickHouse 자체의 대형 버퍼(컬럼 데이터, 해시 테이블, 아레나, 쿼리 파이프라인, IO 버퍼)에서 이루어지는 할당은 항상 memory limit을 준수하며, 이 값과 관계없이 예외를 발생시킬 수 있습니다. 이 설정은 `operator new`를 통해 이루어지는 암시적 할당에만 영향을 줍니다.

값이 `0`(기본값)이면 기존 방식을 유지합니다. 즉, 암시적 `operator new` 할당은 메모리 추적기에 의해 절대 거부되지 않으며, 명시적인 ClickHouse 할당만 `MEMORY_LIMIT_EXCEEDED`를 발생시킬 수 있습니다.

부작용을 피하려면 값을 `max_untracked_memory`보다 크게 설정하는 것이 좋습니다.

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

연결을 끊는 것을 고려할 때 OS CPU 대기(`OSCPUWaitMicroseconds` 메트릭) 시간과 사용 중인(`OSCPUVirtualTimeMicroseconds` 메트릭) 시간 사이의 최소 비율입니다. 확률 계산에는 최소 비율과 최대 비율 사이의 선형 보간이 사용되며, 이 지점에서 확률은 0입니다.
자세한 내용은 [서버 CPU 과부하 시 동작 제어](/ko/concepts/features/configuration/settings/server-overload)를 참조하십시오.
