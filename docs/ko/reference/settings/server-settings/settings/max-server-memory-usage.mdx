이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="max_server_memory_usage">
  ## max_server_memory_usage
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

서버가 사용할 수 있는 최대 메모리 양을 바이트 단위로 지정합니다.

<Note>
  서버의 최대 메모리 사용량은 `max_server_memory_usage_to_ram_ratio` 설정에 따라 추가로 제한됩니다.
</Note>

예외적으로 값이 `0`(기본값)이면 서버는 사용 가능한 모든 메모리를 사용할 수 있습니다(`max_server_memory_usage_to_ram_ratio`에 따른 추가 제한은 제외).

<div id="max_server_memory_usage_to_ram_ratio">
  ## max_server_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.9" changeable_without_restart="No" />

서버가 사용할 수 있는 최대 메모리 양을, 전체 가용 메모리에 대한 비율로 나타낸 값입니다.

예를 들어 값이 `0.9`(기본값)이면 서버가 가용 메모리의 90%까지 사용할 수 있음을 의미합니다.

메모리가 적은 시스템에서 메모리 사용량을 줄일 수 있습니다.
RAM과 스왑이 적은 호스트에서는 [`max_server_memory_usage_to_ram_ratio`](/ko/reference/settings/server-settings/settings/max-server-memory-usage#max_server_memory_usage_to_ram_ratio) 설정값을 1보다 크게 지정해야 할 수 있습니다.

이 비율은 두 가지 방식으로 적용됩니다.

* 시작 시(및 구성 다시 로드 시) 위에서 설명한 대로 서버의 하드 메모리 한도를 전체 물리 RAM의 이 비율로 제한합니다.
* 런타임에는 메모리 워커가 하드 메모리 한도를 `(resident memory + system available memory) * max_server_memory_usage_to_ram_ratio`로 주기적으로 다시 계산하므로, 서버는 같은 호스트에서 실행 중인 다른 프로세스를 위한 여유 용량을 남겨 둡니다. 유한한 메모리 한도가 있는 cgroup 내부에서 실행되는 경우에는 호스트 전체 값 대신 cgroup의 가용 메모리가 사용됩니다. 다른 프로세스가 커지고 가용 메모리가 줄어들면 서버의 하드 한도도 함께 낮아져, 호스트(또는 cgroup)의 메모리가 고갈되기 전에 증가를 제한합니다.

비율을 `0`으로 설정하면 시작 시 제한과 런타임 조정이 모두 비활성화됩니다. 또한 런타임 조정은 Linux 이외의 시스템에서는 적용되지 않으며, 이 설정을 노출하지 않는 `clickhouse-keeper`에서도 효과가 없습니다. 정적인 시작/다시 로드 제한은 유지하면서 런타임 조정만 비활성화하려면(이전 버전의 동작 방식) `memory_worker_dynamic_hard_limit`를 `0`으로 설정하십시오.

<Note>
  서버의 최대 메모리 사용량은 `max_server_memory_usage` 설정에 의해 추가로 제한됩니다.
</Note>
