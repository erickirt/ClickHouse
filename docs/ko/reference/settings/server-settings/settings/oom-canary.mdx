이 설정은 ClickHouse 서버 구성에 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="oom_canary_enable">
  ## oom_canary_enable
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

실험적 기능입니다. OOM 카나리를 활성화합니다. OOM 카나리는 주 ClickHouse 서버 프로세스보다
먼저 Linux OOM killer의 대상이 되도록 유도하는 희생용 자식 프로세스로, 서버가 부하를 줄일 시간을 제공합니다.
Linux &gt;= 5.3(`pidfd_open`용)가 필요하며, 더 오래된 커널에서는 시작 시 카나리가 비활성화됩니다.
OOM 대응은 cgroup v2 `memory.events.local`의 OOM-kill 근거를 필요로 하며, 전역
쿼리 취소, 머지 취소, `system.crash_log` 기록을 수행할 수 있습니다.
서버의 cgroup에서 cgroup v2 `memory.oom.group`이 활성화되어 있으면 카나리는 서버를 보호할 수
없습니다. 이 경우 커널이 서버를 포함한 전체 cgroup을 하나의 단위로 종료하므로
OOM 대응이 실행되지 않습니다. 이 모드에서는 시작 시 경고가 기록됩니다.
프로덕션 검증이 완료될 때까지 동작은 ClickHouse 버전 간에 변경될 수 있습니다.

<div id="oom_canary_initial_backoff_seconds">
  ## oom_canary_initial_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="1" changeable_without_restart="No" />

연속된 OOM 카나리 재시작 사이에 적용되는 초기 백오프 지연 시간(초)입니다.
지연 시간은 재시작할 때마다 2배로 늘어나며, 최대 `oom_canary_max_backoff_seconds`까지 증가합니다.

<div id="oom_canary_max_backoff_seconds">
  ## oom_canary_max_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />

연속적인 OOM 카나리 재시작 사이의 최대 백오프 지연 시간(초)입니다.

<div id="oom_canary_max_rapid_relaunches">
  ## oom_canary_max_rapid_relaunches
</div>

<SettingsInfoBlock type="UInt64" default_value="10" changeable_without_restart="No" />

지속적인 메모리 압박 상태에서 스래싱을 방지하기 위해, 자동 재시작이 비활성화되기 전까지 허용되는 연속적인 빠른 OOM 카나리 재시작의 최대 횟수입니다.
카나리가 `oom_canary_max_backoff_seconds`보다 오래 살아남으면 카운터와 재시작 백오프가 재설정되므로, 오랜 가동 시간 동안 카나리가 드물게만 종료되는 경우에는 결국 비활성화되지 않습니다.
`oom_canary_relaunch`가 true인 경우에만 적용됩니다.

<div id="oom_canary_relaunch">
  ## oom_canary_relaunch
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

값이 true이면, 영구적인 설정 실패나 서버 종료를 제외한 어떤 이유로든 카나리 프로세스가 종료된 후 OOM 카나리가 자동으로 다시 시작됩니다. 여기에는 OOM kill, 기타 시그널, 일시적인 종료가 포함되며, `oom_canary_max_rapid_relaunches` 및 백오프 설정의 영향을 받습니다. OOM 응답 시퀀스 자체는 cgroup v2 `memory.events.local`이 OOM kill의 증거를 제공할 때만 실행됩니다.

<div id="oom_canary_size">
  ## oom_canary_size
</div>

<SettingsInfoBlock type="UInt64" default_value="104857600" changeable_without_restart="No" />

OOM 카나리 자식 프로세스가 할당하고 실제로 접근하는 메모리 영역의 크기(바이트)입니다.
`mlock`으로 이 영역을 잠그는 작업은 best-effort 방식으로 수행됩니다. 이를 위해서는 `CAP_IPC_LOCK` 또는 충분한
`RLIMIT_MEMLOCK`이 필요하며, 잠금에 실패하면 카나리가 경고를 기록하고 메모리는 계속
할당된 상태로 유지되지만 스왑될 수 있습니다.
기본값은 100 MB(104857600)입니다. 값이 클수록 카나리가 OOM 대상으로 더 우선 선택될 가능성이 높아집니다.
