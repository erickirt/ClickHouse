이 설정은 ClickHouse 서버를 구성하는 항목이며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="allow_feature_tier">
  ## allow_feature_tier
</div>

<SettingsInfoBlock type="UInt32" default_value="0" changeable_without_restart="Yes" />

사용자가 각 기능 티어에 해당하는 설정을 변경할 수 있는지 제어합니다.

* `0` - 모든 설정 변경이 허용됩니다(실험적, 베타, 프로덕션).
* `1` - 베타 및 프로덕션 기능 설정만 변경할 수 있습니다. 실험적 설정 변경은 거부됩니다.
* `2` - 프로덕션 기능 설정만 변경할 수 있습니다. 실험적 또는 베타 설정 변경은 거부됩니다.

이는 모든 `EXPERIMENTAL` / `BETA` 기능에 `readonly` 제약을 설정하는 것과 같습니다.

<Note>
  값이 `0`이면 모든 설정을 변경할 수 있음을 의미합니다.
</Note>

<div id="allow_impersonate_user">
  ## allow_impersonate_user
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />IMPERSONATE 기능(EXECUTE AS target&#95;user)을 활성화하거나 비활성화합니다. 이 설정은 더 이상 사용을 권장하지 않습니다.

<div id="allow_implicit_no_password">
  ## allow_implicit_no_password
</div>

`IDENTIFIED WITH no_password`를 명시적으로 지정하지 않는 한 비밀번호 없는 사용자의 생성을 금지합니다.

```xml
<allow_implicit_no_password>1</allow_implicit_no_password>
```

<div id="allow_no_password">
  ## allow_no_password
</div>

안전하지 않은 비밀번호 유형인 no&#95;password의 허용 여부를 설정합니다.

```xml
<allow_no_password>1</allow_no_password>
```

<div id="allow_plaintext_password">
  ## allow_plaintext_password
</div>

평문 비밀번호 타입(안전하지 않음)의 허용 여부를 설정합니다.

```xml
<allow_plaintext_password>1</allow_plaintext_password>
```

<div id="allow_use_jemalloc_memory">
  ## allow_use_jemalloc_memory
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />jemalloc 메모리를 사용할 수 있도록 허용합니다.
