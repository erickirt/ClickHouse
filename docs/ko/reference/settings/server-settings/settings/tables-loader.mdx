이 설정은 ClickHouse 서버 구성을 위한 것이며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

백그라운드 풀에서 비동기 로드 작업을 수행하는 스레드 수를 설정합니다. 백그라운드 풀은 서버 시작 후 해당 테이블을 기다리는 쿼리가 없는 경우, 테이블을 비동기로 로드하는 데 사용됩니다. 테이블이 많은 경우에는 백그라운드 풀의 스레드 수를 적게 유지하는 것이 유리할 수 있습니다. 이렇게 하면 동시 쿼리 실행을 위해 CPU 리소스를 확보할 수 있습니다.

<Note>
  `0` 값은 사용 가능한 모든 CPU를 사용함을 의미합니다.
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

포그라운드 풀에서 로드 작업를 수행하는 스레드 수를 설정합니다. 포그라운드 풀은 server가 포트에서 수신 대기를 시작하기 전에 table을 동기적으로 로드할 때와, 대기 중인 table을 로드할 때 사용됩니다. 포그라운드 풀은 백그라운드 풀보다 우선순위가 높습니다. 즉, 포그라운드 풀에서 작업이 실행 중인 동안에는 백그라운드 풀에서 어떤 작업도 시작되지 않습니다.

<Note>
  값이 `0`이면 사용 가능한 모든 CPU를 사용합니다.
</Note>
