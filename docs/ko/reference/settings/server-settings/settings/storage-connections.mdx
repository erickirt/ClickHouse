이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스 코드에서 자동 생성됩니다.

<div id="storage_connections_hard_limit">
  ## storage_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />이 한도에 도달한 상태에서 생성을 시도하면 예외가 발생합니다. 하드 제한을 비활성화하려면 0으로 설정하십시오. 이 한도는 스토리지 연결에 적용됩니다.

<div id="storage_connections_rcvbuf">
  ## storage_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />스토리지 연결(복제, 분산 쿼리)의 `SO_RCVBUF` 옵션 크기입니다. 0보다 큰 값으로 설정하면 수신 버퍼에 대한 커널 TCP 자동 튜닝을 재정의합니다. 0 = 커널 기본값(자동 튜닝)입니다. 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에만 자동 튜닝이 복원되며, 기존 풀 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="storage_connections_sndbuf">
  ## storage_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />스토리지 연결(복제, 분산 쿼리)의 `SO_SNDBUF` 옵션 크기입니다. 0보다 큰 값으로 설정하면 전송 버퍼에 대한 커널 TCP 자동 튜닝을 재정의합니다. 0 = 커널 기본값(자동 튜닝). 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에만 자동 튜닝이 복원되며, 기존 풀 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="storage_connections_soft_limit">
  ## storage_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />이 한도를 초과하는 연결은 유지 시간이 크게 짧아집니다. 이 한도는 스토리지 연결에 적용됩니다.

<div id="storage_connections_store_limit">
  ## storage_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />이 제한을 초과하는 연결은 사용 후 초기화됩니다. 연결 캐시를 비활성화하려면 0으로 설정하십시오. 이 제한은 스토리지 연결에 적용됩니다.

<div id="storage_connections_warn_limit">
  ## storage_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />사용 중인 연결 수가 이 한도를 초과하면 Warning 메시지가 로그에 기록됩니다. 이 한도는 스토리지 연결에 적용됩니다.
