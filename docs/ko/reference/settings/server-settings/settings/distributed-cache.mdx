이 설정은 ClickHouse 서버를 구성하기 위한 항목이며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="distributed_cache_apply_throttling_settings_from_client">
  ## distributed_cache_apply_throttling_settings_from_client
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />캐시 서버가 클라이언트로부터 전달받은 스로틀링 설정을 적용할지 여부입니다.

<div id="distributed_cache_keep_up_free_connections_ratio">
  ## distributed_cache_keep_up_free_connections_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.1" changeable_without_restart="No" />Distributed Cache가 유휴 상태로 유지하려는 활성 연결 수의 소프트 리밋입니다. 사용 가능한 연결 수가 distributed&#95;cache&#95;keep&#95;up&#95;free&#95;connections&#95;ratio * max&#95;connections 아래로 내려가면, 연결 수가 다시 이 리밋을 넘을 때까지 가장 오래 활동이 없었던 연결부터 닫힙니다.

<div id="distributed_cache_write_pool_size">
  ## distributed_cache_write_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />동시에 백그라운드 스레드에서 실행할 수 있는 Distributed Cache 쓰기 요청의 최대 수입니다(모든 쿼리 전체 기준). 한도에 도달하면 쓰기 작업은 백그라운드 스레드가 아니라 캐시를 통해 인라인으로(호출한 스레드에서) 처리되므로, 슬롯을 기다리며 차단되지 않고 스레드 수가 무제한으로 늘어나지도 않습니다.
