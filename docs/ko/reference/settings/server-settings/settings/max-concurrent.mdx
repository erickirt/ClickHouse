이 설정은 ClickHouse 서버 구성에 사용되며 ClickHouse 소스에서 자동 생성됩니다.

<div id="max_concurrent_insert_queries">
  ## max_concurrent_insert_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

동시에 실행할 수 있는 전체 삽입 쿼리 수의 제한입니다.

<Note>
  값이 `0`(기본값)이면 제한이 없음을 의미합니다.

  이 설정은 런타임에 수정할 수 있으며 즉시 적용됩니다. 이미 실행 중인 쿼리에는 영향을 주지 않습니다.
</Note>

<div id="max_concurrent_queries">
  ## max_concurrent_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

동시에 실행할 수 있는 전체 쿼리 수에 대한 제한입니다. `INSERT` 및 `SELECT` 쿼리의 제한과 사용자별 최대 쿼리 수 제한도 함께 고려해야 합니다.

관련 항목:

* [`max_concurrent_insert_queries`](/ko/reference/settings/server-settings/settings/max-concurrent#max_concurrent_insert_queries)
* [`max_concurrent_select_queries`](/ko/reference/settings/server-settings/settings/max-concurrent#max_concurrent_select_queries)
* [`max_concurrent_queries_for_all_users`](/ko/reference/settings/session-settings#max_concurrent_queries_for_all_users)

<Note>
  값이 `0`이면(기본값) 제한이 없음을 의미합니다.

  이 설정은 런타임에 수정할 수 있으며 즉시 적용됩니다. 이미 실행 중인 쿼리에는 영향을 주지 않습니다.
</Note>

<div id="max_concurrent_select_queries">
  ## max_concurrent_select_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

동시에 실행되는 SELECT 쿼리의 총 개수를 제한합니다.

<Note>
  값이 `0`(기본값)이면 제한이 없음을 의미합니다.

  이 설정은 런타임에 수정할 수 있으며 즉시 적용됩니다. 이미 실행 중인 쿼리는 변경되지 않습니다.
</Note>
