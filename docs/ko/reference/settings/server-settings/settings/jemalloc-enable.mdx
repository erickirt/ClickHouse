이 설정은 ClickHouse 서버 구성에 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="jemalloc_enable_background_threads">
  ## jemalloc_enable_background_threads
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />jemalloc의 백그라운드 스레드를 활성화합니다. Jemalloc은 사용하지 않는 메모리 페이지를 정리하기 위해 백그라운드 스레드를 사용합니다. 이 기능을 비활성화하면 성능이 저하될 수 있습니다.

<div id="jemalloc_enable_global_profiler">
  ## jemalloc_enable_global_profiler
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />모든 스레드에 대해 jemalloc의 할당 프로파일러를 활성화합니다. Jemalloc은 샘플링된 할당과 해당 샘플링된 할당의 모든 메모리 해제를 샘플링합니다.
프로파일은 SYSTEM JEMALLOC FLUSH PROFILE을 사용해 플러시할 수 있으며, 할당 분석에 활용할 수 있습니다.
샘플은 구성 jemalloc&#95;collect&#95;global&#95;profile&#95;samples&#95;in&#95;trace&#95;log 또는 쿼리 설정 jemalloc&#95;collect&#95;profile&#95;samples&#95;in&#95;trace&#95;log를 사용해 system.trace&#95;log에 저장할 수도 있습니다.
[할당 프로파일링](/ko/concepts/features/performance/allocation-profiling)을 참조하십시오.
