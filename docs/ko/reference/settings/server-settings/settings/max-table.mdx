이 설정은 ClickHouse 서버를 구성하며, ClickHouse 소스에서 자동 생성됩니다.

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

테이블 수가 이 값보다 크면 서버는 예외를 발생시킵니다.

다음 테이블은 개수에 포함되지 않습니다:

* view
* remote
* dictionary
* system

다음 데이터베이스 엔진의 테이블만 개수에 포함됩니다:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  값이 `0`이면 제한이 없음을 의미합니다.
</Note>

**예시**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

attached 상태인 테이블의 수가 지정된 값을 초과하면 ClickHouse 서버가 `system.warnings` 테이블에 경고 메시지를 추가합니다.

**예시**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

테이블 삭제를 제한하는 설정입니다.

[MergeTree](/ko/reference/engines/table-engines/mergetree-family/mergetree) 테이블의 크기가 `max_table_size_to_drop`(바이트 단위)를 초과하면 [`DROP`](/ko/reference/statements/drop) 쿼리나 [`TRUNCATE`](/ko/reference/statements/truncate) 쿼리로는 삭제할 수 없습니다.

<Note>
  `0` 값은 아무런 제한 없이 모든 테이블을 삭제할 수 있음을 의미합니다.

  이 설정은 적용하기 위해 ClickHouse 서버를 재시작할 필요가 없습니다. 이 제한을 비활성화하는 다른 방법으로는 `<clickhouse-path>/flags/force_drop_table` 파일을 생성하는 방법이 있습니다.
</Note>

**예시**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
