이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="memory_worker_correct_memory_tracker">
  ## memory_worker_correct_memory_tracker
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

메모리 워커가 jemalloc 및 cgroups와 같은 외부 소스의 정보를 바탕으로 내부 메모리 트래커를 보정할지 여부입니다

<div id="memory_worker_decay_adjustment_period_ms">
  ## memory_worker_decay_adjustment_period_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />

메모리 압박 상태가 지속될 때 jemalloc의 `dirty_decay_ms`를 동적으로 조정하기 전까지 경과해야 하는 시간(밀리초)입니다. 메모리 사용량이 이 기간 동안 계속 purge 임계값을 초과하면 메모리를 적극적으로 회수하기 위해 자동 dirty page decay가 비활성화됩니다(`dirty_decay_ms=0`). 사용량이 이 기간 동안 계속 임계값 아래에 머물면 기본 decay 동작이 복원됩니다. 동적 조정을 비활성화하고 jemalloc의 기본 decay 설정을 사용하려면 0으로 설정하세요.

<div id="memory_worker_dynamic_hard_limit">
  ## memory_worker_dynamic_hard_limit
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

메모리 워커가 런타임에 `(resident memory + system available memory) * max_server_memory_usage_to_ram_ratio`를 기준으로 서버의 하드 메모리 한도를 주기적으로 다시 계산할지 여부입니다. 이렇게 하면 동일한 호스트에서 실행 중인 다른 프로세스를 위해 서버가 여유 용량을 남겨 둘 수 있습니다.

비활성화하면 `max_server_memory_usage_to_ram_ratio`는 이전 버전과 마찬가지로 시작 시점과 구성 다시 로드 시점에만 하드 메모리 한도를 정적으로 제한합니다.

`max_server_memory_usage_to_ram_ratio`가 `0`이면 이 설정은 아무런 영향을 미치지 않습니다.

<div id="memory_worker_period_ms">
  ## memory_worker_period_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

메모리 트래커가 기록한 메모리 사용량을 보정하고, 메모리 사용량이 높을 때 사용되지 않는 페이지를 정리하는 메모리 워커의 틱 주기입니다. 0으로 설정하면 메모리 사용량의 출처에 따라 기본값이 사용됩니다

<div id="memory_worker_purge_dirty_pages_threshold_ratio">
  ## memory_worker_purge_dirty_pages_threshold_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.2" changeable_without_restart="No" />

ClickHouse 서버에서 사용할 수 있는 메모리를 기준으로 한 jemalloc 더티 페이지(dirty pages)의 임계값 비율입니다. 더티 페이지 크기가 이 비율을 초과하면 메모리 워커가 더티 페이지 purge를 강제로 수행합니다. 0으로 설정하면 더티 페이지 비율 기준의 강제 purge가 비활성화됩니다.

<div id="memory_worker_purge_total_memory_threshold_ratio">
  ## memory_worker_purge_total_memory_threshold_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.9" changeable_without_restart="No" />

ClickHouse 서버에서 사용 가능한 메모리 대비 jemalloc purge를 수행하는 임계값 비율입니다. 전체 메모리 사용량이 이 비율을 초과하면 메모리 워커가 dirty pages를 강제로 purge합니다. 0으로 설정하면 전체 메모리를 기준으로 하는 강제 purge가 비활성화됩니다.

<div id="memory_worker_rss_speculative_reserve_ratio">
  ## memory_worker_rss_speculative_reserve_ratio
</div>

<SettingsInfoBlock type="Double" default_value="1" changeable_without_restart="No" />

각 `MemoryWorker` 틱마다, 다음 틱에서도 직전 틱과 같은
증가가 발생할 수 있다고 가정하고, 관측된 RSS에
`ratio * min(resident - previous_resident, resident - tracked)`를 추가로
예약합니다(`resident - previous_resident`는 직전 틱 동안의 RSS 증가량입니다).
이 증가분의 상한은 `resident - tracked`로 제한되는데, 이는 전역 메모리 트래커에
포착되지 않는 RSS 부분이기 때문입니다. 이미 추적된 증가분은 일반적인
하드 제한 검사에서 처리됩니다. 이 예약은 전역 하드 제한 검사가
`MemoryTracker::allocImpl`을 통해 참조하는 `rss` 카운터에 적용되므로,
외삽된 값이 `max_server_memory_usage`를 초과하면 이후 할당은 커널 OOM-killer가
동작하기 전에 `MEMORY_LIMIT_EXCEEDED`를 발생시킵니다. 값이 `0`이면
추정 기능이 비활성화되어(`rss = resident`로 되돌아감), 기본값 `1`은 다음
인터벌을 위해 직전 증가분만큼의 headroom을 예약합니다. 새니타이저
환경(`ASan`, `UBSan`, `MSan`, `TSan`)에서는 기본값이 `0`인데, 이는
`resident - tracked` 간극이 tracker의 bookkeeping 지연보다 새니타이저의 shadow /
런타임 오버헤드의 영향을 더 크게 받기 때문입니다.
