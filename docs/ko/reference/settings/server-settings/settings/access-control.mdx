이 설정은 ClickHouse 서버를 구성하는 항목이며 ClickHouse 소스에서 자동 생성됩니다.

<div id="access_control_improvements">
  ## access_control_improvements
</div>

접근 제어 시스템의 선택적 개선 사항에 대한 설정입니다.

| Setting                                         | Description                                                                                                                                                                                                                                                                                                                                                                                          | Default |
| ----------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------- |
| `on_cluster_queries_require_cluster_grant`      | `ON CLUSTER` 쿼리에 `CLUSTER` 권한이 필요한지 설정합니다.                                                                                                                                                                                                                                                                                                                                                           | `true`  |
| `role_cache_expiration_time_seconds`            | 마지막으로 접근한 시점부터 역할이 Role Cache에 저장되는 시간을 초 단위로 설정합니다.                                                                                                                                                                                                                                                                                                                                                 | `600`   |
| `select_from_information_schema_requires_grant` | `SELECT * FROM information_schema.<table>` 실행에 권한이 필요한지, 또는 모든 사용자가 실행할 수 있는지 설정합니다. true로 설정하면 이 쿼리를 실행하려면 일반 테이블과 마찬가지로 `GRANT SELECT ON information_schema.<table>`가 필요합니다.                                                                                                                                                                                                                       | `true`  |
| `select_from_system_db_requires_grant`          | `SELECT * FROM system.<table>` 실행에 권한이 필요한지, 또는 모든 사용자가 실행할 수 있는지 설정합니다. true로 설정하면 이 쿼리를 실행하려면 일반 테이블이 아닌 테이블이 아니라 일반 테이블과 마찬가지로 `GRANT SELECT ON system.<table>`가 필요합니다. 예외적으로 일부 system tables(`tables`, `columns`, `databases`, 그리고 `one`, `contributors` 같은 일부 상수 테이블)는 계속 모든 사용자가 접근할 수 있습니다. 또한 `SHOW` privilege(예: `SHOW USERS`)가 권한 부여된 경우 해당 system table(즉, `system.users`)에도 접근할 수 있습니다. | `true`  |
| `settings_constraints_replace_previous`         | 특정 설정에 대한 settings profile의 제약 조건이, 해당 설정에 대해 이전 제약 조건(다른 profile에 정의됨)의 동작을 새 제약 조건에 설정되지 않은 field까지 포함해 취소할지 설정합니다. 또한 `changeable_in_readonly` 제약 조건 유형도 활성화합니다.                                                                                                                                                                                                                                  | `true`  |
| `table_engines_require_grant`                   | 특정 테이블 엔진으로 테이블을 생성할 때 권한이 필요한지 설정합니다.                                                                                                                                                                                                                                                                                                                                                               | `false` |
| `throw_on_unmatched_row_policies`               | 테이블에 행 정책이 있지만 현재 사용자에 해당하는 정책이 하나도 없을 경우, 테이블을 읽을 때 예외를 발생시킬지 설정합니다.                                                                                                                                                                                                                                                                                                                                | `false` |
| `users_without_row_policies_can_read_rows`      | 허용형 행 정책이 없는 사용자도 `SELECT` 쿼리로 행을 계속 읽을 수 있는지 설정합니다. 예를 들어 사용자 A와 B가 있고 행 정책이 A에 대해서만 정의되어 있다면, 이 설정이 true일 때 사용자 B는 모든 행을 볼 수 있습니다. 이 설정이 false이면 사용자 B는 어떤 행도 볼 수 없습니다.                                                                                                                                                                                                                            | `true`  |

예시:

```xml
<access_control_improvements>
    <throw_on_unmatched_row_policies>true</throw_on_unmatched_row_policies>
    <users_without_row_policies_can_read_rows>true</users_without_row_policies_can_read_rows>
    <on_cluster_queries_require_cluster_grant>true</on_cluster_queries_require_cluster_grant>
    <select_from_system_db_requires_grant>true</select_from_system_db_requires_grant>
    <select_from_information_schema_requires_grant>true</select_from_information_schema_requires_grant>
    <settings_constraints_replace_previous>true</settings_constraints_replace_previous>
    <table_engines_require_grant>false</table_engines_require_grant>
    <role_cache_expiration_time_seconds>600</role_cache_expiration_time_seconds>
</access_control_improvements>
```

<div id="access_control_path">
  ## access_control_path
</div>

ClickHouse 서버가 SQL 명령으로 생성된 사용자 및 역할 구성을 저장하는 폴더 경로입니다.
