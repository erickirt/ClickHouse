이 설정은 ClickHouse 서버를 구성하며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="jemalloc_collect_global_profile_samples_in_trace_log">
  ## jemalloc_collect_global_profile_samples_in_trace_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />jemalloc이 샘플링한 메모리 할당 정보를 system.trace&#95;log에 저장합니다

<div id="jemalloc_max_background_threads_num">
  ## jemalloc_max_background_threads_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="아니요" />생성할 jemalloc 백그라운드 스레드의 최대 수입니다. 0으로 설정하면 jemalloc의 기본값이 사용됩니다

<div id="jemalloc_profiler_sampling_rate">
  ## jemalloc_profiler_sampling_rate
</div>

<SettingsInfoBlock type="UInt64" default_value="19" changeable_without_restart="No" />

jemalloc의 `lg_prof_sample`을 제어합니다. 이는 할당 샘플 사이의 평균 인터벌(바이트 단위)에 대한 밑수 2 로그값입니다.
기본값 19는 512 KiB에 해당합니다. 더 작은 값으로 설정하면 샘플링 빈도가 증가하여(오버헤드는 늘어나지만 더 자세한 정보를 제공함) 더 큰 값으로 설정했을 때보다 더 자주 샘플링됩니다. 반대로 더 큰 값으로 설정하면 샘플링 빈도가 감소합니다.
이 값을 변경하면 `prof.reset`이 호출되어 누적된 모든 프로파일링 통계가 재설정됩니다. 프로파일링이 활성화되어 있어야 합니다(`MALLOC_CONF=prof:true`).
