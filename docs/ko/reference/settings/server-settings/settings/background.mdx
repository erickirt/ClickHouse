이 설정은 ClickHouse 서버를 구성하는 데 사용되며 ClickHouse 소스에서 자동으로 생성됩니다.

<div id="background_buffer_flush_schedule_pool_size">
  ## background_buffer_flush_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="증가만 가능" />백그라운드에서 [Buffer 엔진 테이블](/ko/reference/engines/table-engines/special/buffer)의 플러시 작업을 수행할 때 사용하는 최대 스레드 수입니다.

<div id="background_common_pool_size">
  ## background_common_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />백그라운드에서 [*MergeTree 엔진](/ko/reference/engines/table-engines/mergetree-family/index) 테이블의 다양한 작업(주로 가비지 컬렉션)을 수행할 때 사용할 수 있는 최대 스레드 수입니다.

<div id="background_distributed_schedule_pool_size">
  ## background_distributed_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="증가만 가능" />분산 전송 실행에 사용할 최대 스레드 수입니다.

<div id="background_fetches_pool_size">
  ## background_fetches_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />백그라운드에서 [*MergeTree 엔진](/ko/reference/engines/table-engines/mergetree-family/index) 테이블의 다른 레플리카로부터 데이터 파트를 가져오는 데 사용할 수 있는 최대 스레드 수입니다.

<div id="background_message_broker_schedule_pool_size">
  ## background_message_broker_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="증가만 가능" />메시지 스트리밍용 백그라운드 작업 실행에 사용할 수 있는 최대 스레드 수입니다.

<div id="background_move_pool_size">
  ## background_move_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />백그라운드에서 *MergeTree 엔진 테이블의 데이터 파트를 다른 디스크나 볼륨으로 이동할 때 사용하는 최대 스레드 수입니다.

<div id="background_pool_size">
  ## background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />

MergeTree 엔진을 사용하는 테이블에서 백그라운드 머지와 뮤테이션을 수행하는 스레드 수를 설정합니다.

<Note>
  * 이 설정은 이전 버전과의 호환성을 위해 ClickHouse 서버 시작 시 `default` 프로필 구성으로도 적용할 수 있습니다.
  * 런타임에는 스레드 수를 늘릴 수만 있습니다.
  * 스레드 수를 줄이려면 서버를 다시 시작해야 합니다.
  * 이 설정을 조정하면 CPU 및 디스크 부하를 관리할 수 있습니다.
</Note>

<Danger>
  풀 크기가 작을수록 CPU 및 디스크 리소스 사용량은 줄어들지만, 백그라운드 프로세스가 더 느리게 진행되어 결국 쿼리 성능에 영향을 줄 수 있습니다.
</Danger>

변경하기 전에 다음과 같은 관련 MergeTree 설정도 확인하십시오.

* [`number_of_free_entries_in_pool_to_lower_max_size_of_merge`](/ko/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_lower_max_size_of_merge).
* [`number_of_free_entries_in_pool_to_execute_mutation`](/ko/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_mutation).
* [`number_of_free_entries_in_pool_to_execute_optimize_entire_partition`](/ko/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_optimize_entire_partition)

**예시**

```xml
<background_pool_size>16</background_pool_size>
```
