이 설정은 ClickHouse 서버 구성을 위한 것이며, ClickHouse 소스에서 자동 생성됩니다.

<div id="listen_backlog">
  ## listen_backlog
</div>

<SettingsInfoBlock type="UInt32" default_value="4096" changeable_without_restart="No" />

listen socket의 백로그(대기 중인 연결의 큐 크기)입니다. 기본값 `<4096>`은 Linux 5.4+의 기본값과 동일합니다).

일반적으로는 다음과 같은 이유로 이 값을 변경할 필요가 없습니다.

* 기본값이 충분히 크고,
* 서버에는 클라이언트 연결을 수락하기 위한 별도의 thread가 있습니다.

따라서 `<TcpExtListenOverflows>`(`<nstat>`에서 확인 가능) 값이 0이 아니고 ClickHouse 서버에서 이 카운터가 증가하더라도, 그렇다고 해서 이 값을 늘려야 한다는 뜻은 아닙니다. 이유는 다음과 같습니다.

* 일반적으로 `<4096>`으로 충분하지 않다면 ClickHouse 내부의 스케일링 문제를 의미하므로, issue를 보고하는 편이 더 낫습니다.
* 또한 이것이 서버가 나중에 더 많은 연결을 처리할 수 있음을 뜻하지도 않습니다(설령 가능하더라도, 그 시점에는 클라이언트가 이미 사라졌거나 연결이 끊겼을 수 있습니다).

**예시**

```xml
<listen_backlog>4096</listen_backlog>
```

<div id="listen_host">
  ## listen_host
</div>

요청을 받을 수 있는 호스트를 제한합니다. 서버가 모든 호스트의 요청에 응답하게 하려면 `::`를 지정하십시오.

예시:

```xml
<listen_host>::1</listen_host>
<listen_host>127.0.0.1</listen_host>
```

<div id="listen_reuse_port">
  ## listen_reuse_port
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

여러 서버가 동일한 address:port에서 수신 대기하도록 허용합니다. 요청은 운영 체제가 임의의 서버로 라우팅합니다. 이 설정을 활성화하는 것은 권장되지 않습니다.

**예시**

```xml
<listen_reuse_port>0</listen_reuse_port>
```

<div id="listen_try">
  ## listen_try
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

수신 대기(listen)를 시도하는 동안 IPv6 또는 IPv4 네트워크를 사용할 수 없더라도 서버는 종료되지 않습니다.

**예시**

```xml
<listen_try>0</listen_try>
```
