이 설정은 ClickHouse 서버를 구성하며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="http_connections_hard_limit">
  ## http_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />이 제한에 도달한 상태에서 생성을 시도하면 예외가 발생합니다. 하드 제한을 끄려면 0으로 설정하십시오. 이 제한은 어떤 디스크나 스토리지에도 속하지 않는 HTTP 연결에 적용됩니다.

<div id="http_connections_rcvbuf">
  ## http_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />일반 HTTP 연결의 `SO_RCVBUF` 옵션 크기입니다. 0보다 큰 값으로 설정하면 수신 버퍼에 대한 커널 TCP 자동 튜닝을 재정의합니다. 0 = 커널 기본값(자동 튜닝). 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에 대해서만 자동 튜닝이 복원되며, 기존 연결 풀의 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="http_connections_sndbuf">
  ## http_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />일반 HTTP 연결의 `SO_SNDBUF` 옵션 크기입니다. 0보다 큰 값으로 설정하면 송신 버퍼에 대한 커널 TCP 자동 튜닝을 재정의합니다. 0 = 커널 기본값(자동 튜닝)입니다. 참고: 이 설정을 다시 0으로 변경하면 새로 생성되는 연결에 대해서만 자동 튜닝이 복원되며, 기존 연결 풀의 연결은 다시 생성될 때까지 고정된 버퍼 크기를 유지합니다.

<div id="http_connections_soft_limit">
  ## http_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />이 한도를 초과하는 연결은 수명이 현저히 짧아집니다. 이 한도는 어떤 디스크나 스토리지에도 속하지 않는 HTTP 연결에 적용됩니다.

<div id="http_connections_store_limit">
  ## http_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />이 한도를 초과하는 연결은 사용 후 재설정됩니다. 0으로 설정하면 연결 캐시가 꺼집니다. 이 한도는 어떤 디스크나 스토리지에도 속하지 않는 HTTP 연결에 적용됩니다.

<div id="http_connections_warn_limit">
  ## http_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />사용 중인 연결 수가 이 한도보다 많으면 로그에 경고 메시지가 기록됩니다. 이 한도는 어떤 디스크나 스토리지에도 속하지 않는 HTTP 연결에 적용됩니다.
