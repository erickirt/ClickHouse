<div id="format_display_secrets_in_show_and_select">
  ## format_display_secrets_in_show_and_select
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

테이블, 데이터베이스, 테이블 함수 및 Dictionaries에 대해 `SHOW` 및 `SELECT` 쿼리에서 시크릿을 표시할지 여부를 설정합니다.

시크릿을 보려면 다음 조건도 충족해야 합니다.
[`display_secrets_in_show_and_select` server setting](/ko/reference/settings/server-settings/settings#display_secrets_in_show_and_select)이 활성화되어 있어야 하며,
[`displaySecretsInShowAndSelect`](/ko/reference/statements/grant#displaysecretsinshowandselect) 권한도 있어야 합니다.

가능한 값:

* 0 — 비활성화됨.
* 1 — 활성화됨.

  ## format_geojson_validate_geometry
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "GeoJSON 포맷이 읽기 및 쓰기 시 RFC 7946 지오메트리 유효성(선 및 링의 최소 Point 수, 링 폐쇄, 비어 있지 않은 다중 지오메트리)을 검사할지 여부를 제어하는 새로운 설정"}]}]} />

`GeoJSON` 포맷이 읽기와 쓰기 양쪽에서 RFC 7946 지오메트리 유효성을 검사할지 여부를 제어합니다.

활성화되면(기본값) GeoJSON 도형 규칙을 위반하는 지오메트리는 거부됩니다. 예를 들어 Point가 2개 미만인 `LineString`(또는 `MultiLineString`의 선), Point가 4개 미만이거나 첫 번째 Point와 마지막 Point가 다른(닫히지 않은 링) `Polygon` 또는 `MultiPolygon`의 링, 또는 비어 있는 `MultiLineString`, `Polygon`, `MultiPolygon`이 여기에 해당합니다. 이는 읽기 시(이러한 문서는 거부됨)와 쓰기 시 모두 적용됩니다(입력 형식이 거부할 문서를 생성하는 대신 이러한 ClickHouse 값을 거부함).

비활성화되면 이러한 도형 규칙은 적용되지 않습니다. 즉, 이러한 지오메트리는 있는 그대로 읽고 그대로 기록되므로 퇴화된 지오메트리도 round-trip되지만, 기록된 문서는 유효한 GeoJSON이 아닐 수 있습니다.

검사는 구조만 대상으로 합니다. 즉, Point 수와 링 폐쇄만 확인합니다. 도형의 기하학적 올바름은 검사하지 않습니다. 링 방향(오른손 법칙 / winding order)은 강제되지 않으며, 구조적으로는 유효하지만 기하학적으로는 퇴화된 지오메트리(예: 면적이 0인 다각형, 자기 교차 링, 또는 hole이 외곽 링 바깥에 있는 다각형)도 허용됩니다. 유한하지 않은 좌표(`NaN`, `Inf`)는 JSON 숫자로 표현할 수 없으므로 이 설정과 관계없이 항상 거부됩니다.

<div id="format_json_object_each_row_column_for_object_name">
  ## format_json_object_each_row_column_for_object_name
</div>

[JSONObjectEachRow](/ko/reference/formats/JSON/JSONObjectEachRow) 포맷에서 객체 이름을 저장하거나 기록하는 데 사용할 컬럼 이름입니다.
컬럼 타입은 String이어야 합니다. 값이 비어 있으면 객체 이름으로 기본 이름 `row_{i}`가 사용됩니다.
