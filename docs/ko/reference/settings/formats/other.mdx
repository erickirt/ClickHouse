<div id="allow_special_bool_values_inside_variant">
  ## allow_special_bool_values_inside_variant
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

&quot;on&quot;, &quot;off&quot;, &quot;enable&quot;, &quot;disable&quot; 등의 특수한 텍스트 Bool 값에서 Variant 유형 내부의 Bool 값을 파싱할 수 있도록 합니다.

<div id="check_conversion_from_numbers_to_enum">
  ## check_conversion_from_numbers_to_enum
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

값이 Enum에 없으면 Numbers에서 Enum으로 변환할 때 예외를 발생시킵니다.

가능한 값:

* 0 — 비활성화됨.
* 1 — 활성화됨.

**예시**

```text
CREATE TABLE tab (
  val Enum('first' = 1, 'second' = 2, 'third' = 3)
) ENGINE = Memory;

INSERT INTO tab SETTINGS check_conversion_from_numbers_to_enum = 1 VALUES (4); -- returns an error
```

<div id="insert_distributed_one_random_shard">
  ## insert_distributed_one_random_shard
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

분산 키가 없을 때 [Distributed](/ko/reference/engines/table-engines/special/distributed) 테이블에 데이터를 임의의 세그먼트로 삽입할지 여부를 설정합니다.

기본적으로 세그먼트가 2개 이상인 `Distributed` 테이블에 데이터를 삽입할 때 분산 키가 없으면 ClickHouse 서버는 삽입 요청을 거부합니다. `insert_distributed_one_random_shard = 1`이면 삽입이 허용되며 데이터는 모든 세그먼트 중 하나로 무작위 전달됩니다.

가능한 값:

* 0 — 세그먼트가 여러 개이고 분산 키가 지정되지 않으면 삽입이 거부됩니다.
* 1 — 분산 키가 지정되지 않으면 사용 가능한 모든 세그먼트 중 하나에 무작위로 삽입됩니다.

<div id="interval_output_format">
  ## interval_output_format
</div>

<SettingsInfoBlock type="IntervalOutputFormat" default_value="numeric" />

인터벌 유형의 텍스트 표현에 사용할 출력 형식을 선택할 수 있습니다.

가능한 값:

* `kusto` - KQL 스타일 출력 형식입니다.

  ClickHouse는 인터벌을 [KQL 형식](https://learn.microsoft.com/en-us/dotnet/standard/base-types/standard-timespan-format-strings#the-constant-c-format-specifier)으로 출력합니다. 예를 들어 `toIntervalDay(2)`는 `2.00:00:00`으로 표시됩니다. `IntervalMonth` 및 `IntervalYear`처럼 길이가 가변적인 인터벌 유형은 인터벌당 평균 초 수를 기준으로 처리된다는 점에 유의하십시오.

* `numeric` - 숫자 출력 형식입니다.

  ClickHouse는 인터벌을 내부 숫자 표현으로 출력합니다. 예를 들어 `toIntervalDay(2)`는 `2`로 표시됩니다.

관련 항목:

* [Interval](/ko/reference/data-types/special-data-types/interval)

<div id="max_dynamic_subcolumns_in_json_type_parsing">
  ## max_dynamic_subcolumns_in_json_type_parsing
</div>

<SettingsInfoBlock type="UInt64Auto" default_value="auto" />

JSON 컬럼을 파싱하는 동안 각 컬럼별로 생성할 수 있는 동적 서브컬럼의 최대 개수입니다.

<div id="precise_float_parsing">
  ## precise_float_parsing
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

입력값에 대해 항상 표현 가능한 값 중 가장 가까운 값을 반환하는 정밀한 부동소수점 파싱 알고리즘을 사용합니다. 비활성화하면 더 빠르지만 정확도는 낮은 알고리즘이 사용되며, 정밀한 결과와 최하위 비트에서 차이가 날 수 있습니다.
