<div id="max_parts_in_total">
  ## max_parts_in_total
</div>

<SettingsInfoBlock type="UInt64" default_value="100000" />

테이블의 모든 파티션에 있는 활성 파트의 총개수가
`max_parts_in_total` 값을 초과하면 `INSERT`가
`Too many parts
(N)` 예외와 함께 중단됩니다.

Possible values:

* 임의의 양의 정수.

테이블에 파트가 너무 많으면 ClickHouse 쿼리 성능이 저하되고
ClickHouse 시작 시간이 늘어납니다. 대부분의 경우 이는
설계가 잘못되었기 때문입니다(파티셔닝 전략을 잘못 선택해
파티션이 너무 작게 나뉜 경우).

<div id="max_parts_to_merge_at_once">
  ## max_parts_to_merge_at_once
</div>

<SettingsInfoBlock type="UInt64" default_value="100" />

한 번에 머지할 수 있는 파트의 최대 개수입니다(0이면 비활성화됨). `OPTIMIZE FINAL` 쿼리에는
영향이 없습니다.
