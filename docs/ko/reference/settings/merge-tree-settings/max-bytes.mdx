<div id="max_bytes_to_merge_at_max_space_in_pool">
  ## max_bytes_to_merge_at_max_space_in_pool
</div>

<SettingsInfoBlock type="UInt64" default_value="161061273600" />

사용 가능한 리소스가 충분할 때 여러 파트를 하나의 파트로 머지할 수 있는 최대 총 파트 크기(바이트)입니다. 자동 백그라운드 머지로 생성될 수 있는 최대 파트 크기에 대략 해당합니다. (0은 머지가 비활성화됨을 의미합니다)

Possible values:

* 0 이상의 모든 정수.

머지 스케줄러는 주기적으로 파티션 내 파트의 크기와 개수를 분석하고, 풀에 충분한 여유 리소스가 있으면 백그라운드 머지를 시작합니다. 머지는 소스 파트들의 총 크기가 `max_bytes_to_merge_at_max_space_in_pool`보다 커질 때까지 수행됩니다.

[OPTIMIZE FINAL](/ko/reference/statements/optimize)로 시작된 머지는 `max_bytes_to_merge_at_max_space_in_pool`를 무시합니다(여유 디스크 공간만 고려됩니다).

<div id="max_bytes_to_merge_at_min_space_in_pool">
  ## max_bytes_to_merge_at_min_space_in_pool
</div>

<SettingsInfoBlock type="UInt64" default_value="1048576" />

background 풀에서 사용 가능한 리소스가 최소일 때,
하나의 파트로 머지할 수 있는 최대 총 파트 크기(바이트)입니다.

가능한 값:

* 양의 정수 아무 값이나 가능합니다.

`max_bytes_to_merge_at_min_space_in_pool`는 디스크 공간이 부족한 경우에도
머지할 수 있는 파트의 최대 총 크기(풀 내)를 정의합니다.
이는 작은 파트 수를 줄이고 `Too many parts` 오류가 발생할 가능성을 낮추기 위해 필요합니다.
머지 작업은 머지되는 전체 파트 크기의 2배에 해당하는 디스크 공간을 예약합니다.
따라서 디스크의 여유 공간이 적으면,
여유 공간이 있더라도 그 공간이 이미 진행 중인 큰 머지 작업에 예약되어 있어
다른 머지 작업을 시작할 수 없고, 삽입할 때마다 작은 파트 수가 늘어나는
상황이 발생할 수 있습니다.
