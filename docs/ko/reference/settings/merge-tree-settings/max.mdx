이러한 설정은 [system.merge&#95;tree&#95;settings](/ko/reference/system-tables/merge_tree_settings)에서 확인할 수 있으며, ClickHouse 소스에서 자동 생성됩니다.

<div id="max_avg_part_size_for_too_many_parts">
  ## max_avg_part_size_for_too_many_parts
</div>

<SettingsInfoBlock type="UInt64" default_value="1073741824" />

&#39;parts&#95;to&#95;delay&#95;insert&#39; 및
&#39;parts&#95;to&#95;throw&#95;insert&#39;에 따른 &#39;too many parts&#39; 검사는 해당
파티션의 평균 파트 크기가 지정된 임계값보다 크지 않은 경우에만 활성화됩니다. 평균
파트 크기가 지정된 임계값보다 크면 INSERT는 지연되거나 거부되지
않습니다. 따라서 파트가 더 큰 파트로 성공적으로 머지되면 단일
서버의 단일 테이블에 수백 테라바이트의 데이터를 저장할 수 있습니다. 이는
비활성 파트 또는 전체 파트의 임계값에는 영향을 주지 않습니다.

<div id="max_buckets_in_map">
  ## max_buckets_in_map
</div>

<div id="max_compress_block_size">
  ## max_compress_block_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

테이블에 쓰기 위해 압축하기 전에, 압축되지 않은 데이터 블록의 최대
크기입니다. 이 설정은 전역 설정에서도 지정할 수 있습니다
([max&#95;compress&#95;block&#95;size](/ko/reference/settings/merge-tree-settings/max#max_compress_block_size)
설정 참조). 테이블 생성 시 지정한 값은 이 설정의 전역
값을 재정의합니다.

<div id="max_concurrent_queries">
  ## max_concurrent_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

MergeTree 테이블과 관련된 쿼리를 동시에 실행할 수 있는 최대 개수입니다.
쿼리는 다른 `max_concurrent_queries` 설정의 적용도 받습니다.

Possible values:

* 양의 정수.
* `0` — 제한 없음.

기본값: `0` (제한 없음).

**예시**

```xml
<max_concurrent_queries>50</max_concurrent_queries>
```

<div id="max_file_name_length">
  ## max_file_name_length
</div>

<SettingsInfoBlock type="UInt64" default_value="127" />

해싱하지 않고 파일 이름을 그대로 유지할 수 있는 최대 길이입니다.
이 설정은 `replace_long_file_name_to_hash` 설정이 활성화된 경우에만 적용됩니다.
이 설정값에는 파일 확장자의 길이가 포함되지 않습니다. 따라서
파일 시스템 오류를 방지할 수 있도록 여유를 두고 최대 파일 이름 길이(보통 255
바이트)보다 작게 설정하는 것이 좋습니다.

<div id="max_partitions_to_read">
  ## max_partitions_to_read
</div>

<SettingsInfoBlock type="Int64" default_value="-1" />

하나의 쿼리에서 액세스할 수 있는 파티션의 최대 개수를 제한합니다.

table을 생성할 때 지정한 설정 값은

쿼리 / 세션 / profile 수준에서 쿼리 복잡도 설정 [max&#95;partitions&#95;to&#95;read](/ko/reference/settings/session-settings#max_partitions_to_read)를
지정할 수도 있습니다.
