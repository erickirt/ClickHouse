이 설정은 [system.merge&#95;tree&#95;settings](/ko/reference/system-tables/merge_tree_settings)에서 확인할 수 있으며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="vertical_merge_algorithm_min_bytes_to_activate">
  ## vertical_merge_algorithm_min_bytes_to_activate
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

수직 병합 알고리즘을 활성화하는 데 필요한, 머지 중인 파트의 최소(대략적인) 압축되지 않은 크기(바이트)입니다.

<div id="vertical_merge_algorithm_min_columns_to_activate">
  ## vertical_merge_algorithm_min_columns_to_activate
</div>

<SettingsInfoBlock type="UInt64" default_value="11" />

수직 병합 알고리즘을 활성화하는 데 필요한 PK 이외 컬럼의 최소 개수입니다.

<div id="vertical_merge_algorithm_min_rows_to_activate">
  ## vertical_merge_algorithm_min_rows_to_activate
</div>

<SettingsInfoBlock type="UInt64" default_value="131072" />

수직 병합 알고리즘을 활성화하는 데 필요한
머지 중인 파트의 행 수 합의 최소값(대략적)입니다.

<div id="vertical_merge_optimize_lightweight_delete">
  ## vertical_merge_optimize_lightweight_delete
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "1"},{"label": "새로운 설정"}]}]} />

값이 true이면 경량한 삭제가 수직 병합 시 최적화됩니다.

<div id="vertical_merge_optimize_ttl_delete">
  ## vertical_merge_optimize_ttl_delete
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "1"},{"label": "TTL로 만료된 행을 제거해야 하는 머지에 수직 병합 알고리즘을 허용합니다"}]}]} />

값이 true이면 수직 병합에서 행 TTL 삭제가 최적화됩니다. 수평 병합을 강제하는 대신
TTL 필터를 평가하여 머지 알고리즘에 전달하며, 이 알고리즘은 행 소스에 skip 플래그를 설정합니다.

<div id="vertical_merge_remote_filesystem_prefetch">
  ## vertical_merge_remote_filesystem_prefetch
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

true이면 머지 중 다음 컬럼에 대해 원격 파일 시스템의 데이터를 프리페치하는 기능을 사용합니다
