이러한 설정은 [system.merge&#95;tree&#95;settings](/ko/reference/system-tables/merge_tree_settings)에서 확인할 수 있으며, ClickHouse 소스에서 자동으로 생성됩니다.

<div id="max_delay_to_insert">
  ## max_delay_to_insert
</div>

<SettingsInfoBlock type="UInt64" default_value="1" />

단일 파티션의 활성 파트 수가
[parts&#95;to&#95;delay&#95;insert](/ko/reference/settings/merge-tree-settings/parts-to#parts_to_delay_insert) 값을
초과하는 경우 `INSERT` 지연을 계산하는 데 사용되는 초 단위 값입니다.

가능한 값:

* 모든 양의 정수.

`INSERT` 지연(밀리초)은 다음 공식으로 계산됩니다:

```code
max_k = parts_to_throw_insert - parts_to_delay_insert
k = 1 + parts_count_in_partition - parts_to_delay_insert
delay_milliseconds = pow(max_delay_to_insert * 1000, k / max_k)
```

예를 들어, 파티션에 활성 파트가 299개 있고 parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1인 경우, `INSERT`는
`pow( 1 * 1000, (1 + 299 - 150) / (300 - 150) ) = 1000`
밀리초만큼 지연됩니다.

버전 23.1부터는 수식이 다음과 같이 변경되었습니다:

```code
allowed_parts_over_threshold = parts_to_throw_insert - parts_to_delay_insert
parts_over_threshold = parts_count_in_partition - parts_to_delay_insert + 1
delay_milliseconds = max(min_delay_to_insert_ms, (max_delay_to_insert * 1000)
* parts_over_threshold / allowed_parts_over_threshold)
```

예를 들어, 파티션의 활성 파트가 224개이고 parts&#95;to&#95;throw&#95;insert
= 300, parts&#95;to&#95;delay&#95;insert = 150, max&#95;delay&#95;to&#95;insert = 1,
min&#95;delay&#95;to&#95;insert&#95;ms = 10이면, `INSERT`는 `max( 10, 1 * 1000 *
(224 - 150 + 1) / (300 - 150) ) = 500`밀리초 동안 지연됩니다.

<div id="max_delay_to_mutate_ms">
  ## max_delay_to_mutate_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" />

완료되지 않은 뮤테이션이 많은 경우, MergeTree 테이블의 뮤테이션 최대 지연 시간(밀리초)입니다.
