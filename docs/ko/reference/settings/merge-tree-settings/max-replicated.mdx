이러한 설정은 [system.merge&#95;tree&#95;settings](/ko/reference/system-tables/merge_tree_settings)에서 확인할 수 있으며, ClickHouse 소스 코드에서 자동 생성됩니다.

<div id="max_replicated_fetches_network_bandwidth">
  ## max_replicated_fetches_network_bandwidth
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

[복제된](/ko/reference/engines/table-engines/mergetree-family/replication)
fetch 작업에 대해 네트워크를 통한 데이터 전송의 최대 속도를 초당 바이트 단위로 제한합니다. 이 설정은 서버에 적용되는
[`max_replicated_fetches_network_bandwidth_for_server`](/ko/reference/settings/merge-tree-settings/max-replicated#max_replicated_fetches_network_bandwidth)
설정과 달리 특정 테이블에 적용됩니다.

서버 네트워크와 특정 테이블의 네트워크를 모두 제한할 수 있지만, 이 경우
테이블 수준 설정값은 서버 수준 설정값보다 작아야 합니다.
그렇지 않으면 서버는
`max_replicated_fetches_network_bandwidth_for_server` 설정만 적용합니다.

이 설정은 완전히 정확하게 적용되지는 않습니다.

Possible values:

* 양의 정수입니다.
* `0` — 제한 없음.

기본값: `0`.

**사용법**

새 노드를 추가하거나 교체할 때 데이터를 복제하는 속도를 스로틀링하는 데 사용할 수 있습니다.

<div id="max_replicated_logs_to_keep">
  ## max_replicated_logs_to_keep
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" />

비활성 레플리카가 있을 때 ClickHouse Keeper 로그에 포함될 수 있는 레코드 수입니다. 이 값을 초과하면 비활성 레플리카는 손실된 것으로 간주됩니다.

가능한 값:

* 모든 양의 정수.

<div id="max_replicated_merges_in_queue">
  ## max_replicated_merges_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" />

ReplicatedMergeTree 큐에서 파트를 머지하거나 변경하는 작업을 동시에 몇 개까지 허용할지 지정합니다.

<div id="max_replicated_merges_with_ttl_in_queue">
  ## max_replicated_merges_with_ttl_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1" />

ReplicatedMergeTree 큐에서 TTL이 적용된 파트를 머지하는 작업을
동시에 몇 개까지 허용할지 지정합니다.

<div id="max_replicated_mutations_in_queue">
  ## max_replicated_mutations_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="8" />

ReplicatedMergeTree 큐에서 뮤테이션이 진행 중인 파트에 대한 작업을 동시에
몇 개까지 허용할지 지정합니다.

<div id="max_replicated_sends_network_bandwidth">
  ## max_replicated_sends_network_bandwidth
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

[복제된](/ko/reference/engines/table-engines/mergetree-family/replacingmergetree)
전송 시 네트워크를 통한 데이터 교환의 최대 속도를 초당 바이트 수로
제한합니다. 이 설정은 서버에 적용되는
[`max_replicated_sends_network_bandwidth_for_server`](/ko/reference/settings/merge-tree-settings/max-replicated#max_replicated_sends_network_bandwidth)
설정과 달리 특정 테이블에 적용됩니다.

서버 네트워크와 특정 테이블의 네트워크를 모두 제한할 수 있지만,
이때 테이블 수준 설정값은 서버 수준 설정값보다 작아야 합니다.
그렇지 않으면 서버는
`max_replicated_sends_network_bandwidth_for_server` 설정만 고려합니다.

이 설정이 항상 완벽하게 정확히 적용되는 것은 아닙니다.

가능한 값:

* 양의 정수.
* `0` — 제한 없음.

**사용법**

새 노드를 추가하거나 교체하기 위해 데이터를 복제할 때 속도
스로틀링에 사용할 수 있습니다.
