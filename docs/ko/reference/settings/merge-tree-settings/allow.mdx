<div id="allow_dimensions_outside_sorting_key">
  ## allow_dimensions_outside_sorting_key
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "이제 AggregatingMergeTree는 테이블 생성 시 컬럼이 정렬 키에 속하지도 않고 집계 상태 측정값도 아닌 스키마를 거부합니다. 이전에는 이러한 스키마가 허용되었으며(이전 동작은 값 'true'에 해당함)"}]}]} />

`AggregatingMergeTree`에서는 백그라운드 머지가 정렬 키(sorting key)의 동일한 값을 공유하는 행들을 하나로 합치고, 해당 집계 상태(aggregate state)를 결합합니다. 정렬 키에 속하지도 않고 집계 상태 *측정값*(`AggregateFunction` 또는 `SimpleAggregateFunction`)도 아닌 컬럼은 *차원*입니다. 이러한 컬럼은 머지 후 합쳐진 행 중 하나의 임의 값을 유지하므로, 이를 기준으로 `GROUP BY`를 수행하거나 필터링하는 쿼리에서 잘못된 결과가 조용히 발생할 수 있습니다(참조:
https://github.com/ClickHouse/ClickHouse/issues/751).

기본적으로 이러한 스키마는 테이블 생성 시 거부됩니다. 컬럼이 정렬 키에 함수적으로 종속되어 있어(즉, 합쳐지는 모든 행이 동일한 값을 가짐) 이러한 동작이 의도된 경우에만 이 설정을 활성화하세요.

이 검사는 실제로 집계를 수행하는 테이블(즉, 집계 상태 컬럼이 하나 이상 있는 테이블)에만 적용되며, `allow_tuple_element_aggregation`이 활성화된 경우에는 건너뜁니다.

<div id="allow_floating_point_partition_key">
  ## allow_floating_point_partition_key
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

부동소수점 수를 파티션 키로 사용할 수 있도록 허용합니다.

가능한 값:

* `0` — 부동소수점 파티션 키는 허용되지 않습니다.
* `1` — 부동소수점 파티션 키가 허용됩니다.

<SettingsInfoBlock type="Bool" default_value="0" />

활성화하면 Tuple 컬럼 내부의 개별 원소가 SummingMergeTree, AggregatingMergeTree 및
CoalescingMergeTree에서 머지 시 집계 대상에 포함됩니다. 중첩된 Tuple은 재귀적으로 확장되므로 모든
리프 원소가 각각 독립적으로 집계됩니다. 이 설정은 변경할 수 없으며
테이블 생성 시에 지정해야 합니다.
