이 설정은 [system.merge&#95;tree&#95;settings](/ko/reference/system-tables/merge_tree_settings)에서 확인할 수 있으며, ClickHouse 소스에서 자동 생성됩니다.

<div id="exclude_deleted_rows_for_part_size_in_merge">
  ## exclude_deleted_rows_for_part_size_in_merge
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

활성화하면 머지할 파트를 선택할 때 데이터 파트의 예상 실제 크기(즉, `DELETE FROM`으로 삭제된 행 제외)를 사용합니다. 이 동작은 이 설정을 활성화한 후 실행된 `DELETE FROM`의 영향을 받은 데이터 파트에 대해서만 트리거됩니다.

가능한 값:

* `true`
* `false`

  설정

<div id="exclude_materialize_skip_indexes_on_merge">
  ## exclude_materialize_skip_indexes_on_merge
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.10"},{"label": ""},{"label": "새 설정입니다."}]}]} />

쉼표로 구분된 지정 스킵 인덱스 목록이 머지 중 빌드 및 저장되지 않도록 제외합니다.
[materialize&#95;skip&#95;indexes&#95;on&#95;merge](/ko/reference/settings/merge-tree-settings/materialize#materialize_skip_indexes_on_merge)가 false이면 이 설정은 적용되지 않습니다.

제외된 스킵 인덱스는 명시적인
[MATERIALIZE INDEX](/ko/reference/statements/alter/skipping-index#materialize-index) 쿼리 또는
[materialize&#95;skip&#95;indexes&#95;on&#95;insert](/ko/reference/settings/session-settings#materialize_skip_indexes_on_insert)
세션 설정에 따라 INSERT 중에는 여전히 빌드되고 저장됩니다.

예시:

```sql
CREATE TABLE tab
(
a UInt64,
b UInt64,
INDEX idx_a a TYPE minmax,
INDEX idx_b b TYPE set(3)
)
ENGINE = MergeTree ORDER BY tuple() SETTINGS exclude_materialize_skip_indexes_on_merge = 'idx_a';

INSERT INTO tab SELECT number, number / 50 FROM numbers(100); -- setting has no effect on INSERTs

-- idx_a will be excluded from update during background or explicit merge via OPTIMIZE TABLE FINAL

-- can exclude multiple indexes by providing a list
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = 'idx_a, idx_b';

-- default setting, no indexes excluded from being updated during merge
ALTER TABLE tab MODIFY SETTING exclude_materialize_skip_indexes_on_merge = '';
```
