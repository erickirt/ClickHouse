title: Crear ClickPipe

<Step title="Selecciona el formato de los datos" id="3-select-data-format">
  La UI mostrará una lista de archivos del bucket especificado.
  Selecciona el formato de tus datos (actualmente admitimos un subconjunto de formatos de ClickHouse) e indica si quieres habilitar la ingestión continua.
  Consulta la sección &quot;ingesta continua&quot; en la página de resumen para obtener más información.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step3_object_storage.webp" alt="Configurar el formato de los datos y el topic" />
  </Frame>
</Step>

<Step title="Configura la tabla, el esquema y la configuración" id="5-configure-table-schema-settings">
  En el siguiente paso, puedes seleccionar si quieres ingestar datos en una nueva tabla de ClickHouse o reutilizar una existente.
  Sigue las instrucciones que aparecen en pantalla para modificar el nombre de la tabla, el esquema y la configuración.
  Puedes ver una vista previa en tiempo real de los cambios en la tabla de ejemplo de la parte superior.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4a.webp" alt="Configurar la tabla, el esquema y la configuración" />
  </Frame>

  También puedes personalizar la configuración avanzada con los controles proporcionados.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4a3.webp" alt="Configurar los controles avanzados" />
  </Frame>

  Como alternativa, puedes decidir ingestar tus datos en una tabla de ClickHouse existente.
  En ese caso, la UI te permitirá mapear campos del origen a los campos de ClickHouse de la tabla de destino seleccionada.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4b.webp" alt="Usar una tabla existente" />
  </Frame>

  <Info>
    También puedes mapear [columnas virtuales](/es/reference/functions/table-functions/s3#virtual-columns), como `_path` o `_size`, a campos.
  </Info>
</Step>

<Step title="Configura los permisos" id="6-configure-permissions">
  Por último, puedes configurar los permisos para el usuario interno de ClickPipes.

  **Permisos:** ClickPipes creará un usuario dedicado para escribir datos en una tabla de destino. Puedes seleccionar un rol para este usuario interno usando un rol personalizado o uno de los roles predefinidos:

  * `Full access`: con acceso completo al clúster. Es necesario si usas una vista materializada o un Diccionario con la tabla de destino.
  * `Only destination table`: solo con permisos de `INSERT` para la tabla de destino.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step5.webp" alt="Permisos" />
  </Frame>
</Step>

<Step title="Completa la configuración" id="7-complete-setup">
  Al hacer clic en &quot;Complete Setup&quot;, el sistema registrará tu ClickPipe y podrás verlo en la tabla de resumen.

  <div className="ch-image-sm">
    <Frame>
      <img src="/images/integrations/data-ingestion/clickpipes/cp_success.webp" alt="Aviso de éxito" />
    </Frame>
  </div>

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_remove.webp" alt="Aviso de eliminación" />
  </Frame>

  La tabla de resumen proporciona controles para mostrar datos de ejemplo del origen o de la tabla de destino en ClickHouse.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_destination.webp" alt="Ver destino" />
  </Frame>

  También incluye controles para eliminar el ClickPipe y mostrar un resumen del trabajo de ingestión.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_overview.webp" alt="Ver resumen" />
  </Frame>

  **¡Enhorabuena!** Has configurado correctamente tu primer ClickPipe.
  Si se trata de un ClickPipe configurado para la ingestión continua, se ejecutará de forma continua e ingestará datos en tiempo real desde tu fuente de datos remota.
  De lo contrario, ingestará el lote y finalizará.
</Step>
