  ### アクセスキーを生成する

  ### サービス アカウントの HMAC キーとシークレットを作成する

  **Cloud Storage &gt; Settings &gt; Interoperability** を開き、既存の **Access key** を選択するか、**CREATE A KEY FOR A SERVICE ACCOUNT** を選択します。このガイドでは、新しいサービス アカウント用に新しいキーを作成する手順を説明します。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-create-a-service-account-key.webp" alt="GCS でサービス アカウントの HMAC キーを生成する" />
    </Frame>
  </div>

  ### 新しいサービス アカウントを追加する

  既存のサービス アカウントがないプロジェクトの場合は、**CREATE NEW ACCOUNT** を選択します。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-create-service-account-0.webp" alt="GCS で新しいサービス アカウントを追加する" />
    </Frame>
  </div>

  サービス アカウントの作成は 3 つの手順で行います。最初の手順では、アカウントにわかりやすい名前、ID、説明を設定します。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-create-service-account-a.webp" alt="GCS で新しいサービス アカウント名と ID を定義する" />
    </Frame>
  </div>

  Interoperability の設定ダイアログでは、IAM role の **Storage Object Admin** ロールを推奨します。2 番目の手順でこのロールを選択してください。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-create-service-account-2.webp" alt="GCS で IAM role Storage Object Admin を選択する" />
    </Frame>
  </div>

  3 番目の手順は任意であり、このガイドでは使用しません。ポリシーに応じて、ユーザーにこれらの権限を付与できます。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-create-service-account-3.webp" alt="GCS で新しいサービス アカウントの追加設定を構成する" />
    </Frame>
  </div>

  サービス アカウントの HMAC キーが表示されます。この情報は ClickHouse の設定で使用するため、保存しておいてください。

  <div className="ch-image-md">
    <Frame>
      <img src="/images/integrations/data-ingestion/s3/GCS-guide-key.webp" alt="GCS 用に生成された HMAC キーを取得する" />
    </Frame>
  </div>
