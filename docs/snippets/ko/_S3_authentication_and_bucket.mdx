<Accordion title="S3 버킷과 IAM 사용자 생성">
  이 문서에서는 AWS IAM 사용자를 구성하고, S3 버킷을 생성하며, ClickHouse가 해당 버킷을 S3 디스크로 사용하도록 설정하는 기본적인 방법을 설명합니다.
  실제로 적용할 권한은 보안 팀과 협의하여 결정하시고, 이 문서의 내용은 시작점으로 참고하시기 바랍니다.

  ### AWS IAM 사용자 생성

  다음 단계에서는 서비스 계정 사용자(로그인 사용자가 아닌)를 생성합니다.

  1. AWS IAM Management Console에 로그인합니다.

  2. `Users` 메뉴에서 `Create user`를 선택합니다.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-1.webp" alt="AWS IAM Management Console - 신규 사용자 추가" />
    </Frame>
  </div>

  3. 사용자 이름을 입력하고 자격 증명 유형을 `Access key - Programmatic access`로 설정한 후 `Next: Permissions`를 선택하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-2.webp" alt="IAM user 이름 및 액세스 유형 설정" />
    </Frame>
  </div>

  4. 사용자를 어떤 그룹에도 추가하지 말고 `다음: Tags`를 선택하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-3.webp" alt="IAM 사용자의 그룹 할당 건너뛰기" />
    </Frame>
  </div>

  5. 태그를 추가할 필요가 없다면 `Next: Review`를 선택하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-4.webp" alt="IAM user의 태그 할당 스키핑" />
    </Frame>
  </div>

  6. `Create User`를 클릭하세요

  <Note>
    사용자에게 권한이 없다는 경고 메시지는 무시해도 됩니다. 다음 섹션에서 해당 사용자에게 버킷 권한이 부여됩니다.
  </Note>

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-5.webp" alt="권한 없음 경고가 표시된 IAM user 생성" />
    </Frame>
  </div>

  7. 이제 사용자가 생성되었으므로 `show`를 클릭한 다음 액세스 키와 시크릿 키를 복사하십시오.

  <Note>
    키는 다른 곳에 저장해 두십시오. 비밀 액세스 키는 지금만 확인할 수 있습니다.
  </Note>

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-6.webp" alt="IAM 사용자 액세스 키 보기 및 복사" />
    </Frame>
  </div>

  8. 해지를 클릭한 다음 사용자 화면에서 해당 사용자를 찾습니다.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-7.webp" alt="사용자 목록에서 새로 생성된 IAM 사용자 찾기" />
    </Frame>
  </div>

  9. ARN(Amazon Resource Name)을 복사한 다음, 버킷의 액세스 정책을 구성할 때 사용할 수 있도록 저장합니다.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-8.webp" alt="IAM 사용자의 ARN 복사" />
    </Frame>
  </div>

  ### S3 버킷 생성

  1. S3 버킷 섹션에서 `Create bucket`을 선택하십시오

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-9.webp" alt="S3 버킷 생성 시작" />
    </Frame>
  </div>

  2. 버킷 이름을 입력하고, 나머지 옵션은 기본값으로 그대로 둡니다

  <Note>
    버킷 이름은 조직 내에서만이 아니라 AWS 전체에서 고유해야 하며, 그렇지 않으면 오류가 발생합니다.
  </Note>

  3. `Block all Public Access`는 활성화된 상태로 유지하십시오. 공개 액세스는 필요하지 않습니다.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-a.webp" alt="퍼블릭 액세스 차단이 설정된 S3 버킷 설정" />
    </Frame>
  </div>

  4. 페이지 하단의 `Create Bucket`을 선택하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-b.webp" alt="S3 버킷 생성 완료" />
    </Frame>
  </div>

  5. 링크를 선택하여 ARN을 복사한 다음, 버킷의 액세스 정책을 구성할 때 사용할 수 있도록 저장하십시오.

  6. 버킷이 생성되면 S3 버킷 목록에서 새 S3 버킷을 찾은 다음 해당 링크를 선택하십시오.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-c.webp" alt="버킷 목록에서 새로 생성한 S3 버킷 찾기" />
    </Frame>
  </div>

  7. `Create folder`를 선택하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-d.webp" alt="S3 버킷에 새 폴더를 만드는 모습" />
    </Frame>
  </div>

  8. ClickHouse S3 디스크의 대상 폴더 이름을 입력하고 `Create folder`를 선택합니다

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-e.webp" alt="ClickHouse S3 디스크에 사용할 폴더 이름 설정" />
    </Frame>
  </div>

  9. 이제 버킷 목록에 폴더가 표시됩니다

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-f.webp" alt="S3 버킷에서 새로 생성된 폴더를 보는 모습" />
    </Frame>
  </div>

  10. 새 폴더의 체크박스를 선택하고 `Copy URL`을 클릭하세요. 다음 섹션의 ClickHouse 스토리지 구성에서 사용할 수 있도록 복사한 URL을 저장해 두세요.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-g.webp" alt="ClickHouse 구성에 사용할 S3 폴더 URL 복사" />
    </Frame>
  </div>

  11. `Permissions` 탭을 선택한 다음 `Bucket Policy` 섹션에서 `Edit` 버튼을 클릭하세요

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-h.webp" alt="S3 버킷 정책 구성에 액세스" />
    </Frame>
  </div>

  12. 버킷 정책을 추가하세요. 아래는 예시입니다:

  ```json
  {
    "Version" : "2012-10-17",
    "Id" : "Policy123456",
    "Statement" : [
      {
        "Sid" : "abc123",
        "Effect" : "Allow",
        "Principal" : {
          "AWS" : "arn:aws:iam::921234567898:user/mars-s3-user"
        },
        "Action" : "s3:*",
        "Resource" : [
          "arn:aws:s3:::mars-doc-test",
          "arn:aws:s3:::mars-doc-test/*"
        ]
      }
    ]
  }
  ```

  ```response
  |Parameter | Description | Example Value |
  |----------|-------------|----------------|
  |Version | Version of the policy interpreter, leave as-is | 2012-10-17 |
  |Sid | User-defined policy id | abc123 |
  |Effect | Whether user requests will be allowed or denied | Allow |
  |Principal | The accounts or user that will be allowed | arn:aws:iam::921234567898:user/mars-s3-user |
  |Action | What operations are allowed on the bucket| s3:*|
  |Resource | Which resources in the bucket will operations be allowed in | "arn:aws:s3:::mars-doc-test", "arn:aws:s3:::mars-doc-test/*" |
  ```

  <Note>
    사용할 권한을 결정할 때는 보안 팀과 협의하고, 아래 내용은 출발점으로 삼으십시오.
    정책 및 설정에 대한 자세한 내용은 AWS 문서를 참조하십시오:
    https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-policy-language-overview.html
  </Note>

  13. 정책 구성을 저장하세요.
</Accordion>
