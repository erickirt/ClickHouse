---
title: Créer un ClickPipe
---

<Step title="Sélectionner le format des données" id="3-select-data-format">
  L’interface utilisateur affichera une liste des fichiers du bucket spécifié.
  Sélectionnez votre format de données (nous prenons actuellement en charge un sous-ensemble des formats ClickHouse) et indiquez si vous souhaitez activer l’ingestion continue.
  Consultez la section « ingestion continue » de la page de vue d’ensemble pour plus de détails.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step3_object_storage.webp" alt="Définir le format des données et le topic" />
  </Frame>
</Step>

<Step title="Configurer la table, le schéma et les paramètres" id="5-configure-table-schema-settings">
  À l’étape suivante, vous pouvez choisir d’ingérer les données dans une nouvelle table ClickHouse ou de réutiliser une table existante.
  Suivez les instructions à l’écran pour modifier le nom de votre table, le schéma et les paramètres.
  Vous pouvez voir un aperçu en temps réel de vos modifications dans la table d’exemple en haut de la page.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4a.webp" alt="Définir la table, le schéma et les paramètres" />
  </Frame>

  Vous pouvez également personnaliser les paramètres avancés à l’aide des contrôles fournis.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4a3.webp" alt="Définir les contrôles avancés" />
  </Frame>

  Vous pouvez aussi choisir d’ingérer vos données dans une table ClickHouse existante.
  Dans ce cas, l’interface utilisateur vous permettra de mapper les champs de la source aux champs ClickHouse de la table de destination sélectionnée.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step4b.webp" alt="Utiliser une table existante" />
  </Frame>

  <Info>
    Vous pouvez également mapper des [colonnes virtuelles](/fr/reference/functions/table-functions/s3#virtual-columns), comme `_path` ou `_size`, à des champs.
  </Info>
</Step>

<Step title="Configurer les autorisations" id="6-configure-permissions">
  Enfin, vous pouvez configurer les autorisations de l’utilisateur interne ClickPipes.

  **Autorisations :** ClickPipes créera un utilisateur dédié pour écrire des données dans une table de destination. Vous pouvez sélectionner un rôle pour cet utilisateur interne à l’aide d’un rôle personnalisé ou de l’un des rôles prédéfinis :

  * `Full access` : accorde un accès complet au cluster. Requis si vous utilisez une vue matérialisée ou un Dictionary avec la table de destination.
  * `Only destination table` : accorde uniquement les autorisations `INSERT` sur la table de destination.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_step5.webp" alt="Autorisations" />
  </Frame>
</Step>

<Step title="Terminer la configuration" id="7-complete-setup">
  En cliquant sur « Complete Setup », le système enregistrera votre ClickPipe et vous pourrez le voir apparaître dans le tableau récapitulatif.

  <div className="ch-image-sm">
    <Frame>
      <img src="/images/integrations/data-ingestion/clickpipes/cp_success.webp" alt="Notification de réussite" />
    </Frame>
  </div>

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_remove.webp" alt="Notification de suppression" />
  </Frame>

  Le tableau récapitulatif fournit des contrôles pour afficher des données d’exemple provenant de la source ou de la table de destination dans ClickHouse.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_destination.webp" alt="Afficher la destination" />
  </Frame>

  Il propose également des contrôles pour supprimer le ClickPipe et afficher un récapitulatif de la tâche d’ingestion.

  <Frame>
    <img src="/images/integrations/data-ingestion/clickpipes/cp_overview.webp" alt="Afficher la vue d’ensemble" />
  </Frame>

  **Félicitations !** Vous avez configuré avec succès votre premier ClickPipe.
  S’il s’agit d’un ClickPipe configuré pour l’ingestion continue, il s’exécutera en continu et ingérera les données en temps réel depuis votre source de données distante.
  Sinon, il ingérera le lot puis s’arrêtera.
</Step>
