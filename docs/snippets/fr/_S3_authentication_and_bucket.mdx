<Accordion title="Créez des buckets S3 et un utilisateur IAM">
  Cet article présente les bases de la configuration d&#39;un AWS IAM user, de la création d&#39;un S3 bucket et de la configuration de ClickHouse pour utiliser ce bucket comme S3 disk.
  Nous vous recommandons de travailler avec votre équipe de sécurité pour déterminer les permissions à appliquer, et de considérer ces éléments comme point de départ.

  ### Créer un utilisateur AWS IAM

  Dans les étapes suivantes, vous allez créer un utilisateur de compte de service (et non un utilisateur avec compte de connexion).

  1. Connectez-vous à l’AWS IAM Management Console.

  2. Dans le menu `Users`, sélectionnez `Create user`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-1.webp" alt="AWS IAM Management Console - Ajout d'un nouvel utilisateur" />
    </Frame>
  </div>

  3. Saisissez le nom d’utilisateur et définissez le type d’identifiant sur `Access key - Programmatic access`, puis sélectionnez `Next: Permissions`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-2.webp" alt="Configuration du nom d’utilisateur et du type d’accès pour l’utilisateur IAM" />
    </Frame>
  </div>

  4. N’ajoutez l’utilisateur à aucun groupe ; cliquez sur `Next: Tags`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-3.webp" alt="Ignorer l’attribution d’un groupe à l’utilisateur IAM" />
    </Frame>
  </div>

  5. À moins que vous n&#39;ayez besoin d&#39;ajouter des tags, sélectionnez `Next: Review`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-4.webp" alt="Attribution du tag skipping à l’IAM user" />
    </Frame>
  </div>

  6. Sélectionnez `Create User`

  <Note>
    Le message d’avertissement indiquant que l’utilisateur n’a aucune autorisation peut être ignoré ; les autorisations sur le bucket seront accordées à l’utilisateur dans la section suivante
  </Note>

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-5.webp" alt="Création de l’IAM user avec l’avertissement indiquant l’absence d’autorisations" />
    </Frame>
  </div>

  7. L’utilisateur a maintenant été créé ; cliquez sur `show` et copiez les clés d’accès et les clés secrètes.

  <Note>
    Conservez les clés ailleurs ; c’est la seule fois où la clé d’accès secrète sera disponible.
  </Note>

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-6.webp" alt="Affichage et copie des clés d’accès de l’utilisateur IAM" />
    </Frame>
  </div>

  8. Cliquez sur Fermer, puis recherchez l’utilisateur dans l’écran Utilisateurs.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-7.webp" alt="Repérage de l’IAM user nouvellement créé dans la liste des utilisateurs" />
    </Frame>
  </div>

  9. Copiez l’ARN (Amazon Resource Name) et enregistrez-le pour l’utiliser lors de la configuration de la politique d’accès au bucket.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-8.webp" alt="Copie de l’ARN de l’utilisateur IAM" />
    </Frame>
  </div>

  ### Créer un bucket S3

  1. Dans la section « bucket S3 », sélectionnez `Create bucket`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-9.webp" alt="Lancement de la création du bucket S3" />
    </Frame>
  </div>

  2. Saisissez un nom de bucket, laissez les autres options par défaut

  <Note>
    Le nom du bucket doit être unique dans tout AWS, et pas seulement au sein de l’organisation, sinon une erreur sera générée.
  </Note>

  3. Laissez `Block all Public Access` activé ; l’accès public n’est pas nécessaire.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-a.webp" alt="Configuration des paramètres du bucket S3 avec l’accès public bloqué" />
    </Frame>
  </div>

  4. Sélectionnez `Create Bucket` en bas de la page

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-b.webp" alt="Finalisation de la création du bucket S3" />
    </Frame>
  </div>

  5. Sélectionnez le lien, copiez l’ARN et enregistrez-le pour l’utiliser lors de la configuration de la politique d’accès au bucket.

  6. Une fois le bucket créé, repérez le nouveau bucket S3 dans la liste des buckets S3 et sélectionnez le lien

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-c.webp" alt="Localiser le bucket S3 nouvellement créé dans la liste des buckets" />
    </Frame>
  </div>

  7. Sélectionnez `Create folder`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-d.webp" alt="Création d’un nouveau dossier dans le bucket S3" />
    </Frame>
  </div>

  8. Saisissez un nom de dossier qui servira de destination pour le disk S3 de ClickHouse, puis sélectionnez `Create folder`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-e.webp" alt="Définition du nom du dossier pour l’utilisation du disque S3 ClickHouse" />
    </Frame>
  </div>

  9. Le dossier devrait désormais être visible dans la liste des buckets

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-f.webp" alt="Affichage du nouveau dossier dans le bucket S3" />
    </Frame>
  </div>

  10. Cochez la case du nouveau dossier et cliquez sur `Copy URL` Enregistrez l’URL copiée afin de l’utiliser dans la configuration de stockage de ClickHouse, dans la section suivante.

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-g.webp" alt="Copie de l’URL du dossier S3 pour configurer ClickHouse" />
    </Frame>
  </div>

  11. Sélectionnez l’onglet `Permissions`, puis cliquez sur le bouton `Edit` dans la section `Bucket Policy`

  <div className="ch-image-md">
    <Frame>
      <img src="/images/_snippets/s3/s3-h.webp" alt="Accès à la configuration de la stratégie du bucket S3" />
    </Frame>
  </div>

  12. Ajoutez une stratégie de compartiment, comme dans l’exemple ci-dessous :

  ```json
  {
    "Version" : "2012-10-17",
    "Id" : "Policy123456",
    "Statement" : [
      {
        "Sid" : "abc123",
        "Effect" : "Allow",
        "Principal" : {
          "AWS" : "arn:aws:iam::921234567898:user/mars-s3-user"
        },
        "Action" : "s3:*",
        "Resource" : [
          "arn:aws:s3:::mars-doc-test",
          "arn:aws:s3:::mars-doc-test/*"
        ]
      }
    ]
  }
  ```

  ```response
  |Parameter | Description | Example Value |
  |----------|-------------|----------------|
  |Version | Version of the policy interpreter, leave as-is | 2012-10-17 |
  |Sid | User-defined policy id | abc123 |
  |Effect | Whether user requests will be allowed or denied | Allow |
  |Principal | The accounts or user that will be allowed | arn:aws:iam::921234567898:user/mars-s3-user |
  |Action | What operations are allowed on the bucket| s3:*|
  |Resource | Which resources in the bucket will operations be allowed in | "arn:aws:s3:::mars-doc-test", "arn:aws:s3:::mars-doc-test/*" |
  ```

  <Note>
    Vous devriez travailler avec votre équipe de sécurité pour déterminer les autorisations à utiliser ; considérez celles-ci comme un point de départ.
    Pour plus d’informations sur les politiques et les paramètres, consultez la documentation AWS :
    https://docs.aws.amazon.com/AmazonS3/latest/userguide/access-policy-language-overview.html
  </Note>

  13. Enregistrez la configuration de la règle d’accès.
</Accordion>
