  # Installer ClickHouse en utilisant Homebrew
</div>

<Warning>
  L&#39;installation via les Formulae Homebrew est désormais obsolète et sera désactivée le 2026-09-01.
  Nous recommandons d&#39;utiliser plutôt la méthode [d&#39;installation rapide](/fr/get-started/setup/self-managed/quick-install) qui fonctionne sur toutes les plateformes.
</Warning>

<Steps>
  <Step title="Review recommendations" id="review-recommendations">
    <Recommendations />
  </Step>

  <Step title="Install using the community Homebrew formula" id="install-using-community-homebrew-formula">
    Pour installer ClickHouse sur macOS en utilisant [Homebrew](https://brew.sh/), vous pouvez utiliser
    la [formule Homebrew](https://formulae.brew.sh/cask/clickhouse) communautaire de ClickHouse.

    ```bash
    brew install --cask clickhouse
    ```
  </Step>

  <Step title="Fix the developer verification error in macOS" id="fix-developer-verification-error-macos">
    Si vous installez ClickHouse en utilisant `brew`, vous pouvez rencontrer une erreur provenant de macOS.
    Par défaut, macOS n&#39;exécutera pas les applications ou outils créés par un développeur qui ne peut pas être vérifié.

    Lorsque vous tentez d&#39;exécuter une commande `clickhouse`, vous pouvez voir cette erreur :

    <div className="ch-image-sm">
      <Frame>
        <img src="/images/knowledgebase/fix-the-developer-verification-error-in-macos/dev-verification-error.webp" alt="Dialogue d'erreur de vérification du développeur macOS" />
      </Frame>
    </div>

    Pour contourner cette erreur de vérification, vous devez retirer l&#39;application du bac de quarantaine de macOS soit en trouvant le réglage approprié dans votre fenêtre System Settings, en utilisant le terminal, ou en réinstallant ClickHouse.

    ### System settings process

    Le moyen le plus simple de retirer l&#39;exécutable `clickhouse` du bac de quarantaine est de :

    1. Ouvrir **System Settings**.

    2. Naviguer vers **Privacy &amp; Security** :

       <div className="ch-image-md">
         <Frame>
           <img src="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-default-view.webp" alt="Vue par défaut des réglages Privacy & Security de macOS" />
         </Frame>
       </div>

    3. Faites défiler jusqu&#39;au bas de la fenêtre pour trouver un message indiquant *&quot;clickhouse-macos-aarch64&quot; a été bloqué parce qu&#39;il ne provient pas d&#39;un développeur identifié*.

    4. Cliquez sur **Allow Anyway**.

       <div className="ch-image-md">
         <Frame>
           <img src="/images/knowledgebase/fix-the-developer-verification-error-in-macos/privacy-and-security-screen-allow-anyway.webp" alt="Réglages Privacy & Security de macOS affichant le bouton Allow Anyway" />
         </Frame>
       </div>

    5. Saisissez le mot de passe de votre compte macOS.

    Vous devriez maintenant pouvoir exécuter les commandes `clickhouse` dans votre terminal.

    ### Terminal process

    Parfois, appuyer sur le bouton `Allow Anyway` ne résout pas ce problème, auquel cas vous pouvez aussi effectuer cette procédure en ligne de commande.
    Ou vous pouvez simplement préférer utiliser la ligne de commande !

    Commencez par trouver où Homebrew a installé l&#39;exécutable `clickhouse` :

    ```shell
    which clickhouse
    ```

    Cela devrait afficher quelque chose comme :

    ```shell
    /opt/homebrew/bin/clickhouse
    ```

    Retirez `clickhouse` du bac de quarantaine en exécutant `xattr -d com.apple.quarantine` suivi du chemin obtenu précédemment :

    ```shell
    xattr -d com.apple.quarantine /opt/homebrew/bin/clickhouse
    ```

    Vous devriez maintenant pouvoir exécuter l&#39;exécutable `clickhouse` :

    ```shell
    clickhouse
    ```

    Cela devrait afficher quelque chose comme :

    ```bash
    Use one of the following commands:
    clickhouse local [args]
    clickhouse client [args]
    clickhouse benchmark [args]
    ```
  </Step>

  <Step title="Fix the issue by reinstalling ClickHouse" id="fix-issue">
    Brew dispose d&#39;une option en ligne de commande qui évite de mettre en quarantaine les binaires installés dès le départ.

    Commencez par désinstaller ClickHouse :

    ```shell
    brew uninstall clickhouse
    ```

    Réinstallez maintenant ClickHouse avec `--no-quarantine` :

    ```shell
    brew install --no-quarantine clickhouse
    ```
  </Step>
</Steps>
