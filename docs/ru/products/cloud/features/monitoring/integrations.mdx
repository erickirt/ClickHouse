<div id="clickhouse-cloud-datadog-api-integration">
  ### Интеграция ClickHouse Cloud с API Datadog
</div>

<PrivatePreviewBadge />

Рекомендуемый способ передачи метрик уровня сервиса из ClickHouse Cloud в Datadog.

Вы подключаете свой аккаунт Datadog к ClickHouse Cloud с помощью OAuth-рукопожатия (**Connect Accounts** в плитке интеграции Datadog). Затем ClickHouse Cloud отправляет метрики уровня сервиса в Datadog вместо того, чтобы Datadog напрямую опрашивал сервис.

* **Только в ClickHouse Cloud** — не подходит для OSS ClickHouse
* Datadog **не** считает эти метрики пользовательскими
* **Не** предотвращает переход сервиса в состояние простоя
* Поставляется с предварительно настроенным набором панелей мониторинга и мониторов

Информацию по шагам онбординга см. в [руководстве Datadog по интеграции ClickHouse Cloud](https://docs.datadoghq.com/integrations/clickhouse-cloud/).

<div id="clickhouse-cloud-prometheus-integration">
  ### Интеграция Prometheus с ClickHouse Cloud
</div>

Самый распространённый и универсальный способ собирать телеметрию на уровне сервиса и организации в ClickHouse Cloud.

Настройте Datadog Agent с помощью [интеграции OpenMetrics](https://docs.datadoghq.com/integrations/openmetrics/) для периодического опроса ClickHouse Cloud API и сбора метрик. Подробные сведения о настройке см. на [странице интеграции Prometheus](/ru/integrations/prometheus#integrating-with-datadog).

* **Только в ClickHouse Cloud** — не подходит для OSS ClickHouse
* В Datadog эти метрики учитываются как **пользовательские метрики**
* **Не** предотвращает переход сервиса в режим простоя

<div id="clickhouse-datadog-agent-integration">
  ### Интеграция ClickHouse с Datadog Agent
</div>

Это самый распространённый способ передачи данных из сервисов OSS ClickHouse в Datadog. Он также работает с ClickHouse Cloud, но с некоторыми оговорками.

Datadog Agent периодически опрашивает экземпляр ClickHouse и собирает метрики и журналы. Эта интеграция включает предварительно настроенный набор панелей мониторинга.

**OSS ClickHouse:** журналы и метрики поддерживаются полностью.

**ClickHouse Cloud (частичная поддержка):**

* Только метрики — журналы не поддерживаются
* Требуется `datadog-cluster-agent` вместо `datadog-agent`
* **Не позволит сервису перейти в режим простоя**

<div id="clickhouse-datadog-dbm">
  ### ClickHouse Datadog DBM
</div>

Database Monitoring (DBM) для ClickHouse разрабатывает и поддерживает Datadog. Для этого Datadog Agent настраивается на сбор данных из системных таблиц экземпляра ClickHouse, что позволяет получать информацию о производительности на уровне запросов, аналогичную той, что встроенно доступна в ClickHouse Cloud Console. ClickHouse не участвует в этой интеграции — по всем проблемам обращайтесь в Datadog, а не в ClickHouse.

* Метрики бесплатны; отдельно оплачиваются журналы и ингестия данных из системных таблиц
* **OSS ClickHouse:** полностью поддерживается
* **ClickHouse Cloud (частичная поддержка):** требует `datadog-cluster-agent` вместо `datadog-agent`; **не даст сервису перейти в режим простоя**

<div id="billing-usage-api">
  ## API Billing &amp; Usage
</div>

API Billing &amp; Usage позволяет программно получать доступ к данным о биллинге и использовании вашей организации в Cloud. Это полезно при создании собственных панелей мониторинга затрат или интеграции данных о биллинге в существующие процессы финансовой отчетности.

Полное справочное описание API см. в [документации Billing API](/ru/products/cloud/api-reference/billing/get-organization-usage-costs).
