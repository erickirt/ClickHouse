Позволяет выполнять запросы к данным, хранящимся в базе данных [SQLite](/ru/reference/engines/database-engines/sqlite).

<div id="syntax">
  ## Синтаксис
</div>

```sql
sqlite('db_path', 'table_name')
```

<div id="arguments">
  ## Аргументы
</div>

* `db_path` — Путь к файлу базы данных SQLite. [String](/ru/reference/data-types/string).
* `table_name` — Имя таблицы в базе данных SQLite или запрос, передаваемый в SQLite как есть (см. [Передача запроса вместо имени таблицы](#passing-a-query)). [String](/ru/reference/data-types/string).

<div id="returned_value">
  ## Возвращаемое значение
</div>

* Объект table с теми же столбцами, что и в исходной таблице `SQLite`.

<div id="passing-a-query">
  ## Передача запроса вместо имени таблицы
</div>

Вместо имени таблицы вторым аргументом можно указать `SELECT`-запрос, который передаётся в SQLite как есть. Структура результирующей таблицы автоматически определяется по результату запроса. Запрос можно записать либо как подзапрос, либо обернуть в функцию `query`:

```sql
SELECT * FROM sqlite('sqlite.db', (SELECT col1, col2 FROM table1 WHERE col2 > 1));
SELECT * FROM sqlite('sqlite.db', query('SELECT col1, col2 FROM table1 WHERE col2 > 1'));
```

Такая таблица доступна только для чтения: операция `INSERT` для неё недопустима. Тот же синтаксис поддерживается и движком таблицы [`SQLite`](/ru/reference/engines/table-engines/integrations/sqlite).

<Note>
  Форма подзапроса `(SELECT ...)` разбирается ClickHouse и повторно сериализуется перед отправкой в SQLite. Поэтому она должна быть корректным ClickHouse SQL. Чтобы передать синтаксис, специфичный для SQLite, который ClickHouse не разбирает, используйте форму `query('...')`, текст которой отправляется в SQLite дословно.

  Любые внешние `WHERE`, `LIMIT`, агрегация и т. п. из окружающего запроса к ClickHouse **не** переносятся в переданный запрос — они применяются в ClickHouse после получения полного результата запроса. Чтобы ограничить данные, считываемые из SQLite, поместите фильтр внутрь переданного запроса. При [`external_table_strict_query = 1`](/ru/reference/settings/session-settings#external_table_strict_query) внешний фильтр, который нельзя перенести, вызывает исключение вместо локального применения.
</Note>

<div id="example">
  ## Пример
</div>

```sql title="Query"
SELECT * FROM sqlite('sqlite.db', 'table1') ORDER BY col2;
```

```text title="Response"
┌─col1──┬─col2─┐
│ line1 │    1 │
│ line2 │    2 │
│ line3 │    3 │
└───────┴──────┘
```
