Эти настройки доступны в [system.merge&#95;tree&#95;settings](/ru/reference/system-tables/merge_tree_settings) и автоматически создаются на основе исходного кода ClickHouse.

<div id="max_number_of_merges_with_ttl_in_pool">
  ## max_number_of_merges_with_ttl_in_pool
</div>

<SettingsInfoBlock type="UInt64" default_value="2" />

Если в
пуле число слияний с TTL превышает указанное значение, не назначайте
новые слияния с TTL. Это нужно, чтобы оставить свободные потоки для обычных слияний и
избежать ошибки &quot;Too many parts&quot;

<div id="max_number_of_mutations_for_replica">
  ## max_number_of_mutations_for_replica
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Ограничивает количество мутаций частей для каждой реплики указанным значением.
Ноль означает, что количество мутаций на реплику не ограничено (выполнение
по-прежнему может ограничиваться другими настройками).
