Эти настройки доступны в [system.merge&#95;tree&#95;settings](/ru/reference/system-tables/merge_tree_settings) и автоматически сгенерированы на основе исходного кода ClickHouse.

<div id="materialize_skip_indexes_on_merge">
  ## materialize_skip_indexes_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.1"},{"label": "1"},{"label": "новая настройка"}]}]} />

Если параметр включен, при слиянии для новых частей создаются и сохраняются индексы пропуска данных.
В противном случае их можно создать/сохранить явной командой [MATERIALIZE INDEX](/ru/reference/statements/alter/skipping-index#materialize-index)
или [при INSERT](/ru/reference/settings/session-settings#materialize_skip_indexes_on_insert).

См. также [exclude&#95;materialize&#95;skip&#95;indexes&#95;on&#95;merge](/ru/reference/settings/merge-tree-settings/exclude#exclude_materialize_skip_indexes_on_merge) для более тонкой настройки.

<div id="materialize_statistics_on_merge">
  ## materialize_statistics_on_merge
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "1"},{"label": "новая настройка"}]}]} />

Если параметр включен, при слиянии для новых частей будет создаваться и сохраняться статистика.
В противном случае ее можно создать/сохранить явной командой [MATERIALIZE STATISTICS](/ru/reference/statements/alter/statistics)
или [во время INSERT](/ru/reference/settings/session-settings#materialize_statistics_on_insert)

<div id="materialize_ttl_recalculate_only">
  ## materialize_ttl_recalculate_only
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Пересчитывать информацию о TTL только при выполнении MATERIALIZE TTL
