Эти настройки доступны в [system.merge&#95;tree&#95;settings](/ru/reference/system-tables/merge_tree_settings) и автоматически создаются на основе исходного кода ClickHouse.

<div id="primary_key_compress_block_size">
  ## primary_key_compress_block_size
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="65536" />

Размер блока сжатия для первичного ключа; фактический размер сжимаемого блока.

<div id="primary_key_compression_codec">
  ## primary_key_compression_codec
</div>

<SettingsInfoBlock type="String" default_value="ZSTD(3)" />

Кодек сжатия, используемый для первичного ключа; первичный ключ достаточно мал и кэшируется,
поэтому по умолчанию используется сжатие ZSTD(3).

<div id="primary_key_lazy_load">
  ## primary_key_lazy_load
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

Загружать первичный ключ в память
при первом использовании, а не при инициализации таблицы. Это может сэкономить память
при большом количестве таблиц.

<div id="primary_key_ratio_of_unique_prefix_values_to_skip_suffix_columns">
  ## primary_key_ratio_of_unique_prefix_values_to_skip_suffix_columns
</div>

<SettingsInfoBlock type="Float" default_value="0.9" />

Если значение столбца первичного ключа в части данных изменяется как минимум
с такой частотой, загрузка следующих столбцов в память пропускается. Это позволяет экономить
память, не загружая бесполезные столбцы первичного ключа.
