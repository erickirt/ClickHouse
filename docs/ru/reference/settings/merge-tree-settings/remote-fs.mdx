Эти настройки доступны в [system.merge&#95;tree&#95;settings](/ru/reference/system-tables/merge_tree_settings) и автоматически генерируются из исходного кода ClickHouse.

<div id="remote_fs_execute_merges_on_single_replica_time_threshold">
  ## remote_fs_execute_merges_on_single_replica_time_threshold
</div>

<SettingsInfoBlock type="Seconds" default_value="10800" />

Если значение этой настройки больше нуля и слитая часть находится в общем хранилище, слияние немедленно запускает только одна реплика.

<Note>
  Репликация с нулевым копированием не готова к использованию в продакшне
  По умолчанию репликация с нулевым копированием отключена в ClickHouse версии 22.8 и
  выше.

  Эту возможность не рекомендуется использовать в продакшне.
</Note>

Возможные значения:

* Любое положительное целое число.

<div id="remote_fs_zero_copy_path_compatible_mode">
  ## remote_fs_zero_copy_path_compatible_mode
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

Запускает zero-copy в режиме совместимости в процессе преобразования.

<div id="remote_fs_zero_copy_zookeeper_path">
  ## remote_fs_zero_copy_zookeeper_path
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="String" default_value="/clickhouse/zero_copy" />

Путь в ZooKeeper для не зависящей от таблицы информации zero-copy.
