Эти настройки доступны в [system.merge&#95;tree&#95;settings](/ru/reference/system-tables/merge_tree_settings) и автоматически генерируются на основе исходного кода ClickHouse.

<div id="escape_index_filenames">
  ## escape_index_filenames
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": "1"},{"label": "Экранировать не-ASCII-символы в именах файлов, создаваемых для индексов"}]}]} />

До версии 26.1 специальные символы в именах файлов, создаваемых для вторичных индексов, не экранировались, что могло приводить к проблемам: некоторые символы в именах индексов вызывали появление поврежденных частей. Это изменение добавлено исключительно из соображений совместимости. Не следует менять это значение, если только вы не читаете старые части с индексами, в именах которых используются не-ASCII-символы.
