  ## allow_dimensions_outside_sorting_key
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Теперь AggregatingMergeTree отклоняет при создании таблицы схемы, в которых столбец не входит ни в ключ сортировки, ни в меру — состояние агрегатной функции; ранее такие схемы допускались (старое поведение соответствует значению 'true')."}]}]} />

В `AggregatingMergeTree` фоновые слияния схлопывают строки с одинаковым значением ключа сортировки,
объединяя их состояния агрегатной функции. Столбец, который не входит ни в ключ сортировки, ни в
*меру* — состояние агрегатной функции (`AggregateFunction` или `SimpleAggregateFunction`), считается *измерением*: после
слияния в нём сохраняется произвольное значение из одной из схлопнутых строк, что незаметно приводит к неверным
результатам в запросах, которые используют `GROUP BY` или фильтрацию по этому столбцу (см.
https://github.com/ClickHouse/ClickHouse/issues/751).

По умолчанию такая схема отклоняется при создании таблицы. Включите этот параметр, если столбец
функционально зависит от ключа сортировки (то есть все схлопнутые строки имеют одно и то же значение) и такое
поведение является намеренным.

Проверка применяется только к таблицам, которые действительно агрегируют данные (то есть имеют хотя бы один столбец
с состоянием агрегатной функции), и пропускается, если включён `allow_tuple_element_aggregation`.

<div id="allow_tuple_element_aggregation">
  ## allow_tuple_element_aggregation
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "0"},{"label": "новая настройка"}]}]} />

Если настройка включена, отдельные элементы в столбцах Tuple участвуют в
агрегации при merge в SummingMergeTree, AggregatingMergeTree и
CoalescingMergeTree. Вложенные Tuple рекурсивно разворачиваются, так что все
конечные элементы агрегируются независимо. Эта настройка неизменяема
и должна быть указана при создании таблицы.
