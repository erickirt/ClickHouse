  ## map_serialization_version
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "Добавлена настройка для управления версией сериализации Map"}]}]} />

Управляет методом сериализации, используемым для столбцов `Map`.

Возможные значения:

* basic — использовать стандартную сериализацию для `Map`.
* with&#95;buckets — разбивать ключи по бакетам при сериализации. Использование бакетов улучшает чтение отдельных ключей из `Map`.

Количество бакетов в сериализации `with_buckets` определяется параметрами [max&#95;buckets&#95;in&#95;map](/ru/reference/settings/merge-tree-settings/max#max_buckets_in_map) и [map&#95;buckets&#95;strategy](/ru/reference/settings/merge-tree-settings/map-buckets#map_buckets_strategy).

<div id="map_serialization_version_for_zero_level_parts">
  ## map_serialization_version_for_zero_level_parts
</div>

<SettingsInfoBlock type="MergeTreeMapSerializationVersion" default_value="basic" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "basic"},{"label": "Добавлена настройка для управления версией сериализации Map для частей нулевого уровня"}]}]} />

Эта настройка позволяет указать другую версию сериализации
столбцов `Map` для частей нулевого уровня, создаваемых при вставке.
Это может быть полезно, если для частей нулевого уровня нужно сохранить сериализацию `basic`, чтобы избежать
снижения производительности при вставке, и при этом использовать `with_buckets` для слитых частей.
