  ## table_readonly
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Новая настройка для перевода таблицы в режим только для чтения, предотвращающий вставки и изменения"}]}]} />

Если установлено значение true, таблица переводится в режим только для чтения и не выполняет никаких изменений на диске.

Отклоняются все операции переднего плана, которые могли бы изменить таблицу: вставки, мутации, `OPTIMIZE` и команды партиций, изменяющие данные
(`ATTACH`/`MOVE`/`DROP`/`DROP DETACHED`/`FETCH`/`REPLACE PARTITION`, а также `MOVE PARTITION ... TO TABLE`, если эта таблица является целевой). Операции,
которые не изменяют данные таблицы, такие как `FREEZE`/`UNFREEZE` и `FORGET PARTITION`, остаются разрешёнными.

Фоновые задачи также не планируются: обычные слияния, TTL-слияния (`DELETE`/`MOVE`/повторное сжатие), слияния с повторным сжатием, фоновые мутации
и фоновые перемещения частей — всё это подавляется. В результате таблица с TTL больше не удаляет и не перемещает устаревшие данные, пока эта настройка
включена.

Эту настройку всегда можно отключить с помощью `ALTER TABLE ... MODIFY SETTING table_readonly = 0` (или `RESET SETTING`). Она не поддерживается для `ReplicatedMergeTree`.
