Когда false (по умолчанию), AST-фаззер на стороне сервера (управляется через `ast_fuzzer_runs`) выполняет фаззинг только запросов только для чтения (SELECT, EXPLAIN, SHOW, DESCRIBE, EXISTS). Когда true, фаззингу подвергаются все типы запросов, включая DDL и INSERT.

  ## ast_fuzzer_runs
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Float" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "Новая настройка для включения AST-фаззера на стороне сервера."}]}]} />

Включает AST-фаззер на стороне сервера, который после каждого обычного запроса запускает рандомизированные запросы и отбрасывает их результаты.

* 0: отключено (по умолчанию).
* Значение между 0 и 1 (не включая границы): вероятность запуска одного фаззинг-запроса.
* Значение &gt;= 1: количество фаззинг-запросов, запускаемых на каждый обычный запрос.

Фаззер накапливает фрагменты AST из всех запросов во всех сеансах, со временем создавая всё более интересные мутации. Фаззинг-запросы, завершившиеся ошибкой, просто отбрасываются; результаты клиенту никогда не возвращаются.
