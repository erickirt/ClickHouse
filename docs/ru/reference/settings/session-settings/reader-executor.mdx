Граница дочитывания для экспериментального `ReaderExecutor`: если долгоживущее соединение с источником было сброшено в пределах этого количества байтов от своей правой границы, оно сначала дочитывается до границы, чтобы корректно завершиться и вернуться в пул соединений для повторного использования, а не учитываться как незавершённое соединение.

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "2097152"},{"label": "Новая экспериментальная настройка ReaderExecutor: порог прямого пропуска разрыва при сквозном чтении через удерживаемое соединение с источником."}]}]} />

Порог прямого пропуска разрыва для экспериментального `ReaderExecutor`: разрыв до этого значения пропускается через открытое соединение с источником (сквозным чтением) вместо выполнения отдельного чтения из источника или повторного открытия соединения. Установите значение близким к точке баланса между затратами на пропускную способность и стоимостью запроса, чтобы сквозное чтение оставалось выгодным.

</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "Новая экспериментальная настройка ReaderExecutor (по умолчанию отключена): повторно использовать удерживаемое соединение с источником между последовательными окнами."}]}]} />

Повторно использует в экспериментальном `ReaderExecutor` долгоживущее соединение с источником с ограниченным временем жизни между окнами. Долгоживущее соединение — это соединение, диапазон которого выходит за пределы текущего окна чтения; если настройка отключена, исполнитель не расходует бюджет пула соединений, и для каждого окна открывается краткоживущее одноразовое соединение (путь без сохранения состояния).
