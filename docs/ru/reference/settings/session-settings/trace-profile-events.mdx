<div id="trace_profile_events">
  ## trace_profile_events
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Включает или отключает сбор трассировок стека при каждом обновлении счетчиков событий профилирования вместе с именем события профилирования и значением приращения, а также отправку этих данных в [trace&#95;log](/ru/reference/system-tables/trace_log).

Возможные значения:

* 1 — Трассировка событий профилирования включена.
* 0 — Трассировка событий профилирования отключена.

<div id="trace_profile_events_list">
  ## trace_profile_events_list
</div>

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.1"},{"label": ""},{"label": "новая настройка"}]}]} />

Когда настройка `trace_profile_events` включена, трассируемые события ограничиваются указанным списком имен, разделенных запятыми.
Если `trace_profile_events_list` — пустая строка (по умолчанию), трассируются все события профилирования.

Пример значения: &#39;DiskS3ReadMicroseconds,DiskS3ReadRequestsCount,SelectQueryTimeMicroseconds,ReadBufferFromS3Bytes&#39;

Эта настройка позволяет точнее собирать данные для большого числа запросов, поскольку в противном случае очень большой объем событий может переполнить внутреннюю очередь системного журнала, и часть из них будет отброшена.
