Эти настройки доступны в [system.settings](/ru/reference/system-tables/settings) и автоматически генерируются на основе [исходного файла](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="memory_profiler_sample_max_allocation_size">
  ## memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Собирает случайные выделения памяти размером не больше указанного значения с вероятностью, равной `memory_profiler_sample_probability`. Значение 0 означает, что параметр отключен. Возможно, потребуется установить `max_untracked_memory` в 0, чтобы этот порог работал как ожидается.

<div id="memory_profiler_sample_min_allocation_size">
  ## memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

Собирать случайные выделения памяти размером не меньше указанного значения с вероятностью `memory_profiler_sample_probability`. Значение 0 означает, что параметр отключён. Возможно, стоит установить &#39;max&#95;untracked&#95;memory&#39; в 0, чтобы этот порог работал как ожидается.

<div id="memory_profiler_sample_probability">
  ## memory_profiler_sample_probability
</div>

<SettingsInfoBlock type="Float" default_value="0" />

Собирает случайные выделения и освобождения памяти и записывает их в system.trace&#95;log с trace&#95;type &#39;MemorySample&#39;. Вероятность применяется к каждой операции alloc/free независимо от размера выделения памяти (это можно изменить с помощью `memory_profiler_sample_min_allocation_size` и `memory_profiler_sample_max_allocation_size`). Обратите внимание, что сэмплирование происходит только тогда, когда объём неотслеживаемой памяти превышает &#39;max&#95;untracked&#95;memory&#39;. Для ещё более детального сэмплирования можно установить &#39;max&#95;untracked&#95;memory&#39; в 0.

<div id="memory_profiler_step">
  ## memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="4194304" />

Задаёт шаг профилировщика памяти. Каждый раз, когда использование памяти запросом превышает очередной шаг, заданный в байтах, профилировщик памяти собирает трассировку стека выделения памяти и записывает её в [trace&#95;log](/ru/reference/system-tables/trace_log).

Возможные значения:

* Положительное целое число байтов.

* 0 — для отключения профилировщика памяти.
