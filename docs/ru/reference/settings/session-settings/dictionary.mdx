Эти настройки доступны в [system.settings](/ru/reference/system-tables/settings) и автоматически генерируются из [исходного кода](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

<div id="dictionary_lazy_load">
  ## dictionary_lazy_load
</div>

<SettingsInfoBlock type="BoolAuto" default_value="auto" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "auto"},{"label": "Новая настройка, переопределяющая настройку сервера `dictionaries_lazy_load` для отдельного словаря."}]}]} />

Управляет загрузкой словаря, если указана в секции `SETTINGS` команды `CREATE DICTIONARY`: `1` откладывает загрузку до первого использования, `0` загружает словарь при создании, `'auto'` использует настройку сервера `dictionaries_lazy_load`. Не имеет эффекта, если установлена на уровне сеанса или запроса.

<div id="dictionary_use_async_executor">
  ## dictionary_use_async_executor
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Выполняет конвейер чтения источника словаря в нескольких потоках. Поддерживается только для словарей с локальным источником CLICKHOUSE.
