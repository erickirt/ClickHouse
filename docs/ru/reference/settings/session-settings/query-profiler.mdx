<div id="query_profiler_cpu_time_period_ns">
  ## query_profiler_cpu_time_period_ns
</div>

<SettingsInfoBlock type="UInt64" default_value="1000000000" />

Задаёт период таймера CPU для [профилировщика запросов](/ru/concepts/features/performance/troubleshoot/sampling-query-profiler). Этот таймер учитывает только процессорное время.

Возможные значения:

* Положительное целое число в наносекундах.

  Рекомендуемые значения:

  * 10000000 (100 раз в секунду) наносекунд и более для отдельных запросов.
  * 1000000000 (один раз в секунду) для профилирования в масштабе всего кластера.

* 0 для отключения таймера.

См. также:

* Системная таблица [trace&#95;log](/ru/reference/system-tables/trace_log)

<div id="query_profiler_real_time_period_ns">
  ## query_profiler_real_time_period_ns
</div>

<SettingsInfoBlock type="UInt64" default_value="1000000000" />

Задаёт период таймера реального времени для [профилировщика запросов](/ru/concepts/features/performance/troubleshoot/sampling-query-profiler). Таймер реального времени измеряет фактически прошедшее время.

Возможные значения:

* Положительное целое число в наносекундах.

  Рекомендуемые значения:

  * 10000000 наносекунд (100 раз в секунду) и меньше для отдельных запросов.
  * 1000000000 (один раз в секунду) для профилирования всего кластера.

* 0, чтобы отключить таймер.

См. также:

* системная таблица [trace&#95;log](/ru/reference/system-tables/trace_log)

Значение по умолчанию в Cloud: `3000000000`.
