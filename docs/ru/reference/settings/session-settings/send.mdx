<div id="send_profile_events">
  ## send_profile_events
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.11"},{"label": "1"},{"label": "Новая настройка. Определяет, отправлять ли события профиля клиентам."}]}]} />

Включает или отключает отправку пакетов [ProfileEvents](/ru/resources/develop-contribute/native-protocol/server#profile-events) клиенту.

Этот параметр можно отключить, чтобы уменьшить сетевой трафик для клиентов, которым не нужны события профиля.

Возможные значения:

* 0 — Отключено.
* 1 — Включено.

<div id="send_progress_in_http_headers">
  ## send_progress_in_http_headers
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Включает или отключает HTTP-заголовки `X-ClickHouse-Progress` в ответах `clickhouse-server`.

Дополнительные сведения см. в [описании HTTP-интерфейса](/ru/concepts/features/interfaces/http).

Возможные значения:

* 0 — Отключено.
* 1 — Включено.

<div id="send_table_structure_on_insert_with_inline_data">
  ## send_table_structure_on_insert_with_inline_data
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "1"},{"label": "Новая настройка, определяющая, отправляет ли сервер структуру таблицы для запросов INSERT со встроенными данными."}]}]} />

Если параметр отключён и запрос INSERT содержит встроенные данные, сервер не будет отправлять клиенту по собственному протоколу структуру таблицы и значения столбцов по умолчанию. Вместо этого сервер будет сам разбирать встроенные данные. Это может повысить производительность при большом количестве небольших вставок по собственному протоколу.

<div id="send_timeout">
  ## send_timeout
</div>

<SettingsInfoBlock type="Seconds" default_value="300" />

Тайм-аут отправки данных по сети, в секундах. Если клиенту нужно отправить данные, но за этот промежуток времени не удаётся отправить ни одного байта, генерируется исключение. Если задать этот параметр на клиенте, то на соответствующей стороне соединения на сервере для сокета также будет установлен &#39;receive&#95;timeout&#39;.
