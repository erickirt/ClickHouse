Эти настройки доступны в [system.settings](/ru/reference/system-tables/settings) и автоматически генерируются на основе [исходного кода](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

  ## jemalloc_profile_text_collapsed_use_count

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "0"},{"label": "Новая настройка для агрегирования по количеству выделений вместо байтов в collapsed-формате профиля кучи jemalloc"}]}]} />

При использовании формата вывода `collapsed` для профиля кучи jemalloc агрегирование выполняется по количеству выделений, а не по байтам. Если значение `false` (по умолчанию), каждый стек взвешивается по объёму используемых байтов; если `true` — по количеству активных выделений.

  ## jemalloc_profile_text_output_format

Формат вывода профиля кучи jemalloc в таблице system.jemalloc&#95;profile&#95;text. Может быть: &#39;raw&#39; (необработанный профиль), &#39;symbolized&#39; (формат jeprof с символами) или &#39;collapsed&#39; (формат флеймграфа).

  ## jemalloc_profile_text_symbolize_with_inline

Определяет, включать ли inline-фреймы при символизации профиля кучи jemalloc. Если включено, inline-фреймы добавляются, что может существенно замедлить процесс символизации; если отключено, они пропускаются. Влияет только на форматы вывода &#39;symbolized&#39; и &#39;collapsed&#39;.
