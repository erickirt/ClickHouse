Добавляет сравнения с константой в цепочках AND для улучшения фильтрации. Поддерживаются операторы `<`, `<=`, `>`, `>=`, `=` и их сочетания. Например, `(a < b) AND (b < c) AND (c < 5)` будет преобразовано в `(a < b) AND (b < c) AND (c < 5) AND (b < 5) AND (a < 5)`.

Бюджет работы для оптимизации `optimize_and_compare_chain` на этапе анализа запроса, измеряемый количеством узлов дерева запроса, хешируемых функцией `getTreeHash` (это основная стоимость данной оптимизации). Как только при применении этой оптимизации запрос хеширует больше узлов, чем задано этим значением, она перестаёт применяться для оставшейся части запроса. Это ограничивает время анализа для запросов с очень большим числом или очень длинными цепочками сравнений, объединённых `AND`, где иначе эта оптимизация может занимать основное время анализа, не давая никакого эффекта свёртки. Преждевременная остановка всегда безопасна: она лишь означает отказ от оптимизации и никогда не меняет результаты. Установите `0`, чтобы отключить бюджет (без ограничений).
