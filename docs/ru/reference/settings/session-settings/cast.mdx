<div id="cast_keep_nullable">
  ## cast_keep_nullable
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Включает или отключает сохранение типа данных `Nullable` в операциях [CAST](/ru/reference/functions/regular-functions/type-conversion-functions#CAST).

Если параметр включен и аргумент функции `CAST` имеет тип `Nullable`, результат также преобразуется в тип `Nullable`. Если параметр отключен, результат всегда имеет в точности указанный целевой тип.

Возможные значения:

* 0 — Результат `CAST` имеет в точности указанный целевой тип.
* 1 — Если аргумент имеет тип `Nullable`, результат `CAST` преобразуется в `Nullable(DestinationDataType)`.

**Примеры**

Следующий запрос возвращает результат с в точности указанным целевым типом данных:

```sql
SET cast_keep_nullable = 0;
SELECT CAST(toNullable(toInt32(0)) AS Int32) as x, toTypeName(x);
```

Результат:

```text
┌─x─┬─toTypeName(CAST(toNullable(toInt32(0)), 'Int32'))─┐
│ 0 │ Int32                                             │
└───┴───────────────────────────────────────────────────┘
```

В результате следующего запроса к типу данных пункта назначения применяется модификатор `Nullable`:

```sql
SET cast_keep_nullable = 1;
SELECT CAST(toNullable(toInt32(0)) AS Int32) as x, toTypeName(x);
```

Результат:

```text
┌─x─┬─toTypeName(CAST(toNullable(toInt32(0)), 'Int32'))─┐
│ 0 │ Nullable(Int32)                                   │
└───┴───────────────────────────────────────────────────┘
```
