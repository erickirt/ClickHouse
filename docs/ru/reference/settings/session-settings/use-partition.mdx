  ## use_partition_minmax_for_primary_key_pruning
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "Новая настройка, позволяющая использовать minmax-индекс партиции части для отсечения большего числа гранул при анализе первичного ключа для таблиц `MergeTree`, когда столбец первичного ключа также входит в ключ партиционирования."}]}]} />

Использует границы minmax-индекса партиции для отсечения большего числа гранул при анализе первичного ключа для таблиц `MergeTree`, когда столбец первичного ключа также входит в ключ партиционирования.

Например, в таблице `MergeTree` с `ORDER BY (id, event_time)` и `PARTITION BY toYYYYMM(event_time)` ClickHouse будет использовать minmax-индекс партиции по `event_time` при анализе индекса первичного ключа, чтобы точнее определять, какие гранулы можно отсечь.

Возможные значения:

* 0 — отключено.
* 1 — включено.
