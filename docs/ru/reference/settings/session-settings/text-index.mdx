<div id="text_index_like_max_postings_to_read">
  ## text_index_like_max_postings_to_read
</div>

<SettingsInfoBlock type="UInt64" default_value="50" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "50"},{"label": "новая настройка"}]}]} />

Максимальное количество крупных postings, считываемых при включенной оценке LIKE для текстового индекса с помощью сканирования словаря.

Требует включения `use_text_index_like_evaluation_by_dictionary_scan`.

<div id="text_index_like_min_pattern_length">
  ## text_index_like_min_pattern_length
</div>

<SettingsInfoBlock type="UInt64" default_value="4" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "4"},{"label": "новая настройка"}]}]} />

Минимальная длина буквенно-цифровой подстроки в шаблоне LIKE/ILIKE, необходимая для использования оценки LIKE текстового индекса при сканировании словаря.
Шаблоны короче этого порога соответствуют слишком большому количеству токенов словаря и пропускаются во избежание ресурсоемкого сканирования.

Требует, чтобы `use_text_index_like_evaluation_by_dictionary_scan` был включен.

<div id="text_index_posting_list_apply_mode">
  ## text_index_posting_list_apply_mode
</div>

<SettingsInfoBlock type="TextIndexPostingListApplyMode" default_value="materialize" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.6"},{"label": "materialize"},{"label": "Новая настройка для режима отложенного применения списков вхождений"}]}]} />

Управляет способом применения списков вхождений в запросах, использующих текстовый индекс.
&#39;materialize&#39; (по умолчанию) сразу декодирует списки вхождений в Roaring Bitmaps.
&#39;lazy&#39; использует декодирование по требованию на основе курсора (требуется формат индекса с сериализованным кодеком).
