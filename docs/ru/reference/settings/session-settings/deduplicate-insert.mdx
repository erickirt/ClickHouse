  ## deduplicate_insert
</div>

<SettingsInfoBlock type="DeduplicateInsertMode" default_value="enable" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.2"},{"label": "enable"},{"label": "По умолчанию включает дедупликацию для всех синхронных и асинхронных вставок."}]}]} />

Включает или отключает дедупликацию блоков при `INSERT INTO` (для таблиц Replicated*).
Этот параметр переопределяет параметры `insert_deduplicate` и `async_insert_deduplicate`.
У этого параметра есть три возможных значения:

* disable — Дедупликация отключена для запроса `INSERT INTO`.
* enable — Дедупликация включена для запроса `INSERT INTO`.
* backward&#95;compatible&#95;choice — Дедупликация включается, если для соответствующего типа вставки включен `insert_deduplicate` или `async_insert_deduplicate`.

Включает или отключает дедупликацию блоков для `INSERT SELECT` (для таблиц Replicated*).
Эта настройка переопределяет `insert_deduplicate` и `deduplicate_insert` для запросов `INSERT SELECT`.
У этой настройки есть четыре возможных значения:

* disable — Дедупликация для запроса `INSERT SELECT` отключена.
* force&#95;enable — Дедупликация для запроса `INSERT SELECT` включена. Если результат SELECT нестабилен, генерируется исключение.
* enable&#95;when&#95;possible — Дедупликация включается, если `insert_deduplicate` включен и результат SELECT стабилен, в противном случае отключается.
* enable&#95;even&#95;for&#95;bad&#95;queries - Дедупликация включается, если `insert_deduplicate` включен. Если результат SELECT нестабилен, в журнал записывается предупреждение, но запрос выполняется с дедупликацией. Этот параметр предназначен для обратной совместимости. Рекомендуется использовать другие варианты, так как этот может приводить к неожиданным результатам.
