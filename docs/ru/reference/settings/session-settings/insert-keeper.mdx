<div id="insert_keeper_max_retries">
  ## insert_keeper_max_retries
</div>

<SettingsInfoBlock type="UInt64" default_value="20" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "23.2"},{"label": "20"},{"label": "Включены повторные подключения к Keeper при INSERT, повышена надежность"}]}]} />

Этот параметр задает максимальное число повторных попыток для запросов к ClickHouse Keeper (или ZooKeeper) при вставке в реплицируемый MergeTree. Повторно выполняются только те запросы к Keeper, которые завершились ошибкой из-за сетевой ошибки, тайм-аута сеанса Keeper или тайм-аута запроса.

Возможные значения:

* Положительное целое число.
* 0 — повторные попытки отключены

Повторные попытки запросов к Keeper выполняются через некоторый тайм-аут. Он задается следующими параметрами: `insert_keeper_retry_initial_backoff_ms`, `insert_keeper_retry_max_backoff_ms`.
Первая повторная попытка выполняется по истечении тайм-аута `insert_keeper_retry_initial_backoff_ms`. Последующие тайм-ауты рассчитываются следующим образом:

```
timeout = min(insert_keeper_retry_max_backoff_ms, latest_timeout * 2)
```

Например, если `insert_keeper_retry_initial_backoff_ms=100`, `insert_keeper_retry_max_backoff_ms=10000` и `insert_keeper_max_retries=8`, то значения тайм-аутов будут `100, 200, 400, 800, 1600, 3200, 6400, 10000`.

Помимо отказоустойчивости, повторные попытки также улучшают пользовательский опыт: они позволяют не возвращать ошибку при выполнении INSERT, если Keeper, например, перезапускается из-за обновления.

  ## insert_keeper_retry_initial_backoff_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="100" />

Начальный тайм-аут (в миллисекундах) перед повторной попыткой выполнить неудачный запрос к Keeper во время выполнения запроса INSERT

* Положительное целое число.
* 0 — без тайм-аута

  ## insert_keeper_retry_max_backoff_ms

Максимальный тайм-аут (в миллисекундах) перед повторной попыткой после сбоя запроса к Keeper при выполнении INSERT-запроса

* Положительное целое число.
