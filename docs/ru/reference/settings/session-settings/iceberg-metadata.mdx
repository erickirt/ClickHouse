  ## iceberg_metadata_log_level
</div>

<SettingsInfoBlock type="IcebergMetadataLogLevel" default_value="none" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.9"},{"label": "none"},{"label": "Новая настройка"}]}]} />

Управляет уровнем журналирования метаданных для таблиц Iceberg в журнале system.iceberg&#95;metadata&#95;log.
Обычно эту настройку можно изменять для отладки.

Возможные значения:

* none - Журналирование метаданных отключено.
* metadata - Корневой файл metadata.json.
* manifest&#95;list&#95;metadata - Всё вышеперечисленное + метаданные из avro-файла manifest list, соответствующего снимку.
* manifest&#95;list&#95;entry - Всё вышеперечисленное + записи из avro-файла manifest list.
* manifest&#95;file&#95;metadata - Всё вышеперечисленное + метаданные из обработанных avro manifest files.
* manifest&#95;file&#95;entry - Всё вышеперечисленное + записи из обработанных avro manifest files.

  ## iceberg_metadata_staleness_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Новая настройка, позволяющая использовать кэшированную версию метаданных при операциях READ, чтобы избежать получения данных из удалённого каталога"}]}]} />

Если значение не равно нулю, получение метаданных Iceberg из удалённого каталога пропускается, если существует кэшированный снимок метаданных, более новый, чем заданное окно устаревания. Ноль означает, что из удалённого каталога всегда будет получаться последняя версия метаданных. Установка для этой настройки ненулевого значения снижает задержку операций чтения ценой использования устаревших данных.
