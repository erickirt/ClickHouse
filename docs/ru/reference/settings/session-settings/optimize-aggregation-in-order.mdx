<div id="optimize_aggregation_in_order">
  ## optimize_aggregation_in_order
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Включает оптимизацию [GROUP BY](/ru/reference/statements/select/group-by) в запросах [SELECT](/ru/reference/statements/select/index) для агрегации данных в соответствующем порядке в таблицах [MergeTree](/ru/reference/engines/table-engines/mergetree-family/mergetree).

Возможные значения:

* 0 — оптимизация `GROUP BY` отключена.
* 1 — оптимизация `GROUP BY` включена.

**См. также**

* [Оптимизация GROUP BY](/ru/reference/statements/select/group-by#group-by-optimization-depending-on-table-sorting-key)

<div id="optimize_aggregation_in_order_limit">
  ## optimize_aggregation_in_order_limit
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "Новая настройка, позволяющая применять `LIMIT` на этапе aggregation-in-order для раннего завершения, когда `ORDER BY` является префиксом описания сортировки `GROUP BY`."}]}]} />

Когда настройка включена и активна aggregation in order, `LIMIT` применяется на этапе агрегации, что позволяет завершить выполнение раньше, как только будет сформировано достаточное количество групп. Это уменьшает объём читаемых данных, когда `ORDER BY` совпадает с префиксом ключа `GROUP BY`. Значение `rows_before_limit_at_least` при этом может уменьшиться; если нужны точные значения, используйте `exact_rows_before_limit`.
