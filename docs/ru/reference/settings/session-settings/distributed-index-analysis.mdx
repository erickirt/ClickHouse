  ## distributed_index_analysis

Анализ индексов будет распределяться между репликами.
Полезно при использовании общего хранилища и очень больших объёмов данных в кластере.
Использует реплики из cluster&#95;for&#95;parallel&#95;replicas.

<div id="distributed_index_analysis_only_on_coordinator">
  ## distributed_index_analysis_only_on_coordinator
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.4"},{"label": "0"},{"label": "Новая настройка"}]}]} />

Если включено, распределенный анализ индексов выполняется только на координаторе.
Это предотвращает появление запросов O(N^2), когда предикат содержит подзапросы (например, `IN (SELECT ...)`),
поскольку в противном случае каждая реплика-follower независимо запускала бы собственный распределенный анализ индексов,
но снижает эффективность распределенного анализа индексов, если в подзапросах используются большие таблицы.
