Эти настройки доступны в [system.settings](/ru/reference/system-tables/settings) и автоматически генерируются на основе [исходного кода](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp).

  ## max_skip_unavailable_shards_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Новая настройка, ограничивающая количество сегментов, которые можно пропустить без генерации ошибки при включённом skip_unavailable_shards."}]}]} />

При включённом `skip_unavailable_shards` ограничивает максимальное число сегментов, которые можно пропустить без генерации ошибки.
Если число недоступных сегментов превышает это значение, вместо их пропуска генерируется исключение.

Значение 0 означает отсутствие ограничений (поведение по умолчанию — можно пропускать все недоступные сегменты).

  ## max_skip_unavailable_shards_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.3"},{"label": "0"},{"label": "Новая настройка для ограничения доли сегментов, которые могут быть молча пропущены при включенном `skip_unavailable_shards`."}]}]} />

Когда `skip_unavailable_shards` включен, ограничивает максимальную долю (от 0 до 1) сегментов, которые можно молча пропускать.
Если доля недоступных сегментов от общего числа сегментов превышает это значение, вместо их молчаливого пропуска генерируется исключение.

Значение 0 означает отсутствие ограничения (поведение по умолчанию — можно пропускать все недоступные сегменты).
