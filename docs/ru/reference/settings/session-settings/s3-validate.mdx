  ## s3_validate_etag_on_read

При чтении объекта из S3 (или S3-совместимого хранилища, такого как GCS) проверяйте, что каждый запрос GET возвращает тот же ETag, который был получен при листинге объекта. Одно чтение файла выполняет много диапазонных запросов GET; если объект будет перезаписан между ними (например, если внешний writer перезаписывает фиксированный ключ), иначе чтение может оказаться составленным из двух разных поколений объекта и привести к ошибке контрольной суммы или ошибке разбора. При обнаружении несоответствия чтение завершается с `S3_OBJECT_CHANGED_DURING_READ` вместо возврата несогласованных данных. Отключайте это только для рабочих нагрузок, которые намеренно читают объекты во время их перезаписи и могут допускать несогласованное чтение.
