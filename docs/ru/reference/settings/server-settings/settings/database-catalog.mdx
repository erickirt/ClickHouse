Эти настройки конфигурируют сервер ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="database_catalog_drop_error_cooldown_sec">
  ## database_catalog_drop_error_cooldown_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="5" changeable_without_restart="No" />Если удаление таблицы завершилось ошибкой, ClickHouse подождёт этот тайм-аут перед повторной попыткой.

<div id="database_catalog_drop_table_concurrency">
  ## database_catalog_drop_table_concurrency
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />Размер пула потоков, используемого для удаления таблиц.

<div id="database_catalog_unused_dir_cleanup_period_sec">
  ## database_catalog_unused_dir_cleanup_period_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="86400" changeable_without_restart="Нет" />

Параметр задачи, очищающей каталог `store/` от мусора.
Задаёт периодичность выполнения задачи.

<Note>
  Значение `0` означает «никогда». Значение по умолчанию соответствует 1 дню.
</Note>

<div id="database_catalog_unused_dir_hide_timeout_sec">
  ## database_catalog_unused_dir_hide_timeout_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="3600" changeable_without_restart="Нет" />

Параметр задачи, которая очищает каталог `store/` от мусора.
Если какой-либо подкаталог не используется clickhouse-server и в этот каталог не вносились изменения в течение последних
[`database_catalog_unused_dir_hide_timeout_sec`](/ru/reference/settings/server-settings/settings/database-catalog#database_catalog_unused_dir_hide_timeout_sec) секунд, задача «скроет» его,
удалив все права доступа. Это также работает для каталогов, которые clickhouse-server не
ожидает увидеть внутри `store/`.

<Note>
  Значение `0` означает «немедленно».
</Note>

<div id="database_catalog_unused_dir_rm_timeout_sec">
  ## database_catalog_unused_dir_rm_timeout_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="2592000" changeable_without_restart="No" />

Параметр задачи, которая очищает каталог `store/` от мусора.
Если какой-либо подкаталог не используется clickhouse-server и ранее был &quot;скрыт&quot;
(см. [database&#95;catalog&#95;unused&#95;dir&#95;hide&#95;timeout&#95;sec](/ru/reference/settings/server-settings/settings/database-catalog#database_catalog_unused_dir_hide_timeout_sec))
и этот каталог не изменялся в течение последних
[`database_catalog_unused_dir_rm_timeout_sec`]/operations/server-configuration-parameters/settings#database&#95;catalog&#95;unused&#95;dir&#95;rm&#95;timeout&#95;sec) секунд, задача удалит этот каталог.
Это также работает для каталогов, которые clickhouse-server не
ожидает увидеть внутри `store/`.

<Note>
  Значение `0` означает &quot;никогда&quot;. Значение по умолчанию соответствует 30 дням.
</Note>
