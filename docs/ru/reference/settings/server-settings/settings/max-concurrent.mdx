Эти настройки задают параметры сервера ClickHouse и автоматически сгенерированы на основе исходного кода ClickHouse.

<div id="max_concurrent_insert_queries">
  ## max_concurrent_insert_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Ограничение на общее количество одновременно выполняемых запросов на вставку.

<Note>
  Значение `0` (по умолчанию) означает отсутствие ограничений.

  Этот параметр можно изменить во время работы и он вступит в силу немедленно. Уже выполняющиеся запросы останутся без изменений.
</Note>

<div id="max_concurrent_queries">
  ## max_concurrent_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Ограничение на общее число одновременно выполняемых запросов. Обратите внимание, что также следует учитывать ограничения на запросы `INSERT` и `SELECT`, а также максимальное количество запросов для пользователей.

См. также:

* [`max_concurrent_insert_queries`](/ru/reference/settings/server-settings/settings/max-concurrent#max_concurrent_insert_queries)
* [`max_concurrent_select_queries`](/ru/reference/settings/server-settings/settings/max-concurrent#max_concurrent_select_queries)
* [`max_concurrent_queries_for_all_users`](/ru/reference/settings/session-settings#max_concurrent_queries_for_all_users)

<Note>
  Значение `0` (по умолчанию) означает отсутствие ограничений.

  Этот параметр можно изменять во время работы, и изменения вступят в силу немедленно. Уже выполняющиеся запросы останутся без изменений.
</Note>

<div id="max_concurrent_select_queries">
  ## max_concurrent_select_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Ограничение на общее число одновременно выполняемых SELECT-запросов.

<Note>
  Значение `0` (по умолчанию) означает, что ограничения отсутствуют.

  Этот параметр можно изменять во время выполнения, и он вступает в силу немедленно. Уже выполняющиеся запросы останутся без изменений.
</Note>
