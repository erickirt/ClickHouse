Эти настройки задают параметры сервера ClickHouse и автоматически генерируются на основе исходного кода ClickHouse.

<div id="custom_cached_disks_base_directory">
  ## custom_cached_disks_base_directory
</div>

Эта настройка задаёт путь файлового кэша для пользовательских кэшированных дисков (созданных с помощью SQL).
`custom_cached_disks_base_directory` имеет более высокий приоритет для пользовательских дисков, чем `filesystem_caches_path` (указанный в `filesystem_caches_path.xml`),
который используется, если первый параметр отсутствует.
Путь настройки файлового кэша должен находиться внутри этого каталога,
иначе будет сгенерировано исключение, и диск не будет создан.

<Note>
  Это не повлияет на диски, созданные в более старой версии, для которых сервер был обновлён.
  В этом случае исключение не будет сгенерировано, чтобы сервер мог успешно запуститься.
</Note>

Пример:

```xml
<custom_cached_disks_base_directory>/var/lib/clickhouse/caches/</custom_cached_disks_base_directory>
```

<div id="custom_settings_prefixes">
  ## custom_settings_prefixes
</div>

Список префиксов, используемых для [пользовательских настроек](/ru/concepts/features/configuration/settings/settings-query-level#custom_settings).
Если префиксов несколько, разделяйте их запятыми.

**Пример**

```xml
<custom_settings_prefixes>SQL_</custom_settings_prefixes>
```
