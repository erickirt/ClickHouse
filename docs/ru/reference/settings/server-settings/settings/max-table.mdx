Эти настройки конфигурируют сервер ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Если число таблиц превышает это значение, сервер сгенерирует исключение.

Следующие таблицы не учитываются:

* view
* remote
* dictionary
* system

Учитываются только таблицы в базах данных со следующими движками:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  Значение `0` означает отсутствие ограничений.
</Note>

**Пример**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

Если количество присоединённых таблиц превышает указанное значение, сервер ClickHouse добавит предупреждения в таблицу `system.warnings`.

**Пример**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

Ограничение на удаление таблиц.

Если размер таблицы [MergeTree](/ru/reference/engines/table-engines/mergetree-family/mergetree) превышает `max_table_size_to_drop` (в байтах), вы не сможете удалить её с помощью запроса [`DROP`](/ru/reference/statements/drop) или [`TRUNCATE`](/ru/reference/statements/truncate).

<Note>
  Значение `0` означает, что вы можете удалять любые таблицы без ограничений.

  Для применения этой настройки не требуется перезапуск сервера ClickHouse. Ещё один способ отключить это ограничение — создать файл `<clickhouse-path>/flags/force_drop_table`.
</Note>

**Пример**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
