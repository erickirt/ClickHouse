Эти настройки задают конфигурацию сервера ClickHouse и автоматически сгенерированы на основе исходного кода ClickHouse.

<div id="background_schedule_pool_initial_size">
  ## background_schedule_pool_initial_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />Начальное количество рабочих потоков, выделяемых для каждого фонового пула планировщика. Каждый пул расширяется по мере необходимости вплоть до настроенного предела (`background_schedule_pool_size` и аналогичных параметров), когда нагрузка превышает текущее число воркеров. Более низкие значения уменьшают накладные расходы при запуске и число бездействующих потоков; более высокие устраняют задержку планирования первой задачи на загруженных серверах.

<div id="background_schedule_pool_log">
  ## background_schedule_pool_log
</div>

Содержит информацию обо всех фоновых задачах, выполняемых в различных фоновых пулах.

```xml
<background_schedule_pool_log>
    <database>system</database>
    <table>background_schedule_pool_log</table>
    <partition_by>toYYYYMM(event_date)</partition_by>
    <flush_interval_milliseconds>7500</flush_interval_milliseconds>
    <max_size_rows>1048576</max_size_rows>
    <reserved_size_rows>8192</reserved_size_rows>
    <buffer_size_rows_flush_threshold>524288</buffer_size_rows_flush_threshold>
    <flush_on_crash>false</flush_on_crash>
    <!-- Only tasks longer than duration_threshold_milliseconds will be logged. Zero means log everything -->
    <duration_threshold_milliseconds>0</duration_threshold_milliseconds>
</background_schedule_pool_log>
```

<div id="background_schedule_pool_max_parallel_tasks_per_type_ratio">
  ## background_schedule_pool_max_parallel_tasks_per_type_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.8" changeable_without_restart="No" />Максимальная доля потоков в пуле, которые могут одновременно выполнять задачи одного типа.

<div id="background_schedule_pool_size">
  ## background_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="512" changeable_without_restart="Только увеличение" />Предел количества потоков, используемых для выполнения легковесных периодических операций для таблиц с репликацией, стриминга Kafka, обновления DNS-кэша и подобных задач. Потоки создаются по мере необходимости вплоть до этого предела, поэтому сервер с небольшой фоновой нагрузкой использует значительно меньше потоков, чем задано этим значением.
