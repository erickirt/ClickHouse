Эти настройки относятся к серверу ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="oom_canary_enable">
  ## oom_canary_enable
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="Нет" />

Экспериментальный. Включает OOM-канарейку: жертвенный дочерний процесс, который привлекает Linux OOM killer
раньше основного процесса сервера ClickHouse, давая серверу возможность снизить нагрузку.
Требуется Linux &gt;= 5.3 (для `pidfd_open`); на более старых ядрах канарейка отключается при запуске.
Реакция на OOM требует наличия признаков OOM-kill в cgroup v2 `memory.events.local` и может запускать глобальную
отмену запросов, отмену слияний и запись в `system.crash_log`.
Канарейка не может защитить сервер, если для cgroup сервера включен cgroup v2 `memory.oom.group`:
в этом случае ядро завершает всю cgroup как единое целое, включая сервер,
поэтому реакция на OOM не запускается. При запуске в этом режиме записывается предупреждение.
Поведение может меняться от версии к версии ClickHouse, пока не будет завершена проверка в продакшне.

<div id="oom_canary_initial_backoff_seconds">
  ## oom_canary_initial_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="1" changeable_without_restart="No" />

Начальная задержка в секундах между последовательными перезапусками OOM-канарейки.
При каждом перезапуске задержка удваивается, пока не достигнет `oom_canary_max_backoff_seconds`.

<div id="oom_canary_max_backoff_seconds">
  ## oom_canary_max_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />

Максимальная задержка в секундах между последовательными перезапусками OOM-канарейки.

<div id="oom_canary_max_rapid_relaunches">
  ## oom_canary_max_rapid_relaunches
</div>

<SettingsInfoBlock type="UInt64" default_value="10" changeable_without_restart="Нет" />

Максимальное число последовательных быстрых перезапусков OOM-канарейки, после которого автоматический перезапуск отключается,
чтобы избежать перегрузки при устойчивом дефиците памяти. Счётчик и задержка перед перезапуском сбрасываются,
если канарейка работает дольше, чем `oom_canary_max_backoff_seconds`, поэтому канарейка, которая завершается
лишь изредка после длительного времени непрерывной работы, в итоге не будет отключена.
Применяется только если `oom_canary_relaunch` имеет значение true.

<div id="oom_canary_relaunch">
  ## oom_canary_relaunch
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

Если установлено значение true, OOM-канарейка автоматически перезапускается после завершения процесса канарейки
по любой причине, кроме постоянного сбоя настройки или остановки сервера, — включая завершение из-за OOM, другие
сигналы и временные завершения, — с учетом `oom_canary_max_rapid_relaunches` и параметров задержки.
Сама последовательность реакции на OOM запускается только тогда, когда cgroup v2 `memory.events.local`
предоставляет подтверждение OOM-kill.

<div id="oom_canary_size">
  ## oom_canary_size
</div>

<SettingsInfoBlock type="UInt64" default_value="104857600" changeable_without_restart="No" />

Размер в байтах области памяти, которую выделяет и к которой обращается дочерний процесс OOM-канарейки.
Закрепление этой области с помощью `mlock` выполняется по возможности: для этого требуется `CAP_IPC_LOCK` или достаточное значение
`RLIMIT_MEMLOCK`; если закрепить область не удаётся, канарейка записывает предупреждение в журнал, а память остаётся
выделенной, но может быть выгружена в swap.
Значение по умолчанию — 100 МБ (104857600). Чем больше значение, тем более привлекательной целью для OOM становится канарейка.
