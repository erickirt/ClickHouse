Эти настройки конфигурируют сервер ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="temporary_data_in_cache">
  ## temporary_data_in_cache
</div>

С помощью этой опции временные данные будут храниться в кэше указанного диска.
Здесь нужно указать имя диска с типом `cache`.
В этом случае кэш и временные данные будут использовать одно и то же пространство, а дисковый кэш может быть вытеснен для размещения временных данных.

<Note>
  Для настройки хранения временных данных можно использовать только одну опцию: `tmp_path` ,`tmp_policy`, `temporary_data_in_cache`.
</Note>

**Пример**

И кэш для `local_disk`, и временные данные будут храниться в `/tiny_local_cache` в файловой системе, управляемой `tiny_local_cache`.

```xml highlight={9-16,20}
<clickhouse>
<storage_configuration>
<disks>
<local_disk>
<type>local</type>
<path>/local_disk/</path>
</local_disk>

<tiny_local_cache>
<type>cache</type>
<disk>local_disk</disk>
<path>/tiny_local_cache/</path>
<max_size_rows>10M</max_size_rows>
<max_file_segment_size>1M</max_file_segment_size>
<cache_on_write_operations>1</cache_on_write_operations>
</tiny_local_cache>
</disks>
</storage_configuration>

<temporary_data_in_cache>tiny_local_cache</temporary_data_in_cache>
</clickhouse>
```
