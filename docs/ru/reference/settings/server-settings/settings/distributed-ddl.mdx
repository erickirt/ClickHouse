Эти настройки задают параметры сервера ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="distributed_ddl">
  ## distributed_ddl
</div>

Управляет выполнением [распределённых DDL-запросов](/ru/reference/statements/distributed-ddl) (`CREATE`, `DROP`, `ALTER`, `RENAME`) в кластере.
Работает только если [ZooKeeper](/ru/reference/settings/server-settings/settings/zookeeper#zookeeper) включен.

Настраиваемые параметры в `<distributed_ddl>` включают:

| Параметр               | Описание                                                                                                                                  | Значение по умолчанию                  |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| `cleanup_delay_period` | очистка начинается после получения события о новом узле, если с момента предыдущей очистки прошло не менее `cleanup_delay_period` секунд. | `60` секунд                            |
| `max_tasks_in_queue`   | максимальное количество задач, которое может находиться в очереди.                                                                        | `1,000`                                |
| `path`                 | путь в Keeper для `task_queue` DDL-запросов                                                                                               |                                        |
| `pool_size`            | сколько запросов с предложением `ON CLUSTER` может выполняться одновременно                                                               |                                        |
| `profile`              | профиль, используемый для выполнения DDL-запросов                                                                                         |                                        |
| `task_max_lifetime`    | удалить узел, если его возраст превышает это значение.                                                                                    | `7 * 24 * 60 * 60` (неделя в секундах) |

**Пример**

```xml
<distributed_ddl>
    <!-- Path in ZooKeeper to queue with DDL queries -->
    <path>/clickhouse/task_queue/ddl</path>

    <!-- Settings from this profile will be used to execute DDL queries -->
    <profile>default</profile>

    <!-- Controls how much ON CLUSTER queries can be run simultaneously. -->
    <pool_size>1</pool_size>

    <!--
         Cleanup settings (active tasks will not be removed)
    -->

    <!-- Controls task TTL (default 1 week) -->
    <task_max_lifetime>604800</task_max_lifetime>

    <!-- Controls how often cleanup should be performed (in seconds) -->
    <cleanup_delay_period>60</cleanup_delay_period>

    <!-- Controls how many tasks could be in the queue -->
    <max_tasks_in_queue>1000</max_tasks_in_queue>
</distributed_ddl>
```

<div id="distributed_ddl_use_initial_user_and_roles">
  ## distributed_ddl_use_initial_user_and_roles
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Если включено, запросы с предложением ON CLUSTER будут сохранять и использовать пользователя и роли инициатора для выполнения на удалённых сегментах. Это обеспечивает согласованное управление доступом во всём кластере, но требует, чтобы этот пользователь и эти роли существовали на всех узлах.
