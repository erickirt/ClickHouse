Эти настройки определяют конфигурацию сервера ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="max_server_memory_usage">
  ## max_server_memory_usage
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Максимальный объём памяти в байтах, который может использовать сервер.

<Note>
  Максимальное потребление памяти сервером дополнительно ограничивается настройкой `max_server_memory_usage_to_ram_ratio`.
</Note>

В особом случае значение `0` (по умолчанию) означает, что сервер может использовать всю доступную память (за исключением дополнительных ограничений, накладываемых `max_server_memory_usage_to_ram_ratio`).

<div id="max_server_memory_usage_to_ram_ratio">
  ## max_server_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.9" changeable_without_restart="No" />

Максимальный объём памяти, который серверу разрешено использовать, выраженный как доля всей доступной памяти.

Например, значение `0.9` (по умолчанию) означает, что сервер может использовать 90% доступной памяти.

Позволяет снизить использование памяти на системах с небольшим объёмом памяти.
На хостах с небольшим объёмом оперативной памяти и swap может потребоваться установить [`max_server_memory_usage_to_ram_ratio`](/ru/reference/settings/server-settings/settings/max-server-memory-usage#max_server_memory_usage_to_ram_ratio) больше 1.

Это отношение применяется двумя способами:

* При запуске (и при перезагрузке конфигурации) оно ограничивает жёсткий лимит памяти сервера этой долей от общего объёма физической оперативной памяти, как описано выше.
* Во время работы фоновый воркер памяти периодически пересчитывает жёсткий лимит памяти как `(resident memory + system available memory) * max_server_memory_usage_to_ram_ratio`, чтобы сервер оставлял запас для других процессов, работающих на том же хосте. При работе внутри cgroup с конечным лимитом памяти используется доступная память cgroup вместо значения для всего хоста. По мере того как другие процессы растут, а объём доступной памяти уменьшается, жёсткий лимит сервера также снижается, ограничивая рост до того, как у хоста (или cgroup) закончится память.

Установка значения `0` для этого отношения отключает как ограничение при запуске, так и корректировку во время работы. Корректировка во время работы также является no-op в системах, отличных от Linux, и в `clickhouse-keeper`, где эта настройка недоступна. Чтобы сохранить статическое ограничение при запуске/перезагрузке, но отключить корректировку во время работы (как в предыдущих версиях), установите `memory_worker_dynamic_hard_limit` в `0`.

<Note>
  Максимальное потребление памяти сервером дополнительно ограничивается настройкой `max_server_memory_usage`.
</Note>
