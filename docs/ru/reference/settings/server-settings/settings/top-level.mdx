Эти настройки задают конфигурацию сервера ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="top_level_domains_list">
  ## top_level_domains_list
</div>

Определяет список пользовательских доменов верхнего уровня, которые требуется добавить; каждая запись должна иметь формат `<name>/path/to/file</name>`.

Например:

```xml
<top_level_domains_lists>
    <public_suffix_list>/path/to/public_suffix_list.dat</public_suffix_list>
</top_level_domains_lists>
```

См. также:

* функцию [`cutToFirstSignificantSubdomainCustom`](/ru/reference/functions/regular-functions/url-functions#cutToFirstSignificantSubdomainCustom) и её варианты,
  которые принимают имя пользовательского списка TLD и возвращают часть домена, включающую поддомены верхнего уровня до первого значимого поддомена.

<div id="top_level_domains_path">
  ## top_level_domains_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/top_level_domains/" changeable_without_restart="No" />

Каталог доменов верхнего уровня.

**Пример**

```xml
<top_level_domains_path>/var/lib/clickhouse/top_level_domains/</top_level_domains_path>
```
