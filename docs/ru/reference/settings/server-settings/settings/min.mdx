Эти настройки задают конфигурацию сервера ClickHouse и автоматически генерируются на основе исходного кода ClickHouse.

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Да" />

Минимальный размер в байтах для обычного выделения памяти в C++ (например, стандартными контейнерами, строками, при расширении `std::vector`, умными указателями и т. п.), при котором может быть сгенерировано `MEMORY_LIMIT_EXCEEDED` после достижения `max_server_memory_usage`. Более мелкие обычные выделения памяти по-прежнему учитываются трекером памяти, но им разрешается успешно выполняться даже после превышения лимита, что уменьшает количество ложных сбоев во время очистки и обработки исключений в коде, работающем на грани OOM.

Выделения памяти, выполняемые собственными большими буферами ClickHouse — данные столбцов, хеш-таблицы, арены, конвейеры запросов, буферы ввода-вывода — всегда учитывают лимит памяти и могут сгенерировать исключение независимо от этого значения. Эта настройка влияет только на неявные выделения памяти, проходящие через `operator new`.

Значение `0` (по умолчанию) сохраняет прежнее поведение: для неявных выделений памяти через `operator new` трекер памяти никогда не отказывает, и только явные выделения памяти ClickHouse могут приводить к `MEMORY_LIMIT_EXCEEDED`.

Обратите внимание: чтобы избежать побочных эффектов, рекомендуется задавать значение больше `max_untracked_memory`.

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

Минимальное соотношение между временем ожидания CPU на уровне ОС (метрика `OSCPUWaitMicroseconds`) и временем занятости (метрика `OSCPUVirtualTimeMicroseconds`), при котором может рассматриваться сброс соединений. Для расчета вероятности используется линейная интерполяция между минимальным и максимальным соотношением; в этой точке вероятность равна 0.
Подробнее см. в разделе [Управление поведением при перегрузке CPU сервера](/ru/concepts/features/configuration/settings/server-overload).
