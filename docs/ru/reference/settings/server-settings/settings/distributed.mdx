Эти настройки задают конфигурацию сервера ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="distributed_ddl.cleanup_delay_period">
  ## distributed_ddl.cleanup_delay_period
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />очистка начинается после получения события о новом узле, если с момента предыдущей очистки прошло не менее `<cleanup_delay_period>` секунд.

<div id="distributed_ddl.max_tasks_in_queue">
  ## distributed_ddl.max_tasks_in_queue
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />Максимальное количество задач в очереди.

<div id="distributed_ddl.path">
  ## distributed_ddl.path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/ddl/" changeable_without_restart="No" />путь в Keeper к `<task_queue>` для DDL-запросов

<div id="distributed_ddl.pool_size">
  ## distributed_ddl.pool_size
</div>

<SettingsInfoBlock type="Int32" default_value="1" changeable_without_restart="No" />сколько запросов `<ON CLUSTER>` могут выполняться одновременно

<div id="distributed_ddl.profile">
  ## distributed_ddl.profile
</div>

профиль, используемый для выполнения DDL-запросов

<div id="distributed_ddl.replicas_path">
  ## distributed_ddl.replicas_path
</div>

<SettingsInfoBlock type="String" default_value="/clickhouse/task_queue/replicas/" changeable_without_restart="No" />путь в Keeper к `<task_queue>` для реплик

<div id="distributed_ddl.task_max_lifetime">
  ## distributed_ddl.task_max_lifetime
</div>

<SettingsInfoBlock type="UInt64" default_value="604800" changeable_without_restart="No" />Удаляет узел, если его возраст превышает это значение.
