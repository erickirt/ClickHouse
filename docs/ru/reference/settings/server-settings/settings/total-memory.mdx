Эти настройки конфигурируют сервер ClickHouse и автоматически сгенерированы из исходного кода ClickHouse.

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Собирать случайные выделения памяти размером меньше или равным указанному значению с вероятностью `total_memory_profiler_sample_probability`. 0 означает, что параметр отключён. Рекомендуется установить `max_untracked_memory` в 0, чтобы этот порог работал как ожидается.

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Собирает случайные выделения памяти размером не меньше указанного значения с вероятностью, равной `total_memory_profiler_sample_probability`. 0 означает, что функция отключена. При необходимости установите `max_untracked_memory` в 0, чтобы этот порог работал как ожидается.

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Когда использование памяти сервера превышает каждый следующий порог, заданный в байтах, профилировщик памяти собирает трассировку стека выделения памяти. Ноль означает, что профилировщик памяти отключён. Значения меньше нескольких мегабайт замедлят работу сервера.

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="No" />

Позволяет собирать случайные события выделения и освобождения памяти и записывать их в системную таблицу [system.trace&#95;log](/ru/reference/system-tables/trace_log), где `trace_type` имеет значение `MemorySample`, с указанной вероятностью. Эта вероятность применяется к каждому выделению или освобождению памяти независимо от размера выделения. Обратите внимание, что сэмплирование происходит только тогда, когда объём неотслеживаемой памяти превышает лимит неотслеживаемой памяти (значение по умолчанию — `4` MiB). Его можно уменьшить, уменьшив [total&#95;memory&#95;profiler&#95;step](/ru/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step). Для ещё более детального сэмплирования можно установить `total_memory_profiler_step` равным `1`.

Возможные значения:

* Положительное число типа Double.
* `0` — запись случайных событий выделения и освобождения памяти в системную таблицу `system.trace_log` отключена.
