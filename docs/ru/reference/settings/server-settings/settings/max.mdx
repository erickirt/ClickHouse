Эти настройки задают конфигурацию сервера ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="max_active_parts_loading_thread_pool_size">
  ## max_active_parts_loading_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="64" changeable_without_restart="Yes" />Количество потоков в пуле для загрузки активного набора частей данных при запуске.

<div id="max_authentication_methods_per_user">
  ## max_authentication_methods_per_user
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />

Максимальное количество методов аутентификации, с которыми можно создать пользователя или изменить его параметры.
Изменение этой настройки не влияет на существующих пользователей. Запросы CREATE/ALTER, связанные с аутентификацией, завершатся ошибкой, если число методов превысит предел, указанный в этой настройке.
Запросы CREATE/ALTER, не связанные с аутентификацией, будут выполняться успешно.

<Note>
  Значение `0` означает отсутствие ограничений.
</Note>

<div id="max_os_cpu_wait_time_ratio_to_drop_connection">
  ## max_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

Максимальное соотношение времени ожидания CPU в ОС (метрика `OSCPUWaitMicroseconds`) ко времени занятости CPU (метрика `OSCPUVirtualTimeMicroseconds`), при котором может быть принято решение о разрыве соединений. Подробности см. в разделе [Управление поведением при перегрузке CPU сервера](/ru/concepts/features/configuration/settings/server-overload).

<div id="max_partition_size_to_drop">
  ## max_partition_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

Ограничение на удаление партиций.

Если размер таблицы [MergeTree](/ru/reference/engines/table-engines/mergetree-family/mergetree) превышает [`max_partition_size_to_drop`](/ru/reference/settings/server-settings/settings/max#max_partition_size_to_drop) (в байтах), вы не сможете удалить партицию с помощью запроса [DROP PARTITION](/ru/reference/statements/alter/partition#drop-partitionpart).
Для применения этой настройки не требуется перезапуск ClickHouse server. Ещё один способ отключить это ограничение — создать файл `<clickhouse-path>/flags/force_drop_table`.

<Note>
  Значение `0` означает, что партиции можно удалять без каких-либо ограничений.

  Это ограничение не распространяется на команды drop table и truncate table, см. [max&#95;table&#95;size&#95;to&#95;drop](/ru/reference/settings/session-settings#max_table_size_to_drop)
</Note>

**Пример**

```xml
<max_partition_size_to_drop>0</max_partition_size_to_drop>
```

<div id="max_per_cpu_untracked_memory">
  ## max_per_cpu_untracked_memory
</div>

<SettingsInfoBlock type="UInt64" default_value="8388608" changeable_without_restart="Yes" />

Верхний предел неотслеживаемой памяти в байтах, который все потоки, работающие на одном CPU, могут суммарно удерживать одновременно до её передачи в трекер памяти. Если `max_untracked_memory` ограничивает один поток, то этот параметр ограничивает суммарный объём на CPU, чтобы множество потоков не могли превратить свой лимит на поток в большой общесерверный overcommit. Таким образом, общий объём неотслеживаемой памяти ограничен примерно значением `number_of_cpus * max_per_cpu_untracked_memory`. Значение `0` отключает ограничение на CPU (в этом случае действует только ограничение на поток `max_untracked_memory`). Только для Linux.

<div id="max_waiting_queries">
  ## max_waiting_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

Ограничение на общее число одновременно ожидающих запросов.
Выполнение ожидающего запроса блокируется, пока необходимые таблицы асинхронно загружаются (см. [`async_load_databases`](/ru/reference/settings/server-settings/settings/async-load#async_load_databases).

<Note>
  Ожидающие запросы не учитываются при проверке ограничений, задаваемых следующими настройками:

  * [`max_concurrent_queries`](/ru/reference/settings/server-settings/settings/max-concurrent#max_concurrent_queries)
  * [`max_concurrent_insert_queries`](/ru/reference/settings/server-settings/settings/max-concurrent#max_concurrent_insert_queries)
  * [`max_concurrent_select_queries`](/ru/reference/settings/server-settings/settings/max-concurrent#max_concurrent_select_queries)
  * [`max_concurrent_queries_for_user`](/ru/reference/settings/session-settings#max_concurrent_queries_for_user)
  * [`max_concurrent_queries_for_all_users`](/ru/reference/settings/session-settings#max_concurrent_queries_for_all_users)

  Это сделано, чтобы избежать достижения этих ограничений сразу после запуска сервера.
</Note>

<Note>
  Значение `0` (по умолчанию) означает отсутствие ограничений.

  Эту настройку можно изменять во время выполнения, и она вступает в силу немедленно. Уже выполняющиеся запросы останутся без изменений.
</Note>
