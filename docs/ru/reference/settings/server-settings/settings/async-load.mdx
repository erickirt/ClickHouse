Эти настройки настраивают сервер ClickHouse и автоматически сгенерированы на основе исходного кода ClickHouse.

<div id="async_load_databases">
  ## async_load_databases
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

Асинхронная загрузка баз данных и таблиц.

* Если `true`, все несистемные базы данных с движком `Ordinary`, `Atomic` и `Replicated` будут загружаться асинхронно после запуска сервера ClickHouse. См. таблицу `system.asynchronous_loader`, а также настройки сервера `tables_loader_background_pool_size` и `tables_loader_foreground_pool_size`. Любой запрос, который пытается обратиться к таблице, ещё не загруженной в память, будет ждать запуска именно этой таблицы. Если задача загрузки завершится с ошибкой, запрос повторно сгенерирует исключение (вместо остановки всего сервера, как в случае `async_load_databases = false`). Таблица, которую ожидает хотя бы один запрос, будет загружена с более высоким приоритетом. DDL-запросы к базе данных будут ждать запуска именно этой базы данных. Также стоит задать ограничение `max_waiting_queries` на общее число ожидающих запросов.
* Если `false`, все базы данных загружаются при запуске сервера.

**Пример**

```xml
<async_load_databases>true</async_load_databases>
```

<div id="async_load_system_database">
  ## async_load_system_database
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Асинхронная загрузка системных таблиц. Полезно, если в базе данных `system` много таблиц логов и частей. Не зависит от настройки `async_load_databases`.

* Если установлено значение `true`, все системные базы данных с движками `Ordinary`, `Atomic` и `Replicated` будут загружаться асинхронно после запуска сервера ClickHouse. См. таблицу `system.asynchronous_loader`, настройки сервера `tables_loader_background_pool_size` и `tables_loader_foreground_pool_size`. Любой запрос, который пытается обратиться к системной таблице, ещё не завершившей загрузку, будет ждать запуска именно этой таблицы. Таблица, которую ожидает хотя бы один запрос, будет загружена с более высоким приоритетом. Также можно задать настройку `max_waiting_queries`, чтобы ограничить общее число ожидающих запросов.
* Если установлено значение `false`, системная база данных загружается до запуска сервера.

**Пример**

```xml
<async_load_system_database>true</async_load_system_database>
```
