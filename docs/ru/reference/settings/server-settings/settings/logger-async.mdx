Эти настройки задают конфигурацию сервера ClickHouse и автоматически генерируются на основе исходного кода ClickHouse.

<div id="logger.async">
  ## logger.async
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />Если задано `<true>` (по умолчанию), журналирование выполняется асинхронно (один фоновый поток на каждый канал вывода). В противном случае запись в журнал выполняется в потоке, вызывающем LOG.

<div id="logger.async_queye_max_size">
  ## logger.async_queye_max_size
</div>

<SettingsInfoBlock type="UInt64" default_value="65536" changeable_without_restart="No" />При использовании асинхронного логирования это максимальное количество сообщений, которое будет храниться в очереди в ожидании сброса. Лишние сообщения будут отброшены.
