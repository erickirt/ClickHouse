Эти настройки относятся к серверу ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="No" />

Путь в локальной файловой системе для хранения временных данных при обработке больших запросов.

<Note>
  * Для настройки хранения временных данных можно использовать только один из следующих параметров: tmp&#95;path, tmp&#95;policy, temporary&#95;data&#95;in&#95;cache.
  * Завершающий slash обязателен.
</Note>

**Пример**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

Политика хранения временных данных. Все файлы с префиксом `tmp` будут удалены при запуске.

<Note>
  Рекомендации по использованию объектного хранилища в качестве `tmp_policy`:

  * Используйте отдельный `bucket:path` на каждом сервере
  * Используйте `metadata_type=plain`
  * Также может быть полезно задать TTL для этого бакета
</Note>

<Note>
  * Для настройки хранения временных данных можно использовать только один из следующих параметров: `tmp_path`, `tmp_policy`, `temporary_data_in_cache`.
  * `move_factor`, `keep_free_space_bytes`, `max_data_part_size_bytes` игнорируются.
  * Политика должна содержать ровно *один том*

  Дополнительные сведения см. в документации [движка таблицы MergeTree](/ru/reference/engines/table-engines/mergetree-family/mergetree).
</Note>

**Пример**

Если `/disk1` заполнен, временные данные будут храниться на `/disk2`.

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
