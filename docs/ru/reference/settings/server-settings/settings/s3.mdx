Эти настройки задают конфигурацию сервера ClickHouse и автоматически сгенерированы на основе исходного кода ClickHouse.

<div id="s3_allow_server_credentials_for_system_table_disks">
  ## s3_allow_server_credentials_for_system_table_disks
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

Разрешает динамическому `disk(type = s3, ...)` таблицы в базе данных `system` использовать собственные S3-учетные данные сервера (ambient),
выводя его из-под ограничения `s3_allow_server_credentials_in_user_queries`. Это предназначено для
внутренней инфраструктуры сервера, которая выгружает системные таблицы в S3 с использованием identity сервера (подключаемой
cloud operator к базе данных `system`). В отличие от настройки сеанса, это настройка уровня сервера, поэтому
исключение также действует, когда таблица повторно загружается из metadata при перезапуске. Она ограничена базой данных `system`,
в которой обычные пользователи не могут создавать таблицы, поэтому это не ослабляет ограничение для пользовательских запросов.

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Максимальное количество провайдеров учетных данных S3, которые можно сохранить в кэше

<div id="s3_load_table_anonymously_if_credentials_restricted">
  ## s3_load_table_anonymously_if_credentials_restricted
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

Определяет, что происходит, когда постоянная table `S3` или `S3Queue`, динамический `disk(type = s3, ...)` или
database `DataLakeCatalog` (Glue, BigLake) загружается из существующих metadata (при запуске server или `RESTORE`), и
по его определению должны использоваться собственные S3/Cloud учётные данные server из окружения (ambient), которые запрещены для user
queries параметром `s3_allow_server_credentials_in_user_queries`.

Когда параметр включен (по умолчанию), объект загружается без этих учётных данных вместо прерывания запуска: table
`S3`/`S3Queue` или динамический S3-диск создаётся с анонимным S3 client, а database `DataLakeCatalog`
остаётся с недоступным catalog. Server запускается, но объект остаётся недоступным (его requests
к private бакету отклоняются либо catalog сообщает явную error), пока его учётные данные снова не начнут разрешаться в
допустимый source. Он никогда незаметно не получает обратно identity server. Это позволяет одному такому объекту —
например, созданному в более старой version до появления этого ограничения, или объекту, чья named collection была
позже повторно привязана к `use_environment_credentials = 1`, — не прерывать запуск server.

Когда параметр отключен, загрузка такого объекта завершается с error, что может помешать запуску server. Используйте это только
если вы предпочитаете жёсткий сбой, а не молча недоступную table, disk или database catalog.
