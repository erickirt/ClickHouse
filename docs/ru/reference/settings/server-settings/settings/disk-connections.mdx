Эти настройки служат для настройки сервера ClickHouse и автоматически генерируются из исходного кода ClickHouse.

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />При попытке создания подключения при достижении этого предела генерируется исключение. Установите значение 0, чтобы отключить жёсткое ограничение. Предел применяется к подключениям к диску.

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />Размер параметра SO&#95;RCVBUF для подключений к дискам (S3, Azure, GCS). По умолчанию — 204800 (200 КБ): доступ к объектному хранилищу обычно выполняется в пределах одного региона, где произведение пропускной способности на задержку значительно меньше 100 КБ, поэтому небольшой буфер обеспечивает полную пропускную способность и при этом позволяет избежать многомегабайтных буферов для каждого сокета, которые иначе выделялись бы за счёт автонастройки ядра. Установите 0, чтобы вместо этого использовать автотюнинг TCP ядра для буфера приёма. Примечание: если изменить значение этого параметра обратно на 0, автонастройка восстановится только для вновь создаваемых подключений; существующие подключения в пуле сохранят фиксированные размеры буфера, пока не будут пересозданы.

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />Размер параметра SO&#95;SNDBUF для подключений к диску (S3, Azure, GCS). Если задано значение больше 0, оно переопределяет автотюнинг TCP ядра для буфера отправки. 0 = значение ядра по умолчанию (автотюнинг). Примечание: возврат этого параметра к значению 0 восстанавливает автотюнинг только для вновь создаваемых подключений; существующие подключения из пула сохраняют фиксированные размеры буфера до пересоздания.

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />У подключений, превышающих этот предел, значительно меньшее время жизни. Это ограничение применяется к подключениям к диску.

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />Подключения сверх этого лимита сбрасываются после использования. Установите 0, чтобы отключить кэш подключений. Лимит применяется к подключениям к диску.

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />Если число используемых подключений превышает этот предел, в журнал записываются предупреждающие сообщения. Этот предел применяется к подключениям к диску.
