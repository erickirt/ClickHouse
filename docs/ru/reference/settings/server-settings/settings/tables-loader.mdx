Эти настройки относятся к серверу ClickHouse и автоматически генерируются на основе исходного кода ClickHouse.

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Задаёт количество потоков, выполняющих асинхронные задачи загрузки в фоновом пуле. Фоновый пул используется для асинхронной загрузки таблиц после запуска сервера, если нет запросов, ожидающих эту таблицу. При большом количестве таблиц может быть полезно оставить во фоновом пуле небольшое число потоков. Это позволит зарезервировать ресурсы CPU для параллельного выполнения запросов.

<Note>
  Значение `0` означает, что будут использоваться все доступные CPU.
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

Задаёт количество потоков, выполняющих задачи загрузки в пуле переднего плана. Пул переднего плана используется для синхронной загрузки таблиц до того, как сервер начнёт прослушивать порт, а также для загрузки таблиц, завершения которой ожидают. Пул переднего плана имеет более высокий приоритет, чем фоновый пул. Это означает, что в фоновом пуле не запускается ни одна задача, пока в пуле переднего плана выполняются задачи.

<Note>
  Значение `0` означает, что будут использоваться все доступные CPU.
</Note>
