Эти настройки задают конфигурацию сервера ClickHouse и автоматически генерируются на основе исходного кода ClickHouse.

<div id="jemalloc_collect_global_profile_samples_in_trace_log">
  ## jemalloc_collect_global_profile_samples_in_trace_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />Сохраняет выборочные записи о выделении памяти jemalloc в system.trace&#95;log

<div id="jemalloc_max_background_threads_num">
  ## jemalloc_max_background_threads_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />Максимальное количество создаваемых фоновых потоков jemalloc; значение 0 означает использование значения по умолчанию jemalloc

<div id="jemalloc_profiler_sampling_rate">
  ## jemalloc_profiler_sampling_rate
</div>

<SettingsInfoBlock type="UInt64" default_value="19" changeable_without_restart="No" />

Управляет параметром `lg_prof_sample` в jemalloc — двоичным логарифмом среднего интервала (в байтах) между сэмплами выделения памяти.
Значение по умолчанию 19 соответствует 512 KiB. Меньшее значение увеличивает частоту сэмплирования (больше накладных расходов, больше деталей), а большее — уменьшает.
Изменение этого значения вызывает `prof.reset`, что сбрасывает всю накопленную статистику данных профилирования. Для этого профилирование должно быть включено (`MALLOC_CONF=prof:true`).
