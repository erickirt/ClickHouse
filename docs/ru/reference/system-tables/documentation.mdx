<div id="description">
  ## Описание
</div>

Собирает встроенную документацию по унифицированным компонентам системы в одну таблицу. Каждая строка соответствует одной сущности (функции, движку таблицы, типу данных и так далее) и содержит справочную документацию по этой сущности, отрендеренную в Markdown — то же содержимое, которое публикуется на сайте и доступно через таблицы `system.*` для каждого типа.

`description` формируется из структурированных частей встроенной документации (`description`, `syntax`, аргументов, примеров и так далее), так что в одном столбце содержится полная документация по сущности. Когда встроенная документация содержит всю справочную страницу в своём `description` (как в случае с компонентами, страницы которых на сайте автоматически генерируются на его основе: движки таблиц, движки баз данных, типы данных, форматы, табличные функции), страница публикуется как есть, без добавления разделов, составленных из структурированных полей метаданных, — содержимое страницы уже охватывает этот материал. Псевдонимы отображаются как краткая ссылка на каноническую сущность, например: ``Псевдоним `trunc`.``

Эта таблица в некотором смысле объединяет информацию, доступную в таблицах документации для отдельных типов ([`system.functions`](/ru/reference/system-tables/functions), [`system.table_engines`](/ru/reference/system-tables/table_engines), [`system.data_type_families`](/ru/reference/system-tables/data_type_families) и других). В частности, она предназначена для поддержки интерактивной команды `help` в клиенте, но полезна и сама по себе.

Собираются следующие виды сущностей (значение столбца `type` показано в скобках):

* Функции (`Function`)
* Агрегатные функции (`Aggregate Function`)
* Табличные функции (`Table Function`)
* Движки таблиц (`Table Engine`)
* Движки баз данных (`Database Engine`)
* Типы данных (`Data Type`)
* Структуры словарей (`Dictionary Layout`)
* Источники словарей (`Dictionary Source`)
* Комбинаторы агрегатных функций (`Aggregate Function Combinator`)
* Типы индексов пропуска данных (`Data Skipping Index`)
* Типы дисков (`Disk Type`)
* Настройки (`Setting`)
* Настройки MergeTree (`MergeTree Setting`)
* Настройки сервера (`Server Setting`)
* Форматы (`Format`)
* Кодеки сжатия (`Compression Codec`)
* События профиля (`Profile Event`)
* Текущие метрики (`Current Metric`)
* Асинхронные метрики (`Asynchronous Metric`)
* Системные таблицы (`System Table`)

Для настроек (любого вида) документацией служит описание самой настройки вместе с её типом и значением по умолчанию; устаревшие настройки не показываются. Для системных таблиц документацией служит комментарий к таблице вместе со списком её столбцов (имя, тип и описание каждого).

В столбце `source` хранится путь к исходному файлу, в котором определена документация сущности, относительно корня репозитория. Для большинства сущностей он фиксируется автоматически в месте, где создаётся объект документации (в месте регистрации компонента); для типов, документация по которым находится в одном исходном файле (например, настройки, события профиля и текущие метрики), указывается этот файл.

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `name` ([String](/ru/reference/data-types/index)) — Имя сущности, например `domainWithoutWWW` или `MergeTree`.
* `type` ([Enum8(&#39;Function&#39; = 1, &#39;Aggregate Function&#39; = 2, &#39;Table Function&#39; = 3, &#39;Table Engine&#39; = 4, &#39;Database Engine&#39; = 5, &#39;Data Type&#39; = 6, &#39;Dictionary Layout&#39; = 7, &#39;Dictionary Source&#39; = 8, &#39;Aggregate Function Combinator&#39; = 9, &#39;Data Skipping Index&#39; = 10, &#39;Disk Type&#39; = 11, &#39;Setting&#39; = 12, &#39;MergeTree Setting&#39; = 13, &#39;Server Setting&#39; = 14, &#39;Format&#39; = 15, &#39;Compression Codec&#39; = 16, &#39;Profile Event&#39; = 17, &#39;Current Metric&#39; = 18, &#39;Asynchronous Metric&#39; = 19, &#39;System Table&#39; = 20)](/ru/reference/data-types/index)) — Вид сущности, например `Function` или `Table Engine`.
* `description` ([String](/ru/reference/data-types/index)) — Справочная документация по сущности, представленная в формате Markdown и собранная из встроенной документации (то же содержимое, что опубликовано на сайте), включая синтаксис, примеры и другие структурированные разделы, если они есть.
* `source` ([String](/ru/reference/data-types/index)) — Путь к исходному файлу, в котором определена документация сущности, относительно корня репозитория. Пусто, если расположение исходного файла неизвестно.

<div id="example">
  ## Пример
</div>

Ознакомьтесь с документацией по конкретному объекту:

```sql title="Query"
SELECT description
FROM system.documentation
WHERE type = 'Table Engine' AND name = 'MergeTree'
FORMAT TSVRaw;
```

Одно и то же имя может относиться к нескольким видам сущностей (например, есть и табличная функция `file`, и источник данных словаря `file`), поэтому удобно искать это имя сразу по всем видам:

```sql title="Query"
SELECT type, name
FROM system.documentation
WHERE name = 'file'
ORDER BY type;
```

Подсчитайте количество описанных сущностей каждого вида:

```sql title="Query"
SELECT type, count()
FROM system.documentation
GROUP BY type
ORDER BY count() DESC;
```
