<div id="description">
  ## Описание
</div>

Содержит сведения обо всех настройках файлового кэша

<div id="columns">
  ## Столбцы
</div>

{/*AUTOGENERATED_START*/}

* `cache_name` ([String](/ru/reference/data-types/string)) — Имя кэша
* `path` ([String](/ru/reference/data-types/string)) — Путь к каталогу кэша
* `max_size` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальный размер кэша
* `max_elements` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальное количество элементов кэша, например сегментов файла (ограничивает количество файлов в файловой системе)
* `max_file_segment_size` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальный размер одного сегмента файла
* `boundary_alignment` ([UInt64](/ru/reference/data-types/int-uint)) — Выравнивание сегмента файла
* `cache_on_write_operations` ([UInt8](/ru/reference/data-types/int-uint)) — Включает сквозное кэширование при записи (кэширование при INSERT и MERGE)
* `cache_policy` ([String](/ru/reference/data-types/string)) — Политика вытеснения кэша
* `slru_size_ratio` ([Float64](/ru/reference/data-types/float)) — Отношение размеров защищённых и probationary-элементов в политике кэша SLRU
* `background_download_threads` ([UInt64](/ru/reference/data-types/int-uint)) — Количество потоков фоновой загрузки. Значение 0 отключает фоновую загрузку
* `background_download_queue_size_limit` ([UInt64](/ru/reference/data-types/int-uint)) — Размер очереди фоновой загрузки. Значение 0 отключает фоновую загрузку
* `background_download_max_file_segment_size` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальный размер, который может быть загружен в фоновом режиме
* `load_metadata_threads` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальное количество потоков, используемых для загрузки метаданных кэша при запуске сервера
* `load_metadata_asynchronously` ([UInt8](/ru/reference/data-types/int-uint)) — Включает асинхронную загрузку метаданных при запуске сервера
* `keep_free_space_size_ratio` ([Float64](/ru/reference/data-types/float)) — Доля свободного пространства, которую кэш будет пытаться поддерживать в фоновом режиме
* `keep_free_space_elements_ratio` ([Float64](/ru/reference/data-types/float)) — Доля свободных элементов, которую кэш будет пытаться поддерживать в фоновом режиме
* `keep_free_space_remove_batch` ([UInt64](/ru/reference/data-types/int-uint)) — Размер батча удаления элементов кэша, выполняемого фоновым потоком для поддержания доли свободного пространства/элементов
* `invalidated_entries_cleanup_interval_ms` ([UInt64](/ru/reference/data-types/int-uint)) — Интервал бездействия в миллисекундах для фоновой задачи, которая очищает помеченные недействительными (лениво удалённые) элементы очереди с приоритетом
* `invalidated_entries_cleanup_threshold` ([UInt64](/ru/reference/data-types/int-uint)) — Количество накопленных помеченных недействительными элементов очереди с приоритетом, при котором запускается их фоновое удаление
* `invalidated_entries_cleanup_remove_batch` ([UInt64](/ru/reference/data-types/int-uint)) — Максимальное количество помеченных недействительными элементов очереди с приоритетом, удаляемых под одной блокировкой записи за одну фоновую итерацию
* `enable_filesystem_query_cache_limit` ([UInt8](/ru/reference/data-types/int-uint)) — Включает ограничение максимального размера кэша, который может быть записан в рамках одного запроса
* `cache_hits_threshold` ([UInt64](/ru/reference/data-types/int-uint)) — Устаревшая настройка
* `enable_bypass_cache_with_threshold` ([UInt8](/ru/reference/data-types/int-uint)) — Не документировано. Не рекомендуется к использованию
* `bypass_cache_threshold` ([UInt64](/ru/reference/data-types/int-uint)) — Не документировано. Не рекомендуется к использованию
* `write_cache_per_user_id_directory` ([UInt8](/ru/reference/data-types/int-uint)) — Внутренняя настройка ClickHouse Cloud
* `allow_dynamic_cache_resize` ([UInt8](/ru/reference/data-types/int-uint)) — Разрешает динамически изменять размер файлового кэша
* `max_size_ratio_to_total_space` ([Float64](/ru/reference/data-types/float)) — Отношение `max_size` к общему объему дискового пространства
* `skip_cache_on_disk_failure` ([UInt8](/ru/reference/data-types/int-uint)) — Если true, при ошибках дискового IO операции файлового кэша будут молча пропускаться. Если false (по умолчанию), ошибки дискового IO приводят к ошибкам запуска.
* `use_split_cache` ([UInt8](/ru/reference/data-types/int-uint)) — Использовать разделение файлов на system/data.
* `split_cache_ratio` ([Float64](/ru/reference/data-types/float)) — Отношение системного сегмента к общему размеру кэша для split&#95;cache.
* `overcommit_eviction_evict_step` ([UInt64](/ru/reference/data-types/int-uint)) — Шаг вытеснения в байтах для политики overcommit eviction. Используется для настроек keep&#95;free&#95;space&#95;*&#95;ratio
* `check_cache_probability` ([Float64](/ru/reference/data-types/float)) — Работает только в debug- или sanitizer-сборке. Проверяет корректность кэша, проходя по всему кэшу и проверяя состояние каждого его элемента
* `expose_prometheus_eviction_metrics` ([UInt8](/ru/reference/data-types/int-uint)) — Публиковать метрики Prometheus для активности вытеснения файлового кэша (`filesystem_cache_evictions_total` и т. д.). По умолчанию выключено. Может переключаться во время выполнения через `SYSTEM RELOAD CONFIG`.
* `expose_prometheus_eviction_metrics_per_user` ([UInt8](/ru/reference/data-types/int-uint)) — Дополнительно публиковать метрики вытеснения для каждого user-id. Требует `expose_prometheus_eviction_metrics`. Мощность растет с числом различных пользователей, выполняющих вытеснение.
* `is_initialized` ([UInt8](/ru/reference/data-types/int-uint)) — Указывает, был ли кэш успешно инициализирован
* `current_size` ([UInt64](/ru/reference/data-types/int-uint)) — Текущий размер кэша
* `current_elements_num` ([UInt64](/ru/reference/data-types/int-uint)) — Текущее количество элементов кэша (сегментов файла)

{/*AUTOGENERATED_END*/}
