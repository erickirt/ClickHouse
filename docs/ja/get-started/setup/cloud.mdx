> ClickHouse を最もすばやく簡単に使い始めるには、[ClickHouse Cloud](https://console.clickhouse.cloud) で新しい
> サービスを作成する方法がおすすめです。このクイックスタートガイドでは、3 つの簡単なステップでセットアップ
> を完了します。

<Steps>
  <Step title="ClickHouse サービスを作成する" id="1-create-a-clickhouse-service">
    [ClickHouse Cloud](https://console.clickhouse.cloud) で無料の ClickHouse サービスを作成するには、以下の手順でサインアップするだけです。

    * [サインアップページ](https://console.clickhouse.cloud/signUp) でアカウントを作成します
    * メールアドレス、Google SSO、Microsoft SSO、AWS Marketplace、Google Cloud、Microsoft Azure を使ってサインアップできます
    * メールアドレスとパスワードでサインアップする場合は、メールで届くリンクから 24 時間以内にメールアドレスを確認してください
    * 作成したユーザー名とパスワードでログインします

    <Image img="/images/_snippets/signup_page.webp" size="md" alt="プランを選択" border />

    <br />

    ログインすると、ClickHouse Cloud のオンボーディングウィザードが開始し、新しい ClickHouse サービスの作成手順が案内されます。サービスをデプロイするリージョンを選択し、新しいサービスに名前を付けます。

    <Image img="/images/_snippets/createservice1.webp" size="md" alt="新しい ClickHouse サービス" border />

    <br />

    デフォルトでは、新しい組織は Scale ティアに設定され、4 vCPU と 16 GiB RAM を備えた 3 つのレプリカが作成されます。[垂直オートスケーリング](/ja/products/cloud/features/autoscaling/vertical) は Scale ティアでデフォルトで有効です。組織のティアは後で &#39;Plans&#39; ページから変更できます。

    必要に応じて、レプリカのスケール範囲となる最小サイズと最大サイズを指定して、サービスのリソースを調整します。準備ができたら、`Create service` を選択します。

    <Image img="/images/_snippets/scaling_limits.webp" size="md" alt="スケーリング制限" border />

    <br />

    これで、ClickHouse Cloud サービスの準備が整い、オンボーディングは完了です。データの取り込みとクエリの開始方法について、引き続きご確認ください。
  </Step>

  <Step title="ClickHouse に接続する" id="2-connect-to-clickhouse">
    ClickHouse に接続する方法は 2 つあります。

    * Web ベースの SQL Console を使用して接続する
    * アプリから接続する

    <br />

    ### SQL Console を使用して接続する

    すぐに使い始めるには、ClickHouse が提供する Web ベースの SQL Console を利用します。オンボーディングが完了すると、この画面にリダイレクトされます。

    <Image img="/images/_snippets/createservice8.webp" size="md" alt="SQL Console" border />

    接続が正しく機能していることを確認するには、クエリタブを開いて簡単なクエリを入力します。

    ```sql
    SHOW databases
    ```

    一覧には、4 つのデータベースに加えて、追加したものがあればそれらも表示されるはずです。

    <Image img="/images/_snippets/show_databases.webp" size="md" alt="SQL Console" border />

    <br />

    以上です。これで、新しい ClickHouseサービスを使い始める準備が整いました。

    ### アプリに接続する

    ナビゲーションメニューの Connect ボタンを押します。モーダルが開き、サービスの認証情報と、インターフェイスまたは言語クライアントから接続するための手順が表示されます。

    <Image img="/images/_snippets/service_connect.webp" size="md" alt="Service Connect" border />

    <br />

    お使いの言語クライアントが見つからない場合は、[Integrations](/ja/integrations/home) の一覧を確認してください。
  </Step>

  <Step title="データの追加" id="3-add-data">
    ClickHouseはデータがあってこそ真価を発揮します。データを追加する方法は複数あり、そのほとんどはナビゲーションメニューからアクセスできるData Sourcesページで利用できます。

    <Image img="/images/_snippets/data_sources.webp" size="md" alt="データソース" border />

    <br />

    以下の方法でデータをアップロードできます：

    * S3、Postgres、Kafka、GCS などのデータソースからのデータ取り込みを開始するために ClickPipe を設定する
    * SQLコンソールを使用する
    * ClickHouse clientを使用する
    * ファイルをアップロードする  - 対応フォーマットには JSON、CSV、TSV が含まれます
    * ファイルのURLからデータをアップロードする

    ### ClickPipes

    [ClickPipes](http://clickhouse.com/docs/integrations/clickpipes) は、多様なソースからのデータ取り込みを数回のクリックで実現するマネージドインテグレーションプラットフォームです。最も要求の厳しいワークロードを想定して設計されており、ClickPipes の堅牢でスケーラブルなアーキテクチャが安定したパフォーマンスと信頼性を実現します。ClickPipes は、長期的なストリーミング用途にも一回限りのデータ読み込みジョブにも対応しています。

    <Image img="/images/_snippets/select_data_source.webp" size="md" alt="データソースを選択" border />

    <br />

    ### SQL Consoleを使用してデータを追加する

    多くのデータベース管理システムと同様に、ClickHouse はテーブルを**データベース**に論理的にグループ化します。ClickHouse で新しいデータベースを作成するには、[`CREATE DATABASE`](/ja/reference/statements/create/database) コマンドを使用します。

    ```sql
    CREATE DATABASE IF NOT EXISTS helloworld
    ```

    次のコマンドを実行して、`helloworld` データベースに `my_first_table` という名前のテーブルを作成します。

    ```sql
    CREATE TABLE helloworld.my_first_table
    (
        user_id UInt32,
        message String,
        timestamp DateTime,
        metric Float32
    )
    ENGINE = MergeTree()
    PRIMARY KEY (user_id, timestamp)
    ```

    上記の例では、`my_first_table` は4つのカラムを持つ [`MergeTree`](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルです：

    * `user_id`: 32ビットの符号なし整数 ([UInt32](/ja/reference/data-types/int-uint))
    * `message`: [String](/ja/reference/data-types/string) 型。これは、他のデータベースシステムで使われる `VARCHAR`、`BLOB`、`CLOB` などの型に相当します
    * `timestamp`: [DateTime](/ja/reference/data-types/datetime) 値で、ある時点を表します
    * `metric`: 32ビット浮動小数点数 ([Float32](/ja/reference/data-types/float))

    <Info>
      **テーブルエンジン**

      テーブルエンジンによって、次の点が決まります。

      * データをどのように、どこに保存するか
      * どのクエリに対応しているか
      * データがレプリケートされるかどうか

      <br />

      選択可能なテーブルエンジンは数多くありますが、単一ノードのClickHouseサーバー上のシンプルなテーブルには、[`MergeTree`](/ja/reference/engines/table-engines/mergetree-family/mergetree) を選ぶのが一般的です。
    </Info>

    #### 主キーの概要

    先に進む前に、ClickHouse における主キーの仕組みを理解しておくことが重要です (主キーの実装は、想像とは異なる部分があるかもしれません！) ：

    * ClickHouse の主キーは、テーブル内の各行で ***一意ではありません***

    ClickHouseテーブルの主キーは、ディスクへの書き込み時にデータがどのようにソートされるかを決定します。8,192行または10MBのデータごと (**index granularity**と呼ばれます) に、主キーの索引ファイルへのエントリが作成されます。このgranularityの概念により、メモリに容易に収まる**スパースインデックス**が生成されます。granuleは、`SELECT`クエリの処理時に読み取られる最小単位のカラムデータのストライプを表します。

    主キーは `PRIMARY KEY` パラメータを使用して定義できます。`PRIMARY KEY` を指定せずにテーブルを定義した場合、`ORDER BY` 句で指定されたタプルがキーになります。`PRIMARY KEY` と `ORDER BY` の両方を指定する場合、主キーはソート順のサブセットでなければなりません。

    主キーはソートキーでもあり、`(user_id, timestamp)` のタプルです。そのため、各カラムファイルに格納されたデータは、`user_id`、次に `timestamp` の順でソートされます。

    ClickHouseのコアコンセプトを詳しく理解するには、[「コアコンセプト」](/ja/concepts/core-concepts/index)を参照してください。

    #### テーブルへのデータ挿入

    ClickHouse では使い慣れた [`INSERT INTO TABLE`](/ja/reference/statements/insert-into) コマンドを使用できますが、[`MergeTree`](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルへの挿入が行われるたびに、ストレージ内に**パート**が作成される点を理解しておくことが重要です。

    <Tip>
      **ClickHouse のベストプラクティス**

      1 回のバッチで大量の行を挿入してください。1 度に数万行、あるいは数百万
      行でもかまいません。心配はいりません。ClickHouse はその規模のデータ量でも簡単に処理でき、サービスへの書き込みリクエスト数を減らすことで、[コスト削減](/ja/concepts/best-practices/selecting-an-insert-strategy#batch-inserts-if-synchronous)にもつながります。
    </Tip>

    <br />

    簡単な例であっても、一度に複数の行を挿入してみましょう：

    ```sql
    INSERT INTO helloworld.my_first_table (user_id, message, timestamp, metric) VALUES
        (101, 'Hello, ClickHouse!',                                 now(),       -1.0    ),
        (102, 'Insert a lot of rows per batch',                     yesterday(), 1.41421 ),
        (102, 'Sort your data based on your commonly-used queries', today(),     2.718   ),
        (101, 'Granules are the smallest chunks of data read',      now() + 5,   3.14159 )
    ```

    <Note>
      `timestamp` カラムには、さまざまな [**Date**](/ja/reference/data-types/date) 関数や [**DateTime**](/ja/reference/data-types/datetime) 関数が使われています。ClickHouse には便利な関数が数百用意されており、[**Functions** セクション](/ja/reference/functions/index) で確認できます。
    </Note>

    動作を確認しましょう：

    ```sql
    SELECT * FROM helloworld.my_first_table
    ```

    ### ClickHouse Clientを使用したデータの追加

    [**clickhouse client**](/ja/concepts/features/interfaces/cli) というコマンドラインツールを使用して、ClickHouse Cloud サービスに接続することもできます。左側のメニューの `Connect` をクリックすると、接続の詳細情報を確認できます。ダイアログが表示されたら、ドロップダウンから `Native` を選択してください。

    <Image img="/images/_snippets/client_details.webp" size="md" alt="clickhouse clientの接続情報" border />

    <br />

    1. [ClickHouse](/ja/concepts/features/interfaces/cli) をインストールします。

    2. `hostname`、`username`、`password` をご自身のものに置き換えて、コマンドを実行します。

    ```bash
    ./clickhouse client --host HOSTNAME.REGION.CSP.clickhouse.cloud \
    --secure --port 9440 \
    --user default \
    --password <password>
    ```

    スマイルマークのプロンプトが表示されれば、クエリを実行する準備完了です！

    ```response
    :)
    ```

    3. 次のクエリを実行して試してみましょう：

    <br />

    ```sql
    SELECT *
    FROM helloworld.my_first_table
    ORDER BY timestamp
    ```

    レスポンスが見やすいテーブル形式で返ってくることに注目してください：

    ```response
    ┌─user_id─┬─message────────────────────────────────────────────┬───────────timestamp─┬──metric─┐
    │     102 │ Insert a lot of rows per batch                     │ 2022-03-21 00:00:00 │ 1.41421 │
    │     102 │ Sort your data based on your commonly-used queries │ 2022-03-22 00:00:00 │   2.718 │
    │     101 │ Hello, ClickHouse!                                 │ 2022-03-22 14:04:09 │      -1 │
    │     101 │ Granules are the smallest chunks of data read      │ 2022-03-22 14:04:14 │ 3.14159 │
    └─────────┴────────────────────────────────────────────────────┴─────────────────────┴─────────┘

    4 rows in set. Elapsed: 0.008 sec.
    ```

    4. [ClickHouseでサポートされている多数の出力フォーマット](/ja/reference/formats/index)のいずれかを指定するには、[`FORMAT`](/ja/reference/statements/select/format)句を追加します。

    <br />

    ```sql
    SELECT *
    FROM helloworld.my_first_table
    ORDER BY timestamp
    FORMAT TabSeparated
    ```

    上記のクエリでは、出力はタブ区切りで返されます：

    ```response
    Query id: 3604df1c-acfd-4117-9c56-f86c69721121

    102 Insert a lot of rows per batch      2022-03-21 00:00:00     1.41421
    102 Sort your data based on your commonly-used queries  2022-03-22 00:00:00     2.718
    101 Hello, ClickHouse!  2022-03-22 14:04:09     -1
    101 Granules are the smallest chunks of data read       2022-03-22 14:04:14     3.14159

    4 rows in set. Elapsed: 0.005 sec.
    ```

    5. `clickhouse client`を終了するには、**exit** コマンドを入力します。

    <br />

    ```bash
    exit
    ```

    ### ファイルのアップロード

    データベースを使い始める際によく行う作業の一つが、既存のファイルからデータをinsertすることです。クリックストリームデータを表すサンプルデータをオンラインで提供しています。このデータには、ユーザーID、訪問したURL、およびイベントのtimestampが含まれています。

    `data.csv` という名前のCSVファイルに次のようなテキストがあるとします：

    ```bash title="data.csv"
    102,This is data in a file,2022-02-22 10:43:28,123.45
    101,It is comma-separated,2022-02-23 00:00:00,456.78
    103,Use FORMAT to specify the format,2022-02-21 10:43:30,678.90
    ```

    1. 次のコマンドで、データを`my_first_table`に挿入します。

    <br />

    ```bash
    ./clickhouse client --host HOSTNAME.REGION.CSP.clickhouse.cloud \
    --secure --port 9440 \
    --user default \
    --password <password> \
    --query='INSERT INTO helloworld.my_first_table FORMAT CSV' < data.csv
    ```

    2. SQL Console でクエリを実行すると、新しい行がテーブルに表示されていることを確認できます:

    <br />

    <Image img="/images/_snippets/new_rows_from_csv.webp" size="md" alt="CSVファイルからの新規行" />

    <br />
  </Step>
</Steps>
