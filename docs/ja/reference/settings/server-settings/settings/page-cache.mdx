これらの設定はClickHouse serverを設定するもので、ClickHouseのソースコードから自動生成されています。

<div id="page_cache_free_memory_ratio">
  ## page_cache_free_memory_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.15" changeable_without_restart="No" />メモリ制限のうち、ユーザー空間ページキャッシュに使用されないまま確保しておく割合です。Linux の `min_free_kbytes` 設定に相当します。

<div id="page_cache_history_window_ms">
  ## page_cache_history_window_ms
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />解放されたメモリをユーザー空間ページキャッシュで再利用できるようになるまでの遅延。

<div id="page_cache_max_size">
  ## page_cache_max_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />ユーザー空間ページキャッシュの最大サイズ。キャッシュを無効にするには 0 に設定します。page&#95;cache&#95;min&#95;size より大きい値を設定すると、合計のメモリ使用量を制限値 (max&#95;server&#95;memory&#95;usage[&#95;to&#95;ram&#95;ratio]) 未満に保ちながら、利用可能なメモリの大部分を使用できるよう、この範囲内でキャッシュサイズが継続的に調整されます。

<div id="page_cache_min_size">
  ## page_cache_min_size
</div>

<SettingsInfoBlock type="UInt64" default_value="104857600" changeable_without_restart="No" />ユーザー空間ページキャッシュの最小サイズ。

<div id="page_cache_policy">
  ## page_cache_policy
</div>

<SettingsInfoBlock type="String" default_value="SLRU" changeable_without_restart="No" />ユーザー空間ページキャッシュのポリシー名。

<div id="page_cache_shards">
  ## page_cache_shards
</div>

<SettingsInfoBlock type="UInt64" default_value="4" changeable_without_restart="No" />ミューテックスの競合を減らすため、ユーザー空間ページキャッシュをこの数の分片にまたがってストライプ化します。実験的機能であり、パフォーマンスが向上する可能性は低いです。

<div id="page_cache_size_ratio">
  ## page_cache_size_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.5" changeable_without_restart="No" />ユーザー空間ページキャッシュにおける保護キューのサイズが、cache 全体のサイズに占める比率です。
