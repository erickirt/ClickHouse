これらの設定はClickHouseサーバーを設定するためのもので、ClickHouseのソースコードから自動生成されています。

<div id="distributed_ddl">
  ## distributed_ddl
</div>

クラスター上で [分散 DDL クエリ](/ja/reference/statements/distributed-ddl) (`CREATE`、`DROP`、`ALTER`、`RENAME`) を実行するための管理を行います。
[ZooKeeper](/ja/reference/settings/server-settings/settings/zookeeper#zookeeper) が有効な場合にのみ機能します。

`<distributed_ddl>` 内で設定できる項目は次のとおりです。

| Setting                | Description                                                                         | Default Value                   |
| ---------------------- | ----------------------------------------------------------------------------------- | ------------------------------- |
| `cleanup_delay_period` | 新しい node イベントを受信したあと、前回のクリーンアップから `cleanup_delay_period` 秒以上経過している場合にクリーンアップを開始します。 | `60` 秒                          |
| `max_tasks_in_queue`   | queue に保持できる task の最大数です。                                                           | `1,000`                         |
| `path`                 | DDL queries 用の `task_queue` を Keeper に配置する path です                                  |                                 |
| `pool_size`            | 同時に実行できる `ON CLUSTER` queries の数です                                                  |                                 |
| `profile`              | DDL queries の実行に使用する profile です                                                     |                                 |
| `task_max_lifetime`    | 経過時間がこの値を超えた node を削除します。                                                           | `7 * 24 * 60 * 60` (秒単位で 1 週間)  |

**例**

```xml
<distributed_ddl>
    <!-- Path in ZooKeeper to queue with DDL queries -->
    <path>/clickhouse/task_queue/ddl</path>

    <!-- Settings from this profile will be used to execute DDL queries -->
    <profile>default</profile>

    <!-- Controls how much ON CLUSTER queries can be run simultaneously. -->
    <pool_size>1</pool_size>

    <!--
         Cleanup settings (active tasks will not be removed)
    -->

    <!-- Controls task TTL (default 1 week) -->
    <task_max_lifetime>604800</task_max_lifetime>

    <!-- Controls how often cleanup should be performed (in seconds) -->
    <cleanup_delay_period>60</cleanup_delay_period>

    <!-- Controls how many tasks could be in the queue -->
    <max_tasks_in_queue>1000</max_tasks_in_queue>
</distributed_ddl>
```

<div id="distributed_ddl_use_initial_user_and_roles">
  ## distributed_ddl_use_initial_user_and_roles
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />有効にすると、ON CLUSTER クエリの実行時に、リモート分片でもイニシエーターのユーザーとロールがそのまま使用されます。これによりクラスター全体で一貫したアクセス制御を維持できますが、該当するユーザーとロールがすべてのノードに存在している必要があります。
