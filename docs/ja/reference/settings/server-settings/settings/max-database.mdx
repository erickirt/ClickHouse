これらの設定は ClickHouseサーバーを構成するもので、ClickHouse のソースコードから自動生成されています。

<div id="max_database_num_to_throw">
  ## max_database_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />データベース数がこの値を超える場合、server は例外をスローします。0 は制限なしを意味します。

<div id="max_database_num_to_warn">
  ## max_database_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="Yes" />

アタッチされているデータベース数が指定した値を超えると、ClickHouse サーバー は `system.warnings` テーブルに警告メッセージを追加します。

**例**

```xml
<max_database_num_to_warn>50</max_database_num_to_warn>
```

<div id="max_database_replicated_create_table_thread_pool_size">
  ## max_database_replicated_create_table_thread_pool_size
</div>

<SettingsInfoBlock type="UInt32" default_value="1" changeable_without_restart="No" />DatabaseReplicated でレプリカの復旧中にテーブルを作成するためのスレッド数。ゼロを指定すると、スレッド数はコア数と同じになります。
