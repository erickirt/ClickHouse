これらの設定は ClickHouseサーバーを構成するもので、ClickHouse のソースコードから自動生成されています。

<div id="listen_backlog">
  ## listen_backlog
</div>

<SettingsInfoBlock type="UInt32" default_value="4096" changeable_without_restart="No" />

listen socket のバックログ (接続待ちの connection の queue サイズ) です。デフォルト値の `<4096>` は Linux 5.4+ の値と同じです。

通常、この値を変更する必要はありません。理由は次のとおりです。

* デフォルト値が十分に大きいこと。
* client からの connection を受け付けるための専用 thread が server にあること。

そのため、`<nstat>` の `<TcpExtListenOverflows>` が 0 以外で、ClickHouse server でこのカウンターが増加していても、この値を増やす必要があることを意味するわけではありません。理由は次のとおりです。

* 通常、`<4096>` では足りない場合、ClickHouse 内部の scaling に問題があることを示しているため、issue を報告するほうが適切です。
* これは、server が後からさらに多くの connections を処理できることを意味するわけではありません (また、たとえ処理できたとしても、その時点では clients がすでにいなくなっているか、切断されている可能性があります) 。

**例**

```xml
<listen_backlog>4096</listen_backlog>
```

<div id="listen_host">
  ## listen_host
</div>

リクエストの送信元として許可するホストを制限します。サーバーがすべてのホストからのリクエストに応答するようにするには、`::` を指定します。

例:

```xml
<listen_host>::1</listen_host>
<listen_host>127.0.0.1</listen_host>
```

<div id="listen_reuse_port">
  ## listen_reuse_port
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

複数のサーバーが同じアドレス:ポートで待ち受けられるようにします。リクエストはオペレーティングシステムによってランダムなサーバーに振り分けられます。この設定を有効にすることは推奨されません。

**例**

```xml
<listen_reuse_port>0</listen_reuse_port>
```

<div id="listen_try">
  ## listen_try
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

IPv6 または IPv4 ネットワークが利用できない状態で listen を試みても、server は終了しません。

**例**

```xml
<listen_try>0</listen_try>
```
