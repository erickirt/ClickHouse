これらの設定はClickHouseサーバーを設定するためのもので、ClickHouseのソースコードから自動生成されています。

<div id="async_load_databases">
  ## async_load_databases
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

データベースとテーブルの非同期読み込み。

* `true` の場合、`Ordinary`、`Atomic`、`Replicated` engine を使用する system database 以外のすべてのデータベースは、ClickHouseサーバー の起動後に非同期でロードされます。`system.asynchronous_loader` table、`tables_loader_background_pool_size` および `tables_loader_foreground_pool_size` サーバー設定 を参照してください。まだロードされていない table にアクセスしようとするクエリは、その table がロードされるまで待機します。load job が失敗した場合、クエリはエラーを再スローします (`async_load_databases = false` の場合のように server 全体を停止することはありません) 。少なくとも 1 つのクエリが待機している table は、より高い優先度でロードされます。database に対する DDLクエリは、その database がロードされるまで待機します。また、待機中のクエリ総数に対する制限として `max_waiting_queries` の設定も検討してください。
* `false` の場合、server の起動時にすべての database がロードされます。

**例**

```xml
<async_load_databases>true</async_load_databases>
```

<div id="async_load_system_database">
  ## async_load_system_database
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

システムテーブルの非同期読み込みを有効にします。`system` データベース内に多数のログテーブルやパーツがある場合に有用です。`async_load_databases` 設定とは独立しています。

* `true` に設定すると、`Ordinary`、`Atomic`、`Replicated` エンジンを使用するすべてのシステムデータベースが、ClickHouseサーバー の起動後に非同期で読み込まれます。`system.asynchronous_loader` テーブル、`tables_loader_background_pool_size` および `tables_loader_foreground_pool_size` サーバー設定を参照してください。まだ読み込まれていないシステムテーブルにアクセスしようとするクエリは、そのテーブルの起動が完了するまで待機します。1 つ以上のクエリが待機しているテーブルは、より高い優先度で読み込まれます。また、待機中のクエリ総数を制限するために、`max_waiting_queries` 設定の利用も検討してください。
* `false` に設定すると、`system` データベースはサーバーの起動前に読み込まれます。

**例**

```xml
<async_load_system_database>true</async_load_system_database>
```
