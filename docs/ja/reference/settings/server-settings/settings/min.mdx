これらの設定は ClickHouse server を構成するためのもので、ClickHouse のソースコードから自動生成されています。

<div id="min_allocation_size_to_throw_on_memory_limit">
  ## min_allocation_size_to_throw_on_memory_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

`max_server_memory_usage` に達した際に `MEMORY_LIMIT_EXCEEDED` の発生を許可する、汎用的な C++ の割り当て (標準コンテナー、文字列、`std::vector` の拡張、スマートポインターなどで行われる種類) の最小サイズを、バイト単位で指定します。これより小さい汎用割り当ても引き続きメモリトラッカーに計上されますが、制限超過後も成功が許可されるため、OOM 近傍でのクリーンアップや例外処理の経路における不要な失敗を減らせます。

ClickHouse 自身の大きなバッファ — カラムデータ、ハッシュテーブル、アリーナ、クエリパイプライン、I/O バッファ — による割り当ては常にメモリ制限に従うため、この値に関係なく例外を発生させることがあります。この設定が影響するのは、`operator new` を通る暗黙的な割り当てだけです。

値 `0` (デフォルト) では、従来の動作が維持されます。つまり、暗黙的な `operator new` の割り当てはメモリトラッカーによって拒否されず、`MEMORY_LIMIT_EXCEEDED` を発生させられるのは明示的な ClickHouse の割り当てだけです。

副作用を避けるため、値は `max_untracked_memory` より大きく設定することを推奨します。

<div id="min_os_cpu_wait_time_ratio_to_drop_connection">
  ## min_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

接続を切断対象と見なすための、OS の CPU 待機時間 (`OSCPUWaitMicroseconds` Metric) と稼働時間 (`OSCPUVirtualTimeMicroseconds` Metric) の最小比率です。確率の計算には最小比率と最大比率の間の線形補間が使用され、この時点での確率は 0 です。
詳細については、[サーバーの CPU 過負荷時の動作制御](/ja/concepts/features/configuration/settings/server-overload) を参照してください。
