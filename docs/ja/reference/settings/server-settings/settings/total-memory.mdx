これらの設定は ClickHouse server を設定するためのもので、ClickHouse のソースコードから自動生成されています。

<div id="total_memory_profiler_sample_max_allocation_size">
  ## total_memory_profiler_sample_max_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />指定した値以下のサイズの割り当てを、`total_memory_profiler_sample_probability` と同じ確率でランダムに収集します。0 は無効を意味します。このしきい値を想定どおりに機能させるには、&#39;max&#95;untracked&#95;memory&#39; を 0 に設定するとよい場合があります。

<div id="total_memory_profiler_sample_min_allocation_size">
  ## total_memory_profiler_sample_min_allocation_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />指定した値以上のサイズのメモリ割り当てを、`total_memory_profiler_sample_probability` で指定した確率でランダムに収集します。0 は無効を意味します。このしきい値を想定どおりに機能させるには、`max_untracked_memory` を 0 に設定するとよい場合があります。

<div id="total_memory_profiler_step">
  ## total_memory_profiler_step
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />サーバーのメモリ使用量が、バイト数で次の各ステップを超えるたびに、memory profiler はその割り当てを行ったスタックトレースを収集します。0 は memory profiler が無効であることを意味します。数メガバイト未満の値を設定すると、サーバーの動作が遅くなります。

<div id="total_memory_tracker_sample_probability">
  ## total_memory_tracker_sample_probability
</div>

<SettingsInfoBlock type="Double" default_value="0" changeable_without_restart="No" />

ランダムな割り当てと解放を収集し、指定した確率で、`trace_type` が `MemorySample` のものとして [system.trace&#95;log](/ja/reference/system-tables/trace_log) システムテーブルに書き込みます。この確率は、割り当てサイズに関係なく、各割り当てまたは解放ごとに適用されます。サンプリングは、未追跡メモリ量が未追跡メモリ制限 (デフォルト値は `4` MiB) を超えた場合にのみ行われる点に注意してください。[total&#95;memory&#95;profiler&#95;step](/ja/reference/settings/server-settings/settings/total-memory#total_memory_profiler_step) を小さくすると、この制限も下げられます。さらにきめ細かくサンプリングするには、`total_memory_profiler_step` を `1` に設定できます。

設定可能な値:

* 正の double 値。
* `0` — `system.trace_log` システムテーブルへのランダムな割り当ておよび解放の書き込みは無効です。
