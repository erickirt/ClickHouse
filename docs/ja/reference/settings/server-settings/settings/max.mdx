これらの設定は ClickHouseサーバーを構成するもので、ClickHouse のソースコードから自動生成されています。

<div id="max_active_parts_loading_thread_pool_size">
  ## max_active_parts_loading_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="64" changeable_without_restart="Yes" />起動時にアクティブなデータパーツ群 (Active なもの) を読み込むためのスレッド数。

<div id="max_authentication_methods_per_user">
  ## max_authentication_methods_per_user
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />

ユーザーの作成時または変更時に設定できる認証方式の最大数です。
この設定を変更しても、既存のユーザーには影響しません。認証関連の CREATE/ALTER クエリは、この設定で指定された上限を超えると失敗します。
認証に関係しない CREATE/ALTER クエリは成功します。

<Note>
  `0` は無制限を意味します。
</Note>

<div id="max_backup_bandwidth_for_server">
  ## max_backup_bandwidth_for_server
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />server 上のすべてのバックアップに対する、1 秒あたりの最大読み取り速度 (バイト/秒) です。0 は無制限を意味します。

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />同時に保持できる軽量スナップショットの最大数。0 は無制限を意味します。スナップショット数がこの上限に達すると、新しいスナップショットを作成しようとした際に例外がスローされます。

<div id="max_http_index_page_size">
  ## max_http_index_page_size
</div>

<SettingsInfoBlock type="UInt64" default_value="10485760" changeable_without_restart="いいえ" />

HTTP 経由でディレクトリ一覧を表示する際に使用される、HTTP インデックスページのレスポンスの最大サイズです。
レスポンスがこの上限を超えると、クエリはエラーで失敗します。

デフォルト: `10485760` (10 MiB)。

<div id="max_os_cpu_wait_time_ratio_to_drop_connection">
  ## max_os_cpu_wait_time_ratio_to_drop_connection
</div>

<SettingsInfoBlock type="Float" default_value="0" changeable_without_restart="Yes" />

接続を切断するかどうかを判断する際に考慮する、OS の CPU 待機時間 (`OSCPUWaitMicroseconds` メトリクス) とビジー時間 (`OSCPUVirtualTimeMicroseconds` メトリクス) の最大比率です。確率の計算には最小比率と最大比率の間の線形補間が使用され、この時点で確率は 1 になります。
詳細については、[server の CPU 過負荷時の動作制御](/ja/concepts/features/configuration/settings/server-overload)を参照してください。

<div id="max_partition_size_to_drop">
  ## max_partition_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

パーティションの削除に関する制限です。

[MergeTree](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルのサイズが [`max_partition_size_to_drop`](/ja/reference/settings/server-settings/settings/max#max_partition_size_to_drop) (バイト単位) を超える場合、[DROP PARTITION](/ja/reference/statements/alter/partition#drop-partitionpart) クエリを使ってパーティションを削除することはできません。
この設定は、適用のために ClickHouseサーバー を再起動する必要はありません。制限を無効にする別の方法として、`<clickhouse-path>/flags/force_drop_table` ファイルを作成することもできます。

<Note>
  値 `0` は、パーティションを制限なく削除できることを意味します。

  この制限は drop table や truncate table には適用されません。[max&#95;table&#95;size&#95;to&#95;drop](/ja/reference/settings/session-settings#max_table_size_to_drop) を参照してください。
</Note>

**例**

```xml
<max_partition_size_to_drop>0</max_partition_size_to_drop>
```

<div id="max_per_cpu_untracked_memory">
  ## max_per_cpu_untracked_memory
</div>

<SettingsInfoBlock type="UInt64" default_value="8388608" changeable_without_restart="Yes" />

1 つの CPU 上で実行中のすべてのスレッドが、memory tracker にフラッシュされる前に一度に保持できる未追跡メモリ量の上限を、バイト単位で指定します。`max_untracked_memory` は単一スレッドを制限するのに対し、これは CPU ごとの合計を制限するため、多数のスレッドが各スレッドの許容量を積み上げてサーバー全体で大きなオーバーコミットを引き起こすことを防ぎます。したがって、未追跡メモリの合計はおおむね `number_of_cpus * max_per_cpu_untracked_memory` によって制限されます。値 `0` は CPU ごとの上限を無効にします (スレッドごとの `max_untracked_memory` のみが適用されます) 。Linux のみ。

<div id="max_waiting_queries">
  ## max_waiting_queries
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

同時に待機できるクエリの総数の上限です。
必要なテーブルが非同期に読み込まれている間は、待機中のクエリの実行はブロックされます ([`async_load_databases`](/ja/reference/settings/server-settings/settings/async-load#async_load_databases) を参照) 。

<Note>
  以下の設定で制御される上限を確認する際、待機中のクエリはカウントされません。

  * [`max_concurrent_queries`](/ja/reference/settings/server-settings/settings/max-concurrent#max_concurrent_queries)
  * [`max_concurrent_insert_queries`](/ja/reference/settings/server-settings/settings/max-concurrent#max_concurrent_insert_queries)
  * [`max_concurrent_select_queries`](/ja/reference/settings/server-settings/settings/max-concurrent#max_concurrent_select_queries)
  * [`max_concurrent_queries_for_user`](/ja/reference/settings/session-settings#max_concurrent_queries_for_user)
  * [`max_concurrent_queries_for_all_users`](/ja/reference/settings/session-settings#max_concurrent_queries_for_all_users)

  これは、server の起動直後にこれらの上限に達するのを防ぐための補正です。
</Note>

<Note>
  `0` (デフォルト) は無制限を意味します。

  この設定は実行時に変更でき、変更は即座に反映されます。すでに実行中のクエリには影響しません。
</Note>

<div id="max_zookeeper_pooled_connections">

`0` を指定すると、プールされた接続は無効になり、単一のセッションを使用し続けます。
