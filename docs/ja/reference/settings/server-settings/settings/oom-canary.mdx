これらの設定は ClickHouseサーバーを構成するもので、ClickHouse のソースコードから自動生成されています。

<div id="oom_canary_enable">
  ## oom_canary_enable
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

実験的機能です。OOM canary を有効にします。これは、メインの ClickHouse サーバープロセスより先に Linux OOM killer の対象となる犠牲的な子プロセスで、サーバーが負荷を軽減するための猶予を与えます。
Linux &gt;= 5.3 (`pidfd_open` のため) が必要です。古いカーネルでは、起動時に canary は無効化されます。
OOM response には、cgroup v2 `memory.events.local` に OOM-kill の証跡があることが必要で、グローバルなクエリのキャンセル、マージのキャンセル、および `system.crash_log` への書き込みを実行する場合があります。
サーバーの cgroup で cgroup v2 `memory.oom.group` が有効になっている場合、canary はサーバーを保護できません。この場合、カーネルはサーバーを含む cgroup 全体を 1 つの単位として kill するため、OOM response は実行されません。このモードでは、起動時に警告がログに記録されます。
本番環境での検証が完了するまでは、ClickHouse のバージョン間で動作が変わる可能性があります。

<div id="oom_canary_initial_backoff_seconds">
  ## oom_canary_initial_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="1" changeable_without_restart="No" />

連続する OOM canary の再起動の間に適用される、初期バックオフ遅延 (秒) です。
この遅延は再起動のたびに 2 倍になり、`oom_canary_max_backoff_seconds` まで増加します。

<div id="oom_canary_max_backoff_seconds">
  ## oom_canary_max_backoff_seconds
</div>

<SettingsInfoBlock type="UInt64" default_value="60" changeable_without_restart="No" />

連続する OOM canary の再起動間における、秒単位の最大バックオフ時間。

<div id="oom_canary_max_rapid_relaunches">
  ## oom_canary_max_rapid_relaunches
</div>

<SettingsInfoBlock type="UInt64" default_value="10" changeable_without_restart="No" />

継続的なメモリ逼迫下でスラッシングを避けるため、自動再起動が無効になるまでに許容される、OOM canary の連続する短時間での再起動回数の上限です。
canary が `oom_canary_max_backoff_seconds` より長く存続すると、カウンターと再起動バックオフはリセットされます。そのため、長い稼働時間の中で散発的にしか停止しない canary が、最終的に無効化されることはありません。
`oom_canary_relaunch` が true の場合にのみ適用されます。

<div id="oom_canary_relaunch">
  ## oom_canary_relaunch
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />

true の場合、OOM canary は、永続的なセットアップ障害またはサーバーのシャットダウン以外の理由で canary プロセスが終了すると、自動的に再起動されます。これには、OOM kill、その他のシグナル、一時的な終了が含まれ、`oom_canary_max_rapid_relaunches` および backoff 設定の制約を受けます。OOM response 自体が実行されるのは、cgroup v2 の `memory.events.local` が OOM-kill の発生を示す場合に限られます。

<div id="oom_canary_size">
  ## oom_canary_size
</div>

<SettingsInfoBlock type="UInt64" default_value="104857600" changeable_without_restart="No" />

OOM canary の子プロセスが確保してアクセスするメモリ領域のサイズ (バイト単位) 。
`mlock` による領域のロックはベストエフォートです。これには `CAP_IPC_LOCK` または十分な
`RLIMIT_MEMLOCK` が必要です。ロックに失敗した場合、canary は警告をログに出力し、メモリは
確保されたままですが、スワップアウトされる可能性があります。
既定値は 100 MB (104857600) です。値を大きくすると、canary は OOM 時の対象になりやすくなります。
