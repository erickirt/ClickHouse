これらの設定はClickHouse サーバーを構成するものであり、ClickHouseのソースから自動生成されています。

<div id="allow_feature_tier">
  ## allow_feature_tier
</div>

<SettingsInfoBlock type="UInt32" default_value="0" changeable_without_restart="Yes" />

ユーザーが各機能ティアに関連する設定を変更できるかどうかを制御します。

* `0` - すべての設定の変更が許可されます (実験的、ベータ、本番) 。
* `1` - ベータおよび本番の機能設定のみ変更できます。実験的な設定への変更は拒否されます。
* `2` - 本番の設定のみ変更できます。実験的またはベータの設定への変更は拒否されます。

これは、すべての `EXPERIMENTAL` / `BETA` 機能に readonly 制約を設定するのと同等です。

<Note>
  値が `0` の場合は、すべての設定を変更できます。
</Note>

<div id="allow_impersonate_user">
  ## allow_impersonate_user
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />IMPERSONATE 機能 (EXECUTE AS target&#95;user) を有効または無効にします。この設定は非推奨です。

<div id="allow_implicit_no_password">
  ## allow_implicit_no_password
</div>

&#39;IDENTIFIED WITH no&#95;password&#39; が明示的に指定されている場合を除き、パスワードなしのユーザー作成を禁止します。

```xml
<allow_implicit_no_password>1</allow_implicit_no_password>
```

<div id="allow_no_password">
  ## allow_no_password
</div>

安全性の低い `no&#95;password` タイプのパスワードを許可するかどうかを設定します。

```xml
<allow_no_password>1</allow_no_password>
```

<div id="allow_plaintext_password">
  ## allow_plaintext_password
</div>

平文パスワード型 (安全ではない) の使用を許可するかどうかを設定します。

```xml
<allow_plaintext_password>1</allow_plaintext_password>
```

<div id="allow_use_jemalloc_memory">
  ## allow_use_jemalloc_memory
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />jemallocメモリを使用できるようにします。
