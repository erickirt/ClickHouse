これらは ClickHouse サーバー を構成する設定で、ClickHouse のソースコードから自動生成されています。

<div id="max_server_memory_usage">
  ## max_server_memory_usage
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

サーバー が使用できるメモリの最大量です。バイト単位で指定します。

<Note>
  サーバー の最大メモリ使用量は、設定 `max_server_memory_usage_to_ram_ratio` によってさらに制限されます。
</Note>

特殊なケースとして、値 `0` (デフォルト) は、サーバー が使用可能なすべてのメモリを消費できることを意味します (`max_server_memory_usage_to_ram_ratio` による追加の制限を除く) 。

<div id="max_server_memory_usage_to_ram_ratio">
  ## max_server_memory_usage_to_ram_ratio
</div>

<SettingsInfoBlock type="Double" default_value="0.9" changeable_without_restart="No" />

サーバーが使用できるメモリの最大量を、利用可能な総メモリに対する比率で表したものです。

たとえば、値 `0.9` (デフォルト) は、サーバーが利用可能なメモリの 90% まで使用できることを意味します。

メモリ容量の少ないシステムで、メモリ使用量を抑えるのに役立ちます。
RAM とスワップが少ないホストでは、[`max_server_memory_usage_to_ram_ratio`](/ja/reference/settings/server-settings/settings/max-server-memory-usage#max_server_memory_usage_to_ram_ratio) を 1 より大きい値に設定する必要がある場合があります。

この比率は、次の 2 つの方法で適用されます。

* 起動時 (および設定の再読み込み時) には、前述のとおり、サーバーのハードメモリ制限の上限を、物理 RAM 全体に対するこの割合に設定します。
* 実行時には、background memory worker が `(resident memory + system available memory) * max_server_memory_usage_to_ram_ratio` に基づいてハードメモリ制限を定期的に再計算し、同じホスト上で動作するほかのプロセスのためのヘッドルームを確保します。有限のメモリ制限を持つ cgroup 内で実行している場合は、ホスト全体の値ではなく cgroup の利用可能メモリが使用されます。ほかのプロセスの使用量が増えて利用可能メモリが減ると、サーバーのハード制限もそれに合わせて引き下げられ、ホスト (または cgroup) のメモリが枯渇する前に増加が抑えられます。

比率を `0` に設定すると、起動時の上限制限と実行時の調整の両方が無効になります。実行時の調整は、Linux 以外のシステムでは no-op であり、この設定を公開していない `clickhouse-keeper` でも同様です。静的な起動時／再読み込み時の上限制限は維持しつつ、実行時の調整 (以前のバージョンの動作) だけを無効にするには、`memory_worker_dynamic_hard_limit` を `0` に設定してください。

<Note>
  サーバーの最大メモリ消費量は、設定 `max_server_memory_usage` によってさらに制限されます。
</Note>
