これらの設定はClickHouseサーバーを構成するもので、ClickHouseのソースコードから自動生成されています。

<div id="distributed_cache_apply_throttling_settings_from_client">
  ## distributed_cache_apply_throttling_settings_from_client
</div>

<SettingsInfoBlock type="Bool" default_value="1" changeable_without_restart="No" />cache server がクライアントから受信したスロットリング設定を適用するかどうか。

<div id="distributed_cache_keep_up_free_connections_ratio">
  ## distributed_cache_keep_up_free_connections_ratio
</div>

<SettingsInfoBlock type="Float" default_value="0.1" changeable_without_restart="No" />分散キャッシュが空き接続として維持しようとする接続数のソフト制限です。空き接続数が distributed&#95;cache&#95;keep&#95;up&#95;free&#95;connections&#95;ratio * max&#95;connections を下回ると、その数がこの制限を再び上回るまで、最後のアクティビティが最も古い接続から順に閉じられます。

<div id="distributed_cache_write_pool_size">
  ## distributed_cache_write_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />バックグラウンドスレッドで同時に実行できる分散キャッシュの書き込みリクエストの最大数です (全クエリ合計) 。この上限に達すると、書き込みはバックグラウンドスレッドではなく、インライン (呼び出し元のスレッド上) で cache を経由して処理されるため、スロットの空きを待ってブロックされることも、スレッドが無制限に作成されることもありません。
