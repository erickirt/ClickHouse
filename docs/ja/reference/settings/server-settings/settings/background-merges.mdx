これらの設定はClickHouseサーバーを設定するためのもので、ClickHouseのソースから自動生成されています。

<div id="background_merges_mutations_concurrency_ratio">
  ## background_merges_mutations_concurrency_ratio
</div>

<SettingsInfoBlock type="Float" default_value="2" changeable_without_restart="No" />

スレッド数に対する、同時実行可能なバックグラウンドマージおよびミューテーション数の比率を設定します。

たとえば、比率が 2 で [`background_pool_size`](/ja/reference/settings/server-settings/settings/background#background_pool_size) が 16 に設定されている場合、ClickHouse は 32 件のバックグラウンドマージを同時実行できます。これは、バックグラウンド処理を中断して後回しにできるためです。小規模なマージを優先して実行できるようにするために、この仕組みが必要です。

<Note>
  この比率は実行時に増やすことしかできません。下げるにはサーバーを再起動する必要があります。

  [`background_pool_size`](/ja/reference/settings/server-settings/settings/background#background_pool_size) 設定と同様に、[`background_merges_mutations_concurrency_ratio`](/ja/reference/settings/server-settings/settings/background-merges#background_merges_mutations_concurrency_ratio) も、後方互換性のために `default` プロファイルから適用できます。
</Note>

<div id="background_merges_mutations_scheduling_policy">
  ## background_merges_mutations_scheduling_policy
</div>

<SettingsInfoBlock type="String" default_value="round_robin" changeable_without_restart="No" />

バックグラウンドマージおよびミューテーションのスケジューリング方法を定めるポリシーです。設定可能な値は `round_robin` と `shortest_task_first` です。

バックグラウンドスレッドプールで次に実行するマージまたはミューテーションを選択する際に使用されるアルゴリズムです。ポリシーはサーバーを再起動せずに実行時に変更できます。
後方互換性のため、`default` プロファイルから適用することもできます。

設定可能な値:

* `round_robin` — すべての同時実行のマージおよびミューテーションを、飢餓が発生しないようラウンドロビン順で実行します。小さいマージは、マージするブロック数が少ないため、大きいものより速く完了します。
* `shortest_task_first` — 常に、より小さいマージまたはミューテーションから実行します。マージとミューテーションには、結果サイズに基づいて優先度が割り当てられます。サイズの小さいマージは、大きいものより常に優先されます。このポリシーにより小さなパーツのマージを可能な限り高速にできますが、`INSERT` が大量に発生するパーティションでは、大きなマージが無期限に飢餓状態になる可能性があります。
