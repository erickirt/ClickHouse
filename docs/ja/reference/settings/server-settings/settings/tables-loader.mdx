これらの設定は ClickHouse server を設定するためのもので、ClickHouse のソースコードから自動生成されています。

<div id="tables_loader_background_pool_size">
  ## tables_loader_background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

background pool で非同期のロードジョブを実行するスレッド数を設定します。background pool は、サーバー起動後、そのテーブルを待機しているクエリがない場合に、テーブルを非同期にロードするために使用されます。テーブル数が多い場合は、background pool のスレッド数を少なくしておくと有効なことがあります。これにより、同時実行のクエリ実行に CPU リソースを確保できます。

<Note>
  `0` は、利用可能なすべての CPU が使用されることを意味します。
</Note>

<div id="tables_loader_foreground_pool_size">
  ## tables_loader_foreground_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />

フォアグラウンドプールでロードジョブを実行するスレッド数を設定します。フォアグラウンドプールは、server がポートで待ち受けを開始する前に table を同期的にロードするため、および待機対象の table をロードするために使用されます。フォアグラウンドプールは background pool より優先度が高くなっています。つまり、フォアグラウンドプールで job が実行中の間は、background pool ではどの job も開始されません。

<Note>
  `0` は、使用可能なすべての CPU が使用されることを意味します。
</Note>
