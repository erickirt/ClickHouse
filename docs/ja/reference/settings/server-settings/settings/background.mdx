これらの設定は ClickHouse server を設定するためのもので、ClickHouse のソースコードから自動生成されています。

<div id="background_buffer_flush_schedule_pool_size">
  ## background_buffer_flush_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="Increase only" />[Buffer エンジンのテーブル](/ja/reference/engines/table-engines/special/buffer) に対してバックグラウンドで flush 操作を実行する際に使用されるスレッドの最大数。

<div id="background_common_pool_size">
  ## background_common_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />バックグラウンドで [*MergeTree-engine](/ja/reference/engines/table-engines/mergetree-family/index) テーブルに対して各種の操作 (主にガベージコレクション) を実行するために使用されるスレッドの最大数です。

<div id="background_distributed_schedule_pool_size">
  ## background_distributed_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="増加のみ" />分散送信の実行に使用するスレッドの最大数です。

<div id="background_fetches_pool_size">
  ## background_fetches_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />[*MergeTree-engine](/ja/reference/engines/table-engines/mergetree-family/index) テーブルで、別のレプリカからデータパーツをバックグラウンドでフェッチする際に使用されるスレッドの最大数です。

<div id="background_message_broker_schedule_pool_size">
  ## background_message_broker_schedule_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="増加のみ" />メッセージストリーミング向けのバックグラウンド処理の実行に使用するスレッドの最大数です。

<div id="background_move_pool_size">
  ## background_move_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="8" changeable_without_restart="No" />バックグラウンドで *MergeTree-engine tables のデータパーツを別のディスクまたはボリュームに移動する際に使用されるスレッドの最大数。

<div id="background_pool_size">
  ## background_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="16" changeable_without_restart="No" />

MergeTree エンジンを使用するテーブルで、バックグラウンドマージおよびミューテーションを実行するスレッド数を設定します。

<Note>
  * この設定は、後方互換性のため、ClickHouse server の起動時に `default` プロファイル設定から適用することもできます。
  * 実行時に増やせるのはスレッド数のみです。
  * スレッド数を減らすには、サーバーを再起動する必要があります。
  * この設定を調整することで、CPU とディスクの負荷を管理できます。
</Note>

<Danger>
  プールサイズを小さくすると CPU とディスクの使用量は抑えられますが、バックグラウンド処理の進行が遅くなり、最終的にはクエリパフォーマンスに影響する可能性があります。
</Danger>

変更する前に、次のような関連する MergeTree 設定も確認してください。

* [`number_of_free_entries_in_pool_to_lower_max_size_of_merge`](/ja/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_lower_max_size_of_merge).
* [`number_of_free_entries_in_pool_to_execute_mutation`](/ja/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_mutation).
* [`number_of_free_entries_in_pool_to_execute_optimize_entire_partition`](/ja/reference/settings/merge-tree-settings#number_of_free_entries_in_pool_to_execute_optimize_entire_partition)

**例**

```xml
<background_pool_size>16</background_pool_size>
```
