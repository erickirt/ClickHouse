これらはClickHouse サーバーを設定するための項目で、ClickHouseのソースコードから自動生成されています。

<div id="tmp_path">
  ## tmp_path
</div>

<SettingsInfoBlock type="String" default_value="/var/lib/clickhouse/tmp/" changeable_without_restart="いいえ" />

大規模なクエリの処理時に使用する一時データを保存するための、local filesystem 上のパスです。

<Note>
  * 一時データの保存先を設定するには、tmp&#95;path、tmp&#95;policy、temporary&#95;data&#95;in&#95;cache のうち 1 つだけを使用できます。
  * 末尾のスラッシュは必須です。
</Note>

**例**

```xml
<tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
```

<div id="tmp_policy">
  ## tmp_policy
</div>

一時データ用ストレージのポリシーです。`tmp` プレフィックスを持つすべてのファイルは、起動時に削除されます。

<Note>
  `tmp_policy` としてオブジェクトストレージを使用する際の推奨事項:

  * 各サーバーで別々の `bucket:path` を使用する
  * `metadata_type=plain` を使用する
  * この bucket に対して有効期限 (TTL) を設定することもできます
</Note>

<Note>
  * 一時データストレージの設定に使用できるオプションは 1 つだけです: `tmp_path` ,`tmp_policy`, `temporary_data_in_cache`.
  * `move_factor`, `keep_free_space_bytes`,`max_data_part_size_bytes` は無視されます。
  * ポリシーには *ちょうど 1 つの volume* が含まれている必要があります

  詳細については、[MergeTree Table Engine](/ja/reference/engines/table-engines/mergetree-family/mergetree) のドキュメントを参照してください。
</Note>

**例**

`/disk1` がいっぱいになると、一時データは `/disk2` に保存されます。

```xml highlight={13-20,24}
<clickhouse>
<storage_configuration>
<disks>
<disk1>
<path>/disk1/</path>
</disk1>
<disk2>
<path>/disk2/</path>
</disk2>
</disks>

<policies>
<tmp_two_disks>
<volumes>
<main>
<disk>disk1</disk>
<disk>disk2</disk>
</main>
</volumes>
</tmp_two_disks>
</policies>
</storage_configuration>

<tmp_policy>tmp_two_disks</tmp_policy>
</clickhouse>
```
