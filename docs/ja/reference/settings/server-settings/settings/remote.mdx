これらの設定はClickHouseサーバーを構成するもので、ClickHouseのソースコードから自動生成されています。

<div id="remote_servers">
  ## remote_servers
</div>

[Distributed](/ja/reference/engines/table-engines/special/distributed) テーブルエンジンおよび `cluster` テーブル関数で使用されるクラスター構成です。

**例**

```xml
<remote_servers incl="clickhouse_remote_servers" />
```

`incl` 属性の値については、「[設定ファイル](/ja/concepts/features/configuration/server-config/configuration-files)」セクションを参照してください。

<div id="remote_url_allow_hosts">
  ## remote_url_allow_hosts
</div>

URL 関連のストレージエンジンおよびテーブル関数で使用を許可する host の一覧です。

`\<host\>` xml tag を使って host を追加する場合:

* name は DNS resolution の前にチェックされるため、URL 内の表記と完全に一致するように指定する必要があります。例: `<host>clickhouse.com</host>`
* URL でポートが明示的に指定されている場合は、host:port 全体がまとめてチェックされます。例: `<host>clickhouse.com:80</host>`
* host をポートなしで指定した場合は、その host の任意のポートが許可されます。例: `<host>clickhouse.com</host>` を指定すると、`clickhouse.com:20` (FTP)、`clickhouse.com:80` (HTTP)、`clickhouse.com:443` (HTTPS) などが許可されます。
* host を IP アドレスとして指定した場合は、URL で指定されたとおりにチェックされます。例: `[2a02:6b8:a::a]`。
* リダイレクトがあり、かつリダイレクトのサポートが enabled の場合は、すべてのリダイレクト (`location` フィールド) がチェックされます。

例:

```sql
<remote_url_allow_hosts>
    <host>clickhouse.com</host>
</remote_url_allow_hosts>
```
