これらの設定はClickHouseサーバーの構成に使用され、ClickHouseのソースコードから自動生成されています。

<div id="max_prefixes_deserialization_thread_pool_free_size">
  ## max_prefixes_deserialization_thread_pool_free_size
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

プレフィックスのデシリアライゼーション用スレッドプール内の**アイドル状態**のスレッド数が `max_prefixes_deserialization_thread_pool_free_size` を超えると、ClickHouse はアイドル状態のスレッドが占有しているリソースを解放し、プールサイズを縮小します。必要に応じて、スレッドは再度作成されます。

<div id="max_prefixes_deserialization_thread_pool_size">
  ## max_prefixes_deserialization_thread_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="Yes" />

ClickHouse は、MergeTree の wide パーツ内のファイルプレフィックスからカラムおよびサブカラムのメタデータを並列に読み取るために、スレッドプールのスレッドを使用します。`max_prefixes_deserialization_thread_pool_size` は、このプール内のスレッドの最大数を制限します。
