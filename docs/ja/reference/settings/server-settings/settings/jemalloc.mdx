これらの設定は ClickHouse サーバーを構成するもので、ClickHouse のソースコードから自動生成されています。

<div id="jemalloc_collect_global_profile_samples_in_trace_log">
  ## jemalloc_collect_global_profile_samples_in_trace_log
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />jemalloc でサンプリングされた割り当てを system.trace&#95;log に保存します

<div id="jemalloc_max_background_threads_num">
  ## jemalloc_max_background_threads_num
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="No" />生成する jemalloc のバックグラウンドスレッドの最大数です。0 に設定すると、jemalloc のデフォルト値が使用されます

<div id="jemalloc_profiler_sampling_rate">
  ## jemalloc_profiler_sampling_rate
</div>

<SettingsInfoBlock type="UInt64" default_value="19" changeable_without_restart="No" />

jemalloc の `lg_prof_sample` を制御します。これは、割り当てサンプル間の平均間隔 (バイト単位) の底2対数です。
デフォルト値の 19 は 512 KiB に相当します。これをより小さい値に設定するとサンプリング頻度が増加し (オーバーヘッドは増えますが、より詳細になります) 、より大きい値にすると低下します。
この値を変更すると `prof.reset` が呼び出され、蓄積されたすべてのプロファイリング統計がリセットされます。プロファイリングが有効になっている必要があります (`MALLOC_CONF=prof:true`) 。
