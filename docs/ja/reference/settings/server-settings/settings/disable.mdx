これらの設定はClickHouse serverを設定するためのもので、ClickHouseのソースコードから自動生成されています。

<div id="disable_insertion_and_mutation">
  ## disable_insertion_and_mutation
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />

insert/alter/delete クエリを無効にします。挿入やミューテーションが読み取り性能に影響しないよう、ノードを読み取り専用にする必要がある場合に有効にします。この設定にかかわらず、外部エンジン (S3、DataLake、MySQL、PostrgeSQL、Kafka など) への insert は許可されます。

<div id="disable_internal_dns_cache">
  ## disable_internal_dns_cache
</div>

<SettingsInfoBlock type="Bool" default_value="0" changeable_without_restart="No" />内部DNSキャッシュを無効にします。Kubernetes など、インフラストラクチャの変更が頻繁に発生するシステムで ClickHouse を運用する場合に推奨されます。

<div id="disable_tunneling_for_https_requests_over_http_proxy">
  ## disable_tunneling_for_https_requests_over_http_proxy
</div>

既定では、`HTTP` プロキシ経由で `HTTPS` リクエストを送る際に、トンネリング (つまり `HTTP CONNECT`) が使用されます。この設定を使うと、これを無効にできます。

**no&#95;proxy**

既定では、すべてのリクエストがプロキシを経由します。特定のホストに対してこれを無効にするには、`no_proxy` 変数を設定する必要があります。
これは、list リゾルバおよび remote リゾルバでは `<proxy>` 句内で、environment リゾルバでは環境変数として設定できます。
IP アドレス、ドメイン、サブドメイン、および完全にバイパスするための `'*'` ワイルドカードをサポートしています。先頭のドットは curl と同様に取り除かれます。

**Example**

以下の設定では、`clickhouse.cloud` とそのすべてのサブドメイン (例: `auth.clickhouse.cloud`) へのリクエストでプロキシをバイパスします。
先頭にドットが付いている GitLab についても同様です。`gitlab.com` と `about.gitlab.com` はどちらもプロキシをバイパスします。

```xml
<proxy>
    <no_proxy>clickhouse.cloud,.gitlab.com</no_proxy>
    <http>
        <uri>http://proxy1</uri>
        <uri>http://proxy2:3128</uri>
    </http>
    <https>
        <uri>http://proxy1:3128</uri>
    </https>
</proxy>
```
