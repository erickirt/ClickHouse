これらの設定はClickHouse サーバーを構成するもので、ClickHouseのソースコードから自動生成されています。

<div id="disk_connections_hard_limit">
  ## disk_connections_hard_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />この制限に達した状態で接続の作成を試みると、例外がスローされます。ハード制限を無効にするには 0 に設定してください。この制限はディスクへの接続に適用されます。

<div id="disk_connections_rcvbuf">
  ## disk_connections_rcvbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="204800" changeable_without_restart="Yes" />ディスク (S3、Azure、GCS) 接続における `SO_RCVBUF` オプションのサイズです。デフォルト値は 204800 (200 KB) です。オブジェクトストレージには同一 Region 内からアクセスし、帯域幅遅延積は 100 KB を大きく下回るため、小さめの上限値でも最大スループットを確保でき、指定しない場合にカーネルのソケットごとのバッファ自動調整によって割り当てられる数 MiB のバッファを避けられます。代わりに受信バッファでカーネルの TCP 自動調整を使用するには、0 に設定します。注: この設定を 0 に戻しても、自動調整が復元されるのは新しく作成される接続のみです。既存のプールされた接続は、再作成されるまで固定されたバッファサイズを保持します。

<div id="disk_connections_sndbuf">
  ## disk_connections_sndbuf
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />ディスク (S3、Azure、GCS) 接続における SO&#95;SNDBUF オプションのサイズです。0 より大きい値を設定すると、送信バッファに対するカーネルの TCP 自動調整を上書きします。0 = カーネルのデフォルト (自動調整) 。注: この設定を 0 に戻しても、自動調整が復元されるのは新たに作成された接続のみで、既存のプールされた接続は再作成されるまで固定のバッファサイズを保持します。

<div id="disk_connections_soft_limit">
  ## disk_connections_soft_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="No" />この制限を超える接続は、有効期間が大幅に短くなります。この制限は、ディスクへの接続に適用されます。

<div id="disk_connections_store_limit">
  ## disk_connections_store_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="10000" changeable_without_restart="No" />この上限を超える接続は、使用後にリセットされます。接続 cache を無効にするには、0 に設定します。この上限は、ディスクへの接続に適用されます。

<div id="disk_connections_warn_limit">
  ## disk_connections_warn_limit
</div>

<SettingsInfoBlock type="UInt64" default_value="8000" changeable_without_restart="No" />使用中の接続数がこの制限を上回ると、警告メッセージがログに記録されます。この制限はディスクへの接続に適用されます。
