これらの設定はClickHouse サーバーを構成するためのもので、ClickHouseのソースコードから自動生成されています。

<div id="max_table_num_to_throw">
  ## max_table_num_to_throw
</div>

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />

テーブル数がこの値を超えると、サーバーは例外をスローします。

次のテーブルはカウント対象外です:

* ビュー
* remote
* Dictionary
* system

カウント対象となるのは、次のデータベースエンジンのテーブルのみです:

* Atomic
* Ordinary
* Replicated
* Lazy

<Note>
  値 `0` は、制限がないことを意味します。
</Note>

**例**

```xml
<max_table_num_to_throw>400</max_table_num_to_throw>
```

<div id="max_table_num_to_warn">
  ## max_table_num_to_warn
</div>

<SettingsInfoBlock type="UInt64" default_value="5000" changeable_without_restart="Yes" />

アタッチされているテーブル数が指定した値を超えると、ClickHouse サーバー は `system.warnings` テーブルに警告メッセージを追加します。

**例**

```xml
<max_table_num_to_warn>400</max_table_num_to_warn>
```

<div id="max_table_size_to_drop">
  ## max_table_size_to_drop
</div>

<SettingsInfoBlock type="UInt64" default_value="50000000000" changeable_without_restart="Yes" />

テーブル削除に関する制限です。

[MergeTree](/ja/reference/engines/table-engines/mergetree-family/mergetree) テーブルのサイズが `max_table_size_to_drop` (バイト単位) を超える場合、[`DROP`](/ja/reference/statements/drop) クエリまたは [`TRUNCATE`](/ja/reference/statements/truncate) クエリでは削除できません。

<Note>
  値が `0` の場合は、すべてのテーブルを制限なく削除できます。

  この設定を適用するために ClickHouse サーバー を再起動する必要はありません。制限を無効にする別の方法として、`<clickhouse-path>/flags/force_drop_table` ファイルを作成する方法もあります。
</Note>

**例**

```xml
<max_table_size_to_drop>0</max_table_size_to_drop>
```
