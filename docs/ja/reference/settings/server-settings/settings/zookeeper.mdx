これらの設定はClickHouseサーバーを構成するもので、ClickHouseのソースから自動生成されています。

<div id="zookeeper">
  ## zookeeper
</div>

ClickHouse が [ZooKeeper](http://zookeeper.apache.org/) クラスターと連携するための設定です。ClickHouse は、レプリケートテーブルを使用する際に、レプリカのメタデータを保存するため ZooKeeper を使用します。レプリケートテーブルを使用しない場合、このパラメーターセクションは省略できます。

以下の設定はサブタグで指定できます。

| Setting                                         | Description                                                                                                                                                                                                                                                                                      |
| ----------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `node`                                          | ZooKeeper のエンドポイント。複数のエンドポイントを設定できます。例: `<node index="1"><host>example_host</host><port>2181</port></node>`。`index` 属性は、ZooKeeper クラスターへの接続を試みる際の `node` の順序を指定します。                                                                                                                              |
| `operation_timeout_ms`                          | 1 回の操作に対する最大タイムアウト (ミリ秒) 。                                                                                                                                                                                                                                                                       |
| `session_timeout_ms`                            | クライアントセッションの最大タイムアウト (ミリ秒) 。                                                                                                                                                                                                                                                                     |
| `root` (optional)                               | ClickHouse サーバー が使用する znode 群のルートとなる znode。                                                                                                                                                                                                                                                    |
| `fallback_session_lifetime.min` (optional)      | プライマリが利用できない場合にフォールバック先ノードへ接続する ZooKeeper セッションの存続時間の最小値 (load-balancing) 。秒単位で設定します。デフォルト: 3 時間。                                                                                                                                                                                                |
| `fallback_session_lifetime.max` (optional)      | プライマリが利用できない場合にフォールバック先ノードへ接続する ZooKeeper セッションの存続時間の最大値 (load-balancing) 。秒単位で設定します。デフォルト: 6 時間。                                                                                                                                                                                                |
| `identity` (optional)                           | 指定した znode にアクセスするために ZooKeeper が要求するユーザー名とパスワード。                                                                                                                                                                                                                                                |
| `use_compression` (optional)                    | `true` に設定すると、Keeper プロトコルで圧縮を有効にします。                                                                                                                                                                                                                                                            |
| `use_xid_64` (optional)                         | 64 ビットの transaction ID を有効にします。拡張 transaction ID 形式を有効にするには `true` に設定します。デフォルト: `false`。                                                                                                                                                                                                        |
| `pass_opentelemetry_tracing_context` (optional) | Keeper リクエストへの OpenTelemetry トレーシングコンテキストの伝播を有効にします。有効にすると Keeper 操作のトレーシングスパンが作成され、ClickHouse と Keeper をまたいだ分散トレーシングが可能になります。詳細は [Tracing ClickHouse Keeper Requests](/ja/guides/oss/deployment-and-scaling/monitoring/opentelemetry#tracing-clickhouse-keeper-requests) を参照してください。デフォルト: `false`。 |

また、ZooKeeper ノードの選択アルゴリズムを指定する `zookeeper_load_balancing` 設定 (optional) もあります。

| Algorithm Name                  | Description                                                                       |
| ------------------------------- | --------------------------------------------------------------------------------- |
| `random`                        | ZooKeeper ノードのうち 1 つをランダムに選択します。                                                  |
| `in_order`                      | 最初の ZooKeeper ノードを選択し、利用できない場合は 2 番目、その次は 3 番目というように順に選択します。                      |
| `nearest_hostname`              | server の hostname に最も近い hostname を持つ ZooKeeper ノードを選択します。hostname は名前の接頭辞で比較されます。 |
| `hostname_levenshtein_distance` | `nearest_hostname` と同様ですが、hostname をレーベンシュタイン距離で比較します。                            |
| `first_or_random`               | 最初の ZooKeeper ノードを選択し、利用できない場合は残りの ZooKeeper ノードから 1 つをランダムに選択します。                |
| `round_robin`                   | 最初の ZooKeeper ノードを選択し、再接続が発生した場合は次のノードを選択します。                                     |

**設定例**

```xml
<zookeeper>
    <node>
        <host>example1</host>
        <port>2181</port>
    </node>
    <node>
        <host>example2</host>
        <port>2181</port>
    </node>
    <session_timeout_ms>30000</session_timeout_ms>
    <operation_timeout_ms>10000</operation_timeout_ms>
    <!-- Optional. Chroot suffix. Should exist. -->
    <root>/path/to/zookeeper/node</root>
    <!-- Optional. Zookeeper digest ACL string. -->
    <identity>user:password</identity>
    <!--<zookeeper_load_balancing>random / in_order / nearest_hostname / hostname_levenshtein_distance / first_or_random / round_robin</zookeeper_load_balancing>-->
    <zookeeper_load_balancing>random</zookeeper_load_balancing>
    <!-- Optional. Enable 64-bit transaction IDs. -->
    <use_xid_64>false</use_xid_64>
    <!-- Optional. Enable OpenTelemetry tracing context propagation. -->
    <pass_opentelemetry_tracing_context>false</pass_opentelemetry_tracing_context>
</zookeeper>
```

<div id="zookeeper_log">
  ## zookeeper_log
</div>

[`zookeeper_log`](/ja/reference/system-tables/zookeeper_log) システムテーブルの設定です。

以下の設定はサブタグで設定できます。

<SystemLogParameters />

**例**

```xml
<clickhouse>
    <zookeeper_log>
        <database>system</database>
        <table>zookeeper_log</table>
        <flush_interval_milliseconds>7500</flush_interval_milliseconds>
        <ttl>event_date + INTERVAL 1 WEEK DELETE</ttl>
    </zookeeper_log>
</clickhouse>
```
