これらの設定はClickHouseサーバーを構成するもので、ClickHouseのソースコードから自動生成されています。

<div id="parts_killer_max_condemned_parts_per_batch">
  ## parts_killer_max_condemned_parts_per_batch
</div>

<SettingsInfoBlock type="NonZeroUInt64" default_value="100000" changeable_without_restart="No" />

1 回のクリーンアップサイクルで SharedMergeTree テーブルが Keeper から取得する廃棄対象パーツの最大数です。バッチの上限を設定することで、一覧取得と multi-get を Keeper の操作タイムアウト内に収められるため、テーブルに非常に大量の廃棄対象パーツのバックログが蓄積している場合でも、クリーンアップは継続して進行できます。0 より大きい値である必要があります。`0` を指定すると、Keeper は毎サイクル空の一覧を返すため、クリーンアップは恒久的に停止します。ClickHouse Cloud でのみ利用できます

<div id="parts_killer_pool_size">
  ## parts_killer_pool_size
</div>

<SettingsInfoBlock type="UInt64" default_value="128" changeable_without_restart="No" />

shared merge tree のパーツキラーによるクリーンアップ処理用のスレッド数。ClickHouse Cloud でのみ利用できます
