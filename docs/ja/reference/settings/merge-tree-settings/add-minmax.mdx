これらの設定は [system.merge&#95;tree&#95;settings](/ja/reference/system-tables/merge_tree_settings) で参照でき、ClickHouse のソースコードから自動生成されています。

<div id="add_minmax_index_for_block_number_column">
  ## add_minmax_index_for_block_number_column
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "0"},{"label": "新しい設定"}]}]} />

有効にすると、永続仮想カラム `_block_number` に対して暗黙的な min-max (スキッピング) 索引が追加されます。
これを有効にするには `enable_block_number_column = 1` が必要です。この索引が構築されるのはマージ時のみであり、
挿入時には構築されません。挿入時点ではブロック番号は暫定的で、定数値に対する索引になってしまうためです。

<div id="add_minmax_index_for_block_offset_column">
  ## add_minmax_index_for_block_offset_column
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.5"},{"label": "0"},{"label": "新しい設定"}]}]} />

有効にすると、永続仮想カラム `_block_offset` に対する暗黙的な min-max (スキッピング) 索引が追加されます。
有効にするには `enable_block_offset_column = 1` が必要です。索引はデータ挿入時ではなく、マージ時にのみ構築されます。
