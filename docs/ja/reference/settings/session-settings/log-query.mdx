クエリ設定をquery&#95;logとOpenTelemetryのspanログに記録します。

<div id="log_query_threads">
  ## log_query_threads
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

クエリスレッドのログ記録を設定します。

クエリスレッドは [system.query&#95;thread&#95;log](/ja/reference/system-tables/query_thread_log) テーブルに記録されます。この設定は、[log&#95;queries](/ja/reference/settings/session-settings/log-queries#log_queries) が true の場合にのみ有効です。この設定を有効にすると、ClickHouse が実行するクエリのスレッドが、サーバー設定パラメータ [query&#95;thread&#95;log](/ja/reference/settings/server-settings/settings#query_thread_log) のルールに従って記録されます。

設定可能な値:

* 0 — 無効。
* 1 — 有効。

**例**

```text
log_query_threads=1
```

<div id="log_query_views">
  ## log_query_views
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

クエリビューのログ記録を設定します。

この設定を有効にして ClickHouse が実行するクエリに関連するビュー (materialized view または live view) がある場合、それらはサーバー設定パラメータ [query&#95;views&#95;log](/ja/reference/settings/server-settings/settings#query_views_log) に記録されます。

例:

```text
log_query_views=1
```
