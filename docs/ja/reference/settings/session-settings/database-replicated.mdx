これらの設定は [system.settings](/ja/reference/system-tables/settings) で利用でき、[ソース](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp) から自動生成されています。

<div id="database_replicated_allow_explicit_uuid">
  ## database_replicated_allow_explicit_uuid
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "Replicated データベースのテーブルで UUID を明示的に指定できないようにする新しい設定が追加されました"}]}]} />

0 - Replicated データベースのテーブルで UUID を明示的に指定することを許可しません。1 - 許可します。2 - 許可しますが、指定された UUID は無視され、代わりにランダムな UUID が生成されます。

<div id="database_replicated_allow_heavy_create">
  ## database_replicated_allow_heavy_create
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.7"},{"label": "0"},{"label": "Replicated データベースエンジンでは、長時間実行されるDDLクエリ（CREATE AS SELECT と POPULATE）は禁止されていました"}]}]} />

Replicated データベースエンジンで、長時間実行されるDDLクエリ (CREATE AS SELECT と POPULATE) を許可します。DDL queue が長時間ブロックされる可能性がある点に注意してください。

<div id="database_replicated_allow_only_replicated_engine">
  ## database_replicated_allow_only_replicated_engine
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

Replicated エンジンのデータベースでは、レプリケートテーブルのみ作成を許可します

Cloud でのデフォルト値: `1`。

<div id="database_replicated_allow_replicated_engine_arguments">
  ## database_replicated_allow_replicated_engine_arguments
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "24.9"},{"label": "0"},{"label": "デフォルトでは明示的な引数を許可しない"}]}]} />

0 - Replicated データベース内の *MergeTree テーブルで、ZooKeeper パスおよびレプリカ名を明示的に指定することを許可しません。1 - 許可します。2 - 許可しますが、指定されたパスは無視して、代わりにデフォルトのパスを使用します。3 - 許可し、警告も記録しません。

<div id="database_replicated_always_detach_permanently">
  ## database_replicated_always_detach_permanently
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

データベースエンジンが Replicated の場合、DETACH TABLE は DETACH TABLE PERMANENTLY として実行されます

<div id="database_replicated_enforce_synchronous_settings">
  ## database_replicated_enforce_synchronous_settings
</div>

<SettingsInfoBlock type="Bool" default_value="0" />

一部のクエリで同期的な待機を強制します (関連項目: database&#95;atomic&#95;wait&#95;for&#95;drop&#95;and&#95;detach&#95;synchronously、mutations&#95;sync、alter&#95;sync) 。これらの設定を有効にすることは推奨されません。

<div id="database_replicated_initial_query_timeout_sec">
  ## database_replicated_initial_query_timeout_sec
</div>

<SettingsInfoBlock type="UInt64" default_value="300" />

初期 DDL クエリが、Replicated database で以前の DDL queue のエントリが処理されるまで待機する時間を、秒単位で設定します。

設定可能な値:

* 正の整数。
* 0 — 無制限。
