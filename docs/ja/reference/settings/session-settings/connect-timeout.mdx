<div id="connect_timeout">
  ## connect_timeout
</div>

<SettingsInfoBlock type="Seconds" default_value="10" />

<div id="connect_timeout_with_failover_ms">
  ## connect_timeout_with_failover_ms
</div>

<SettingsInfoBlock type="Milliseconds" default_value="1000" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "23.4"},{"label": "1000"},{"label": "async connect に伴いデフォルトの接続タイムアウトを延長"}]}]} />

クラスター定義で &#39;shard&#39; および &#39;replica&#39; セクションを使用している場合に、Distributed table engine がリモートサーバーへ接続する際のタイムアウトをミリ秒単位で指定します。
接続に失敗した場合は、複数のレプリカへの接続が数回試行されます。
