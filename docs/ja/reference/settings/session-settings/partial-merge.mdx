<div id="partial_merge_join_left_table_buffer_bytes">
  ## partial_merge_join_left_table_buffer_bytes
</div>

<SettingsInfoBlock type="UInt64" default_value="0" />

0 以外の場合、partial merge join における左テーブルのブロックを、より大きなブロックにまとめます。join を実行する各スレッドで、指定したメモリの最大 2 倍を使用します。

<div id="partial_merge_join_rows_in_right_blocks">
  ## partial_merge_join_rows_in_right_blocks
</div>

<SettingsInfoBlock type="UInt64" default_value="65536" />

[JOIN](/ja/reference/statements/select/join) クエリの partial merge join アルゴリズムにおいて、右側の join データの ブロック サイズを制限します。

ClickHouse サーバー:

1. 右側の join データを、指定された行数を上限とする ブロック に分割します。
2. 各 ブロック に最小値と最大値の索引を作成します。
3. 可能であれば、準備済みの ブロック をディスクにアンロードします。

設定可能な値:

* 任意の正の整数。推奨値の範囲: [1000, 100000]。
