<div id="compile_aggregate_expressions">
  ## compile_aggregate_expressions
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

集約関数のネイティブコードへの JIT コンパイルを有効または無効にします。この設定を有効にすると、パフォーマンスが向上する場合があります。

設定可能な値:

* 0 — JIT コンパイルを使用せずに集約を行います。
* 1 — JIT コンパイルを使用して集約を行います。

**関連項目**

* [min&#95;count&#95;to&#95;compile&#95;aggregate&#95;expression](/ja/reference/settings/session-settings/min-count#min_count_to_compile_aggregate_expression)

  ## compile_expressions
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "25.5"},{"label": "1"},{"label": "JITコンパイラを支えるLLVMインフラストラクチャは、この設定をデフォルトで有効にできるほど十分に安定していると考えています。"}]}]} />

<div id="compile_regular_expressions">
  ## compile_regular_expressions
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1"},{"label": "`match` や `extract` などの関数で使用される単純な正規表現の JIT コンパイルを有効にする新しい設定。"}]}]} />

`match` や `extract` などの関数で使用される単純な正規表現をネイティブコードにコンパイルします。サポート対象外のパターンは、自動的に汎用エンジンにフォールバックします。

<div id="compile_sort_description">
  ## compile_sort_description
</div>

<SettingsInfoBlock type="Bool" default_value="1" />

ソート記述をネイティブコードにコンパイルします。
