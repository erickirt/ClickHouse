これらの設定は [system.settings](/ja/reference/system-tables/settings) で確認でき、[ソースコード](https://github.com/ClickHouse/ClickHouse/blob/master/src/Core/Settings.cpp) から自動生成されています。

<div id="reader_executor_max_tail_for_drain">
  ## reader_executor_max_tail_for_drain
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="1048576" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "1048576"},{"label": "新しい実験的な ReaderExecutor 設定: 切断された長時間接続を完了させるためのドレイン上限。"}]}]} />

実験的な `ReaderExecutor` のドレイン上限です。長時間維持されるソース接続が右端境界まで残りこのバイト数以内の時点で切断された場合は、先にその境界まで読み切ることで、未完了の接続として扱うのではなく、完了済みとして再利用可能な状態で接続プールに戻します。

<div id="reader_executor_min_bytes_for_seek">
  ## reader_executor_min_bytes_for_seek
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="UInt64" default_value="2097152" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "2097152"},{"label": "新しい実験的な ReaderExecutor 設定: 維持中のソース接続でブリッジする際の前方ギャップの上限。"}]}]} />

実験的な `ReaderExecutor` における前方ギャップの上限です。これ以下のギャップは、別途ソース読み取りを発行したり再オープンしたりする代わりに、開いているソース接続上でスキップされます (ブリッジ / そのまま読み進める) 。ブリッジがコスト面で有利になるよう、帯域幅とリクエストのコストの損益分岐点付近に設定してください。

<div id="reader_executor_use_long_connections">
  ## reader_executor_use_long_connections
</div>

<ExperimentalBadge />

<SettingsInfoBlock type="Bool" default_value="0" />

<VersionHistory rows={[{"id": "row-1","items": [{"label": "26.7"},{"label": "0"},{"label": "新しい実験的な ReaderExecutor 設定（デフォルトではオフ）: 連続するウィンドウ間で保持中のソース接続を再利用します。"}]}]} />

実験的な `ReaderExecutor` で、ウィンドウをまたいで保持期間に上限のある長時間ソース接続を再利用します。長時間接続とは、range が現在の読み取りウィンドウを超える接続を指します。無効化すると、executor は接続プールの予算を消費せず、各ウィンドウごとに短命なワンショット接続 (stateless パス) を開きます。
