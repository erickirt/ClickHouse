These settings configure the ClickHouse server and are autogenerated from ClickHouse source.

## storage_connections_hard_limit {#storage_connections_hard_limit}

<SettingsInfoBlock type="UInt64" default_value="200000" changeable_without_restart="No" />Exception is thrown at a creation attempt when this limit is reached. Set to 0 to turn off hard limitation. The limit applies to the storages connections.

## storage_connections_rcvbuf {#storage_connections_rcvbuf}

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />The size of the SO_RCVBUF option for storage connections (replication, distributed queries). If set to a value greater than 0, overrides the kernel TCP autotuning for the receive buffer. 0 = kernel default (autotuning). Note: changing this setting back to 0 restores autotuning only for newly created connections; existing pooled connections retain fixed buffer sizes until they are recreated.

## storage_connections_sndbuf {#storage_connections_sndbuf}

<SettingsInfoBlock type="UInt64" default_value="0" changeable_without_restart="Yes" />The size of the SO_SNDBUF option for storage connections (replication, distributed queries). If set to a value greater than 0, overrides the kernel TCP autotuning for the send buffer. 0 = kernel default (autotuning). Note: changing this setting back to 0 restores autotuning only for newly created connections; existing pooled connections retain fixed buffer sizes until they are recreated.

## storage_connections_soft_limit {#storage_connections_soft_limit}

<SettingsInfoBlock type="UInt64" default_value="100" changeable_without_restart="No" />Connections above this limit have significantly shorter time to live. The limit applies to the storages connections.

## storage_connections_store_limit {#storage_connections_store_limit}

<SettingsInfoBlock type="UInt64" default_value="1000" changeable_without_restart="No" />Connections above this limit reset after use. Set to 0 to turn connection cache off. The limit applies to the storages connections.

## storage_connections_warn_limit {#storage_connections_warn_limit}

<SettingsInfoBlock type="UInt64" default_value="500" changeable_without_restart="No" />Warning massages are written to the logs if number of in-use connections are higher than this limit. The limit applies to the storages connections.
